## Summary

Keep open. Current `main` still has a docs/runtime mismatch for `exec` host overrides: `docs/tools/exec.md` says per-call `host=node` is allowed from `auto`, but the runtime and tests reject that override whenever a sandbox runtime is available.

## Evidence

- **Current docs overstate `auto` behavior:** The exec docs say: "Per-call `host=node` is allowed from `auto`; per-call `host=gateway` is only allowed when no sandbox runtime is active." That text does not include the sandbox-active restriction that the runtime applies to `host=node`.
  - file: [docs/tools/exec.md:66](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/docs/tools/exec.md#L66)
  - command: `nl -ba docs/tools/exec.md | sed -n '1,120p'`
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Runtime rejects `node` override from `auto` when sandbox is active:** `isRequestedExecTargetAllowed` returns `false` when `configuredTarget === "auto"`, `sandboxAvailable` is true, and `requestedTarget` is either `"gateway"` or `"node"`.
  - file: [src/agents/bash-tools.exec-runtime.ts:184](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/agents/bash-tools.exec-runtime.ts#L184)
  - command: `nl -ba src/agents/bash-tools.exec-runtime.ts | sed -n '176,193p'`
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Tests lock in the runtime restriction:** The current test suite explicitly expects `resolveExecTarget` to throw for `host=node` from `auto` when a sandbox is available, with the message telling users to set `tools.exec.host=node` instead.
  - file: [src/agents/bash-tools.exec-runtime.test.ts:134](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/agents/bash-tools.exec-runtime.test.ts#L134)
  - command: `nl -ba src/agents/bash-tools.exec-runtime.test.ts | sed -n '121,145p'`
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Mismatch exists on current main, not only in old deployed docs:** The current docs file, runtime file, and runtime test file were all introduced together in commit `78a431ed224b6f5b9d58672346551a5b411ad200` on April 24, 2026, so the inconsistency is present in the checked-out `main` source itself.
  - command: `git show --stat --summary 78a431ed224b6f5b9d58672346551a5b411ad200 -- docs/tools/exec.md src/agents/bash-tools.exec-runtime.ts src/agents/bash-tools.exec-runtime.test.ts`
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Issue discussion narrowed the original report but does not resolve current main:** The reporter acknowledged this may have started as stale docs-site content, but the current repository source still contains the mismatch. The Apr 5 comment claiming runtime/tests allowed `host=node` from `auto` no longer matches the current test file on `main`.
