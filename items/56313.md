## Summary

Issue #56313 is a domain-specific market-data/broker integration request that fits OpenClaw's external plugin/ClawHub model, not core `main`. Current docs and vision explicitly keep optional/commercial capabilities out of core, and the existing plugin SDK already provides the seams needed to implement this externally.

## Close Comment

Closing from a Codex review. This looks like a useful idea, but it fits the external plugin/ClawHub path rather than OpenClaw core.

Concrete scope anchors on current `main`:
- `VISION.md:54` says core stays lean and optional capability should usually ship as plugins.
- `VISION.md:103` explicitly excludes commercial service integrations that do not clearly fit the model-provider category.
- `docs/plugins/building-plugins.md:16` says new plugins do not need to be added to this repo and should be published on ClawHub or npm.
- `docs/plugins/sdk-overview.md:85` already exposes the seams an external implementation would need: agent tools, HTTP routes, gateway methods, and background services.

Given that this request is for finance/market-data broker integrations and routing around external feeds, the conservative maintainer path is to build it as a community plugin or skill and publish it through ClawHub instead of expanding core scope. If you pursue that and hit a missing generic SDK seam, a new focused issue for that seam would be actionable.
