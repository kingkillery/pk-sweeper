## Close Comment

Closing from a Codex review because the visible-feedback part of this report is already implemented on current `main`. Feishu replies now wire `onReplyStart` through the shared typing pipeline and add/remove the Feishu typing reaction around the run (`extensions/feishu/src/reply-dispatcher.ts:118`). OpenClaw’s typing defaults are also immediate for DMs and mentions (`src/auto-reply/reply/typing-mode.ts:18`, `docs/concepts/typing-indicators.md:14`), and the Feishu docs still document `channels.feishu.typingIndicator` as enabled by default (`docs/channels/feishu.md:257`, `docs/channels/feishu.md:400`). There is regression coverage for fresh Feishu messages starting typing on reply start (`extensions/feishu/src/reply-dispatcher.test.ts:238`). The changelog shows the Feishu typing behavior was already shipped by `2026.2.27`, with instant DM/mention typing shipped by `2026.2.13` (`CHANGELOG.md:3570`, `CHANGELOG.md:5917`). If you still see silent turns on a current build, please open a new issue with the current version, relevant `channels.feishu.*` config, and a minimal repro/log sample.
