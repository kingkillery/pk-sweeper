## Summary

Keep open. Current main still resolves the parent/target agent model before `agents.defaults.subagents.model` in both the shared subagent helper and isolated cron path, and checked-in tests currently assert that opposite precedence. PR #58823 is not implemented on main and still represents a plausible fix for the concrete #58822 regression, though it conflicts with the behavior intentionally merged in #58003 and needs maintainer product judgment.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to decide the intended product contract for main versus named agents, then land a narrow fix across `src/agents/model-selection.ts` and `src/cron/isolated-agent/model-selection.ts` with regression tests for explicit spawn override, per-agent `subagents.model`, global `agents.defaults.subagents.model`, and fallback to agent primary model. If #58003's named-agent behavior should remain, implement that distinction explicitly and update docs/tests to match.
