## Summary

Current `main` already handles reasoning-only and empty embedded assistant turns without silently dropping the user message: strict-agentic models get bounded continuation retries, and other cases fall back to an explicit incomplete-turn error. Regression tests cover the non-strict-agentic path, and the changelog shows this shipped in `v2026.4.14`.

## Evidence

- **Incomplete-turn fallback now covers empty/reasoning-only terminal turns:** `resolveIncompleteTurnPayloadText` returns an explicit user-facing error when there is no visible payload and the last assistant turn is reasoning-only or otherwise empty, unless it was an intentional silent reply (`NO_REPLY`) or a clean messaging-tool send.
