## Summary

Keep open. The report's specific 'stale cached Subagent Context' theory is not proven from the issue body, but current `main` still has no guard that validates a spawned subagent task against the requester intent before injecting it into the child prompt. There is also no evidence of a fix on `main` since the latest release, so there is not enough high-confidence evidence to close this conservatively.

## Evidence

- **`sessions_spawn` requires and forwards a caller-provided task:** The tool reads `task` as a required input and treats it as the subagent work item; there is no separate sanitization or cross-check against the parent request at this layer.
  - file: [src/agents/tools/sessions-spawn-tool.ts:183](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/src/agents/tools/sessions-spawn-tool.ts#L183)
