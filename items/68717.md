## Summary

Keep open. Current `main` still treats global cron failure alerts as opt-in, and the public config docs also document `cron.failureAlert.enabled` defaulting to `false`, so this PR is neither implemented on `main` nor obsolete. The PR discussion improved the patch by preserving per-job overrides and adding the missing default-path test, but the remaining question is product direction, not maintainership cleanup.

## Evidence
