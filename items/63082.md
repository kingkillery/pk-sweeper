## Summary

Keep open. Current `main` still hardcodes shell completion profile paths and related user-facing hints, so the underlying bug is not implemented on `main`. The PR is also stale against the current code layout and has unresolved review feedback on bash fallback behavior, so there is not strong evidence for any conservative close reason.

## Evidence

- **Current runtime still ignores ZDOTDIR/XDG_CONFIG_HOME:** `getShellProfilePath` in `src/cli/completion-runtime.ts` still returns `HOME/.zshrc`, `HOME/.bashrc`, and `HOME/.config/fish/config.fish` directly, without consulting `ZDOTDIR` or `XDG_CONFIG_HOME`. `installCompletion` separately hardcodes the same paths, so install/check remain inconsistent with non-default shell config directories.
