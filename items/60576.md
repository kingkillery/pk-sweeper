## Summary

Keep open. Current main still lacks the PR's null/undefined unhandled-rejection guard, so the reported crash path remains plausible and the PR contains useful, narrowly scoped work with regression tests.

## Evidence

- **Current main still exits after unmatched rejection classification:** In installUnhandledRejectionHandler, current main checks custom handlers, AbortError, fatal/config errors, and transient errors, then logs "Unhandled promise rejection" and calls exitWithTerminalRestore. There is no early return for reason === undefined or reason === null before the final exit path.

That supports the PR author's claim that undefined/null can bypass classifiers.
