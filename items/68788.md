## Summary

Keep open. Current `main` still sends the default Claude CLI command (`claude`) through the supervisor child-spawn path, but that path only shim-resolves `npm`/`pnpm`/`yarn`/`npx` and does not reuse the existing Windows `cmd.exe` wrapper used elsewhere for `.cmd`/`.bat` launches. The reporter’s follow-up about a second `EINVAL` failure matches the current code, so this is still an active bug rather than conservative-close material.

  - file: [extensions/anthropic/cli-backend.ts:30](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/extensions/anthropic/cli-backend.ts#L30)

## Risks / Open Questions

- Windows users on the default `claude-cli` backend can still hit `spawn claude ENOENT` before the CLI starts.
- A narrow allowlist-only fix would likely still fail on patched Node with `spawn EINVAL` unless the supervisor path also adopts the existing Windows `.cmd` invocation wrapper.
- I did not run a live Windows reproduction in this Linux checkout; the conclusion is based on current `main`, the latest shipped release, and the issue discussion, which all align.
