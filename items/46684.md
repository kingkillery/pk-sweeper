## Summary

Current `main` does not implement action-level tool policy for the `gateway` tool. `tools.allow`/`tools.deny` still match only the tool name (`gateway`), while `restart`, `config.get`, `config.schema.lookup`, `config.apply`, and `config.patch` remain internal `action` values on that single tool. Main has partial mitigations (`commands.restart=false` and guarded config mutations), but those do not satisfy the request to deny write/restart actions while allowing read-only gateway actions.
