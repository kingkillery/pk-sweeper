## Summary

Keep open. Current `main` still rejects arbitrary bootstrap file basenames in `bootstrap-extra-files`, the docs and tests still describe that restriction, and the linked PR #39174 was closed without merging.

## Evidence

- **Runtime still hardcodes bootstrap basename allowlist:** `VALID_BOOTSTRAP_NAMES` is defined and `loadExtraBootstrapFilesWithDiagnostics()` skips any matched path whose basename is not in that set, reporting `invalid-bootstrap-filename` / `unsupported bootstrap basename`.
