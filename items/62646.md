## Summary

Keep open. Current `main` does not implement the post-`ensureCollections()` managed-collection pruning from PR #62646, so this is not `implemented_on_main`. The PR also is not a safe cleanup close: its bundled `--mask` default conflicts with current shipped behavior/docs, and the review discussion raised unresolved regression risks around older QMD builds with incomplete `collection list` metadata and retry behavior during missing-collection repair.
