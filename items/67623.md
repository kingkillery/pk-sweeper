## Summary

Keep PR #67623 open. Current main still keeps `SANDBOX_BACKEND_FACTORIES` as a module-local `Map`, while sandbox backend registration is exposed through the plugin SDK and used by the bundled OpenShell plugin. The linked issue #67610 remains an open, concrete report of plugin-registered sandbox backends not being found at runtime, and the PR’s final diff addresses the exact registry-sharing boundary by switching to the existing `resolveGlobalMap` helper. This is not cleanup-close material; it needs maintainer review and preferably a regression test for the SDK/core module-instance boundary.

## Best Possible Solution

Keep the PR open. The best maintainer path is to land this fix or an equivalent `resolveGlobalMap(Symbol.for(...))` registry change, plus a focused seam/integration test that registers a sandbox backend through the plugin SDK path and verifies `requireSandboxBackendFactory` finds it from the runtime lookup path.
