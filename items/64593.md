## Summary

Keep open. Current `main` still launches the managed browser with a plain `child_process.spawn(...)`, only tracks the direct Chrome process, and only tests direct SIGTERM/SIGKILL shutdown. I did not find evidence in code, docs, or local history that adopted Chromium/crashpad zombies under a PID 1 gateway were fixed on `main`.

## Evidence

- **Managed browser launch still uses plain child spawn:** `launchOpenClawChrome()` builds args and calls `spawn(preparedSpawn.command, preparedSpawn.args, { stdio: ["ignore", "ignore", "pipe"], env: preparedSpawn.env })`, then returns only that direct `proc` handle. There is no visible process-group, subreaper, or adopted-child cleanup path here.
  - file: [extensions/browser/src/browser/chrome.ts:284](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/browser/src/browser/chrome.ts#L284)

That covers the direct spawned browser process only; I found no companion logic here for reaping or tracking orphaned Chromium helper/crashpad children after parent exit.
