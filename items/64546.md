## Summary

Keep open. Current `main` still uses a bot-token-derived Mattermost interaction secret, so the reported forgery risk has not been fixed on `main`. The PR moves in the right direction, but the recorded review correctly identifies a cross-process regression: a random per-process fallback would break button tokens generated in one process and verified in another.
