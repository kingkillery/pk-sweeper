## Summary

Keep open. Current `main` still has the two pre-PR behaviors this change targets: reset-hook fallback only recognizes slash-prefixed `/new|/reset`, and gateway session reset still reuses `currentEntry.sessionFile` instead of forcing a fresh transcript path. Existing tests on `main` also codify the old custom-`sessionFile` reuse behavior, so there is not strong evidence to close this as already implemented or obsolete.

## Evidence
