## Summary

Built-in EDD stays out of core scope. Current main already provides the plugin-layer pieces this workflow needs: skills are versioned bundles that can include supporting files, and the skills watcher exposes a public plugin SDK change listener. Per VISION, optional capability like regression testing should live as a ClawHub plugin rather than move into OpenClaw core.

## Close Comment

Thanks for the detailed write-up. This was reviewed with Codex against current `main`, and the conservative maintainer call is that this belongs in the ClawHub/plugin layer rather than OpenClaw core. `VISION.md:54` says optional capability should usually ship as plugins and the bar for adding optional plugins to core is intentionally high, and `VISION.md:68` says new skills should go to ClawHub first. Skills are also already treated as bundles with supporting files in `docs/tools/clawhub.md:125`, so an `edd.yaml` sidecar can ship with a skill today without a core format change. Given that the `/edd` workflow already exists as a separate plugin, we’re closing this as ClawHub/plugin scope rather than taking it into core.
