## Summary

Keep open. Current `main` still contains the blanket allowlist heredoc approval gate this PR removes, while the shell-analysis layer already distinguishes safe quoted heredocs from unsafe expanding ones. The bug therefore still appears live on `main`, and duplicate open PRs are not enough to conservatively auto-close this one.

## Evidence

- **Blanket heredoc gate still present on main:** `processGatewayAllowlist()` still computes `hasHeredocSegment`, turns that into `requiresHeredocApproval` whenever allowlist analysis passed, ORs it into `requiresAsk`, and pushes the blanket heredoc warning. That matches the behavior this PR is trying to remove.
