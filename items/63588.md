## Summary

Keep open. Current `main` still drops `Authorization` on Slack CDN redirects in the Slack media download path, and the tests on `main` explicitly lock in that behavior, so #63523 is not implemented on `main`. PR #63588 contains a plausible fix direction, but the last revision is not ready for conservative cleanup because it still carries unresolved review findings and a broken test import (`test/helpers/plugins/fetch-mock.js` does not exist in the repo).
