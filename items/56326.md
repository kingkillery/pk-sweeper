## Summary

Current `main` still exposes ACP-only fields in the public `sessions_spawn` schema and still rejects `streamTo`/`resumeSessionId` before subagent dispatch, so the reported `runtime="subagent"` failure mode remains present rather than implemented or obsolete.

## Evidence

- **Mixed public schema still present:** `SessionsSpawnToolSchema` is still a single object containing both subagent-facing fields and ACP-only fields, including `resumeSessionId`, `cwd`, and `streamTo` alongside normal subagent params.
  - file: [src/agents/tools/sessions-spawn-tool.ts:100](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/agents/tools/sessions-spawn-tool.ts#L100)
  - command: `nl -ba src/agents/tools/sessions-spawn-tool.ts | sed -n '80,280p'`
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Non-ACP guard still hard-fails before dispatch:** The tool still returns an error when `streamTo` or `resumeSessionId` is present and `runtime !== "acp"`, which matches the issue report's failure mode for schema-following models.
  - file: [src/agents/tools/sessions-spawn-tool.ts:229](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/agents/tools/sessions-spawn-tool.ts#L229)
