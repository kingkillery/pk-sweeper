## Summary

Keep open. Current `main` still aborts auto provider selection unless an adapter explicitly implements `shouldContinueAutoSelection`, while the adapter contract keeps that hook optional. That means the PR is not implemented on `main`, and the reported narrow third-party adapter failure mode remains plausible.

## Evidence

- **Current auto-selection still hard-stops by default:** `shouldContinueAutoSelection()` still returns `adapter.shouldContinueAutoSelection?.(err) ?? false`, and `createEmbeddingProvider()` throws on the first adapter error unless that helper returns true. There is no fallback heuristic on `main`.
