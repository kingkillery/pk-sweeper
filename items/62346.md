## Summary

Keep open. Current `main` still uses the old `incomplete-text -> prefill: true` behavior, so this PR is not implemented on `main`, and the PR discussion surfaced a plausible regression in the proposed change: `incomplete-text` is also returned for mixed assistant messages that contain an empty text block alongside other meaningful content, so dropping the whole last assistant turn could discard valid output.

## Evidence

- **Current main still prefills incomplete-text recovery:** `sanitizeThinkingForRecovery` on `main` returns the original messages with `prefill: true` when the last assistant assessment is `incomplete-text`, which is the exact behavior this PR proposes to change.
