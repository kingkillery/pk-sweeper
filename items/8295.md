## Summary

Keep open. Current `main` does not implement Telegram `allowBots` parity: the Telegram config types and schema have no `allowBots` field, Telegram docs only describe user/group allowlists, and the runtime explicitly says Telegram bot-to-bot support exists for `channel_post` channels but not for groups.

## Evidence

- **Telegram config types lack allowBots:** `TelegramAccountConfig`, `TelegramTopicConfig`, and `TelegramGroupConfig` expose group/user allowlist controls but no `allowBots` option anywhere in the Telegram config surface.
  - file: [src/config/types.telegram.ts:94](https://github.com/openclaw/openclaw/blob/c0a7b6a510758645c5267734b984f4885a605b67/src/config/types.telegram.ts#L94)
  - command: `nl -ba src/config/types.telegram.ts | sed -n '80,220p'`

- **Telegram schema lacks allowBots:** The Telegram Zod schema defines `groups`, `allowFrom`, `groupAllowFrom`, and `groupPolicy`, but no `allowBots` field is accepted for Telegram.
  - file: [src/config/zod-schema.providers-core.ts:230](https://github.com/openclaw/openclaw/blob/c0a7b6a510758645c5267734b984f4885a605b67/src/config/zod-schema.providers-core.ts#L230)
