## Summary

Keep open. Current `main` still implements the reported noisy behavior: doctor scans sibling home directories for `.openclaw` and warns on any existing directory without filtering by ownership or readability, and the docs describe that cross-home warning as intended behavior.

## Evidence

- **Cross-home scan has no ownership/readability filter:** `findOtherStateDirs()` walks `/Users` on macOS or `/home` on Linux, builds `<user>/.openclaw` candidates, and adds any candidate where `existsDir()` succeeds. It does not check readability, permissions, or uid ownership before returning the directory.
  - file: [src/commands/doctor-state-integrity.ts:205](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/src/commands/doctor-state-integrity.ts#L205)
  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)
- **Doctor always warns on any discovered sibling state dirs:** `noteStateIntegrity()` adds every result from `findOtherStateDirs(stateDir)` into `extraStateDirs` and emits the `Multiple state directories detected` warning whenever that set is non-empty.
  - file: [src/commands/doctor-state-integrity.ts:797](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/src/commands/doctor-state-integrity.ts#L797)
  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)
- **User-facing docs still describe cross-home detection:** The doctor docs say the check warns when multiple `~/.openclaw` folders exist across home directories or when `OPENCLAW_STATE_DIR` points elsewhere, matching the current noisy behavior rather than the proposed owner/readability filtering.
  - file: [docs/gateway/doctor.md:342](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/docs/gateway/doctor.md#L342)
  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)
- **No opt-out flag or config surfaced in current source/docs search:** A repo search for `doctor.ignoreStateDirs`, `ignoreStateDirs`, and `--scope=home` returned no matches under `src/` or `docs/`, so none of the proposed mitigations appear to exist on `main`.
  - command: `rg -n "doctor --scope=home|ignoreStateDirs|doctor\.ignoreStateDirs|scope=home" src docs`
