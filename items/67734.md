## Summary

Keep PR #67734 open. Current main still constructs Feishu probe clients with the SDK default logger, and the setup/status path still calls that probe before showing channel status. The PR is a focused, still-relevant bug fix for #67497 with proposed regression coverage and manual verification in the discussion.

## Best Possible Solution

Keep the PR open and review/merge a focused version of its approach: add a probe-only silent SDK logger, avoid mutating the shared runtime client cache when a custom logger is supplied, and keep the regression tests that prove both the probe call and client-cache behavior. This directly addresses #67497 and belongs in the Feishu plugin code rather than as cleanup outside the repository.
