## Summary

Keep open. Current main has not implemented the requested WhatsApp `group:` target prefix behavior: the active WhatsApp normalizer strips only `whatsapp:`, and tests still assert `group:<jid>` and `whatsapp:group:<jid>` are invalid. The PR is draft and targets the old `src/whatsapp` path, but the underlying change is concrete and still plausibly useful, so it is not closeable as implemented, duplicate, incoherent, or out-of-repo.

## Best Possible Solution

Keep the PR open only as a concrete but stale signal. The maintainable path is to either port the prefix handling to `extensions/whatsapp/src/normalize-target.ts` and update the WhatsApp target tests for `group:<jid>`, `whatsapp:group:<jid>`, and repeated prefixes, or explicitly reject that alias with tests/docs explaining that WhatsApp message targets must be bare group JIDs.
