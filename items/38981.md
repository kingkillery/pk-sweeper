## Summary

Keep open. The checked-out `main` does not contain `workspaceConfig.allowedExternalPaths`, does not document it, and still hard-rejects workspace file reads/writes and bootstrap loads that resolve outside the workspace root. This PR is not implemented on `main`, and the stale PR still contains non-landed work.

## Evidence

- **No config surface on main:** `AgentDefaultsConfig` still exposes `workspace?: string` and then jumps straight to `skills?`/`repoRoot?`; there is no `workspaceConfig.allowedExternalPaths` field on current `main`. `AgentConfig` likewise has no per-agent `workspaceConfig` block.
  - file: [src/config/types.agent-defaults.ts:176](https://github.com/openclaw/openclaw/blob/4d7c4b3298920c0e6e2c5cc8bd7a1546e8122abe/src/config/types.agent-defaults.ts#L176)

There is no allowlist resolution path for trusted external symlink targets.
