## Summary

Current main does not provide strong evidence that issue #39248 is fixed. The code still routes non-main session keys into sandboxed execution, but the post-release changes I found in the subagent path address context-engine preparation and gateway scope negotiation (#59428), not Docker-backed subagent initialization under `agents.defaults.sandbox.mode: "non-main"`. The test coverage I found also does not exercise the reported path.

## Evidence
