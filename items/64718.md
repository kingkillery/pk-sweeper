## Summary

Keep open. Current `main` still does not apply sender-aware exec defaults in auto-reply, and the PR discussion identifies a remaining bypass in the proposed fix: returning a partial `agentExecDefaults` object still lets downstream exec resolution inherit permissive `security`/`ask` defaults. This is not a conservative close candidate.

## Evidence

## Risks / Open Questions

- The underlying hardening is still absent on current `main`, so non-owner auto-reply turns can still inherit permissive gateway exec defaults when no explicit safer policy is present.
- Closing this PR would discard an active security hardening attempt even though the request remains unimplemented on `main`.
- If the PR is revived, it still needs an explicit fix and regression coverage for partial `agentExecDefaults` objects, not just the no-overrides case.
