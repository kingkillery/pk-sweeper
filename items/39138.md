## Summary

Keep open. Current `main` still auto-runs pending outbound delivery recovery on gateway startup and replays queued entries based on retry/backoff state, not message age. I found no TTL/expiry check, max-replay-count limit, or user-facing config to suppress stale replays, so the reported behavior remains plausible and is not implemented away on `main`.
