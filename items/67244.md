## Summary

Keep open. Current main still appears to have the reported source-level gaps: ACP backend lookup in the explicit/local path depends on an already-started in-process backend service, and the sessions_yield terminal path still returns parent-run metadata without descendant completion reconciliation or final JSON cleanup.

## Best Possible Solution

Keep the issue open. The best fix is source-level: ensure the explicit embedded ACP path can register or start the bundled acpx backend in the current process before requireRuntimeBackend lookup, then add a targeted regression for `openclaw agent --json` with `runtime: "acp"` plus `sessions_yield` so descendant completion updates finalAssistantVisibleText and clears stale replayInvalid/livenessState metadata.
