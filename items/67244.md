## Summary

Keep open. Current `main` appears to have addressed the process-local ACP backend visibility side by preloading runtime plugins before embedded runs, but the explicit `agent --json` path still returns the first `end_turn` result metadata directly. I did not find any source-level descendant-drain/final-state reconciliation for yielded runs in that path, so the stale `replayInvalid`/`livenessState` half of #67244 is not clearly implemented on `main`.

## Evidence

- **Embedded runs preload runtime plugins:** `runEmbeddedPiAgent` calls `ensureRuntimePluginsLoaded(...)` before execution, which resolves the runtime plugin registry for the current process.
  - file: [src/agents/pi-embedded-runner/run.ts:309](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/src/agents/pi-embedded-runner/run.ts#L309)
  - sha: [6f72b74cecf8](https://github.com/openclaw/openclaw/commit/6f72b74cecf8d1fec5c147ae932161bf0f37a68b)
- **Bundled ACPX registers an in-process ACP backend:** The ACPX service registers the `acpx` runtime backend with `registerAcpRuntimeBackend(...)` and logs readiness once healthy, which is the mechanism that makes backend lookup visible inside the process that loaded the plugin.
  - file: [extensions/acpx/src/service.ts:122](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/extensions/acpx/src/service.ts#L122)
  - sha: [6f72b74cecf8](https://github.com/openclaw/openclaw/commit/6f72b74cecf8d1fec5c147ae932161bf0f37a68b)
- **Yielded runs still return immediate replay/liveness metadata:** `runEmbeddedPiAgent` derives `replayInvalid` and `livenessState` from the current attempt and returns them together with `stopReason: "end_turn"` when `yieldDetected` is set; there is no descendant wait or second-pass reconciliation in this return path.
  - file: [src/agents/pi-embedded-runner/run.ts:2209](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/src/agents/pi-embedded-runner/run.ts#L2209)
  - sha: [6f72b74cecf8](https://github.com/openclaw/openclaw/commit/6f72b74cecf8d1fec5c147ae932161bf0f37a68b)
- **Explicit agent JSON output forwards `result.meta` unchanged:** The command path emits a lifecycle end for the first result and then the JSON delivery path serializes `result.meta` directly into the output envelope, so any stale `replayInvalid`/`livenessState` from the yielded result would be surfaced as-is.
  - file: [src/agents/agent-command.ts:942](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/src/agents/agent-command.ts#L942)
  - sha: [6f72b74cecf8](https://github.com/openclaw/openclaw/commit/6f72b74cecf8d1fec5c147ae932161bf0f37a68b)
- **JSON envelope does not do follow-up reconciliation:** `deliverAgentCommandResult` writes `buildOutboundResultEnvelope({ payloads, meta: result.meta })` directly for `--json` output.
  - file: [src/agents/command/delivery.ts:324](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/src/agents/command/delivery.ts#L324)
  - sha: [6f72b74cecf8](https://github.com/openclaw/openclaw/commit/6f72b74cecf8d1fec5c147ae932161bf0f37a68b)
- **A descendant-drain helper exists, but it is wired elsewhere:** `waitForAgentRunsToDrain` explicitly refreshes pending run IDs until descendants stop changing, and current usage shown here is in isolated cron follow-up logic rather than the explicit agent command path.
  - file: [src/agents/run-wait.ts:185](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/src/agents/run-wait.ts#L185)
  - sha: [6f72b74cecf8](https://github.com/openclaw/openclaw/commit/6f72b74cecf8d1fec5c147ae932161bf0f37a68b)
- **Existing descendant-drain usage is not in `agent --json`:** The isolated-agent follow-up path waits for descendant runs to drain before final synthesis, which highlights the missing equivalent in the explicit agent JSON flow.
  - file: [src/cron/isolated-agent/subagent-followup.ts:100](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/src/cron/isolated-agent/subagent-followup.ts#L100)
  - sha: [6f72b74cecf8](https://github.com/openclaw/openclaw/commit/6f72b74cecf8d1fec5c147ae932161bf0f37a68b)
