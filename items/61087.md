## Summary

Keep open. Current `main` still does not advertise Discord `upload-file`, so the discoverability gap remains, and the proposed PR has unresolved review concerns around metadata/reply passthrough and missing tests.

## Evidence

- **Discord discovery still omits `upload-file`:** On current `main`, `describeDiscordMessageTool` adds `read`, `edit`, and `delete` under the Discord `messages` gate, but not `upload-file`. The reported gap is still present in `extensions/discord/src/channel-actions.ts`.
  - file: [extensions/discord/src/channel-actions.ts:88](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/extensions/discord/src/channel-actions.ts#L88)
  - command: `nl -ba extensions/discord/src/channel-actions.ts | sed -n '88,120p'`
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Agent-facing tool descriptions depend on advertised actions:** The shared message tool description enumerates per-channel actions from `listCrossChannelSchemaSupportedMessageActions(...)` and prints them in the agent-visible tool description. If Discord does not advertise `upload-file`, agents will not see that capability called out.
  - file: [src/agents/tools/message-tool.ts:607](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/agents/tools/message-tool.ts#L607)
  - command: `nl -ba src/agents/tools/message-tool.ts | sed -n '600,650p'`
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Current Discord send path already supports attachment params and reply/session metadata:** `handleDiscordMessageAction` already accepts `media`/`path`/`filePath` for `send` and forwards `replyTo`, `__sessionKey`, and `__agentId` to `sendMessage`. That matches the PR's stated root cause: the backend capability exists, but discoverability is incomplete.
  - file: [extensions/discord/src/actions/handle-action.ts:70](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/extensions/discord/src/actions/handle-action.ts#L70)

That means the PR is still useful, but not ready to close as already handled.
