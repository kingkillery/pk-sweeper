## Summary

Current `main` does not clear user-sourced `authProfileOverride`/`modelOverride` on `/new`, `/reset`, or gateway `sessions.reset`. The code explicitly preserves those fields, and tests lock that behavior in, so issue #62587 is not implemented on `main` and should remain open.

## Evidence

- **Reset-preservation helper keeps user overrides:** `resolveResetPreservedSelection()` preserves `providerOverride`, `modelOverride`, and `authProfileOverride` when the stored source is `user`, and also treats legacy model overrides with no source as user-driven.
