## Summary

Keep open. Current `main` and the latest shipped release still lack the Matrix-specific loop guards described in PR #61038: `room.message` ingress is still forwarded without a synchronous self-message gate, inbound `m.notice` is not generically filtered, and normal reply delivery still routes through `m.text` by default. The PR discussion also indicates the proposed fix was otherwise sound, with only a minor observability comment outstanding.

## Evidence
