## Summary

Keep PR #61038 open. The linked bug report is closed, but current main still does not contain the PR's structural Matrix self-loop protections: the Matrix event monitor forwards room messages to the handler without a cached self-sender gate, the handler only has the existing async self-user check, and generic Matrix reply delivery still sends text through the default m.text path rather than m.notice for tool/block output. The PR therefore appears to contain still-useful work rather than being implemented, obsolete, or superseded.
