## Summary

Keep open. Current `main` does not contain this PR's transcript-repair/tool-ID normalization, so the reported OpenAI-compatible `functions exec:0` / `functions exec` pairing problem is not implemented on `main` and remains plausible.

## Evidence

- **Transcript repair still only trims tool names:** `sanitizeToolCallBlock()` trims whitespace only, and `repairToolCallInputs()` only redacts when the trimmed name is exactly `sessions_spawn`. There is no stripping of `functions.` or `functions ` prefixes, so prefixed names like `functions.sessions_spawn` or `functions exec` are not normalized in this path.
  - file: [src/agents/session-transcript-repair.ts:100](https://github.com/openclaw/openclaw/blob/c971c58fc7592344539ac3cf87c7c4229be9978e/src/agents/session-transcript-repair.ts#L100)
  - command: `nl -ba src/agents/session-transcript-repair.ts | sed -n '80,220p'; nl -ba src/agents/session-transcript-repair.ts | sed -n '280,360p'`
  - sha: [c971c58fc759](https://github.com/openclaw/openclaw/commit/c971c58fc7592344539ac3cf87c7c4229be9978e)
- **Tool-result name normalization does not handle provider prefixes:** `normalizeToolResultName()` only applies `normalizeOptionalString()` and fallback substitution. It does not rewrite `functions.exec` or `functions exec` to the canonical tool name, so result/name pairing is still missing this normalization on `main`.
  - file: [src/agents/session-transcript-repair.ts:194](https://github.com/openclaw/openclaw/blob/c971c58fc7592344539ac3cf87c7c4229be9978e/src/agents/session-transcript-repair.ts#L194)
