## Summary

Current `main` still does not implement WhatsApp sticker sending. The WhatsApp outbound send path routes all `image/*` media, including WebP, through the normal image payload, the channel only advertises `react` and `poll` actions, and the linked PR #7502 was closed without merging.

## Evidence

- **Outbound WhatsApp send path has no sticker branch:** `extensions/whatsapp/src/inbound/send-api.ts` routes any `mediaType.startsWith("image/")` payload to `{ image: mediaBuffer, caption, mimetype }` and never emits Baileys `{ sticker: ... }`. That means WebP outbound media is still treated as an image on current `main`.
