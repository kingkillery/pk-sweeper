## Summary

Keep open. Current `main` fixes part of the report: interval embeds now use `memory.qmd.update.embedTimeoutMs`, `embedInterval <= 0` suppresses the periodic embed timer, and the config reference documents both knobs. But the core complaint is not fully resolved because OpenClaw still intentionally runs `qmd embed` even when `searchMode: "search"`, so the reported timeout/build loop can still happen on affected environments.
