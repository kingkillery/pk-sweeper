## Summary

Keep open. Current `main` still has the reported Zalo bug: `callZaloApi` parses every HTTP response with `response.json()` before checking `response.ok`, so non-JSON 4xx/5xx bodies can still throw a `SyntaxError` instead of a typed `ZaloApiError`. The PR also carries still-relevant follow-ups around preserving structured error details and separating HTTP 408 from app-level polling timeouts.

## Evidence

That means HTML/plain-text error bodies from proxies or upstream failures still bypass `ZaloApiError` and can fail with JSON parsing errors.
