## Summary

Keep open. Current `main` still does not provide a first-class agent self-compaction path. `/compact` remains gated to authorized senders, `session_status` still only exposes status/model override behavior, and the agent-facing `gateway` tool cannot call `sessions.compact` even though that RPC exists internally.

## Evidence

- **`session_status` has no compaction action:** The tool schema only accepts `sessionKey` and `model`; there is no `action` field or compaction parameter.
  - file: [src/agents/tools/session-status-tool.ts:55](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/agents/tools/session-status-tool.ts#L55)
