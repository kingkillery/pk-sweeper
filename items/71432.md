## Summary

The request is a higher-level multi-agent orchestration feature that current maintainer direction keeps out of OpenClaw core. Main already exposes isolated multi-agent routing plus plugin/session primitives that are intended for this kind of workflow, so this should be pursued as a ClawHub/plugin implementation rather than a core issue.

## Risks / Open Questions

- A future first-party product decision could still choose to add native shared multi-agent conversation UX, but current documented scope points to plugin/ClawHub rather than core.
- A plugin may still need small new host APIs later if authors hit missing seams, but that is narrower than accepting this full orchestration feature into core now.
