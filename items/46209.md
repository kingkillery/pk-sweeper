## Summary

Keep open. Current `main` still lets `derivedTitle` come from the first transcript user message before considering `label` or `origin.label`, and ACP poll consumers prefer that `derivedTitle` over `displayName`/`label`. The PR's core fix is not present on `main`; the only discussion in the provided thread was a minor stale-test-description note, not evidence that the behavior was already resolved another way.

## Evidence

- **Current title precedence still ignores labels:** `deriveSessionTitle` on `main` checks `displayName`, then `subject`, then `firstUserMessage`, then falls back to the session-id prefix. It does not check `entry.label` or `entry.origin?.label` before transcript-derived text.
