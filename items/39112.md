## Summary

Keep open. Current `main` does not implement the PR’s Ollama usage fallback, and the dashboard still depends on non-zero transcript usage, so the reported gap remains plausible and useful.

## Evidence

- **Ollama native stream still zeros missing counters:** `buildAssistantMessage` in `extensions/ollama/src/stream.ts` still sets usage from `response.prompt_eval_count ?? 0` and `response.eval_count ?? 0`. If native `/api/chat` omits those fields, transcript usage stays zero instead of using any estimate.
