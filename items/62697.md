## Summary

Keep open. Current `main` does not implement WhatsApp newsletter/channel handling: the bundled WhatsApp plugin only models `direct` and `group` chats, the inbound monitor drops non-group JIDs that cannot be normalized to E.164/LID, and there is no `newsletterPolicy` or documented newsletter access-control path.

## Evidence

- **Plugin capabilities only expose direct/group chat types:** The bundled WhatsApp plugin declares only `chatTypes: ["direct", "group"]`; there is no newsletter/channel chat type or capability surface.
  - file: [extensions/whatsapp/src/shared.ts:211](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/extensions/whatsapp/src/shared.ts#L211)
  - command: `nl -ba extensions/whatsapp/src/shared.ts | sed -n '206,218p'`
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **Inbound monitor drops unsupported non-group JIDs before routing:** Inbound normalization treats a message as group only when the JID is a WhatsApp group JID. Otherwise it tries to resolve the remote JID to a direct sender ID and returns `null` when that fails, so unsupported JID classes are ignored before access-control or reply handling.
  - file: [extensions/whatsapp/src/inbound/monitor.ts:364](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/extensions/whatsapp/src/inbound/monitor.ts#L364)
  - command: `nl -ba extensions/whatsapp/src/inbound/monitor.ts | sed -n '352,424p'`
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **JID normalization only accepts phone/LID forms:** `jidToE164` recognizes only `@s.whatsapp.net`, `@hosted`, `@lid`, and `@hosted.lid`. `resolveJidToE164` returns `null` for other suffixes, which would exclude `@newsletter` JIDs from the existing direct-message path.
  - file: [extensions/whatsapp/src/targets-runtime.ts:98](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/extensions/whatsapp/src/targets-runtime.ts#L98)
  - command: `nl -ba extensions/whatsapp/src/targets-runtime.ts | sed -n '98,148p'`
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **User-facing WhatsApp policy surface has no newsletter policy:** The public WhatsApp docs describe only DM and group access controls (`dmPolicy`, `allowFrom`, `groupPolicy`, `groupAllowFrom`). There is no documented `newsletterPolicy` or newsletter-specific behavior.
  - file: [docs/channels/whatsapp.md:155](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/docs/channels/whatsapp.md#L155)
