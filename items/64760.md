## Summary

Current `main` already has the mitigation the reporter’s follow-up comment pointed out: when the skills prompt exceeds budget, OpenClaw switches to a compact catalog that omits descriptions and it exposes size/count limits for that catalog. But the requested optimization is still not implemented: compact mode still emits every visible skill’s `<name>` plus `<location>`, and there is no names-only or lazy-match prompt mode in the current config/docs. This is still a live optimization request, not a high-confidence close.
