## Summary

Current `main` already reads Comfy plugin-owned config and plugin-owned cloud API keys, but it does not implement this PR's additional compatibility behavior of treating `models.providers.comfy` as a runtime override when plugin config is present. The PR still carries distinct behavior and is not strong enough to auto-close as already implemented or obsolete.

## Evidence
