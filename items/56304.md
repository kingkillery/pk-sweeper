## Summary

## Close Comment

Closing as `clawhub` after a Codex review of current `main`. The feature request is clear, but current project scope points this toward an external plugin rather than a bundled core change: `VISION.md:54-59` says optional capability should usually ship as plugins and that the bar for adding optional plugins to core is intentionally high, while `VISION.md:99-103` keeps non-core commercial integrations out of the default merge path for now. Current main already exposes web search as a plugin capability through `api.registerWebSearchProvider(...)` (`docs/plugins/architecture.md:49-50`), and the provider-plugin guide explicitly routes that kind of integration to ClawHub publishing (`docs/plugins/sdk-provider-plugins.md:679-689`). Also, the bundled Qianfan plugin on main is still only the model-provider surface (`extensions/qianfan/index.ts:7-30`). Given that, Baidu web search fits the supported ClawHub plugin path better than expanding bundled OpenClaw core.
