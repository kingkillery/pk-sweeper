## Summary

Current `main` has the `subagent_ended` hook and per-run outcome bookkeeping, but it still does not implement any two-strike/N-strike enforcement, failure counter persistence, or `agents.defaults` failure-policy config. The issue is not implemented on `main` and should remain open.

## Evidence

- **Run record has no failure counter:** `SubagentRunRecord` includes lifecycle/output fields such as `outcome`, `endedReason`, and hook bookkeeping, but no `consecutiveFailureCount` or similar persisted strike field.
  - file: [src/agents/subagent-registry.types.ts:6](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/agents/subagent-registry.types.ts#L6)
