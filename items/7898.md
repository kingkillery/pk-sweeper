## Summary

Dify support is not implemented on current main, and the request fits OpenClaw's external plugin or harness path rather than bundled core scope.

## Evidence

- **Vision keeps optional capability out of core:** Current project vision says OpenClaw has an extensive plugin API, core should stay lean, optional capability should usually ship as plugins, external plugins should be hosted in their own repo, and the bar for adding optional plugins to core is intentionally high.

- **Vision excludes this kind of core integration:** The roadmap guardrails explicitly say OpenClaw will not merge commercial service integrations that do not clearly fit the model-provider category, and will not merge heavy orchestration layers that duplicate existing agent/tool infrastructure.
  - file: [VISION.md:99](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/VISION.md#L99)

## Risks / Open Questions

- If maintainers want to make a deliberate product exception to the current `VISION.md` guardrails, this issue should stay open instead of being closed for scope.
- A Dify integration may need an agent harness rather than only a provider plugin if it owns workflow or session state, but that still fits the external plugin path rather than core bundling.

## Close Comment

Closing this as a Codex maintainer review because Dify is a better fit for an external OpenClaw plugin than for OpenClaw core. Current scope guidance says optional capability should usually ship as plugins and be hosted or maintained outside this repo (`VISION.md:54`), and the project explicitly avoids merging commercial service integrations that do not clearly fit the model-provider category plus heavy orchestration layers (`VISION.md:99`). The current SDK/docs already provide the extension path for third-party provider plugins and native agent harnesses (`docs/plugins/building-plugins.md:16`, `docs/plugins/sdk-provider-plugins.md:11`, `docs/plugins/sdk-agent-harness.md:20`). There is no Dify implementation on current `main`, so the right next step is a third-party Dify plugin published via ClawHub or npm rather than a bundled core feature.
