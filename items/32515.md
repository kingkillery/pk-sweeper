## Summary

Keep #32515 open. Current main has related Anthropic auth support, including API-key setup, Claude CLI credential reuse, `ANTHROPIC_OAUTH_TOKEN`, and manual `sk-ant-oat01-*` setup-token handling, but it does not implement the requested one-click Anthropic browser OAuth flow. The issue comment’s suggested minimum improvement, token-prefix auto-detection between `sk-ant-api03-*` and `sk-ant-oat*`, is only partially present in runtime request handling and not as an onboarding/auth-selection UX that removes the manual choice.

## Evidence

- **Issue request and discussion:** #32515 asks for `openclaw onboard` to open a browser, let the user click "Sign in with Anthropic", detect Pro/subscription versus API credits, and configure auth automatically.
