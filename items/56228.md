## Summary

Keep open. Current `main` still drops reply context for Discord referenced messages whose visible text lives only in Components v2 blocks. `resolveDiscordMessageText()` does not read `message.components`, and `resolveReplyContext()` returns `null` when that resolver returns an empty string.

## Evidence

- **Resolver ignores Components v2 payloads:** `resolveDiscordMessageText()` only builds text from `message.content`, media placeholders, the first embed, and optional fallback text before returning. There is no branch that inspects `message.components` or Discord v2 text-display nodes.
  - file: [extensions/discord/src/monitor/message-utils.ts:615](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/extensions/discord/src/monitor/message-utils.ts#L615)
  - command: `nl -ba extensions/discord/src/monitor/message-utils.ts | sed -n '615,635p'`
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Empty resolver result drops reply context entirely:** `resolveReplyContext()` calls `resolveDiscordMessageText(referenced, { includeForwarded: true })` and returns `null` when that text is empty, so a Components-v2-only referenced message produces no `ReplyToBody`.
  - file: [extensions/discord/src/monitor/reply-context.ts:17](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/extensions/discord/src/monitor/reply-context.ts#L17)
  - command: `nl -ba extensions/discord/src/monitor/reply-context.ts | sed -n '15,35p'`
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Referenced Discord payload is preserved, so missing extraction is the gap:** Fetched `referenced_message` objects are merged onto the Carbon `Message` instance wholesale, with mention normalization layered on top. That means the referenced payload is available to later code; the loss happens in text extraction, not in preflight hydration.
  - file: [extensions/discord/src/monitor/message-handler.preflight.ts:282](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/extensions/discord/src/monitor/message-handler.preflight.ts#L282)

  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Existing tests do not cover Components v2 referenced text extraction:** The `resolveDiscordMessageText()` tests cover forwarded snapshots, mentions, stickers, and embed fallback cases, but there is no case for a message whose text exists only in Discord Components v2 blocks.
  - file: [extensions/discord/src/monitor/message-utils.test.ts:825](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/extensions/discord/src/monitor/message-utils.test.ts#L825)
