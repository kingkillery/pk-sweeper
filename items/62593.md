## Summary

Keep open. Current `main` no longer uses the PR's old Matrix-only branch, but the replacement Slack delivery hook still prefixes `parentConversationId` verbatim. The codebase already carries canonical parent ids like `channel:parent-1`, so this can still produce malformed targets such as `channel:channel:...`. That is not strong evidence that the requested Slack thread-delivery fix is implemented on `main`, and the PR also mixes unrelated QMD/doc changes.
