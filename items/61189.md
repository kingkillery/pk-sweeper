## Summary

Keep open. Current main still treats a richer macOS app node as the intended behavior, while remote mode also starts a headless node-host service. That means the reported `clientId: "node-host"`-only surface is still plausible as a real regression or routing bug, and I did not find strong evidence on `main` or in current docs/release notes that this was intentionally changed or fully fixed.

## Evidence
