## Summary

Keep open. The report is thin, but current main only proves display-side sanitization for leaked tool-call text; it does not prove the underlying tool-execution failure on the reporter's provider/model path is fixed or no longer reproducible.

## Evidence

- **Issue report is still too underspecified to verify or dismiss:** The issue only gives `upgrade 2026.4.2`, `start gateway`, and a generic symptom. It does not identify the actual upstream provider/model behind `openclaw gateway websocket`, and it includes no logs, screenshots, or transcript showing whether the model emitted plain-text pseudo-tool calls versus structured tool calls.
  - command: `provided issue context`
- **Current main strips leaked tool-call text from visible replies:** `sanitizeAssistantVisibleText` removes Minimax tool XML, raw `<tool_call>`/`<tool_result>` XML, and downgraded `[Tool Call: ...]` text before user-visible delivery. That addresses leakage in rendered output, but it is a sanitizer, not evidence that the underlying tool call was executed.
