## Summary

Keep open. On the current checkout HEAD (`86099ec62a41ef61209efef16406cd6e9ece15a1`), OpenClaw still uses a static `100000` default for `session.parentForkMaxTokens`, silently skips parent-thread forking with only a warn log when the parent is too large, and requires manual override tuning. The requested auto-tune or user-visible warning behavior is not implemented on main.

## Evidence
