## Summary

Keep open. Current `main` has exec allowlist enforcement, but it lives in host-local exec approvals (`~/.openclaw/exec-approvals.json` / `openclaw approvals allowlist ...`), not in the requested `tools.exec.allowlist` config key. The strict `tools.exec` schema still does not define an `allowlist` field, so the issue is not implemented as filed.

## Evidence

- **`tools.exec` schema is strict and has no `allowlist` field:** `ToolExecBaseShape` defines `host`, `security`, `ask`, `node`, `pathPrepend`, `safeBins`, `strictInlineEval`, and related fields, but no `allowlist`; `ToolExecSchema` then wraps that shape with `.strict()`, so unknown keys such as `tools.exec.allowlist` are still rejected.
