## Summary

Keep open. Current `main` still lacks the proposed `audio/webm`, `video/webm`, and `video/x-matroska` mappings, and the media-save path still derives filenames from `extensionForMime(...)`, so a WebM/MKV buffer without a usable original extension can still be persisted without the correct suffix. I did not find an equivalent fix on `main`, and the existing tests do not cover these MIME cases.

## Evidence
