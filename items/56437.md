## Summary

Keep open. Current main already preserves normal fallback execution context better than the report suggests: fallback runs reuse the same prompt/workspace path, rebuild the system prompt with bootstrap files, persist fallback model selection into session state, and surface fallback notices/runtime model labels. But the requested feature itself is not implemented: agent model config still only accepts `primary` and `fallbacks`, there is no native `fallbackRunbook` field, and the doctor docs only document model-reference validation for `hooks.gmail.model`, not `agents.defaults.model.fallbacks` or agent fallback IDs.
