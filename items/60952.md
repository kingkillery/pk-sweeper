## Summary

Keep open. On current `main` (`59523e66da03591bc7bcc183a11f690fa0aae641`), the MS Teams plugin only special-cases Adaptive Card `invoke` traffic (`adaptiveCard/action`) and poll votes. Ordinary Teams `message` activities still derive inbound body from stripped text or attachment placeholders, and then drop the activity when that body is empty. There is no generic `messageBack`/`Action.Submit` path that forwards `activity.value` to the agent, so the reported gap still appears real. The issue timeline also points to open PR #64503, which suggests active but not-yet-landed work rather than an already-resolved item.
