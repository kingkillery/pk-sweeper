## Summary

Keep open. This is a recent non-maintainer feature request for core CLI/node execution ergonomics. Current main documentation still says shell execution on nodes should use the exec tool with host=node instead of openclaw nodes run, so the requested restoration is not implemented. The request also targets OpenClaw core CLI/gateway behavior rather than a ClawHub skill/plugin.

## Evidence

- **Issue request:** Issue #59337 asks to restore the removed openclaw nodes run style so users can directly execute shell commands on paired nodes from the CLI, especially for batch remote command execution. The issue is recent, has only the enhancement label, and has no maintainer/protected labels in the provided GitHub context.
