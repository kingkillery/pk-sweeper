## Summary

Keep open. Current `origin/main` at `be1d656514aa485073287cea50e81e1d0e4163cb` still has the exec/system-event heartbeat routing behavior this PR is trying to fix: exec completion wakes are scoped only by session key and do not clear pinned heartbeat routing fields. The PR is a focused, recent bug fix with targeted regression coverage and reviewer feedback that is not a close reason.

## Best Possible Solution

Keep PR #71582 open. The best path is to preserve the bug fix direction, preferably consolidate the duplicated exec-event wake helper into a shared routing/helper module, then run the repo's changed gates for the touched heartbeat, gateway, and agent surfaces before landing.
