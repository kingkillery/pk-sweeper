## Close Comment

Closing as implemented on current OpenClaw. Codex review found the exact pre-tool boundary handling already in the shipped code: `src/agents/pi-embedded-subscribe.handlers.tools.ts:696` flushes pending assistant block replies before tool execution, `src/auto-reply/reply/agent-runner-execution.ts:1210` wires `onBlockReply`/`onBlockReplyFlush` into the embedded agent path, and `src/auto-reply/reply/agent-runner-payloads.ts:223` suppresses duplicate finals after that direct flush. There is also an explicit regression test for this path in `src/agents/pi-embedded-subscribe.subscribe-embedded-pi-session.calls-onblockreplyflush-before-tool-execution-start-preserve.test.ts:9` and `:53`, covering assistant text emitted before a tool call and verifying it is flushed on `tool_execution_start`. This implementation is present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). If this still reproduces on `v2026.4.22` or newer, please open a fresh issue with updated Discord/Slack repro steps and logs from that version.
