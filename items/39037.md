## Summary

Keep open. Current `main` does not implement CTWA referral extraction or expose `ctwa_clid` in WhatsApp inbound context, agent prompt metadata, or persisted session metadata.

## Evidence

- **WhatsApp inbound model has no referral field:** `WebInboundMessage` carries reply, mention, location, media, and `untrustedStructuredContext`, but no dedicated `referral`, `ctwaClid`, or similar field to transport Click-to-WhatsApp metadata.
  - file: [extensions/whatsapp/src/inbound/types.ts:54](https://github.com/openclaw/openclaw/blob/84fc00afdacbbb67e39fc1341242eedad09243df/extensions/whatsapp/src/inbound/types.ts#L54)
  - sha: [84fc00afdacb](https://github.com/openclaw/openclaw/commit/84fc00afdacbbb67e39fc1341242eedad09243df)
- **Inbound assembly only attaches contact structured context:** When OpenClaw builds the inbound WhatsApp message object, the only structured metadata it adds is contact-card data under `untrustedStructuredContext`. There is no extraction or attachment of `contextInfo.referral` or `ctwaClid` here.
  - file: [extensions/whatsapp/src/inbound/monitor.ts:564](https://github.com/openclaw/openclaw/blob/84fc00afdacbbb67e39fc1341242eedad09243df/extensions/whatsapp/src/inbound/monitor.ts#L564)
  - sha: [84fc00afdacb](https://github.com/openclaw/openclaw/commit/84fc00afdacbbb67e39fc1341242eedad09243df)
- **ContextInfo extraction is generic but not used for CTWA referral:** The extractor can find `contextInfo`, but the surrounding WhatsApp inbound code only consumes that path for mentions and quoted-message context; there is no code reading a referral payload or click id from `contextInfo`.
  - file: [extensions/whatsapp/src/inbound/extract.ts:150](https://github.com/openclaw/openclaw/blob/84fc00afdacbbb67e39fc1341242eedad09243df/extensions/whatsapp/src/inbound/extract.ts#L150)
  - sha: [84fc00afdacb](https://github.com/openclaw/openclaw/commit/84fc00afdacbbb67e39fc1341242eedad09243df)
- **Prompt context only forwards whatever structured metadata the inbound message already has:** `buildWhatsAppInboundContext` passes `params.msg.untrustedStructuredContext` into the agent context. Because the inbound message never gets CTWA referral data attached, nothing related to `ctwa_clid` reaches the prompt/preamble.
  - file: [extensions/whatsapp/src/auto-reply/monitor/inbound-dispatch.ts:110](https://github.com/openclaw/openclaw/blob/84fc00afdacbbb67e39fc1341242eedad09243df/extensions/whatsapp/src/auto-reply/monitor/inbound-dispatch.ts#L110)
  - sha: [84fc00afdacb](https://github.com/openclaw/openclaw/commit/84fc00afdacbbb67e39fc1341242eedad09243df)
- **Session metadata persistence does not store arbitrary WhatsApp referral data:** `deriveSessionMetaPatch()` only derives group/session origin metadata. It does not persist `UntrustedStructuredContext`, referral objects, or a session-level `ctwa_clid` attribute.
  - file: [src/config/sessions/metadata.ts:181](https://github.com/openclaw/openclaw/blob/84fc00afdacbbb67e39fc1341242eedad09243df/src/config/sessions/metadata.ts#L181)
  - sha: [84fc00afdacb](https://github.com/openclaw/openclaw/commit/84fc00afdacbbb67e39fc1341242eedad09243df)
- **No shipped implementation found by repo search:** A repo-wide search for `ctwaClid`, `ctwa_clid`, `click-to-whatsapp`, and `referral` returned no matches under `extensions/whatsapp`, `src`, `docs`, or `test`, which is consistent with the missing code paths above.
  - command: `rg -n "ctwaClid|ctwa_clid|click-to-whatsapp|referral" extensions/whatsapp src docs test`
  - sha: [84fc00afdacb](https://github.com/openclaw/openclaw/commit/84fc00afdacbbb67e39fc1341242eedad09243df)
