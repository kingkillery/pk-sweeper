## Summary

Keep open. Current `main` is still image-only across the upload, preview, API, and gateway parsing paths, and this PR only widens the file input accept list. As written, non-image uploads would still be previewed/rendered as images on the client and then dropped or misclassified by the gateway contract.

## Evidence

  - file: [ui/src/ui/chat/attachment-support.ts:1](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/ui/src/ui/chat/attachment-support.ts#L1)
