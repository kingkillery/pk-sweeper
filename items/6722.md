## Summary

Keep open. Current `main` still does not implement WhatsApp link-preview enablement: the Baileys socket setup omits the preview flags, there is no preview-specific send path in the WhatsApp plugin, and there are no matching code/history hits showing this was added on `main`.

## Evidence

- **Baileys socket options omit link-preview flags:** `createWaSocket()` calls `makeWASocket()` with auth, version, logger, browser, syncFullHistory, markOnlineOnConnect, agent, and fetchAgent, but no `generateHighQualityLinkPreview` or `linkPreviewImageThumbnailWidth`.
  - file: [extensions/whatsapp/src/session.ts:149](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/extensions/whatsapp/src/session.ts#L149)
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Current test coverage does not assert preview enablement:** The socket test inspects the `makeWASocket()` call and only asserts `printQRInTerminal: false` plus logger shape; there is no test covering link-preview options.
  - file: [extensions/whatsapp/src/session.test.ts:166](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/extensions/whatsapp/src/session.test.ts#L166)
