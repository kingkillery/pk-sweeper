## Summary

Current `main` still matches the reported contradiction: `sessions_spawn` exposes `streamTo` in the shared tool schema, but the execution path rejects `streamTo` unless `runtime === "acp"`. The same rejection is locked in by a unit test, so this is not implemented on `main` and should stay open.

## Evidence

- **Shared schema still exposes `streamTo`:** `SessionsSpawnToolSchema` includes `streamTo: optionalStringEnum(SESSIONS_SPAWN_ACP_STREAM_TARGETS)`, so the parameter is still advertised on the generic `sessions_spawn` tool surface.
  - file: [src/agents/tools/sessions-spawn-tool.ts:125](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/src/agents/tools/sessions-spawn-tool.ts#L125)
  - command: `nl -ba src/agents/tools/sessions-spawn-tool.ts | sed -n '100,130p'`
  - sha: [7875092f4ddc](https://github.com/openclaw/openclaw/commit/7875092f4ddc6f2dbe218bea8506c9ac1da87c39)
- **Execution still rejects subagent use:** The tool normalizes `streamTo`, then immediately returns `streamTo is only supported for runtime=acp; got runtime=${runtime}` when `runtime !== "acp"`. This is the exact failure described in the issue.
  - file: [src/agents/tools/sessions-spawn-tool.ts:229](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/src/agents/tools/sessions-spawn-tool.ts#L229)
