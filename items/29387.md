## Summary

Keep open. Current main documents that bootstrap files belong in each agent workspace, and that is the right workaround, but the reported behavior is not fixed: bootstrap resolution still loads only `workspaceDir`, there is no `agentDir` merge/override, and I found no startup warning for bootstrap-named files placed under `agentDir`. The linked fixes in the discussion (#29450 warning-only and #29545 agentDir override) are both recorded in the provided GitHub context as closed and unmerged, and their SHAs are not present in this checkout.
