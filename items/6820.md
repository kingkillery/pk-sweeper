## Summary

Keep open. Current `main` still excludes `openai-codex/gpt-5.2` from the OpenAI Codex provider's `xhigh` thinking profile, so the reported mismatch is not implemented away on `main`.

## Evidence

- **Codex xhigh list still omits base GPT-5.2:** `OPENAI_CODEX_XHIGH_MODEL_IDS` includes `gpt-5.2-codex` but not `gpt-5.2`, while `OPENAI_CODEX_MODERN_MODEL_IDS` does include `gpt-5.2`.
  - file: [extensions/openai/openai-codex-provider.ts:101](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/extensions/openai/openai-codex-provider.ts#L101)
  - command: `nl -ba extensions/openai/openai-codex-provider.ts | sed -n '96,122p'`
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Thinking profile gates xhigh off that list:** The OpenAI Codex provider only appends `{ id: "xhigh" }` when `matchesExactOrPrefix(modelId, OPENAI_CODEX_XHIGH_MODEL_IDS)` is true, so `openai-codex/gpt-5.2` will not advertise `xhigh` from this provider hook.
  - file: [extensions/openai/openai-codex-provider.ts:451](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/extensions/openai/openai-codex-provider.ts#L451)
