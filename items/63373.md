## Summary

Current `main` has partial hardening for transcript-leak cases, but the evidence is not strong enough to close #63373. OpenClaw now strips XML tool-call payloads and the `[Tool Call: ...]` / `[Tool Result ...]` downgraded formats from visible assistant text, and it separately hid synthetic transcript-repair tool results for UI history reloads. However, the exact bracketed `[TOOL_CALL] ... [/TOOL_CALL]` form reported in this issue is not handled or tested on `main`, and there is no clear proof that Telegram/channel delivery now blocks that exact corruption output before send.
