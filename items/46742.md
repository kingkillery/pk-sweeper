## Summary

Current `main` already implements the requested behavior: snapshot/read-only plugin loads no longer replace active runtime state, and the gateway config schema path now uses manifest metadata instead of booting plugin runtime.

Closing from this Codex review because current `main` already covers the requested fix. The loader now treats read-only loads as explicit snapshots: `src/plugins/loader.ts:1830` requires `activate:false` loads to also use `cache:false`, and `src/plugins/loader.ts:1890` skips clearing global plugin state unless the load is activating.
