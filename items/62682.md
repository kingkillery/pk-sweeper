## Summary

Keep open. Current `main` still lacks the terminal-abort behavior this PR proposes: timeout aborts are normalized into retryable failover, HTTP client disconnects still abort without a distinguishing reason, and the main fallback entrypoints do not pass an abort signal into `runWithModelFallback`. The PR discussion shows earlier review concerns were addressed, but the underlying behavior is not present on `c997a9f9788e0a96e8eab05e7ae94959e76cdb4a`.

## Evidence
