## Summary

Keep open. Current main has a visible New session control, but it is wired to the existing `/new` chat reset flow, not the PR/linked issue’s requested Control UI workflow for creating a separate named session via `sessions.create` and switching to it. The PR discussion also contains unresolved review concerns about the proposed modal implementation, so this is not a conservative cleanup close.
