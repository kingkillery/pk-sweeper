## Summary

Keep open. Current main still rejects Feishu webhook requests with missing signature headers before parsing `url_verification` payloads, and the existing e2e test explicitly expects unsigned challenge requests to return `401 Invalid signature`. The PR’s requested behavior is therefore not implemented or superseded in this checkout.

## Evidence

- **Current handler rejects before challenge parsing:** `monitorWebhook` validates the Feishu webhook signature before JSON parsing or `Lark.generateChallenge`; invalid or missing signatures return `401 Invalid signature`, so an unsigned token-bearing `url_verification` challenge cannot reach challenge handling.
