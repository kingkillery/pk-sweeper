## Summary

Keep open. Current `main` already preserves forum-topic threading for auth/group-policy rejection replies, but it still does not register Telegram commands for `all_group_chats`, still clears only the default command scope when native commands are disabled, and still sends plugin-command `"Command not found."` replies without thread params. That means the PR is only partially superseded, not implemented on `main`.
