## Summary

Keep open. Current `main` still does not implement the PR's claimed Discord markdown-preservation behavior across chunk boundaries: the Discord chunker is fence-aware plus a special-case reasoning-italics rebalance, the core markdown chunker is fence-only, and table conversion still hardcodes an empty blockquote prefix. This is neither implemented on `main` nor obsolete.
