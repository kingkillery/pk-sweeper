## Summary

Keep open. Current `main` does not implement channel/topic delivery for `sessions_spawn({ runtime: "acp" })`. The tool still rejects channel-delivery fields, docs still say to use `message`/`sessions_send` instead, and ACP run-mode delivery is intentionally routed back through the parent announce path rather than directly to a channel/topic. The existing session-binding adapter infrastructure does make the request coherent, but it has not been wired into `sessions_spawn` as proposed.
