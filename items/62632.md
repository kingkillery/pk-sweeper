## Summary

Keep open. Current `main` has partial repair for stale absolute `sessionFile` transcript paths, but it still persists and reuses `spawnedWorkspaceDir` as an arbitrary absolute string, then feeds that directory into workspace creation with `mkdir(..., { recursive: true })`. I did not find a startup migration or `doctor` check that rewrites or flags stale persisted workspace paths from Docker state copies. The maintainer triage comments on the issue already called this a real bug, and the current code still matches that assessment.
