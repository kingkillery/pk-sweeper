## Summary

Keep open. Current `main` has adjacent monitoring for channel health and cron failure alerts, but it does not expose the blocker API (`sessions.activity.get/list` or activity state-change events) and does not implement a gateway-side session watchdog/alert pipeline for stale runs, waiting states, or child-session convergence.

## Evidence

- **Gateway session RPC surface lacks any activity methods:** `src/gateway/server-methods-list.ts` enumerates the session RPCs on current `main` and includes `sessions.list`, `sessions.subscribe`, `sessions.messages.subscribe`, `sessions.preview`, and mutation methods, but nothing like `sessions.activity.get` or `sessions.activity.list`.
  - file: [src/gateway/server-methods-list.ts:77](https://github.com/openclaw/openclaw/blob/84fc00afdacbbb67e39fc1341242eedad09243df/src/gateway/server-methods-list.ts#L77)
  - command: `nl -ba src/gateway/server-methods-list.ts | sed -n '70,110p'`
  - sha: [84fc00afdacb](https://github.com/openclaw/openclaw/commit/84fc00afdacbbb67e39fc1341242eedad09243df)
- **Session protocol schema has no activity or attention-state contract:** `src/gateway/protocol/schema/sessions.ts` defines list/resolve/create/send/abort/patch/reset/delete/compaction schemas and session row fields like `spawnedBy` and `parentSessionKey`, but no session-activity schema, no waiting-state/attention fields, and no activity query methods.
  - file: [src/gateway/protocol/schema/sessions.ts:38](https://github.com/openclaw/openclaw/blob/84fc00afdacbbb67e39fc1341242eedad09243df/src/gateway/protocol/schema/sessions.ts#L38)

- **Live session events are limited to transcript and lifecycle changes:** `src/gateway/server-session-events.ts` only broadcasts `session.message` and `sessions.changed`. The payload mirrors session row metadata, but there is no separate activity/statechange event stream matching the issue's dependency on `sessions.activity` updates.
  - file: [src/gateway/server-session-events.ts:83](https://github.com/openclaw/openclaw/blob/84fc00afdacbbb67e39fc1341242eedad09243df/src/gateway/server-session-events.ts#L83)

- **Existing built-in watchdog is for channel health, not session activity:** `src/gateway/channel-health-monitor.ts` implements a background monitor that evaluates channel transport health and restarts unhealthy channel accounts. It does not inspect session waiting states, child-session trees, or emit operator alerts for stuck sessions.

There is no documented operator-facing session watchdog, session alerting config, or `sessions.activity` surface.
