## Summary

Keep open. Current `main` at `6f72b74cecf8d1fec5c147ae932161bf0f37a68b` still nulls `snapshot.raw` in the structured-redaction fallback path, and the Control UI still interprets `raw: null` as disabling Raw mode and forcing a switch back to form mode. That matches the reported regression rather than showing an implemented fix on `main`.

## Evidence

- **Snapshot fallback still clears raw text:** `redactConfigSnapshot` computes `redactedRaw`, calls `shouldFallbackToStructuredRawRedaction(...)`, and then explicitly sets `redactedRaw = null` before returning `raw: redactedRaw`. This is the exact behavior the issue reports.
  - file: [src/config/redact-snapshot.ts:442](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/src/config/redact-snapshot.ts#L442)
  - command: `nl -ba src/config/redact-snapshot.ts | sed -n '420,467p'`
  - sha: [6f72b74cecf8](https://github.com/openclaw/openclaw/commit/6f72b74cecf8d1fec5c147ae932161bf0f37a68b)
- **Fallback predicate is about round-trip safety, not displayability:** `shouldFallbackToStructuredRawRedaction` only checks whether the redacted raw text can be parsed and restored back to the original config. A `true` result means unsafe raw editing/round-trip, not that the redacted string cannot be shown read-only.
  - file: [src/config/redact-snapshot.raw.ts:21](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/src/config/redact-snapshot.raw.ts#L21)
  - command: `nl -ba src/config/redact-snapshot.raw.ts | sed -n '1,35p'`
  - sha: [6f72b74cecf8](https://github.com/openclaw/openclaw/commit/6f72b74cecf8d1fec5c147ae932161bf0f37a68b)
- **Control UI still forces form mode when raw is missing:** `applyConfigSnapshot` sets `rawAvailable` from `typeof snapshot.raw === "string"`; when false and the UI is in raw mode, it forcibly switches `state.configFormMode` back to `form`.
  - file: [ui/src/ui/controllers/config.ts:79](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/ui/src/ui/controllers/config.ts#L79)
  - command: `nl -ba ui/src/ui/controllers/config.ts | sed -n '79,98p'`
  - sha: [6f72b74cecf8](https://github.com/openclaw/openclaw/commit/6f72b74cecf8d1fec5c147ae932161bf0f37a68b)
- **Current tests codify the forced fallback behavior:** Controller and browser tests still assert that `raw: null` forces form mode and shows `Raw mode disabled (snapshot cannot safely round-trip raw text).` That indicates the reported behavior is still expected on current `main`, not already fixed.
  - file: [ui/src/ui/controllers/config.test.ts:115](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/ui/src/ui/controllers/config.test.ts#L115)
