## Summary

Keep open. The report is recent, author is not protected, and has no protected labels, but the available evidence does not prove current main fully fixes the exact KVM/user-bus failure path for `openclaw gateway start`. The current docs and related prior fixes show the canonical service process should be foreground `openclaw gateway`, and older `not-found` handling was fixed, but this issue reports OpenClaw 2026.4.1 failing on `Failed to connect to bus: 找不到介质`, which is a distinct systemd user-bus availability failure.
