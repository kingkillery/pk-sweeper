## Summary

Keep PR #59642 open. The linked regression is not implemented on current main: legacy flat auth data is only read from legacy auth.json, not from a flat auth-profiles.json at the same path. The PR discussion also shows useful but unresolved review concerns on the PR head, so it should not be auto-closed as implemented or stale cleanup.

## Evidence

- **Current main does not parse same-file legacy auth-profiles.json:** loadAuthProfileStoreForAgent first tries loadPersistedAuthProfileStore(agentDir), then falls through to inheritance and the separate legacy auth.json migration path. There is no fallback that parses resolveAuthStorePath(agentDir) as a legacy flat store after canonical parsing fails.
  - file: [src/agents/auth-profiles/store.ts:189](https://github.com/openclaw/openclaw/blob/b6af40f1f11e08d3c73d79c678351306bef4acbe/src/agents/auth-profiles/store.ts#L189)
  - command: `nl -ba src/agents/auth-profiles/store.ts | sed -n '1,280p'`
  - sha: [b6af40f1f11e](https://github.com/openclaw/openclaw/commit/b6af40f1f11e08d3c73d79c678351306bef4acbe)
- **Canonical parser rejects flat auth-profiles.json:** loadPersistedAuthProfileStore reads resolveAuthStorePath(agentDir) and calls coercePersistedAuthProfileStore; coercePersistedAuthProfileStore returns null unless the raw object has a profiles object, so the flat example from #59629 would not load here.
  - file: [src/agents/auth-profiles/persisted.ts:482](https://github.com/openclaw/openclaw/blob/b6af40f1f11e08d3c73d79c678351306bef4acbe/src/agents/auth-profiles/persisted.ts#L482)

The path constants define AUTH_PROFILE_FILENAME as auth-profiles.json and LEGACY_AUTH_FILENAME as auth.json, confirming this fallback does not handle a flat auth-profiles.json file.
