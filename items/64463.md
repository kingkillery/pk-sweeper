## Summary

Keep open. Current `main` exposes session reset, maintenance, and per-run timeout controls, but it does not define or enforce `session.maxDurationMinutes` or `session.maxTokensPerSession`. The persisted session usage fields are last-run/context snapshots rather than cumulative per-session budgets, so the requested kill-switch behavior is still missing.
