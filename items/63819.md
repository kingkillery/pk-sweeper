## Summary

Keep open. Current `main` still has the same lifecycle-error cleanup gap described in #63819: a non-error follow-up event can cancel the delayed terminal-error finalization without recording a terminal session state, leaving the session at `running`. I did not find a shipped or on-`main` fix for that path.
