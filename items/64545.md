## Summary

Keep open. Current `main` and the latest shipped tag `v2026.4.23` still derive the Mattermost interaction signing secret from the bot token using a public constant, and the docs/tests explicitly describe that behavior. Source-IP allowlisting narrows exploitability, but it does not disprove the reported signing weakness.

## Evidence

## Risks / Open Questions

- The report's statement that the HMAC check is the 'sole access control' is too broad; current code also enforces callback source-IP checks, and default non-loopback deployments fail closed to loopback-only sources until `allowedSourceIps` is configured.
