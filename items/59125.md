## Summary

Keep issue #59125 open. The linked PR discussion identifies a plausible targeted fix, but current main still appears to return invalid non-null approval modes unchanged at the forwarding consumption point, so the reported invalid `approvals.exec.mode` behavior is not implemented on main with high confidence.

## Evidence

- **current-main-still-passes-invalid-mode:** Current `src/infra/exec-approval-forwarder.ts` defines `DEFAULT_MODE = "session"`, but `normalizeMode(mode)` returns `mode ?? DEFAULT_MODE` without checking the known mode set. `resolveForwardTargets` then only handles `session`, `targets`, and `both`, so an invalid non-null mode can still skip both routing branches and produce no forwarding targets.
