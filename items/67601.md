## Summary

Current main already implements the user-facing fix behind PR #67601. The transport/session metadata path reported in #67442 is blocked by shipped session-envelope stripping, non-inline default Dreaming storage, and existing promotion filters for dreaming artifacts. The PR's proposed patch is now obsolete and also carries stale/unrelated changes outside the memory-core fix surface.

## Best Possible Solution

Close PR #67601 as obsolete and keep the shipped main implementation. Any remaining staged-dream-candidate leakage should be handled through the narrower canonical follow-up #68774 rather than merging this older broad patch with unrelated workflow changes.
