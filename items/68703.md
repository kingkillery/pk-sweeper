## Summary

Keep open. Current `main` still appears to allow Discord guild-admin actions to run without requester authorization checks: the generic dispatcher only enforces trusted-sender requirements when a channel plugin opts in, the Discord adapter does not opt in for these actions, the guild-admin action path does not forward `requesterSenderId` except for moderation, and the guild runtime performs role/channel/event mutations behind feature gates only. The issue timeline also points to an open follow-up PR (#68716), so this does not look obsolete or already resolved.

## Evidence

- **Trusted-sender guard is opt-in at dispatch time:** `dispatchChannelMessageAction` throws only when `plugin.actions.requiresTrustedRequesterSender(...)` returns true. Without a plugin hook, tool-driven actions proceed directly to `handleAction`.
