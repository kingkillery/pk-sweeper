## Summary

Keep open. Current `main` still has the split the report describes: Telegram multi-account prompt/topic/skill lookup reloads per-account config where root `channels.telegram.groups` is suppressed, while group policy and `requireMention` still fall back to the channel-level groups map. The suppression behavior is explicitly locked in tests, there is no compensating warning path in the inspected Telegram docs/runtime files, and the issue already has a linked open PR (#64650).

## Evidence

- **Multi-account account merge drops root groups:** `mergeTelegramAccountConfig()` explicitly suppresses channel-level `groups` when 2+ Telegram accounts are configured: `const groups = account.groups ?? (isMultiAccount ? undefined : channelGroups);`. That means prompt/topic/skill lookup via resolved account config cannot see root `channels.telegram.groups` in multi-account mode.
  - file: [extensions/telegram/src/account-config.ts:31](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/telegram/src/account-config.ts#L31)
  - command: `nl -ba extensions/telegram/src/account-config.ts | sed -n '1,120p'`
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **Prompt/topic lookup uses the merged account config:** `resolveTelegramGroupConfig()` reloads config through `resolveTelegramAccount(...).config` and then reads `freshTelegramCfg.groups`, so it inherits the suppression above for `systemPrompt`, `skills`, and `topics` resolution.
  - file: [extensions/telegram/src/bot-core.ts:505](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/telegram/src/bot-core.ts#L505)
  - command: `nl -ba extensions/telegram/src/bot-core.ts | sed -n '495,560p'`
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **Group policy still falls back to root groups:** Shared group-policy resolution still does `return accountGroups ?? channelConfig.groups;`, so account-scoped group policy and `requireMention` continue to see root `channels.telegram.groups` even when multi-account prompt lookup does not.
  - file: [src/config/group-policy.ts:304](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/config/group-policy.ts#L304)

  - file: [extensions/telegram/src/accounts.test.ts:471](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/telegram/src/accounts.test.ts#L471)
