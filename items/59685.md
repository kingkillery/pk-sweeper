## Summary

Keep open. Current main does not appear to contain the PR's core behavior: `doctor.memory.status` still reports only embedding health, `status` has no gateway memory fallback path, and `ensureMemoryRuntime` still goes directly to config-based plugin loading instead of first reusing active gateway plugin state. The PR also has concrete supporting reproduction evidence from another plugin (`memory-qdrant`) and resolved review comments for the main regressions raised during review.

## Evidence

- **Current main lacks split runtime health in doctor payload:** On current main, `doctor.memory.status` returns `embedding` health only in both success and failure paths; it does not include the PR's proposed `runtime: { ok: true/false }` field, so status cannot distinguish a healthy gateway runtime from an embedding probe failure.
