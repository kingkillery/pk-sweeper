## Summary

Keep open. Current main does not implement the PR's requested behavior: status still reports an enabled memory plugin as unavailable when the local memory snapshot is missing, and the gateway doctor payload still conflates runtime health with embedding probe health.

## Evidence

- **Current memory runtime still skips active-registry-first reuse:** `ensureMemoryRuntime` checks `getMemoryRuntime()` and then immediately calls `resolveRuntimePluginRegistry` with config-derived runtime load options. It does not first reuse the already-active gateway registry, which is the main fix described by the PR and related issue.
