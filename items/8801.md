## Summary

Current `main` already implements the requested post-start gateway lifecycle hook: internal hooks can listen to `gateway:startup` after channels start, and plugins can use the typed public `gateway_start` hook with startup context and cron access. This shipped in `v2026.4.22`.

## Close Comment

Closing this as implemented on current `main` after a Codex review. OpenClaw already has the requested post-start hook surface: internal hooks can listen to `gateway:startup` after channels start in `src/gateway/server-startup-post-attach.ts:268`, and plugins can use the public typed `gateway_start` hook with startup config, workspace dir, and cron access in `src/gateway/server-startup-post-attach.ts:503` plus `src/plugins/hooks.ts:1038`. The user docs also describe `gateway:startup` and the bundled `boot-md` startup hook in `docs/automation/hooks.md:46` and `docs/cli/hooks.md:328`. This behavior is documented in the `v2026.4.22` release notes at `CHANGELOG.md:202`, so there is not an open implementation gap for this request anymore.
