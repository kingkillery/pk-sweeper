## Summary

Keep open. Current main still rejects plain `OSTYPE=linux` and `OSTYPE=linux-musl` in `scripts/install.sh`, while the docs advertise `install.sh` as the macOS/Linux/WSL installer. The PR's one-line broadening from `linux-gnu*` to `linux*` directly addresses the reported openSUSE compatibility gap and is not implemented or superseded on main.

## Best Possible Solution

Keep the PR open or recreate its one-line fix on a maintainer branch: change `detect_os_or_die()` to accept `linux*`, then run the changed installer gates before landing. A small regression check for `OSTYPE=linux` and `OSTYPE=linux-musl` would make the compatibility contract explicit.
