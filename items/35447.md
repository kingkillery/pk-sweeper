## Summary

Keep open. Current main still lacks per-agent/session caller context for cron list, update, remove, and run paths. The cron tool does populate agentId/sessionKey when adding jobs, which confirms the issue's premise, but the service contract and gateway handlers still operate by global job ID without ownership checks. Related PRs #36517 and #49174 attempted this fix but are closed/unmerged; #49175 only tracks a narrower cron.runs leak and #49183 is a broader tool-policy RFC, so neither supersedes the original request.
