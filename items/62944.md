## Summary

Keep open. Current `main` still hardcodes `timeoutMs: 30_000` inside the image tool, so the tool bypasses the configured image-understanding timeout. The report's config path is slightly stale (`tools.media.models.timeoutSeconds` vs current `tools.media.image.timeoutSeconds`), but the underlying bug is still present on `main` and there is no evidence in the provided issue context of a landed fix.

## Evidence

- **Image tool still hardcodes 30s timeout:** `runImagePrompt()` passes `timeoutMs: 30_000` to `describeImages()` and `describeImage()` on all image paths, instead of reading config.
  - file: [src/agents/tools/image-tool.ts:221](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/agents/tools/image-tool.ts#L221)

  - file: [src/media-understanding/runner.entries.ts:363](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/media-understanding/runner.entries.ts#L363)
