## Summary

Keep open. Current `main` already localizes the Control UI, but Telegram built-in command names are still defined in English and the documented Telegram customization surface cannot override those native commands. This is a real, still-open feature request rather than conservative cleanup material.

## Evidence

- **Control UI i18n already exists:** The docs explicitly say the Control UI localizes from browser locale, exposes a language picker, and supports multiple locales including `fr`, `es`, and `de`. That covers the Web UI portion of the request, but not Telegram commands.
  - file: [docs/web/control-ui.md:96](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/docs/web/control-ui.md#L96)

  - file: [docs/channels/telegram.md:315](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/docs/channels/telegram.md#L315)

## Risks / Open Questions

- The request mixes two scopes: Web UI i18n is already present, but Telegram built-in command localization is not. Closing the whole issue would lose the still-unimplemented Telegram portion.
- This may ultimately belong in plugin/channel-owned work, but current `main` does not provide a complete user-facing implementation or documented workaround for localized built-in Telegram commands.
