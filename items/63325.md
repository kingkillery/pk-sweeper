## Summary

Keep open. Current `main` still has the empty-`delta` SSE drop path on `/v1/chat/completions`, so the reported content-loss bug is not implemented/fixed in the checked-out code.

## Evidence

- **OpenAI SSE handler still drops empty deltas:** `src/gateway/openai-http.ts:685-703` resolves assistant chunk text, then returns early on falsy content. That means a replace event with `delta: ""` is ignored and never emitted to SSE clients.
  - file: [src/gateway/openai-http.ts:685](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/gateway/openai-http.ts#L685)
  - command: `nl -ba src/gateway/openai-http.ts | sed -n '680,710p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Delta resolver prefers empty `delta` over full `text`:** `src/gateway/agent-event-assistant-text.ts:3-6` returns `evt.data.delta` whenever it is a string, even if it is the empty string, and only falls back to `evt.data.text` when `delta` is not a string.
  - file: [src/gateway/agent-event-assistant-text.ts:3](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/gateway/agent-event-assistant-text.ts#L3)
  - command: `nl -ba src/gateway/agent-event-assistant-text.ts`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Upstream assistant stream events can legitimately be replace events:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:325-345` defines assistant stream payloads with both `text` and `delta`, plus optional `replace`. The current tests explicitly expect a replace event shaped as `{ text: "Done.", delta: "", replace: true }` at `src/agents/pi-embedded-subscribe.handlers.messages.test.ts:715-723`.
  - file: [src/agents/pi-embedded-subscribe.handlers.messages.test.ts:715](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/agents/pi-embedded-subscribe.handlers.messages.test.ts#L715)
  - command: `nl -ba src/agents/pi-embedded-subscribe.handlers.messages.ts | sed -n '325,345p'; nl -ba src/agents/pi-embedded-subscribe.handlers.messages.test.ts | sed -n '712,730p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Related PR identified the same unfixed gateway path:** Issue timeline cross-references PR `#63894` (`a046e7f`), whose summary says `/v1/chat/completions` SSE dropped content when `<final>`/`<think>` tags crossed chunk boundaries because the handler used raw `delta` and discarded replace events. That PR was closed rather than merged, which matches the unfixed code still present on `main`.
  - command: `web open https://github.com/openclaw/openclaw/pull/63894`
  - sha: [a046e7fbfd9b](https://github.com/openclaw/openclaw/commit/a046e7fbfd9b08d27730b86cbd400299c0506b66)
