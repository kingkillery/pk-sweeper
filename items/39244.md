## Summary

Keep open. Current `main` does not implement the proposed bubblewrap namespace sandbox or `tools.exec.nsSandbox` config, so this PR is not obsolete or already shipped. At the same time, current `main` already hardens `safeBins` with trusted-path checks and resolved-executable enforcement, so the PR body’s absolute-path-bypass rationale looks weaker than when the PR was opened; that argues for refreshed review, not closure.

## Evidence

- **No bwrap implementation on current main:** A repo search for `nsSandbox`, `getBwrapPath`, `buildBwrapArgs`, and `exec-bwrap-sandbox` returned no matches under `src`, so current `main` has no bubblewrap sandbox module or wiring.
