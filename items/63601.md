## Summary

Keep open. On current `main` (`86f8c826e20a01d1bba67c0ba6e73182fbf0455f`), the `message:received` internal-hook path still calls `fireAndForgetHook(triggerInternalHook(...))` and never inspects or routes the resulting hook event’s `messages` array, so `event.messages.push(...)` replies are still dropped. The two linked fix PRs described exactly this gap, but both PRs are closed rather than merged. ([]())
