## Summary

Keep open. Current `main` does not implement channel-body external-content wrapping, so this is not `implemented_on_main`. The PR is still useful and its second commit addresses the Greptile `summaryLine`/thread-context feedback, but the remaining trust-boundary logic still needs maintainer review because it derives wrapping from `OriginatingChannel ?? Provider` and only treats `node`, `cli`, and `exec-event` as trusted.
