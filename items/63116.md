## Summary

Current `main` does not show a delivery-mirror-specific fix for transcript growth. OpenClaw still mirrors outbound assistant text back into the same session transcript as `openclaw/delivery-mirror`, while the replay/compaction pipeline does not strip those entries before building model context. The only delivery-mirror special-casing I found is for UI/usage hygiene, which is not strong evidence that the underlying overflow/loop report is resolved.
