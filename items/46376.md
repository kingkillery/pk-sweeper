## Summary

Do not close. Current `main` added a disconnected-abort queue, but it still drops active-run tracking after reconnect. `ui/src/ui/app-gateway.ts:315` resets `chatRunId` to `null` on `hello`, and the chat view only shows Stop when `chatRunId` is truthy in `ui/src/ui/app-render.ts:2264`. That means the core reported regression, “Stop button disappears after gateway restart,” still matches current code. Main does now have explicit interrupt paths for background/subagent runs (`/stop`, `/kill`, `/subagents kill`, `sessions.abort`, and `POST /sessions/:sessionKey/kill`), so the “zero interrupt mechanism” part is only partially obsolete, not fully resolved.

## Evidence

That loses the state the Stop button depends on.
