## Summary

Keep open. Current `main` still hard-fails `sessions_spawn` when `runtime` is `subagent` and `streamTo: "parent"` is present, which is the exact behavior this PR is trying to relax for strict/schema-materializing callers. I did not find evidence that the fix is already on `main` or that the report is obsolete.
