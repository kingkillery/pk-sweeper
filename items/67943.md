## Summary

Current main still has the reported stale-lineage path. `initSessionState` preserves `spawnedBy`, `parentSessionKey`, `spawnDepth`, `subagentRole`, and `subagentControlScope` from a reused session entry, and `agentCommand` can still pass `sessionEntry.spawnedBy` into the agent run for an ordinary top-level session. PR #67946 is an exact linked fix attempt, but it is open, unmerged, and currently dirty, so the bug should remain tracked rather than be closed as implemented.

## Best Possible Solution

Keep #67943 open until the stale-lineage fix lands on main. The likely fix belongs in `src/auto-reply/reply/session.ts` and `src/agents/agent-command.ts`: preserve subagent lineage only for actual subagent/ACP sessions, keep normal parent-thread metadata such as `parentSessionKey`/`forkedFromParent` where needed, and add regression coverage proving a normal top-level session with stale stored lineage can still use `sessions_spawn` without a false depth-limit rejection.
