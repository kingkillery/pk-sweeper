## Summary

Keep open. Current `main` still encodes the reported behavior: Android onboarding only exposes `Finish` when both operator and node are connected, while runtime status explicitly treats `Connected (operator offline)` as a connected state. The issue discussion is consistent with the code, and the linked PR #61269 remains unmerged.
