## Summary

Keep open. Current main still lacks per-agent dreaming control, and this PR is a focused implementation attempt for the still-open #67413 feature gap rather than obsolete cleanup. The branch also still needs maintainer/code-review attention around the all-agents-disabled fallback before it would be safe to land.

## Best Possible Solution

Keep the PR open for maintainer review. The likely product path is to finish the per-agent dreaming contract in config types, strict Zod schema, generated schema/help, docs, and resolveMemoryDreamingWorkspaces, including a regression test for all agents disabled and a clear decision on whether per-agent schedules remain in #67413 as follow-up.
