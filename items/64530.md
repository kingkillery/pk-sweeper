## Summary

Keep open. Current `main` still relies on prompt-time/in-memory transcript compensation rather than append-only reopen/startup repair, and restart recovery explicitly refuses assistant-tail transcripts. Orphan trailing user turns are merged into the next prompt and removed from the session leaf instead of being preserved with an explicit recovery row.

## Evidence

- **Restart recovery only resumes `user|tool|toolResult` tails:** `isMainSessionResumable()` returns true only when the last meaningful transcript message is `user`, `tool`, or `toolResult`; an interrupted assistant tail with unresolved tool calls is not resumable.

  - command: `nl -ba src/agents/main-session-restart-recovery.ts | sed -n '1,260p'`
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **Assistant-tail sessions are marked failed, not repaired on reopen:** When restart recovery sees a marked session whose transcript tail is not resumable, it calls `markSessionFailed(...)` with reason `transcript tail is not resumable` instead of appending synthetic `toolResult` recovery rows.
  - file: [src/agents/main-session-restart-recovery.ts:219](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/agents/main-session-restart-recovery.ts#L219)

## Risks / Open Questions

- Users can still hit interrupted assistant-tail sessions that restart recovery marks failed instead of explicitly repairing at reopen.
- Current prompt-time compensation can mask persisted transcript corruption without making the transcript itself deterministic and append-only.
- Orphan trailing user handling still mutates active session state by removing the leaf, which does not match the issue's requested preservation semantics.
