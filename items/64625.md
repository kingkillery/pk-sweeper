## Summary

Current `main` still appears to have the reported Slack custom-invoke bug. The Slack plugin always uses the custom `invoke` path, that path still forwards only raw `toolContext`, and the runtime still expects `mediaLocalRoots`/`mediaReadFile` on the action context for `uploadFile` and media sends. I did not find evidence of a landed fix between the latest release and `main`, so this should stay open.

## Evidence
