## Summary

Keep #67379 open. Current main still drops agent-prefixed subagent session keys during QMD scope normalization, so the default deny scope has no channel/chatType to match and QMD search is skipped for those sessions. The issue is recent, has no protected-label or maintainer-author auto-close blocker, and the provided related items do not supersede this specific QMD subagent scope bug.

## Best Possible Solution

Keep the issue open and fix the QMD scope contract for spawned subagent session keys. The narrowest product path is to make `agent:<id>:subagent:<uuid>` inherit the parent/root agent session scope or otherwise classify it explicitly, then add regression coverage in `src/memory-host-sdk/host/qmd-scope.test.ts` and the mirrored package test if that surface remains active. If the default-scope policy changes in #70297, handle that as a separate policy decision rather than using it to close this normalization bug.
