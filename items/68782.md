## Summary

Keep open. Current `main` already addressed part of the reported pain by removing eager bundled runtime-dependency installation and by narrowing runtime activation with allow/deny plus startup planning, but the actual request in #68782 is still not implemented: `openclaw setup` has no selective channel/plugin install UX, discovery still scans bundled/global/workspace roots, and the bundled extension set is still shipped as part of OpenClaw.

## Evidence

- **Bundled postinstall is no longer eager:** The bundled-plugin postinstall path now explicitly says core package install must not install every bundled extension dependency unless the legacy eager-install escape hatch is enabled; used extensions are repaired later via `openclaw doctor --fix`. This means the issue body's postinstall description is already outdated on current `main`.
  - file: [scripts/postinstall-bundled-plugins.mjs:3](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/scripts/postinstall-bundled-plugins.mjs#L3)
  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)
- **User docs confirm on-demand runtime dependency repair:** The public plugin docs say packaged installs do not eagerly install every bundled plugin runtime dependency tree and only repair the declared runtime dependencies for active bundled plugins before import.
  - file: [docs/tools/plugin.md:64](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/docs/tools/plugin.md#L64)
  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)
- **Selective install/setup UX is absent:** `openclaw setup` is documented only with workspace and onboarding flags. There is no documented `--channels`, plugin picker, or other selective install surface matching the issue proposal.
  - file: [docs/cli/setup.md:20](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/docs/cli/setup.md#L20)
  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)
- **Runtime allow/deny gating already exists:** Plugins config already exposes `allow` and `deny`, and activation logic disables plugins that are not allowlisted once `plugins.allow` is populated. So the runtime strict-loading idea is partly already present.
  - file: [src/plugins/config-activation-shared.ts:188](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/src/plugins/config-activation-shared.ts#L188)
  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)
- **Discovery still scans standard plugin roots:** Current discovery still walks bundled and global plugin directories (plus workspace/config paths elsewhere in the function). That means the repo has not implemented the proposed 'skip auto-scanning undeclared plugins' behavior at discovery time.
  - file: [src/plugins/discovery.ts:1187](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/src/plugins/discovery.ts#L1187)
  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)
- **Startup is already narrower than the issue describes:** Gateway startup now filters configured channel plugins through allow/deny and activation checks before starting them. That partially reduces the startup-overhead claim, but it is not the same as install-time selective packaging.
  - file: [src/plugins/gateway-startup-plugin-ids.ts:118](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/src/plugins/gateway-startup-plugin-ids.ts#L118)
  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)
- **Bundled extension footprint is still shipped in-tree:** The repository still contains a large bundled extension set including `discord`, `feishu`, `telegram`, `whatsapp`, and many others, so selective artifact installation is not implemented at the packaging level.
  - command: `find extensions -maxdepth 1 -mindepth 1 -type d -printf '%f\n' | sort`
  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)
