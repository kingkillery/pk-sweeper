## Summary

Keep open. Current main exposes manual/provider-native thinking selection, but not the requested failure-count-based auto-escalation and revert behavior. The plugin-first project direction is real, yet the current public plugin seams do not provide a clean, deterministic per-run thinking override for this exact workflow, so a `clawhub` close would be premature.

## Evidence

- **Current behavior is manual `/think`, not failure-adaptive:** The thinking docs define explicit `/think` levels, provider-native `adaptive` where supported, and manual session/default resolution. They do not document any consecutive-failure escalation or revert-on-success behavior.
  - file: [docs/tools/thinking.md:10](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/docs/tools/thinking.md#L10)

- **Core run path only resolves explicit/default thinking:** `get-reply-run.ts` extracts an inline think hint, otherwise resolves the default thinking level, then validates/remaps unsupported levels. There is no counter for repeated failures, no escalation ladder, and no success-triggered reset in this path.
