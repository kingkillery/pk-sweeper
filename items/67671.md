## Summary

Keep open. Current main has a CLI `cron show <job-id>` helper, but it still pages through `cron.list` and filters client-side. The requested direct `cron.get` gateway RPC and agent/tool read path are not present, and the related PR #67677 is closed unmerged rather than a shipped canonical fix.

## Best Possible Solution

Add a minimal additive `cron.get` read path backed by `CronService.getJob`: protocol schema/validator, gateway handler, method registration and read-scope entry, clean not-found error, CLI `openclaw cron get <id>` or a `show` alias that uses the new RPC, cron agent-tool action support, targeted CLI/protocol/server/tool tests, and docs updates.
