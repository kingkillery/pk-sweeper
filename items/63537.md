## Summary

Keep open. Current `main` still reloads chat history for every terminal `final` chat event, including events with no `message`, so the reported `/new` and `/reset` behavior is not implemented on `main`. The PR also has unresolved review issues: it does not update the existing test that still expects reload-on-undefined, and the proposed `!payload.message` guard would also skip reload for other falsy malformed payloads.
