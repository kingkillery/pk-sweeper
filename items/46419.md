## Close Comment

Closing as `clawhub` based on this Codex review against current `main` (`56fe2aab9c87596515f2280c8060dc740ea64a88`). The current project direction keeps optional observability features in the plugin ecosystem rather than adding more core-only debug surfaces: `VISION.md:54-60` says optional capability should usually ship as plugins, and `docs/plugins/community.md:77-86` already points users to the official `@opik/opik-openclaw` trace plugin. Current public hooks already cover the lifecycle needed for this kind of integration: `llm_input` / `llm_output` / `agent_end` in `src/plugins/hook-types.ts:179-218`, plus compaction and tool hooks in `src/plugins/hook-types.ts:221-225` and `src/plugins/hook-types.ts:318-359`, with explicit conversation-access gating in `src/config/types.plugins.ts:6-10`. Since this PR adds a new core logger and env surface in `src/agents/*` instead of extending the plugin path, it does not fit the current core-scope bar. If richer tracing is still wanted, the better next step is an external plugin or a narrowly scoped public tracing seam proposal.
