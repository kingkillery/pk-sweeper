## Summary

Keep open. Current `main` still intentionally feeds queued system events back into the next agent prompt and still defaults exec-completion heartbeats to user-visible relay text when delivery is available. Webchat uses the default heartbeat visibility settings, which still ship with `showAlerts: true`, so I do not have high-confidence evidence that the reported web UI feedback loop is fixed on `main`.
