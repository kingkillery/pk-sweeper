## Summary

Keep open. Current main still does not expose a per-turn model call count in the `/usage` response footer: the formatter only renders input/output/cache/cost fields, the normalized usage/meta types have no call-count field, and the accumulator tracks aggregate and last-call token counts but not the number of model calls. The related PRs called out in the issue discussion (#34588, #34598, #38449) are useful prior implementation attempts, but the provided GitHub context says they were closed unmerged, so they do not supersede the still-missing feature.
