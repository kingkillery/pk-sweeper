## Summary

Current `main` still routes macOS Talk mode playback through local ElevenLabs/MLX/system synthesizers after fetching `talk.config`, while the docs still say Talk playback goes through gateway `talk.speak`. The issue remains valid and should stay open.

## Evidence

- **Talk docs still promise gateway `talk.speak` playback:** `docs/nodes/talk.md` says Talk mode step 4 is to speak via `talk.speak`, and the Notes section says the gateway resolves Talk playback through `talk.speak`.
