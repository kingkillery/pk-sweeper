## Summary

Keep open. Current main has related Plugin SDK hooks for `llm_output` and subagent lifecycle, but the documented internal hook system still does not expose turn-complete or subagent lifecycle events, and the available plugin payloads do not clearly satisfy the issue’s requested fields like cost, duration, task, and per-subagent token totals.

## Evidence

- **Internal hooks still do not list turn/subagent events:** The public internal-hooks docs enumerate only command, session compaction/patch, agent bootstrap, gateway startup, and message events. There is no documented `turn.complete`, `subagent.spawn`, or `subagent.complete` equivalent in the internal hook surface.
  - file: [docs/automation/hooks.md:34](https://github.com/openclaw/openclaw/blob/c0a7b6a510758645c5267734b984f4885a605b67/docs/automation/hooks.md#L34)
  - command: `nl -ba docs/automation/hooks.md | sed -n '24,70p'`
  - sha: [c0a7b6a51075](https://github.com/openclaw/openclaw/commit/c0a7b6a510758645c5267734b984f4885a605b67)
- **Internal hook event model is still coarse:** `InternalHookEventType` is limited to `command | session | agent | gateway | message`, which leaves no dedicated turn-completion or subagent lifecycle event family in the internal hook mechanism the issue references.
  - file: [src/hooks/internal-hook-types.ts:1](https://github.com/openclaw/openclaw/blob/c0a7b6a510758645c5267734b984f4885a605b67/src/hooks/internal-hook-types.ts#L1)
  - command: `nl -ba src/hooks/internal-hook-types.ts`
  - sha: [c0a7b6a51075](https://github.com/openclaw/openclaw/commit/c0a7b6a510758645c5267734b984f4885a605b67)
- **Plugin SDK has partial equivalents:** Current main defines plugin hooks `llm_input`, `llm_output`, `session_start`, `session_end`, `subagent_spawning`, `subagent_spawned`, and `subagent_ended`, so there is overlap with the request but on a different hook surface.
  - file: [src/plugins/hook-types.ts:56](https://github.com/openclaw/openclaw/blob/c0a7b6a510758645c5267734b984f4885a605b67/src/plugins/hook-types.ts#L56)
  - command: `nl -ba src/plugins/hook-types.ts | sed -n '50,120p'`
  - sha: [c0a7b6a51075](https://github.com/openclaw/openclaw/commit/c0a7b6a510758645c5267734b984f4885a605b67)

That is materially short of the issue’s proposed payloads.

## Risks / Open Questions

- The issue body references the older internal hook system (`gateway.startup`, `command.new`, `command.reset`), so treating plugin-hook support as a full fix would likely over-close it.
- If maintainers consider Plugin SDK hooks sufficient for this request, the remaining gap is still the missing requested fields like cost, task, duration, and per-subagent token accounting.
