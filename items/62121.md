## Summary

Keep open. Current main has adjacent fixes for phased commentary buffering and Telegram cron delivery collapsing to the final visible answer, but the code still relies on `phase: "commentary"` / `phase: "final_answer"` metadata or tagged reasoning blocks. There is no generic sanitizer for plain untagged assistant preamble text, cron payloads still do not expose `enforceFinalTag`, and the current DeepSeek-specific provider work is limited to V4 models rather than `deepseek-chat`.
