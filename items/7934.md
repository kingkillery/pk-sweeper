## Summary

Current `main` already supports plugin-driven model selection for agent runs. The legacy `before_agent_start` hook result includes `modelOverride`/`providerOverride`, and new work is directed to the explicit `before_model_resolve` hook.

## Close Comment

Closing as implemented on current `main`. A Codex review found that plugins can already control model selection through the legacy `before_agent_start` hook via `modelOverride` and `providerOverride` (`src/plugins/hook-before-agent-start.types.ts:14`, `src/plugins/hook-before-agent-start.types.ts:66`), and the runtime applies those values before model resolution (`src/agents/pi-embedded-runner/run/setup.ts:58`). Priority behavior is covered in `src/plugins/hooks.before-agent-start.test.ts:121`, and the docs now point new work to `before_model_resolve` while keeping `before_agent_start` as a compatibility path (`docs/plugins/architecture.md:93`). This also appears in the changelog under `2026.2.17` as `Plugins: add before_agent_start model/provider overrides before resolution.` (`CHANGELOG.md:4574`, `CHANGELOG.md:4608`). The exact proposed field name did not land, but the requested capability is already available on shipped OpenClaw releases through `modelOverride` or the newer `before_model_resolve` hook.
