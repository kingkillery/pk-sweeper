## Summary

Keep PR #60868 open. Current main has partial wrapped-error handling through `formatErrorMessage()`, but it does not implement the PR’s requested full `.error` / `.cause` chain message collector, and the affected overflow-classification paths still consume one formatted string rather than checking each chain segment. The PR also has substantive review discussion that should be resolved rather than auto-closed.
