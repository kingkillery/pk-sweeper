## Summary

Keep open. Current main still runs the git update flow before any gateway service restart/preflight, so a same-install gateway can remain live while `git rebase`/checkout and build replace `dist` in place. The PR is not obsolete: it targets a real remaining gap, though its latest auto-stop approach needs the existing review feedback addressed before merge.

## Best Possible Solution

Keep the PR open and ask for a revision that implements a conservative git-mode guard in `src/cli/update-cli/update-command.ts`: detect a same-install running gateway before in-place git checkout/build work, stop or block only when restart recovery is available, handle wrapper and direct dist entrypoints, and guarantee the service is restarted/restored on success, skip, and error. Add focused `src/cli/update-cli.test.ts` coverage for same-install absolute/relative/wrapper commands, different installs, `--no-restart`, dirty/skipped updates, and update failures.
