## Summary

Current main already archives reset/rollover transcripts and lets operators retain `*.reset.*` files via `session.maintenance.resetArchiveRetention`, but the compaction-truncation path still does not wire archival or retention when `truncateAfterCompaction` rewrites a session JSONL. Given the issue title, proposal, and related open PR #63161, the request is not implemented well enough to close conservatively.
