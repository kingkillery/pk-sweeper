## Summary

Keep open. Current main has overlapping live provider probing through `openclaw models status --probe`, including configured providers and models.json custom-provider credentials, but it does not expose the proposed `models custom-test` command and does not clearly cover the PR's stated behavior of verifying every configured custom model with a failing exit for failed verifications. The PR also has substantive unresolved review concerns, so it should be reworked or explicitly superseded by a maintainer-owned direction rather than auto-closed.

## Best Possible Solution

Keep the PR open for maintainer/product judgment. The best path is to rework the idea on top of the existing `models status --probe`/`runAuthProbes` runtime path, adding any missing all-custom-model target selection and failure-exit semantics if maintainers want that behavior, instead of adding a separate onboarding-helper-based probe command.
