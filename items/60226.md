## Summary

Current `main` does not implement non-image WebChat attachment injection. The shipped WebChat UI still only accepts `image/*`, and the Gateway attachment parser still drops non-image payloads before they can reach agent context. This report remains actionable rather than obsolete; the timeline’s later cross-reference to issue #70438 also suggests maintainers are still treating configurable non-image WebChat handling as open work.
