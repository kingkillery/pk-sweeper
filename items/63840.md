## Summary

Keep open. Current `main` does not implement the DM-root thread preservation described by the PR, and the latest review context still shows an unresolved correctness gap for D-prefixed DMs when Slack omits `channel_type`, so this is not safe for cleanup closure.

## Evidence

- **Current main still drops explicit thread targeting outside replyToMode=all:** `extensions/slack/src/threading.ts:20` computes `isThreadReply` as false when `thread_ts === ts`, and `extensions/slack/src/threading.ts:23` only sets `messageThreadId` for true thread replies or `replyToMode === "all"`. That means DM assistant root messages with `thread_ts === ts` still have no explicit `MessageThreadId` on current `main` for `off`/`first`/`batched`.
