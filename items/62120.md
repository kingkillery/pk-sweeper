## Summary

Keep open. Current `main` clearly supports WeChat via the external `@tencent-weixin/openclaw-weixin` plugin and still routes `openclaw channels login` through the CLI preaction/bootstrap path before handing off to plugin `auth.login`, but I did not find a core-side fix or changelog entry that specifically resolves the reported pre-QR hang on `2026.4.5`. The report is also recent and has concrete reproduction steps, so conservative maintainer cleanup should not close it yet.
