## Summary

Keep open. Current `main` has only narrow mobile offline queues: Android voice turns are buffered in an in-memory `ArrayDeque` and flushed on reconnect, and iOS queues only watch quick replies. There is no general, persistent offline command outbox for iOS/Android chat or voice commands, so the feature request is not implemented on `main`.

## Evidence

That is a reconnect-only in-memory queue, not a persistent general command outbox.

## Risks / Open Questions

- Android's existing offline voice queue appears process-local only; queued turns would be lost on app restart or crash.
- iOS currently has no equivalent general offline queue for chat or talk commands, so closing this as implemented would be a false positive.
