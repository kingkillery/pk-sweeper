## Summary

Keep open. Current main still treats skill env requirements as satisfied only by process env, OpenClaw skill config env, or the skill primary apiKey config; it does not appear to support arbitrary credential files such as ~/.config/ima/* or skill-owned setup-status hooks. The issue is recent, has no protected labels/maintainer author association, and remains a plausible core/API enhancement rather than a stale cleanup close.

## Evidence

- **Issue context:** Issue #59078 reports a false positive setup-needed state when ima-skill can read credentials from ~/.config/ima/client_id and ~/.config/ima/api_key, while its metadata declares requires.env for IMA_OPENAPI_CLIENTID and IMA_OPENAPI_APIKEY. No comments or timeline discussion beyond the enhancement label were provided.
  - sha: [44114328b4e7](https://github.com/openclaw/openclaw/commit/44114328b4e73074ac72126f1230ddbf9c1ea8b5)
- **Current env gate implementation:** shouldIncludeSkill delegates requires.env to evaluateRuntimeEligibility with hasEnv satisfied only by process.env[envName], skills.entries.<skillKey>.env[envName], or skills.entries.<skillKey>.apiKey when the metadata primaryEnv matches. There is no file probe or skill self-report hook in this path.
  - file: [src/agents/skills/config.ts:97](https://github.com/openclaw/openclaw/blob/44114328b4e73074ac72126f1230ddbf9c1ea8b5/src/agents/skills/config.ts#L97)
  - sha: [44114328b4e7](https://github.com/openclaw/openclaw/commit/44114328b4e73074ac72126f1230ddbf9c1ea8b5)
- **Current docs contract:** The skills docs describe requires.env as an env var that must exist or be provided in OpenClaw config, and requires.config as openclaw.json paths that must be truthy. That does not cover arbitrary external credential files such as ~/.config/ima/*.
  - file: [docs/tools/skills.md:378](https://github.com/openclaw/openclaw/blob/44114328b4e73074ac72126f1230ddbf9c1ea8b5/docs/tools/skills.md#L378)
  - sha: [44114328b4e7](https://github.com/openclaw/openclaw/commit/44114328b4e73074ac72126f1230ddbf9c1ea8b5)
- **Config override docs:** The documented workaround path is skills.entries.<key>.env or apiKey in ~/.openclaw/openclaw.json, injected for an agent run, not automatic discovery of a skill-specific credentials file.
  - file: [docs/tools/skills.md:449](https://github.com/openclaw/openclaw/blob/44114328b4e73074ac72126f1230ddbf9c1ea8b5/docs/tools/skills.md#L449)
  - sha: [44114328b4e7](https://github.com/openclaw/openclaw/commit/44114328b4e73074ac72126f1230ddbf9c1ea8b5)
