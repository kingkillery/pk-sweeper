## Summary

Keep open. Current main has partial diagnostics and some related fixes, but it does not fully resolve the reported observable problem: a stale/mismatched gateway token can still be selected, the gateway call path does not retry with local config credentials after a token mismatch, and the "newer OpenClaw" warning still intentionally fires for truly newer config versions. The provided issue thread also has a recent report that the bug persisted as of 2026.4.21, while the related PR #29676 was closed unmerged.
