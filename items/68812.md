## Summary

Current `main` does not implement the requested configurability. The non-batch memory embedding path still hardcodes index concurrency (`4`) plus retry base/max delays (`500`/`8000` ms), while the resolved config surface and public docs only expose `memorySearch.remote.batch.*` settings for provider batch APIs. The issue is also linked to an open PR (#68822) in the provided timeline rather than a merged fix.
