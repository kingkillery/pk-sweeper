## Summary

Keep open. Current `main` still does not implement explicit `tools.web.search.fallbacks` support for web search providers: the runtime remains fail-fast for explicit selections, the tests still lock in that behavior, and the public config/docs surface has no `fallbacks` field for web search.

## Evidence

- **Current runtime still disables fallback for explicit provider selection:** `runWebSearch` on `main` sets `allowFallback` to the negation of `hasExplicitWebSearchSelection(...)`, so explicit config or caller-selected providers still fail fast instead of using configured fallback providers.
  - file: [src/web-search/runtime.ts:307](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/web-search/runtime.ts#L307)
