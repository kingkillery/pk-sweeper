## Summary

Keep open. Current main contains the later #70294 transport-gate fix for malformed openai-completions tool-call arguments, but it does not implement this PR's parser-level Unicode smart-quote delimiter normalization. The existing repair path still parses raw or balanced-prefix JSON directly, the shared balanced JSON scanner only tracks ASCII quotes, and the focused tests cover openai-completions enablement with ASCII JSON rather than Unicode structural quotes.

## Best Possible Solution

Keep the PR open or replace it with a maintainer-owned smaller parser fix. The concrete path is to add focused tests for Unicode smart quotes used as JSON structural delimiters, plus tests preserving smart quotes inside normal ASCII-delimited string content with leading/trailing repair junk, then implement normalization in the tool-call repair parser or shared balanced-json helper without broadening repair beyond the existing guarded stream wrapper.
