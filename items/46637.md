## Summary

Keep open. Current `main` still converts `reasoning_content` into assistant thinking blocks, preserves those blocks on same-model replay, and does not apply any `openai-completions`-specific thinking-block stripping/sanitization before rebuilding history. The strongest code evidence supports an unresolved replay-policy gap for strict OpenAI-compatible backends like oMLX.

## Evidence

- **`reasoning_content` is still turned into replayable thinking:** The OpenAI-compatible stream parser maps `reasoning_content` / `reasoning` / `reasoning_text` deltas into `{ kind: "thinking", signature: field, text: value }`, so this provider-specific reasoning is still captured in assistant turns.

  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
