## Summary

Keep open. Current `main` still uses `.trim()`-only echo-text normalization in the iMessage sent-message cache, the inbound parse path does not remove NUL-prefixed text, and `main` lacks the NUL regression tests described in the PR. The report remains plausible and the PR targets the implicated code path directly, so there is not strong evidence to close it as already implemented, obsolete, or unreproducible.

## Evidence

- **Current main still lacks NUL stripping in echo-cache normalization:** `normalizeEchoTextKey()` only normalizes CRLF and trims whitespace before caching/lookup. A leading NUL byte would survive this function and produce a different cache key from the outbound `hello world` entry.
  - file: [extensions/imessage/src/monitor/echo-cache.ts:26](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/extensions/imessage/src/monitor/echo-cache.ts#L26)
  - command: `nl -ba extensions/imessage/src/monitor/echo-cache.ts | sed -n '26,31p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Inbound iMessage parsing does not strip NUL bytes on main:** `parseIMessageNotification()` only applies `stripImessageLengthPrefixedUtf8Text()` to `message.text` and `reply_to_text`; there is no separate NUL-byte cleanup before the echo-cache comparison path.
  - file: [extensions/imessage/src/monitor/parse-notification.ts:82](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/extensions/imessage/src/monitor/parse-notification.ts#L82)

- **Main test coverage does not include the reported NUL-prefix case:** `monitor-provider.echo-cache.test.ts` covers CRLF/trim normalization, message-id matching, and TTL behavior, but not echoed text with leading NUL bytes. The regression test claimed by the PR is not present on `main`.
  - file: [extensions/imessage/src/monitor/monitor-provider.echo-cache.test.ts:9](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/extensions/imessage/src/monitor/monitor-provider.echo-cache.test.ts#L9)
