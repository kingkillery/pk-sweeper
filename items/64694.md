## Summary

Keep open. Current `main` still lacks a default provider-catalog timeout outside live mode, so this PR is not implemented and the reported hang remains plausible. The code path on `fd65caf4b0bad1b0ac27a82d349c22e92167e7bc` still returns `null` for non-live discovery, passes that through to the timeout wrapper, and then awaits provider catalog hooks without a deadline.

## Evidence
