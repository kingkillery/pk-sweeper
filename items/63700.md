## Summary

Keep open. Current `main` still implements `openclaw infer model run` as a text-only prompt path with no `--file` input surface, while the existing file-based infer commands are split into separate image/audio/video flows and do not provide a single multimodal model-run turn.

## Evidence

- **Capability metadata still defines `model.run` as text-only:** `model.run` is described as `Run a one-shot text inference turn through the agent runtime` and its declared flags are only `--prompt`, `--model`, `--local`, `--gateway`, and `--json`.

- **CLI registration exposes no `--file` option for `infer model run`:** The `model run` command requires `--prompt` and only adds `--model`, `--local`, `--gateway`, and `--json`; there is no multimodal file flag on current `main`.
  - file: [src/cli/capability-cli.ts:1259](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/cli/capability-cli.ts#L1259)
