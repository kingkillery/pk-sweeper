## Summary

Keep open. Current main has the missing-entry helper work for agent edits, but the implementation is not strong enough to close this PR: creating the first explicit `agents.list` entry for a non-default discovered agent can still silently change the default agent and shrink the visible agent set after save.

## Evidence

- **UI helper creates only the edited agent entry:** `ensureAgentConfigEntry` appends just `{"id": <agentId>}` when no explicit entry exists. There is no safeguard to preserve the current default agent before creating the first `agents.list` item.
  - file: [ui/src/ui/controllers/config.ts:256](https://github.com/openclaw/openclaw/blob/ed6094b3014c56882da15a7942130b00ea410c6e/ui/src/ui/controllers/config.ts#L256)
  - command: `nl -ba ui/src/ui/controllers/config.ts | sed -n '235,270p'`
  - sha: [ed6094b3014c](https://github.com/openclaw/openclaw/commit/ed6094b3014c56882da15a7942130b00ea410c6e)
- **Core default resolution still falls back to the first explicit agent entry:** `resolveDefaultAgentId` uses the first `agents.list` entry when no entry has `default: true`. A newly created one-entry list therefore becomes the default agent.
  - file: [src/agents/agent-scope-config.ts:78](https://github.com/openclaw/openclaw/blob/ed6094b3014c56882da15a7942130b00ea410c6e/src/agents/agent-scope-config.ts#L78)

  - sha: [ed6094b3014c](https://github.com/openclaw/openclaw/commit/ed6094b3014c56882da15a7942130b00ea410c6e)
- **Gateway agent list scopes to explicit agents once any `agents.list` exists:** `listGatewayAgentsBasic` builds `allowedIds` from `explicitIds + defaultId` whenever `agents.list` is non-empty, so after creating a single explicit entry the `agents.list` RPC stops showing other disk-only agents.
  - file: [src/gateway/agent-list.ts:69](https://github.com/openclaw/openclaw/blob/ed6094b3014c56882da15a7942130b00ea410c6e/src/gateway/agent-list.ts#L69)
