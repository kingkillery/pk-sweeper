## Summary

Keep open. Current `main` only covers the trailing-token cron-delivery case; the broader own-line `NO_REPLY` handling from PR #46626 is not implemented across the other cited paths, and the proposed `parseReplyDirectives` change would also need follow-up to avoid dropping channelData-only payloads.

## Evidence

- **Silent helper is still exact-match only on main:** `isSilentReplyText` only matches the exact token, and `isSilentReplyPayloadText` only layers exact-token plus JSON-envelope handling. There is no `isSilentTokenOnOwnLine`-style helper on current `main`, so `"text\n\nNO_REPLY"` is not treated as silent by the shared token helpers.
