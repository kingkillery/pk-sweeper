## Summary

Keep open. The issue is recent, non-protected, and actionable: it reports a confusing WeChat plugin rename/install-metadata case with a concrete environment, old/new plugin ids, and requested migration behavior. Current public docs clarify the supported WeChat plugin identity, but I did not find enough evidence that current main automatically migrates or cleans the old `openclaw-weixin` install when installing `openclawwechat`. Conservative cleanup should not close this without maintainer/plugin-owner judgment or a verified fix.

## Evidence

- **Issue report is concrete:** Issue #58660 reports `openclawwechat` install confusion with an existing `openclaw-weixin` v1.0.3 directory on OpenClaw 2026.3.28, including observed install output and requested cleanup/id-mapping guidance.
  - command: `gh/web issue view openclaw/openclaw#58660`
- **Discussion supports plugin-id mismatch:** The provided comment independently identifies the same core concern: `openclaw-weixin` vs `openclawwechat` id mismatch, no automatic cleanup of the old plugin directory, and missing migration guidance. It also notes a working path through ClawHub for a newer Tencent Weixin plugin when no old install existed.
  - command: `provided GitHub context comments for #58660`

That helps users, but does not by itself prove old `openclawwechat` installs are migrated or cleaned.
