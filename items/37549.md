## Summary

Keep open. Current `main` does not contain the subsystem file-logger refresh from this PR, so there is not enough evidence to close it as implemented or obsolete. The underlying stale-child-logger behavior still appears plausible on `main`, and this PR also appears incomplete because `raw()` still has the same cached-child pattern.

## Evidence

- **Current `main` still caches subsystem file loggers indefinitely in normal log emission:** `createSubsystemLogger` stores `fileLogger` in a closure and only initializes it when null before `logToFile`; there is no generation/version check or other invalidation when the root logger changes.
  - file: [src/logging/subsystem.ts:327](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/src/logging/subsystem.ts#L327)

  - sha: [7d4718373698](https://github.com/openclaw/openclaw/commit/7d471837369885c12d540d6bafa319ff41129b89)
- **`raw()` path also keeps the stale cached child:** `raw()` uses the same cached `fileLogger` and only recreates it when null, so long-lived subsystem loggers using `raw()` would continue writing through the old child logger after a root rebuild.
  - file: [src/logging/subsystem.ts:414](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/src/logging/subsystem.ts#L414)
