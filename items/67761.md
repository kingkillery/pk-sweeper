## Summary

Keep #67761 open. Current main still contains the exact truncated-preview behavior the PR removes in Slack, MS Teams, and Mattermost inbound message handlers, while the full message body is also delivered separately through the channel envelope/BodyForAgent path. The PR is a focused, still-relevant fix for linked issue #67503, not implemented on main and not superseded by another item in the provided context.

## Best Possible Solution

Keep the PR open and either merge or adapt its narrow production change: Slack, MS Teams, and Mattermost should enqueue only the inbound notification label while preserving full message content in the existing user turn/BodyForAgent path and retaining preview only for logs or return-shape compatibility. Add focused regression assertions for all three handlers if feasible before landing.
