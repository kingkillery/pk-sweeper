## Summary

Close as ClawHub/plugin work. PR #67571 adds a one-off core parser for the external Kesha Voice Kit CLI, but current OpenClaw direction keeps optional/vendor media-understanding behavior out of core when plugin surfaces are sufficient. The existing media-understanding provider API can own an audio transcription provider, and generic CLI media entries already support arbitrary commands; the PR body also notes Kesha plain-text mode works without this parser.

## Best Possible Solution

Do not merge the Kesha-specific parser into OpenClaw core. The best path is an external Kesha media-understanding plugin or bundle/wrapper published through ClawHub, owned alongside the Kesha tool, using api.registerMediaUnderstandingProvider(...) or generic CLI configuration to format the transcript.
