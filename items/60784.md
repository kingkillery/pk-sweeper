## Summary

Current main and the v2026.4.24 release already implement the observable fix from PR #60784: child loggers no longer clear the parent logger threshold when no child override is supplied, and regression coverage exists.

## Evidence

- **Current main fixes child logger inheritance:** `getChildLogger` now computes `minLevel` as `opts?.level ? levelToMinLevel(opts.level) : base.settings.minLevel`, so a child logger without an explicit level receives the parent threshold instead of `undefined`.
  - file: [src/logging/logger.ts:498](https://github.com/openclaw/openclaw/blob/a5f5608d060b23477b4522c1ba4effab2f7eb1c6/src/logging/logger.ts#L498)
