## Summary

Keep PR #61134 open. Current main still calls `gateway.disconnect()` from the Discord abort path without catching synchronous throws, while the existing supervisor fixes only handle emitted `error` events. The PR still appears to address a real remaining gap rather than being implemented or clearly superseded.

## Evidence

- **Abort path still calls disconnect without catch:** On current main, `onAbort()` calls `finishResolve()`, and `finishResolve()` calls `gateway?.disconnect?.()` inside a `try/finally` with cleanup and resolve, but no `catch`. A synchronous throw from this abort listener would still escape the event callback.
