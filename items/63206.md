## Summary

Keep open. Current `main` still appears to skip symlinked plugin directories during plugin discovery, so this PR is neither implemented on `main` nor obsolete.

## Evidence

- **Current discovery logic skips non-file, non-directory dirents:** On `main` at `86f8c826e20a01d1bba67c0ba6e73182fbf0455f`, `discoverInDirectory` only handles `entry.isFile()` and `entry.isDirectory()`. A symlinked plugin directory returned as a symbolic-link dirent will fall through `if (!entry.isDirectory()) continue;` and be skipped.
  - file: [src/plugins/discovery.ts:855](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/plugins/discovery.ts#L855)

  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **No matching symlink discovery coverage on current main:** The nearby discovery tests on `main` cover regular global/workspace extensions and workspace recursion boundaries, but there is no corresponding test for a symlinked plugin directory in the scanned extensions root.
  - file: [src/plugins/discovery.test.ts:274](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/plugins/discovery.test.ts#L274)
