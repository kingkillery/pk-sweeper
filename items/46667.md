## Summary

Keep open. Current `main` still has no `openai-completions` replay cleanup that drops or downgrades replayed thinking/reasoning blocks, and the transport-side text sanitizer does not scrub control characters. The issue's `JSON.stringify` root-cause claim is not supported, but there is not strong evidence that the underlying multi-turn replay incompatibility has been fixed on `main` or in `v2026.4.22`.

For `openai-completions`, the code falls through with `sanitizedToolCalls` unchanged, and thinking blocks are only removed when `policy.dropThinkingBlocks` is set.
