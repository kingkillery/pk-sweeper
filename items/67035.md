## Summary

Keep open. Current `main` contains some related Control UI follow-up fixes, but the concrete failure modes from this thread are not fully covered: the chat textarea still lacks the focus/composition handling described in the investigation, the `error` chat path still clears streamed text without preserving a partial assistant message, and the issue timeline later cross-references another open report for the same "messages only appear after reload" symptom.

## Evidence

- **Textarea still uses plain draft binding with no focus/composition handlers:** The chat composer textarea is still rendered from `props.draft` and only wires `@keydown`, `@input`, and `@paste`. There is no `inputFocused` state and no `@focus`, `@blur`, `@compositionstart`, or `@compositionend` handling matching the issue thread's concrete workaround for swallowed/delayed input.
