## Summary

Keep open. Current main mitigates part of the dirty-tree symptom by hashing only tracked A2UI source inputs and the current checked-in hash matches those inputs, so unchanged clean builds can skip rebundling. But the generator is still install-layout-sensitive: the Rolldown config still prefers `ui/node_modules` before repo-root `node_modules`, and no current test proves the generated bundle is byte-stable across those layouts. The issue comment's cited files are still the right surface, and related PR #68934 remains relevant to the missing singleton dependency alias work rather than proving this issue is shipped/fixed.

## Best Possible Solution

Keep #67547 open. The best fix is to make `apps/shared/OpenClawKit/Tools/CanvasA2UI/rolldown.config.mjs` resolve A2UI bundle dependencies from one canonical root, or alias the full singleton dependency set consistently (`lit`, `@lit/*`, `@lit-labs/signals`, `signal-utils`, and `signal-polyfill`) so `ui/node_modules` presence cannot change module identity. Add a regression check that forces the rebundle path and proves the bundle/hash are stable with and without `ui/node_modules`, then refresh the tracked A2UI artifacts. Coordinate with the related #68934 alias work if that PR is used as the implementation path.
