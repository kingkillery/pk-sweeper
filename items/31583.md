## Summary

Keep #31583 open. Current main still does not fully satisfy the reported behavior for `skills.entries.*.env`: sensitive user-configured keys can still be blocked unless declared by skill metadata, and sandbox exec still builds an environment that does not include applied skill env values.

## Evidence

- **Sensitive keys are only allowlisted from skill metadata:** `allowedSensitiveKeys` is populated from `primaryEnv` and `requiredEnv`; the keys from `skillConfig.env` are copied into `pendingOverrides` but are not added to the allowlist before sanitization. That matches the later issue comment showing `GOG_KEYRING_PASSWORD` blocked.
