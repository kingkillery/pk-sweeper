## Summary

This RFC is outside current OpenClaw core scope. Main explicitly keeps multi-agent orchestration and agent-hierarchy frameworks out of core, while exposing plugin/skill seams that already support subagent coordination and task-flow orchestration. The RFC’s `sessions_send` reply-routing blocker is also obsolete on current main.

## Evidence

- **Core scope excludes this class of feature:** `VISION.md` says core should stay lean, optional capability should usually ship as plugins, new skills should go to ClawHub first, and OpenClaw will not merge agent-hierarchy frameworks or heavy orchestration layers that duplicate existing agent/tool infrastructure.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/VISION.md#L54)
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Multi-agent design is intentionally isolated, not shared-state-first:** `docs/concepts/multi-agent.md` defines each agent as an isolated workspace/state/session scope and explicitly says credentials are not shared automatically across agents.
  - file: [docs/concepts/multi-agent.md:8](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/docs/concepts/multi-agent.md#L8)
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **The reported `sessions_send` limitation is outdated:** Current docs say `sessions_send` can either fire-and-forget or wait for a reply, and can run a bounded reply-back loop after the target responds.
  - file: [docs/concepts/session-tool.md:68](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/docs/concepts/session-tool.md#L68)
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Visible-channel announce delivery already exists:** `docs/tools/acp-agents.md` documents the current A2A fallback for `sessions_send`: wait for the target reply, optionally exchange follow-up turns, produce an announce message, and deliver that announce to the visible channel/thread.
  - file: [docs/tools/acp-agents.md:349](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/docs/tools/acp-agents.md#L349)
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)

  - file: [docs/plugins/sdk-runtime.md:87](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/docs/plugins/sdk-runtime.md#L87)

## Risks / Open Questions

- If maintainers decide to change `VISION.md` and bring first-class team orchestration into core, this issue would no longer be out of scope.
- A narrower follow-up issue for any remaining concrete `sessions_send` delivery bug on current main could still be valid, but this bundled RFC is not the right core tracking item.
