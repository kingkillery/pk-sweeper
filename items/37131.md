## Summary

Current `main` now has partial direct tool-invocation support, but not the full generic chat/slash/cron flow requested in #37131. OpenClaw ships a direct `POST /tools/invoke` HTTP surface and skill-level slash commands that can dispatch straight to a tool, but there is still no built-in generic `/tool` command, cron payloads still cannot store a direct `toolCall` kind, and several high-risk tools named in the issue are still denied on the HTTP surface by default.
