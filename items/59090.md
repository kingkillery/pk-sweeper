## Summary

Keep open. The issue is recent, concrete, author is not a maintainer, has no protected labels, and current main still appears to use plain X509TrustManager implementations in GatewayTls.kt while the Android network security config still includes domain-specific entries for ts.net/openclaw.local. That matches the reported failure mode rather than proving it fixed or obsolete.
