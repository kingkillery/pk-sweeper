## Summary

Current `main` does not implement the requested Feishu in-place card update flow. The Feishu card interaction envelope/context still lacks any `updateCard`/`uc` payload, and the card-action handler never calls the existing card patch helper during button handling. This issue is also recent and linked to an implementation PR in the provided timeline, so there is not strong evidence for auto-closing.

## Evidence

- **Interaction context has no update payload field:** `buildFeishuCardInteractionContext` only serializes operator/chat/session/expiry/chat-type fields (`u`, `h`, `s`, `e`, `t`). There is no `updateCard` or `uc` support in the builder described by the issue.
