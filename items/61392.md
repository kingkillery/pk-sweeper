## Summary

Close PR #61392 as superseded by the linked open issue #61389. The PR's remaining useful request is already tracked there, while this implementation is not safe to merge as-is: it would hard-code a stale threshold floor that conflicts with existing interval-relative validation, omits generated schema updates, and describes `channelMaxRestartsPerHour: 0` as unlimited even though runtime behavior treats it as zero allowed restarts.

## Evidence
