## Summary

Keep open. The requested capability remains missing on current main: cron webhook delivery still uses the SSRF-guarded fetch without passing any policy/config override, and current tests explicitly assert that private cron webhook URLs are blocked. The issue is concrete, recent, non-maintainer-authored, and has no protected label, but it is not stale and does not fit a conservative close reason.
