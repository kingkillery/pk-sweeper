## Summary

Keep PR #16085 open. Current main still implements Signal through native signal-cli JSON-RPC + SSE only: there is no `apiMode`, no adapter/container client, no REST `/v2/send` path, and no WebSocket `/v1/receive/{account}` stream path. The linked issue #10240 remains the canonical open request, and this PR is a plausible implementation path rather than obsolete. It may need maintainer review and scope tightening before merge, but none of the conservative close reasons apply.
