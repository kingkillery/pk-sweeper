## Summary

Keep open. Current `main` accepts Tailscale Serve identity as a WebSocket auth method, but the Control UI pairing-skip logic still only bypasses pairing for trusted-proxy auth, `auth.mode="none"`, or `dangerouslyDisableDeviceAuth`; there is no landed Tailscale-authenticated operator path. Recent issue comments, including a maintainer read on 2026-04-24, say the bug is still real and the fix needs narrower scope/tests.
