## Summary

Keep open. Current main still has a key-shape mismatch for explicit non-`agent:` session keys: inbound/session-init continues using the raw external key, while gateway store resolution canonicalizes bare keys into `agent:<default-agent>:...` and pruning code deletes the raw alias. The shipped regression coverage only proves the narrower `agent:main`/main-alias repair, not arbitrary raw external keys like the report describes.
