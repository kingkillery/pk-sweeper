## Summary

Current `main` does not implement configurable file or directory modes for OpenClaw-managed state. The cited code paths still hardcode owner-only `0o600`/`0o700`, there is no `OPENCLAW_FILE_MODE`/`OPENCLAW_DIR_MODE`/`OPENCLAW_SKIP_CHMOD` or config `fileMode`/`dirMode` surface, and docs/tests/changelog still describe and enforce that hardening behavior. This is therefore not `implemented_on_main`, not `cannot_reproduce`, and should remain open as an unsatisfied feature request.

## Evidence

## Risks / Open Questions

- This request cuts across current security assumptions, docs, tests, and repair tooling; if implemented later, it will need a deliberate policy change rather than a one-off patch.
- Some helpers already accept internal `mode` parameters, but there is no user-facing configuration surface today, so partial code reuse should not be mistaken for the requested feature being shipped.
