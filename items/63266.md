## Summary

Keep open. Current `main` still assumes one canonical workspace (`~/.openclaw/workspace`) and does not show a real implementation for legacy/split workspace-dir detection. The doctor docs claim legacy workspace-dir warnings exist, but the shipped detector is a stub that always returns no legacy dirs, and its test only covers that no-op behavior. I found no code or docs handling the reported `workspace-local` vs `workspace-local/local` split or migrating secrets/config between those locations.
