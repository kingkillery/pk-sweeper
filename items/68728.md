## Summary

Keep open. Current `main` still marks Ollama and self-hosted reasoning support with name heuristics that do not match `gemma4`, and the Ollama `/api/show` capability data is not used to set `reasoning: true`.

## Evidence

- **Ollama still uses heuristic-only reasoning detection:** `queryOllamaModelShowInfo` reads `/api/show` capabilities and `enrichOllamaModelsWithContext` carries them forward, but `buildOllamaModelDefinition` still sets `reasoning` only from `isReasoningModelHeuristic(modelId)`, which is `/r1|reasoning|think|reason/i` and does not match `gemma4`.
  - file: [extensions/ollama/src/provider-models.ts:210](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/extensions/ollama/src/provider-models.ts#L210)

- **Docs describe the same unfixed behavior:** The Ollama provider docs on current `main` say reasoning detection is still a model-name heuristic using only `r1`, `reasoning`, and `think`, while `/api/show` capability detection is documented for context/vision only.
  - file: [docs/providers/ollama.md:155](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/docs/providers/ollama.md#L155)
