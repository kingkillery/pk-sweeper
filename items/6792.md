## Summary

Keep open. Current `main` does not implement a manifest-level `configPatch` field or any install-time path that reads a plugin manifest patch and merges it into user config. The only shipped `configPatch` support is for provider-auth/onboarding results, and `plugins install` still persists only enablement, allowlist adjustment, install metadata, and slot selection.

## Evidence

- **Manifest contract has no top-level `configPatch`:** `PluginManifest` lists fields such as `activation`, `setup`, `contracts`, `configContracts`, and `channelConfigs`, but no manifest `configPatch` field. The public manifest docs match that field list and likewise omit `configPatch`.
  - file: [src/plugins/manifest.ts:157](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/plugins/manifest.ts#L157)
