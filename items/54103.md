## Summary

Close as duplicate_or_superseded. The requested configurable WhatsApp message-send retry is already tracked by open PR #54183, which explicitly says it closes #54103 and implements channel/per-account retry config for WhatsApp send paths. Current main has partial automatic WhatsApp retry/reconnect handling, but not the requested `channels.whatsapp.retry`/`sendRetry` config surface, so #54183 is the canonical remaining work item.

## Best Possible Solution

Close #54103 as superseded by #54183 and keep review, tests, and any remaining design decisions on that PR. If #54183 is abandoned, reopen or file a focused issue for the missing `channels.whatsapp.retry` channel/account config surface.
