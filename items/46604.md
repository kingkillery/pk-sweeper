## Summary

Current `main` does not implement a separate Communication Panel and Tool Activity Panel. Tool events are still routed into the same chat timeline, with only partial mitigations (`smart scroll` and a show/hide tool-calls toggle).

## Evidence

- **Tool messages still enter the main chat item stream:** `buildChatItems` appends `toolMessages` into the same `items` array used for chat history and streamed assistant text, so tool activity is still part of the single conversation timeline.
