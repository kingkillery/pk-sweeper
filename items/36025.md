## Summary

The repo vision pushes optional capability to plugins/ClawHub, and the SDK already exposes the needed extension points to bypass the LLM and own dispatch.

## Close Comment

Closing this as a Codex maintainer review under `clawhub` scope. This is a custom automation/plugin request rather than an OpenClaw core feature gap: `VISION.md:54-70` says optional capability should usually live in plugins and that new skills should go to ClawHub first. Current `main` already exposes the hooks needed for this pattern: `docs/plugins/sdk-overview.md:85-96` documents `registerCommand(...)` (bypasses the LLM), `registerHook(...)`, and `registerHttpRoute(...)`, `src/plugins/types.ts:2005-2009` says plugin commands run before agent invocation, and `docs/plugins/sdk-overview.md:211` plus `src/auto-reply/reply/dispatch-from-config.ts:712-743` show `reply_dispatch` can claim a message and skip the default model path entirely. For a keyword like "open tv" that calls local HTTP and replies directly, the intended path is a small custom skill/plugin published or installed via ClawHub, not a core OpenClaw issue.
