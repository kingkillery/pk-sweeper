## Summary

Keep open. Current main partially addresses #67369 with `openclaw sessions --agent`, `--all-agents`, `--active <minutes>`, and `--json`, plus a richer `sessions_list` agent tool/API path, but it does not implement the requested stale-session query shape (`--older-than` / `--newer-than`) or size-threshold filtering (`--larger-than`). Because the core CLI owns this session-list surface and meaningful requested filters remain missing, this is not a conservative close.

## Best Possible Solution

Keep #67369 open and implement the missing query semantics in the core session-list surface: add duration parsing for `--older-than` and `--newer-than` (or map `--newer-than` cleanly onto existing `--active`), define what `--larger-than` means for sessions (transcript byte size versus token count), expose the same filters through the gateway `sessions.list` schema if automation/API parity is desired, then update `docs/cli/sessions.md` and targeted command/API tests.
