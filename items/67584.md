## Summary

Keep open. Current main still does not implement the requested memory-wiki per-org vault templating: `vault.path` is resolved once at plugin registration, `wiki_status`/`wiki_lint`/`wiki_apply` are static tools closing over that config, and docs/tests only cover literal or `~` paths. The PR is the active combined implementation for open issue #66003 and supersedes the closed precursor PRs #66134 and #66149, so it is not obsolete.

## Best Possible Solution

Keep #67584 open as the active implementation path for #66003. The best maintainer path is to review the PR's fail-closed templating behavior, rebase/validate it against current main, and merge if CI and product review agree; if maintainers prefer a broader SDK contract for memory supplements instead of skipping them for templated vaults, track that explicitly rather than closing this PR as obsolete.
