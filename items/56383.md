## Summary

Keep open. Current `main` still lacks the cursor-based chat history pagination proposed here across the gateway contract, handler, web UI controller/view state, and generated Swift models.

## Evidence

- **Protocol schema still has no cursor field:** `ChatHistoryParamsSchema` on `main` accepts only `sessionKey`, `limit`, and `maxChars`; there is no `before` cursor parameter.
  - file: [src/gateway/protocol/schema/logs-chat.ts:26](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/gateway/protocol/schema/logs-chat.ts#L26)

- **Gateway handler still serves a single bounded tail page:** The `chat.history` handler reads `{ sessionKey, limit, maxChars }`, slices `rawMessages` with `rawMessages.slice(-max)`, and responds with `messages` only; it does not filter by a cursor or return pagination metadata such as `cursor`/`hasMore`.
  - file: [src/gateway/server-methods/chat.ts:1874](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/gateway/server-methods/chat.ts#L1874)
