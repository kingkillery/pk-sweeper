## Summary

Current `main` does not already contain this hardening. `MediaAttachmentCache` still uses its own `fs.open`/`handle.readFile` path for local attachments and still writes temp files without an explicit mode, so this PR remains a live, non-duplicative change and should not be auto-closed.

## Evidence

- **Attachment cache still bypasses `readLocalFileSafely` on main:** `MediaAttachmentCache.getBuffer()` delegates to a private `readLocalBuffer()` helper rather than `readLocalFileSafely`. That helper manually opens and reads the file, so the exact `fs-safe` identity/path-mismatch protections proposed by the PR are not on `main`.
  - file: [src/media-understanding/attachments.cache.ts:119](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/src/media-understanding/attachments.cache.ts#L119)
  - command: `nl -ba src/media-understanding/attachments.cache.ts | sed -n '109,140p'`
  - sha: [ec8dbc459558](https://github.com/openclaw/openclaw/commit/ec8dbc4595583816f71af2568df61dbe7c090e1f)
- **Current local-read implementation is custom, not the shared hardened helper:** `readLocalBuffer()` uses `fs.open`, `handle.stat()`, `fs.realpath()`, and `handle.readFile()`. It checks regular-file/root constraints and `maxBytes`, but it does not call `readLocalFileSafely()` or `openVerifiedLocalFile()`, which is the change this PR is proposing.
  - file: [src/media-understanding/attachments.cache.ts:383](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/src/media-understanding/attachments.cache.ts#L383)

  - sha: [ec8dbc459558](https://github.com/openclaw/openclaw/commit/ec8dbc4595583816f71af2568df61dbe7c090e1f)
- **Shared fs-safe helper provides stronger verified-open semantics:** `readLocalFileSafely()` goes through `openLocalFileSafely()`/`openVerifiedLocalFile()`, which performs verified open + realpath/stat handling and returns the canonical `realPath`. That helper exists on `main`, but the attachment cache is not using it.
  - file: [src/infra/fs-safe.ts:327](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/src/infra/fs-safe.ts#L327)
