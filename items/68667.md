## Summary

Current `main` still enqueues hook completion/error system events into the default main session regardless of `agentId`, so the cross-agent isolation bug described in PR #68667 is still present and not adequately covered by current tests. The PR remains materially useful cleanup rather than something already implemented or safely auto-closable.

## Evidence

- **Current main still hard-codes default completion session:** `dispatchAgentHook` captures `mainSessionKey = resolveMainSessionKeyFromConfig()` and then uses that same key for both success and error `enqueueSystemEvent(...)` calls. `value.agentId` never participates in completion-event routing on current `main`.
  - file: [src/gateway/server/hooks.ts:45](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/src/gateway/server/hooks.ts#L45)

## Risks / Open Questions

- Closing this PR would leave an apparent cross-agent data-leak path on current `main`, where hook summaries for non-default agents are still enqueued into the default agent session.
- Current tests do not lock in the intended per-agent completion-event routing, so future regressions in this exact area would remain easy to miss until coverage like the PR's is merged.
