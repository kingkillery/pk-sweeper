## Risks / Open Questions

- The PR is partially obsolete because `main` already has the model/thinking/header-layout work, so it likely needs a rebase onto the current `ui/src/ui/chat/session-controls.ts` structure rather than a straight merge.
- The agent-switching logic shown in the PR diff may route some users into unintended empty sessions when the gateway advertises `sessionDefaults.mainSessionKey` instead of a bare `mainKey`.
