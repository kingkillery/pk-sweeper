## Summary

Keep open. Current `main` still reproduces the reported bug: `parseConfigValue` only recognizes plain integers/decimals, so scientific-notation inputs like `1e5` are returned as strings. The PR remains actionable, and the earlier heartbeat-test review noise was already removed from the branch per the PR timeline.
