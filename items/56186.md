## Summary

Keep open. Current `main` still drops channel-config save errors during the post-save refresh path, so PR #56186 addresses a real unfixed UI bug rather than already-landed behavior.

## Evidence

- **Current save handler still lacks error preservation:** `handleChannelConfigSave` on `main` calls `saveConfig`, then immediately `loadConfig` and `loadChannels`, with no capture/restore of the save error.
  - file: [ui/src/ui/app-channels.ts:39](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/ui/src/ui/app-channels.ts#L39)
  - command: `nl -ba ui/src/ui/app-channels.ts | sed -n '39,48p'`
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Config save stores failure on `lastError`:** `saveConfig` funnels through `submitConfigChange`, which writes save failures to `state.lastError` in both missing-hash and exception paths.
  - file: [ui/src/ui/controllers/config.ts:150](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/ui/src/ui/controllers/config.ts#L150)
  - command: `nl -ba ui/src/ui/controllers/config.ts | sed -n '146,165p'`
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Refresh clears the saved error:** `loadChannels` unconditionally resets `state.channelsError = null` before requesting status, so a restored message is required if the UI wants to keep showing the failed save error after refresh.
  - file: [ui/src/ui/controllers/channels.ts:10](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/ui/src/ui/controllers/channels.ts#L10)
  - command: `nl -ba ui/src/ui/controllers/channels.ts | sed -n '10,26p'`
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **No existing test covers this regression path:** A targeted search under `ui/src/ui` finds config-controller tests, but nothing covering `handleChannelConfigSave` or preservation of save errors across the refresh sequence.
  - file: [ui/src/ui](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/ui/src/ui)

## Risks / Open Questions

- The bug is still present in `main`: a failed `config.set` can set `lastError`, then `handleChannelConfigSave` immediately clears visible error state via `loadConfig`/`loadChannels`, leaving the user without feedback.
- There is no regression test around this handler today, so the behavior can regress silently even if fixed later.
