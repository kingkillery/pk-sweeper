## Summary

Keep open. Current `main` exposes compaction lifecycle hooks, but there is still no prune/context-prune hook or event on the session-pruning path, so the feature request is not implemented on `main`.

## Evidence

- **Plugin hook registry has compaction hooks only:** `PluginHookName` and `PLUGIN_HOOK_NAMES` include `before_compaction` and `after_compaction`, but no prune/session-prune hook.
  - file: [src/plugins/hook-types.ts:56](https://github.com/openclaw/openclaw/blob/c0a7b6a510758645c5267734b984f4885a605b67/src/plugins/hook-types.ts#L56)
  - sha: [c0a7b6a51075](https://github.com/openclaw/openclaw/commit/c0a7b6a510758645c5267734b984f4885a605b67)
- **Internal hooks docs list no prune event:** The documented internal hook events include `session:compact:before` and `session:compact:after`, but there is no `session:prune` or `context:prune` event.
  - file: [docs/automation/hooks.md:34](https://github.com/openclaw/openclaw/blob/c0a7b6a510758645c5267734b984f4885a605b67/docs/automation/hooks.md#L34)
  - sha: [c0a7b6a51075](https://github.com/openclaw/openclaw/commit/c0a7b6a510758645c5267734b984f4885a605b67)
- **Session-pruning implementation emits no hook:** The context-pruning extension calls `pruneContextMessages(...)`, updates TTL state, and returns rewritten messages. It does not call the global hook runner or emit a prune lifecycle event.
  - file: [src/agents/pi-hooks/context-pruning/extension.ts:5](https://github.com/openclaw/openclaw/blob/c0a7b6a510758645c5267734b984f4885a605b67/src/agents/pi-hooks/context-pruning/extension.ts#L5)
  - sha: [c0a7b6a51075](https://github.com/openclaw/openclaw/commit/c0a7b6a510758645c5267734b984f4885a605b67)
- **Compaction path explicitly wires hook callbacks:** The embedded compaction handlers call `runBeforeCompaction(...)` and `runAfterCompaction(...)`, which highlights that compaction is hooked while pruning is not.
  - file: [src/agents/pi-embedded-subscribe.handlers.compaction.ts:22](https://github.com/openclaw/openclaw/blob/c0a7b6a510758645c5267734b984f4885a605b67/src/agents/pi-embedded-subscribe.handlers.compaction.ts#L22)
  - sha: [c0a7b6a51075](https://github.com/openclaw/openclaw/commit/c0a7b6a510758645c5267734b984f4885a605b67)
- **Session-pruning docs still describe only in-memory trimming:** The pruning docs describe cache-TTL tool-result trimming and note that pruning is in-memory only; they do not document any prune lifecycle hook or emitted metadata for pruned content.
  - file: [docs/concepts/session-pruning.md:9](https://github.com/openclaw/openclaw/blob/c0a7b6a510758645c5267734b984f4885a605b67/docs/concepts/session-pruning.md#L9)
  - sha: [c0a7b6a51075](https://github.com/openclaw/openclaw/commit/c0a7b6a510758645c5267734b984f4885a605b67)
- **Targeted search found no prune lifecycle hook names:** A repo search for `session:prune|context:prune|before_prune|after_prune|prune lifecycle` returned no matches.
