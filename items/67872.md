## Summary

Keep PR #67872 open. Current main still drops Slack no-mention channel messages after recording pending history, so the requested silent-ingest `message:received` hook behavior is not implemented. The PR contains focused useful work, but the existing review correctly flags that its implementation must use Slack's `channels.slack.channels` config surface rather than the generic `groups` policy helper before it can be treated as ready or obsolete.

## Best Possible Solution

Keep the PR open and ask for revision. The best path is to implement Slack mention-skip `message:received` emission in `extensions/slack/src/monitor/message-handler/prepare.ts`, backed by regression coverage, while resolving ingest from Slack's valid `channels.slack.channels`/account-scoped channel config surface or explicitly extending that schema/docs if `ingest` is intended to become a Slack per-channel option.
