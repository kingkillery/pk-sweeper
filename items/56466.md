## Summary

Keep open. Current `main` still rejects wildcard plugin API ranges: `satisfiesComparator()` has no `*`/`x`/`X` fast-path, `satisfiesPluginApiRange()` still routes through that parser, and the ClawHub install path still fails when the range check returns false. The proposed PR remains relevant and is not implemented on `main`.

## Evidence

- **Wildcard handling is absent in current comparator:** `satisfiesComparator()` only handles empty input, caret ranges, and operator-based semver parsing. There is no branch for `*`, `x`, or `X`, so those tokens fall into `compareSemver(version, target)` and fail when the target is not a concrete semver.
  - file: [src/infra/clawhub.ts:302](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/infra/clawhub.ts#L302)
  - command: `nl -ba src/infra/clawhub.ts | sed -n '292,330p'`
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Wildcard tokens do not parse as comparable semver:** `parseComparableSemver()` only accepts `v?MAJOR.MINOR.PATCH` with optional prerelease/build metadata. `*`, `x`, and `X` do not match this regex, so `compareSemver(..., "*")` returns `null`, which makes the comparator return `false`.
  - file: [src/infra/semver-compare.ts:19](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/infra/semver-compare.ts#L19)

- **Plugin API compatibility still uses the unfixed parser:** `satisfiesPluginApiRange()` directly returns `satisfiesSemverRange(pluginApiVersion, pluginApiRange)`, so wildcard ranges still flow into the unfixed comparator on current `main`.
  - file: [src/infra/clawhub.ts:669](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/infra/clawhub.ts#L669)
