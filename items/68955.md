## Summary

Keep open. Current `main` still routes Telegram native slash commands like `/new` through a separate handler that does not wire the ack-reaction or typing callbacks used by normal inbound Telegram messages. Telegram native commands are still auto-enabled by default, so the reported UX gap remains plausible on current `main`.
