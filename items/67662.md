## Summary

Keep PR #67662 open. Current `main` still has the bug the PR targets: both incomplete-turn terminal cooldown writes call `maybeMarkAuthProfileFailure` without the active `modelId`, while the auth-profile cooldown implementation depends on `modelId` to preserve or write `cooldownModel`. The linked issue #67661 is the directly relevant open bug report, and the PR’s two-line production change plus harness assertions remain useful rather than obsolete or superseded.

## Best Possible Solution

Keep PR #67662 open and have a maintainer either merge/rebase it or apply the equivalent scoped fix on current `main`: pass the active `modelId` in both incomplete-turn `maybeMarkAuthProfileFailure` calls and retain regression coverage that asserts reasoning-only exhausted and generic incomplete-turn failure writes include the model id. The linked issue #67661 should remain the canonical bug until that fix lands.
