## Summary

Keep open. Current `main` still routes ordinary `web_fetch` through the strict pinned-DNS path, not the env-proxy path, and the requested opt-in config is not exposed in the current schema or docs.

## Evidence

- **`web_fetch` call site does not pass env-proxy opt-in:** `runWebFetch()` calls `fetchWithWebToolsNetworkGuard()` with URL, redirects, timeout, lookup, policy, and headers, but does not pass any `useEnvProxy` or similar flag.
  - file: [src/agents/tools/web-fetch.ts:415](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/agents/tools/web-fetch.ts#L415)
  - command: `nl -ba src/agents/tools/web-fetch.ts | sed -n '390,460p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Guarded fetch only uses env proxy when explicitly requested:** `fetchWithWebToolsNetworkGuard()` selects `withTrustedEnvProxyGuardedFetchMode()` only when `useEnvProxy` is truthy; otherwise it uses `withStrictGuardedFetchMode()`. The separate `withTrustedWebToolsEndpoint()` helper hard-codes `useEnvProxy: true`, which matches the issue report's contrast.
  - file: [src/agents/tools/web-guarded-fetch.ts:37](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/agents/tools/web-guarded-fetch.ts#L37)
  - command: `nl -ba src/agents/tools/web-guarded-fetch.ts | sed -n '1,140p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Current tests assert the direct-path behavior:** `web-tools.fetch.test.ts` explicitly verifies that untrusted `web_fetch` keeps DNS pinning even when `HTTP_PROXY` is set, and asserts the dispatcher is not an `EnvHttpProxyAgent`. Lower-level SSRF tests also state env proxy is ignored by default unless trusted proxy mode is explicitly enabled.
  - file: [src/agents/tools/web-tools.fetch.test.ts:257](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/agents/tools/web-tools.fetch.test.ts#L257)
  - command: `nl -ba src/agents/tools/web-tools.fetch.test.ts | sed -n '240,290p'; nl -ba src/infra/net/fetch-guard.ssrf.test.ts | sed -n '1000,1036p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **No user-facing config knob exists on current main:** The published `web_fetch` config docs list `enabled`, `provider`, `maxChars`, `maxCharsCap`, `maxResponseBytes`, `timeoutSeconds`, `cacheTtlMinutes`, `maxRedirects`, `readability`, and `userAgent`; the generated config schema similarly has no `tools.web.fetch.useEnvProxy` or `tools.web.fetch.useTrustedEnvProxy` entry.
  - file: [docs/tools/web-fetch.md:60](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/docs/tools/web-fetch.md#L60)
