## Summary

Keep open. Current main and the latest release ship mitigations for stale TUI activity state, but the TUI still drives `streaming`/`idle` from terminal chat or lifecycle events rather than from "last visible content token." That means delayed `chat.final` / lifecycle completion can still leave the indicator active after text stops, only bounded by a 30s watchdog.

## Evidence

- **Issue discussion already frames this as an acknowledged TUI UX/timing problem, not an incoherent report:** Provided GitHub context includes a maintainer comment on 2026-04-15 calling it a known TUI UX/timing issue and suggesting decoupling the visual streaming indicator from the terminal `chat.done`/lifecycle timing or adding a separate finalizing state. Another commenter reports a named-worker-agent configuration that makes the delay worse.
