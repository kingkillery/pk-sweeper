## Summary

Keep open. Current `main` still stages inbound media under the global state `media/` root, the agent config schema has no per-agent `mediaDir` or equivalent override, and the multi-agent docs still describe agents as isolated by workspace/state/session only. This request is therefore not implemented on `main`, not stale, and not a ClawHub/plugin-scope item.

## Evidence

- **Global media store remains process-wide:** `src/media/store.ts` resolves media writes from `resolveConfigDir()` and `saveMediaBuffer()` writes to `path.join(resolveMediaDir(), subdir)`, so inbound media still lands under the shared state `media/` directory rather than an agent-scoped path.
