## Summary

Keep open. Current main does not include the PR's CLI skills snapshot refresh behavior, and the reported stale-session path still appears plausible: the CLI agent path reads the existing skills snapshot version and only rebuilds when that version has already changed, but it does not force a one-shot bump before reusing a persisted session. The PR also has unresolved review feedback about starting a persistent watcher from the one-shot CLI path, so it remains useful work but should not be auto-closed as implemented or superseded.
