## Summary

Keep open. Current `main` at `56fe2aab9c87596515f2280c8060dc740ea64a88` still removes only the cron job from the cron store; it does not clean `sessions.json` or unlink the per-job run log. Isolated cron runs still persist a canonical `agent:<agent>:cron:<jobId>` session entry, the session reaper explicitly keeps that base key, and session listing hides only `:run:` aliases. That means the ghost-session behavior described in #46369 remains plausible on current `main`, so this PR is not stale/implemented and should stay open for maintainer review rather than be conservatively closed.
