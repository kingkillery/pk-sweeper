## Summary

Keep open. Current `main` still lacks the routing/session fixes this PR is trying to add, so it is not stale or already implemented, but the PR head also has unresolved correctness blockers from review: the new 45s timeout floor would over-apply, and the outbound mirror rewrite can bind metadata to the wrong session in cross-target sends.

## Evidence

That means the PR is addressing behavior that is still missing on `main`, so this is not an `implemented_on_main` close.

## Risks / Open Questions

- Closing this PR would discard fixes for behavior that is still missing on current `main`, especially the session-metadata sanitization and A2A timeout hardening described in the PR.
- Merging the PR as-is risks behavioral regressions: non-A2A `sessions_send` calls can be forced to wait 45 seconds, and outbound mirror metadata can be persisted onto the wrong session key for cross-target sends.
