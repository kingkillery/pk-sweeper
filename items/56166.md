## Summary

Keep open. Current `main` still classifies missing-session-entry/session-id runs as orphaned errors and prunes them immediately; there is no transcript-validity check or completed-transcript recovery path in the checked-out code.

## Evidence

- **Current orphan reconciliation still hard-errors and prunes runs:** `reconcileOrphanedRun` unconditionally stamps `status: "error"`, `endedReason: SUBAGENT_ENDED_REASON_ERROR`, sets cleanup fields, and deletes the run from the registry. There is no branch that inspects a transcript or converts the run to `status: "ok"`.
