## Summary

Current `main` does not implement session titles/summaries for `openclaw sessions`. The CLI still lists sessions by technical `key`, there is no `/title` command surface in the repo, and the session-list JSON/docs still expose only key/model-oriented data. This issue should remain open.

## Evidence

- **`openclaw sessions` text output still uses `Key` only:** `sessionsCommand` builds the table header with `Kind`, `Key`, `Age`, `Model`, `Tokens (ctx %)`, and `Flags`, then prints `formatSessionKeyCell(row.key, ...)`. There is no title/summary column or fallback to `displayName`/`label`.
  - file: [src/commands/sessions.ts:221](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/commands/sessions.ts#L221)
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Session-list rows omit stored display metadata:** `SessionDisplayRow` contains `key`, timestamps, token/model fields, and flags only. `toSessionDisplayRows()` copies `sessionId`, token/model fields, etc., but does not include `label` or `displayName`, so even stored session metadata cannot appear in `openclaw sessions` output or its JSON payload.
  - file: [src/commands/sessions-table.ts:5](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/commands/sessions-table.ts#L5)
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Docs still document key-based session listings:** The current sessions CLI docs show JSON examples with only `agentId`, `key`, and `model`, and the command examples/options do not mention session titles, summaries, or manual renaming.
  - file: [docs/cli/sessions.md:37](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/docs/cli/sessions.md#L37)
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Repo has session display metadata, but only for conversation/group labeling:** `SessionEntry` has `label` and `displayName` fields, and `deriveGroupSessionPatch()` populates `displayName` for group/channel sessions. That is narrower than the requested feature and is not wired into `openclaw sessions`.
  - file: [src/config/sessions/types.ts:246](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/config/sessions/types.ts#L246)
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **No manual `/title` command exists on current main:** A repo-wide search for `/title` returned no command implementation or docs for setting a session title manually.
  - command: `rg -n '\"/title\"|`/title`|/title ' src docs ui extensions test --glob '!**/dist/**'`
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
