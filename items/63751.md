## Summary

Current `main` still appears to ignore `models.mode: "replace"` for plugin-discovered implicit providers. Bedrock discovery can still contribute an implicit `amazon-bedrock` catalog entry, and the replace-mode path only skips merging with existing `models.json`; it does not suppress implicit provider discovery. That means the reported selector clutter remains plausible on `main`, even if the exact trigger in this report may have involved hidden AWS env markers or explicit Bedrock discovery config.

## Evidence

- **Replace mode still merges implicit providers first:** `resolveProvidersForModelsJsonWithDeps()` always resolves implicit providers and merges them with explicit `models.providers`. The later `resolveProvidersForMode()` branch only changes whether existing `models.json` entries are preserved in `merge` mode; for `replace`, it simply returns the already-merged provider set.
  - file: [src/agents/models-config.plan.ts:47](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/agents/models-config.plan.ts#L47)
  - command: `nl -ba src/agents/models-config.plan.ts | sed -n '36,83p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Bedrock plugin still contributes an implicit provider:** The Amazon Bedrock plugin catalog hook calls `resolveImplicitBedrockProvider()` and, when it returns a value, feeds that implicit provider into catalog assembly via `mergeImplicitBedrockProvider()`. There is no `models.mode` gate in this hook.
  - file: [extensions/amazon-bedrock/register.sync.runtime.ts:332](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/extensions/amazon-bedrock/register.sync.runtime.ts#L332)

- **Implicit Bedrock discovery is still active when enabled or AWS markers are present:** `resolveImplicitBedrockProvider()` only returns `null` when discovery is explicitly disabled, or when discovery is not forced and no AWS auth marker is found. The public Bedrock docs describe the same behavior: explicit `plugins.entries.amazon-bedrock.config.discovery.enabled: true` forces discovery, otherwise AWS markers like `AWS_PROFILE` or `AWS_ACCESS_KEY_ID` trigger it.
