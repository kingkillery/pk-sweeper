## Summary

Current main does not implement the PR's requested context-engine factory runtime context. The existing contract is still a no-arg `ContextEngineFactory`, `resolveContextEngine` still accepts only optional config and invokes `entry.factory()` with no context, and the run/compact/subagent call sites still resolve engines without `agentDir` or `workspaceDir`. The PR is a coherent additive SDK/API change and is not obsolete, duplicate, or ClawHub-scoped.

## Best Possible Solution

Keep the PR open. The best path is to finish the additive core/plugin-SDK implementation: add `ContextEngineFactoryContext` and resolver options, pass normalized `agentDir` and `workspaceDir` from run, queued compaction, and subagent cleanup paths, preserve backward-compatible no-arg factories, update SDK exports/docs/API baseline/tests/changelog, and resolve the fallback-context review before merge.
