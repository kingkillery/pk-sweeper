## Summary

Current `main` already has partial overlap with this request: the Control UI has a dedicated Usage dashboard, overview quota indicators, and per-assistant-message token/cost/context footers in chat. But it does not appear to have the requested persistent usage/quota widget in the chat header, and the documented built-in provider quota support still omits the issue's named providers (`OpenRouter`, `LLMAPI`, `Ollama`). That is not strong enough for a conservative close.
