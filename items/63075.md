## Summary

Keep open. Current `main` already contains the documented clear-on-disable path for Discord native commands, and the latest release `v2026.4.23` shipped the same behavior, so there is no strong evidence that the issue was fixed after the report. The report remains plausible because the intended cleanup path exists, yet there is no audit evidence here proving that stale Discord UI commands now disappear reliably in practice.
