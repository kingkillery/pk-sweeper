## Summary

Keep open. The RFC is coherent, recent, and still not implemented on current main. OpenClaw has ad hoc freshness signals on a few non-tool/status surfaces, but shared agent tool results, `session_status`, and `memory_search` do not expose the proposed standardized `freshness` envelope.

## Best Possible Solution

Keep the issue open for maintainer design. The likely path is a backwards-compatible core/SDK contract for optional tool-result freshness metadata, documented for plugin authors, with targeted adoption in `session_status`, `memory_search`, and cron/status tools plus tests proving old `content`/`details` consumers still work.
