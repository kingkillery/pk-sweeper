## Summary

Keep open. Current `main` still has the lifecycle mismatch the report describes: `generateAndAppendDreamNarrative` falls back when `subagent.run(...)` hits a request-scoped runtime error, but its `finally` block still calls `subagent.deleteSession({ sessionKey })`, which is exactly the warning-producing cleanup path from the issue. The current tests also encode that behavior as expected, so this is not implemented on `main` and not obsolete.
