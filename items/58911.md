## Summary

Keep open. I did not find source-backed evidence that current main implements the requested Telegram SDK export aliases or release validation. The current package export map still shows nearby channel-specific SDK exports such as LINE, IRC, Matrix, and Mattermost, but no `./plugin-sdk/telegram` or `./plugin-sdk/telegram-runtime` entry; current docs also describe Telegram helper surfaces as an unsettled compatibility area rather than a clearly fixed package contract. The related third-party plugin has added compatibility fallbacks, but that is a workaround in the plugin, not an OpenClaw core/package fix.
