## Summary

Keep open. Current `main` still shows the reported durability gaps in `memory-wiki`: lint does not resolve report-local relative Markdown links, bridge imports still treat cached file existence as sufficient for skip decisions, bridge render fingerprints do not include a renderer contract/version token, and the Markdown link extractor is still not Markdown-aware enough to ignore fenced `[[reply_to_current]]` content. The linked PR #65197 being closed does not correspond to these fixes being present on `main`.

## Evidence

- **Broken-link lint still compares raw link targets without resolving `./` or `../` against the containing page path:** `collectBrokenLinkIssues` builds a flat `validTargets` set and checks each `page.linkTargets` entry directly. There is no normalization against `path.dirname(page.relativePath)`, so a link like `../sources/foo.md` from `reports/stale-pages.md` would still miss.
  - file: [extensions/memory-wiki/src/lint.ts:53](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/memory-wiki/src/lint.ts#L53)
  - command: `nl -ba extensions/memory-wiki/src/lint.ts | sed -n '1,260p'`
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **Current tests lock in non-relative report links from `reports/` pages:** `compile.test.ts` still expects dashboard pages under `reports/` to contain links like `[Alpha](entities/alpha.md)` and `[Alpha](entities/alpha.md): missing updatedAt`. From `reports/stale-pages.md`, those are not path-relative links to sibling top-level directories, so the current tested behavior still matches the bug report rather than the proposed fix.
  - file: [extensions/memory-wiki/src/compile.test.ts:250](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/memory-wiki/src/compile.test.ts#L250)

There is no renderer version token or content-transform version, so renderer-only changes still will not invalidate unchanged source artifacts.
