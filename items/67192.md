## Summary

Keep open. Current `main` still does not show a landed fix for non-string Mistral reasoning deltas: the completions stream path forwards any truthy `choice.delta.content` straight into text assembly without a type guard, and the reasoning fallback only accepts string-valued `reasoning_content`/`reasoning`/`reasoning_text`. Tests cover Mistral request-parameter compat and OpenRouter `reasoning_details`, but not the reported Mistral object/array streaming shape. The latest shipped release contains the same stream logic, and the issue timeline points at still-open fix PRs rather than a merged resolution.

## Evidence

- **Current main still treats any truthy completions `delta.content` as text:** `processOpenAICompletionsStream` checks only `if (choice.delta.content)` and then passes that value directly to `appendTextDelta` / `queuePostToolCallDelta`. There is no `typeof === "string"` guard here, so an object or array-shaped Mistral delta would still flow into the text path instead of a provider-specific wrapper.
