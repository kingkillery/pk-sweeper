## Summary

Keep open. Current main still has the reported vulnerable stream path: non-string `delta.content` is accepted on truthiness and non-string `reasoning_content` is ignored, while the related fix PRs are still open/unmerged in the provided GitHub context.

## Best Possible Solution

Keep #67192 open and land a focused fix in `src/agents/openai-transport-stream.ts`: guard `delta.content` with `typeof === "string"`, parse Mistral non-string reasoning/content block arrays or objects into standard thinking/text deltas, and add regression coverage in `src/agents/openai-transport-stream.test.ts` using the reported Mistral stream shapes. Once merged, close the issue as implemented with the landed SHA and release evidence.
