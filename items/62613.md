## Summary

Keep open. Current `main` still eagerly builds the message-tool schema at module initialization via `buildPollSchema()` while importing `SHARED_POLL_CREATION_PARAM_NAMES` from `src/poll-params.ts`, which matches the failing initialization path described in the report. I did not find a clear fix on `main` or in the `v2026.4.23..HEAD` diff for that area, and this read-only checkout has no built `dist/` artifacts or `node_modules`, so I could not directly run the packaged-CLI reproduction here.
