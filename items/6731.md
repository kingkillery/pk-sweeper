## Summary

Keep open. Current main has opt-in safety controls (`exec-policy` presets, exec approvals, optional sandboxing), but it still documents no-approval host exec as the default, keeps sandboxing off by default, and the hard approval path is driven by policy/allowlist state rather than a dedicated dangerous-operation gate. That is not strong evidence that #6731’s hard-safety request is already satisfied.
