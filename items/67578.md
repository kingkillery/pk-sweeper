## Summary

Keep open. Current main hardens Bonjour cancellation rejections and one ciao interface-change assertion path, but it does not cover the specific synchronous `IP address version must match` assertion reported in #67578. The bundled dependency is still `@homebridge/ciao@1.3.6`, whose `getNetAddress()` assertion is called from `MDNSServer.handleMessage()` before the packet-parse try/catch, while OpenClaw's current Bonjour classifier only handles unhandled rejections for cancellation and a different IPv4 address-change assertion. Related maintainer comments on #71913 and #71918 explicitly left #67578 as the canonical broader ciao crash surface.

## Best Possible Solution

Keep #67578 open as the canonical broader ciao crash issue. The likely fix belongs in the bundled Bonjour path or an upstream/local ciao patch: guard `handleMessage()`/`getNetAddress()` against mixed address/netmask families, classify only the known ciao assertion as non-fatal if it reaches OpenClaw, and add regression coverage for the malformed mDNS/rinfo path without making unrelated uncaught exceptions non-fatal.
