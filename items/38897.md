## Summary

Keep open. Current `main` does not include the Mission Control runtime/dashboard surface from PR #38897, so this is not `implemented_on_main`; and the PR still contains potentially useful core gateway/UI work. It should not be conservatively closed, but it also is not ready to merge as-is because the PR head still has at least two correctness issues and bundles substantial unrelated changes.

## Risks / Open Questions

- The PR head has a startup race around `scheduleDashboardDelta(requestContext)` in `src/gateway/server.impl.ts` that can fire before `requestContext` is assigned.
- The PR head underreports failed tool runs because `ToolActivityRegistry` ignores `data.isError` on `phase: "result"` events.
- The PR bundles unrelated local/docs/infra/test changes with the feature work, increasing review and salvage cost.
