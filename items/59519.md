## Summary

Keep PR #59519 open. Current main does not contain the proposed fix: `buildProgram()` still captures `process.argv`, while `runCli()` computes the normalized `parseArgv` later and parses with that different argv slice. The linked issue #59449 is closed in the provided context, but the source on current main still matches the reported root-cause shape and the PR’s regression test is not present.
