## Summary

Keep open. Current `main` still uses a separate OpenRouter cache-marker path that only tags `system`/`developer` messages and explicitly leaves user conversation content untouched, while the direct Anthropic path still tags the trailing user message and splits the system cache boundary. The bundled tests also lock in that OpenRouter behavior today, so this issue still matches current code. The issue timeline also links an open PR (#63062), which is another signal that the fix is not yet on `main`.
