## Summary

Current main already implements the central fix from #67699: stale chat.history snapshots no longer wipe the optimistic user/assistant tail after tool-assisted final events, tool-stream reset is deferred and run-scoped, and session switches clear old messages before reloading.

## Best Possible Solution

Close #67699 as implemented on main and keep the shipped signature-based optimistic-tail preservation plus run-scoped deferred tool reset. The related #68014 pending-message discussion and #70391 reload-amplifier thread were useful, but the observable disappearing-message behavior is already covered by current source and tests; any remaining compaction/manual-refresh nuance should be filed as a narrower repro if it still occurs.
