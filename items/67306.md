## Summary

Keep open. Current `main` now has the new `openclaw infer` surface, but `infer model run` and `openclaw agent` still only expose plain `--json` output. I found no first-class JSON Schema input flag, no schema-validation path on these CLI surfaces, and no invalid-output repair/retry contract matching the issue.

## Evidence

- **`infer model run` flags do not include schema input:** `model.run` metadata and command registration only advertise `--prompt`, `--model`, transport flags, and `--json`; there is no schema-related CLI option.
  - file: [src/cli/capability-cli.ts:105](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/src/cli/capability-cli.ts#L105)

There is no schema parsing, validation, or repair loop in this path.
