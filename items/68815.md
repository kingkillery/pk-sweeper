## Summary

Keep open. Current `main` shows that the `gateway status`/exec-notification parts of the report have moved since `2026.3.13`, but I could not prove fixes for the Windows crash-dump upload, memory leak, or external `skill-interceptor` update behavior, so this omnibus issue is not safely closable.

## Evidence

- **`gateway status` now treats missing `operator.read` as degraded reachability:** `buildGatewayStatusWarnings()` emits `probe_scope_limited` for missing `operator.read` instead of treating the target as unreachable, matching the report's `Gateway unreachable (missing scope: operator.read)` symptom.
  - file: [src/commands/gateway-status/output.ts:78](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/src/commands/gateway-status/output.ts#L78)
  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)
- **`gateway status` only exits non-zero when nothing is reachable:** Both JSON and text writers call `runtime.exit(1)` only when `reachable.length === 0` / `!ok`, which means a scope-limited but connected gateway should not fail the command anymore.
  - file: [src/commands/gateway-status/output.ts:140](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/src/commands/gateway-status/output.ts#L140)
  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)
- **Regression test covers the missing-scope case:** `runGatewayStatus(..., --json)` is explicitly tested to return `ok: true`, `degraded: true`, and `scopeLimited: true` when the probe error is `missing scope: operator.read`.
  - file: [src/commands/gateway-status.test.ts:346](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/src/commands/gateway-status.test.ts#L346)
  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)
- **Changelog records the gateway-status fix after the reported version:** The changelog says gateway status now separates reachability, capability, and read-probe reporting so scope-limited sessions no longer look fully healthy; the report was against `2026.3.13`, while this entry appears in the `2026.4.19-beta.2` section.
  - file: [CHANGELOG.md:670](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/CHANGELOG.md#L670)
  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)
- **Exec notification suppression now exists in core:** Gateway node-event handling respects `tools.exec.notifyOnExit === false`, and docs expose that setting as the supported way to stop background exec completion events from enqueueing user-visible heartbeat work.
  - file: [src/gateway/server-node-events.ts:632](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/src/gateway/server-node-events.ts#L632)
  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)
- **Stale plugin config entries are intentionally warning-only on current main:** Unknown `plugins.entries.*` ids are handled as warnings with `stale config entry ignored; remove it from plugins config`, which explains part of the report's repeated config-warning output without proving the wider bug bundle is fixed.
  - file: [src/config/validation.ts:1106](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/src/config/validation.ts#L1106)
  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)
- **`skill-interceptor` is not present on `main`:** Repository search returned no matches for `skill-interceptor`, so the auto-reenable complaint appears to involve an out-of-tree plugin/skill rather than a bundled OpenClaw extension that can be verified from this checkout.
  - command: `rg -n "skill-interceptor" src extensions docs test`
