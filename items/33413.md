## Summary

Keep open. Current main has the plumbing to emit tool-start events, but Slack assistant thread status is still only set to the static "is typing..." value at reply start and cleared at reply end. The Slack onToolStart handler currently updates status reactions and draft-preview progress, not assistant.threads.setStatus. The linked PR #33420 directly attempted this wiring, but the provided GitHub context shows it was closed unmerged, and related issue #67129 is about customizable Slack status/loading messages rather than tool-level progress.
