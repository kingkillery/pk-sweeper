## Summary

Keep open. Current `main` still has an unsafe execution path for persisted cron jobs missing `sessionTarget`: load-time normalization does not apply defaults, while `server-cron` and `assertSupportedJobSpec` still call `startsWith` on `job.sessionTarget` directly. The PR's exact `delivery-plan` hunk is partly obsolete because that file is already guarded on `main`, but the underlying bug class is not fully fixed.

## Evidence

- **Load path does not apply defaults:** Persisted cron jobs are normalized on load with `normalizeCronJobInput(raw)` and no options, so `applyDefaults` stays false on this path.
  - file: [src/cron/service/store.ts:44](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/cron/service/store.ts#L44)

  - file: [src/cron/normalize.ts:505](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/cron/normalize.ts#L505)
