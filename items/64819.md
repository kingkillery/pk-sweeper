## Summary

Current `main` already fixed the PR’s core bug premise around `--session-id` collapsing back to the agent main session, but `main` still does not implement the proposed `openclaw agent --spawn` flag or any equivalent CLI `sessions spawn` surface. Conservative cleanup should keep this PR open: the original rationale is partly obsolete, yet the additive feature itself remains unshipped and still needs maintainer product/design judgment.
