## Summary

Keep open. Current `main` still does not implement a synchronous tool-enabled inline agent task. The shipped `llm-task` plugin is explicitly JSON-only and disables tool use, while `sessions_spawn` remains documented and implemented as non-blocking background delegation with announce-based completion delivery. I also found no `agent_task` tool in the current tool catalog.

## Evidence

- **`llm-task` is documented as JSON-only with no tools:** The current docs describe `llm-task` as an optional plugin tool for JSON-only tasks and explicitly say no tools are exposed during the run.
  - file: [docs/tools/llm-task.md:113](https://github.com/openclaw/openclaw/blob/c0a7b6a510758645c5267734b984f4885a605b67/docs/tools/llm-task.md#L113)
  - command: `nl -ba docs/tools/llm-task.md | sed -n '1,180p'`
  - sha: [c0a7b6a51075](https://github.com/openclaw/openclaw/commit/c0a7b6a510758645c5267734b984f4885a605b67)
- **`llm-task` implementation hard-disables tools:** The implementation injects a system prompt line `Do not call tools.` and passes `disableTools: true` into `runEmbeddedPiAgent`, so the closest existing feature still cannot satisfy the request for inline tool access.
  - file: [extensions/llm-task/src/llm-task-tool.ts:219](https://github.com/openclaw/openclaw/blob/c0a7b6a510758645c5267734b984f4885a605b67/extensions/llm-task/src/llm-task-tool.ts#L219)

- **`sessions_spawn` remains async/background-only:** Current docs say `sessions_spawn` creates an isolated session for a background task, is always non-blocking, and posts results later via an announce step. That does not match the requested synchronous inline `agent_task` behavior.
