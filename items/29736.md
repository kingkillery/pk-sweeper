## Summary

Keep #29736 open. Current main still resolves the exec approvals file and socket through `~/.openclaw/...` expansion, and that expansion honors `OPENCLAW_HOME` or OS home, not `OPENCLAW_STATE_DIR`. The issue discussion has one later user report saying it seemed fixed in 2026.4.11, but current source and docs still support the reported split-brain behavior for services/containers that set only `OPENCLAW_STATE_DIR`.
