## Summary

Keep open. Current main partially addresses the private-proxy Telegram regression, but it does not implement the PR's broader requested behavior of stripping the target hostnameAllowlist when validating every explicit proxy. In current main, hostnameAllowlist is cleared only when allowPrivateProxy is true; the non-private/public explicit proxy path still passes the original policy through unchanged, so a public proxy hostname outside the target allowlist can still be rejected.
