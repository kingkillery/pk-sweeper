## Summary

Keep open. Current `main` still contains the main behaviors described in the report: restart draining explicitly rejects new work, subagent announce is still documented as best-effort and can still terminally give up, and I did not find end-to-end evidence that the large-session compaction/drain/announce-loss cascade is fixed. There is partial mitigation for model-config hot reload and stale session-model races, but not enough to conservatively close this item.
