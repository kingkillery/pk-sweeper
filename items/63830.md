## Summary

Keep open. Current main still does not implement the host-side guarantee requested here: `registerService()` in `src/plugins/registry.ts:1166` only appends to `registry.services`, while `service.start()` is invoked later by a single post-attach pass in `src/plugins/services.ts:47` and `src/gateway/server-startup-post-attach.ts:285`. There is no reconcile or immediate-start path for services registered after that pass has already run.

## Evidence

- **Late registration still only mutates the registry:** `src/plugins/registry.ts:1166` through `src/plugins/registry.ts:1193` trims/dedupes the id and pushes the service into `registry.services`, but it does not notify a running gateway or call any start hook.
  - file: [src/plugins/registry.ts:1166](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/plugins/registry.ts#L1166)
  - command: `nl -ba src/plugins/registry.ts | sed -n '1140,1205p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Service startup is a one-time iteration over the current registry snapshot:** `src/plugins/services.ts:47` through `src/plugins/services.ts:54` loops over `params.registry.services` and calls `await service.start(serviceContext)` once for each entry present at that moment. There is no follow-up reconcile pass for later additions.
  - file: [src/plugins/services.ts:47](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/plugins/services.ts#L47)
  - command: `nl -ba src/plugins/services.ts | sed -n '1,220p'`

- **Only one production callsite starts plugin services:** Repo search shows `startPluginServices()` is only called from `src/gateway/server-startup-post-attach.ts:289`, where the gateway performs a single post-attach sidecar startup pass.
  - file: [src/gateway/server-startup-post-attach.ts:289](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/gateway/server-startup-post-attach.ts#L289)
  - command: `rg -n "startPluginServices\(" src`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)

But that is not the same as the requested guarantee that post-start `registerService()` always results in one host-managed `service.start()`.
  - file: [src/gateway/server.impl.ts:770](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/gateway/server.impl.ts#L770)
