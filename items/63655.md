## Summary

Keep open. Current `main` still sends `memory-lancedb` embedding requests through the OpenAI SDK with an optional `baseUrl` for compatible providers, but it does not add any provider-aware response handling for non-OpenAI embedding payloads and there is no test coverage for that path. I did not find evidence on `main` that the reported incompatibility was fixed.

## Evidence

- **Current embedding call is unchanged:** `Embeddings.embed()` still builds `{ model, input, dimensions? }`, calls `this.client.embeddings.create(params)`, and returns `response.data[0].embedding` directly. There is no branch for string-vs-array embeddings, no `encoding_format` handling, and no provider-specific decode guard.
  - file: [extensions/memory-lancedb/index.ts:179](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/extensions/memory-lancedb/index.ts#L179)

- **Plugin still advertises compatible provider base URLs:** The plugin continues to accept and surface `embedding.baseUrl`, and the UI/help text explicitly says it is for compatible providers. That means the reported non-OpenAI provider path is still a supported configuration surface on `main`.
  - file: [extensions/memory-lancedb/config.ts:181](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/extensions/memory-lancedb/config.ts#L181)
