## Summary

Keep open. Current `main` partially addresses the stable-session part of the report, but `/v1/chat/completions` still goes through the normal agent pipeline with full prompt assembly and there is no HTTP-specific `lightContext`/voice-mode path in the compat handler.

## Evidence

- **HTTP compat endpoint still uses the normal agent run path:** The public docs say the OpenAI-compatible HTTP surface executes requests as a normal Gateway agent run, and the handler still resolves a Gateway request context and calls `agentCommandFromIngress(...)` directly for chat completions.
