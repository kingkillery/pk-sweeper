## Summary

Keep #67629 open. Current main already exposes cron identifiers and running state through the core model/API and CLI, but the Control UI cron job list still renders only the last run status, next run, and last run; it does not show each job's identifier or whether `state.runningAtMs` means the job is currently running. Because the issue is vague but recent and salvageable, this should remain open for a focused UI/product decision rather than be auto-closed.

## Best Possible Solution

Keep open and scope it as a cron visibility enhancement. The likely best implementation is to render each cron job's `id` and a computed active state from `job.state?.runningAtMs` in `ui/src/ui/views/cron.ts`, add concise English i18n labels, preserve existing last-run status as separate history state, and add focused `ui/src/ui/views/cron.test.ts` coverage for running, idle, disabled, and identifier display.
