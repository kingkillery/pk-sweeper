## Summary

Keep open. Current main does not implement the PR’s central behavior: when mcporter returns an empty result set, `QmdMemoryManager.search()` proceeds with that empty parsed result instead of retrying direct `qmd query`. The PR discussion also shows the initial review concerns were acknowledged and reportedly addressed, and the provided related items do not supersede this memory-core fallback.

## Best Possible Solution

Keep the PR open. The best path is to land or adapt the guarded mcporter-empty-result fallback on current main, using direct `qmd query` for the retry, preserving the valid empty mcporter result if the fallback fails, reporting `fallback: "mcporter-empty-result"`, and keeping the regression test focused in `extensions/memory-core/src/memory/qmd-manager.test.ts`. If the qmd-main command-surface drift mentioned by the author is broader than this fallback, track that as a separate follow-up rather than closing this PR as obsolete.
