## Summary

Issue #37022 can be closed as implemented on main. OpenClaw already hard-stops runaway single-agent runs with an outer retry/iteration cap, returns a structured `retry_limit` error when that cap is hit, and also ships configurable tool-loop and agent-to-agent ping-pong protections. The exact `recursion_limit` parameter name is not exposed, but the requested infinite-loop safeguard is already present.

## Close Comment

Closing this from a Codex review because current OpenClaw already has the requested safety guard on `main`. The embedded runner applies a hard outer retry/iteration cap in `src/agents/pi-embedded-runner/run/helpers.ts:68` and enforces it in `src/agents/pi-embedded-runner/run.ts:633`, where it stops the run and returns a structured `retry_limit` error via `src/agents/pi-embedded-runner/run/retry-limit.ts:27` instead of continuing indefinitely. There is also configurable repeated-tool-loop protection in `src/config/types.tools.ts:165` and `src/agents/pi-tools.before-tool-call.ts:146`, plus agent-to-agent reply-turn caps in `src/config/schema.base.generated.ts:19671`. The behavior is covered by tests at `src/agents/pi-embedded-runner/run.overflow-compaction.test.ts:340`. So while there is not a user-facing parameter literally named `recursion_limit`, the infinite-loop prevention requested here is already implemented and shipped.
