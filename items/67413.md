## Summary

Current main does not implement per-agent dreaming configuration. Dreaming is still resolved from global memory-plugin config, one managed sweep schedule fans out across all configured agent workspaces, and the strict per-agent config schema has no `dreaming` property. The linked PR #67420 is open and only covers an `enabled: false` opt-out, not the requested per-agent schedule, so it does not fully supersede this issue.

## Best Possible Solution

Keep #67413 open. The product/code path should add a typed and documented `agents.list[].dreaming` override with strict schema support, at minimum `enabled`, and explicitly define schedule semantics. If per-agent schedules are accepted, memory-core likely needs per-agent or per-schedule managed cron reconciliation plus resolver/runtime tests for opt-out, shared-workspace dedupe, and schedule fan-out.
