## Summary

Keep open. The issue is clear, recent, not protected by maintainer-authored/label rules, and current main still appears to require SQLite for builtin memory search. The code has an FTS-only search path when no embedding provider is available, but that path is still SQLite-backed and does not provide the requested file-based scan of MEMORY.md and memory/*.md when node:sqlite itself is unavailable.

## Evidence

- **SQLite is still required to open the builtin memory database:** openMemoryDatabaseAtPath imports DatabaseSync from node:sqlite, calls requireNodeSqlite(), then constructs new DatabaseSync(dbPath, { allowExtension }). That means missing node:sqlite still blocks the builtin memory manager before any file-scan fallback could run.
  - file: [extensions/memory-core/src/memory/manager-db.ts:3](https://github.com/openclaw/openclaw/blob/44114328b4e73074ac72126f1230ddbf9c1ea8b5/extensions/memory-core/src/memory/manager-db.ts#L3)
  - command: `web inspection of main source; local exec unavailable due sandbox bwrap failure`
  - sha: [44114328b4e7](https://github.com/openclaw/openclaw/commit/44114328b4e73074ac72126f1230ddbf9c1ea8b5)
- **Existing fallback is SQLite FTS-only, not file-based search:** MemoryIndexManager.search handles the no-embedding-provider case by requiring this.fts.enabled and this.fts.available, then calling searchKeyword against the SQLite FTS table. This preserves keyword search only when the SQLite index exists and FTS is available; it does not scan MEMORY.md or memory/*.md without SQLite.
  - file: [extensions/memory-core/src/memory/manager.ts:1](https://github.com/openclaw/openclaw/blob/44114328b4e73074ac72126f1230ddbf9c1ea8b5/extensions/memory-core/src/memory/manager.ts#L1)
  - command: `web inspection of main source; local exec unavailable due sandbox bwrap failure`
  - sha: [44114328b4e7](https://github.com/openclaw/openclaw/commit/44114328b4e73074ac72126f1230ddbf9c1ea8b5)
- **Search helper is database-backed:** searchKeyword accepts a DatabaseSync handle and queries the configured FTS table. The fallback keyword scoring in this helper ranks SQLite rows; it is not a filesystem grep over workspace memory files.
  - file: [extensions/memory-core/src/memory/manager-search.ts:1](https://github.com/openclaw/openclaw/blob/44114328b4e73074ac72126f1230ddbf9c1ea8b5/extensions/memory-core/src/memory/manager-search.ts#L1)
