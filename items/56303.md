## Summary

Keep open. The specific bug report does not match current `main` because `sessions_spawn` no longer contains channel-name hardcoding; thread-bound behavior is delegated through generic `subagent_spawning` hooks implemented by channel plugins. But the underlying request is still not implemented: callers must still opt into `thread: true`, and the SDK/channel entry contract does not expose metadata for automatic thread/session coercion. This is a salvageable feature request, not a strong close.
