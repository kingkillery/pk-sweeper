## Summary

Keep open. Current `main` still implements `openclaw models status --probe` as a provider/profile auth probe, not a per-model `--probe --all` health check across configured primary and fallback models. The issue timeline’s linked PR (#63259) also indicates this work is still separate rather than already shipped on `main`.
