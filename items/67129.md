## Summary

Keep open. Current `main` still hardcodes Slack assistant thread status text, forwards only a single `status` string to Slack, and exposes no `channels.slack` config field for custom assistant `status` or `loading_messages`. The only documented tuning here is the separate `typingReaction` fallback, which does not implement this request.

## Evidence

  - file: [extensions/slack/src/monitor/message-handler/dispatch.ts:363](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/extensions/slack/src/monitor/message-handler/dispatch.ts#L363)
