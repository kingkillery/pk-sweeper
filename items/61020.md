## Summary

Keep open. Current `main` still has the stale-replay path described in PR #61020, and the proposed `.slice(prePromptMessageCount)` fix has an unresolved compaction-safety concern already raised in review, so there is not strong evidence for a conservative close.

## Risks / Open Questions

- Current `main` still appears able to replay a prior-turn assistant response when an idle timeout fires before any current-turn tokens stream.
- Closing this PR would discard a plausible bug report that is still reflected in current code paths.
- Merging the PR as written may introduce a different regression on compaction-shortened snapshots unless the offset logic is made compaction-safe and covered by tests.
