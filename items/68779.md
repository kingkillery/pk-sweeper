## Summary

Keep open. Current `main` adds useful guardrails around existing-session automation, but the core instability described in #68779 is not fully resolved: the tight attach/reachability constants are still present, Chrome MCP reconnect logic only auto-retries one narrow stale-`list_pages` case, and the existing-session navigation guard still uses the same short recheck window.

## Evidence
