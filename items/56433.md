## Summary

Current `main` already handles inbound images with text-only reply models by preserving the attachment, running automatic media understanding, and preferring `agents.defaults.imageModel` or provider image defaults for image analysis when needed.

## Close Comment

Closing from a Codex review: current `main` already covers this behavior through the existing media-understanding and `agents.defaults.imageModel` path. `src/auto-reply/reply/get-reply.ts:247` runs media understanding for inbound media, `src/media-understanding/runner.ts:549` prefers configured `imageModel` entries before the active reply model, and `src/gateway/chat-attachments.ts:275` preserves image attachments for text-only models instead of dropping them. The current docs also describe `imageModel` as fallback routing when the selected/default model cannot accept image input in `docs/gateway/config-agents.md:339` and `docs/nodes/media-understanding.md:139`, and the unreleased changelog calls out both fixes in `CHANGELOG.md:62`. The exact config shape proposed here was not added, but the requested capability is already implemented on `main`, so this issue no longer needs to stay open.
