## Summary

Keep open. Current `main` does not implement optional per-message timestamps in the TUI chat view. There is no `--timestamps` CLI flag, no settings toggle for timestamps, and the history/rendering path drops message timestamps instead of displaying them.

## Evidence

- **CLI surface has no timestamp option:** `registerTuiCli` defines `--local`, `--url`, `--token`, `--password`, `--session`, `--deliver`, `--thinking`, `--message`, `--timeout-ms`, and `--history-limit`; there is no `--timestamps` or similar flag.
  - file: [src/cli/tui-cli.ts:14](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/cli/tui-cli.ts#L14)
  - command: `nl -ba src/cli/tui-cli.ts | sed -n '1,120p'`
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **TUI settings omit any timestamp toggle:** The settings overlay only exposes `Tool output` and `Show thinking`. No timestamp visibility setting exists.
  - file: [src/tui/tui-command-handlers.ts:211](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/tui/tui-command-handlers.ts#L211)
  - command: `nl -ba src/tui/tui-command-handlers.ts | sed -n '210,250p'`
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **History rendering ignores message timestamps:** `loadHistory()` iterates messages and renders user/assistant entries from extracted text only via `chatLog.addUser(text)` and `chatLog.finalizeAssistant(text)`. It never reads or formats `message.timestamp` for display.
  - file: [src/tui/tui-session-actions.ts:311](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/tui/tui-session-actions.ts#L311)

  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Chat log components render text-only messages:** `ChatLog.addUser()` and `addPendingUser()` construct `UserMessageComponent(text)` only. Timestamps are used internally only for pending-user reconciliation, not UI output.
  - file: [src/tui/components/chat-log.ts:105](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/tui/components/chat-log.ts#L105)
