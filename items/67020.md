## Summary

Keep open. On the provided main commit `c7f18a6b9d76a31f4daed5096c0a25b3794c76ac`, Slack still accepts and documents `dmHistoryLimit` but never threads it into the Slack monitor runtime, and DM prompt assembly still excludes non-threaded DM history behind `isRoomish` gates. The latest release `v2026.4.23` still shows the same behavior, so there is no strong basis to close this as already fixed or obsolete.
