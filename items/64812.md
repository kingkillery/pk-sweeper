## Summary

Keep open. Current `main` still drops arbitrary `agents.defaults.*.params` fields like `user` on the generic runner path; only a small stream-option allowlist plus a few explicit payload wrappers are implemented. The issue is recent, concrete, and matches current code.

## Evidence

- **Generic extra-param path is still allowlisted:** `createStreamFnWithExtraParams()` only forwards `temperature`, `maxTokens`, `transport`, `openaiWsWarmup`, `cachedContent`/`cached_content`, and cache-retention-derived options. There is no passthrough for `user` or other arbitrary request-body fields.

It does not patch `payload.user`.
