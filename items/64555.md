## Summary

Keep open: current `main` does not contain the linked PR's lazy-resolution fix for WhatsApp auth paths, so there is not strong evidence that the `--profile` boundary leak is implemented or obsolete.

## Evidence

- **WhatsApp auth dir is still eagerly computed inside the extension:** `extensions/whatsapp/src/auth-store.ts` still exports `WA_WEB_AUTH_DIR = resolveDefaultWebAuthDir()` at module load. The linked PR for this issue proposed replacing that with a lazy string-like wrapper, but that change is not present in current `main`.
  - file: [extensions/whatsapp/src/auth-store.ts:39](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/whatsapp/src/auth-store.ts#L39)
  - command: `nl -ba extensions/whatsapp/src/auth-store.ts | sed -n '33,42p'`
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **Accounts path resolution still uses ambient OAuth resolution directly:** `extensions/whatsapp/src/accounts.ts` still calls `resolveOAuthDir()` directly for auth-dir discovery and default/legacy auth-dir resolution. The linked PR patch proposed routing these calls through a dedicated helper as part of the profile-state fix, but current `main` still has the old form.
  - file: [extensions/whatsapp/src/accounts.ts:53](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/whatsapp/src/accounts.ts#L53)
  - command: `nl -ba extensions/whatsapp/src/accounts.ts | sed -n '48,90p'`
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **Linked PR patch exists, but its core hunks are absent from main:** The read-only patch for PR #64563 shows the intended fix: lazy auth-dir resolution in `extensions/whatsapp/src/auth-store.ts`, helper-based OAuth-root resolution in `extensions/whatsapp/src/accounts.ts`, and a new `extensions/whatsapp/src/auth-store.lazy-dir.test.ts`. Current `main` lacks that new test file and still matches the pre-fix code in the touched files.
  - command: `curl -L --silent https://patch-diff.githubusercontent.com/raw/openclaw/openclaw/pull/64563.patch | sed -n '1,260p'; rg --files extensions/whatsapp/src | grep 'auth-store\.lazy-dir\.test\.ts$' || true`
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **Only the core barrel export is lazy:** `src/channel-web.ts` already wraps `WA_WEB_AUTH_DIR` lazily at the core barrel boundary, but the extension's own `auth-store.ts` export is still eager. That means current `main` reflects only a partial version of the linked fix, not a clear end-to-end resolution of this issue.
