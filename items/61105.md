## Summary

Keep open. Current `main` still has the reported gap: `tools.exec.pathPrepend` is applied only when exec builds a command environment, while `hasBinary()` and other startup/probe paths read `process.env.PATH` directly. The PR is directionally useful, but the review discussion also surfaced a real reload-safety concern that should be resolved before cleanup/closure.

## Risks / Open Questions

- Closing this PR would drop a still-relevant fix: current `main` does not make `tools.exec.pathPrepend` visible to `hasBinary()`-based eligibility checks or media CLI probes.
- Merging as-is may leave stale PATH entries across in-process config reloads unless the implementation also handles removal/refresh semantics and adds targeted tests around `src/config/io.ts`.
