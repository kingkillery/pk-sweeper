## Summary

Keep PR #61105 open. Current main still does not apply `tools.exec.pathPrepend` to `process.env.PATH` during config load, while `hasBinary()` still reads `process.env.PATH` directly. The PR addresses a real remaining mismatch between exec-time PATH handling and PATH-based readiness/autodetection checks.
