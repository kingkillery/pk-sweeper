## Summary

Current `main` still only exposes the health-monitor interval as the global `gateway.channelHealthCheckMinutes` setting. Per-channel/account `healthMonitor` config remains schema-limited to `{ enabled?: boolean }`, and the gateway startup path still derives `checkIntervalMs` solely from the global gateway field with a default of 5 minutes. I found no repo evidence that issue #62583 has already been implemented or obsoleted, so it should remain open.
