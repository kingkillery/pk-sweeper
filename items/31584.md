## Summary

Current main has partial non-interactive Copilot auth via environment variables, but it does not implement the requested non-interactive onboarding/login flow or a Copilot token flag. The GitHub Copilot provider is still registered only with a device-code method, docs still say the login command requires an interactive TTY, and the generic non-interactive onboarding path errors when a provider method does not implement runNonInteractive.

## Evidence

- **GitHub Copilot docs still mark login as interactive-only:** The provider docs state the device-login flow requires an interactive TTY and should not be run in a non-interactive script or CI pipeline.
