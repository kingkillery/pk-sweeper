## Summary

Keep open. Current `main` still cleans only package rename directories under the global `node_modules` root and verifies package contents after update; it does not resolve, verify, or repair the public global `openclaw` bin link or clean hidden bin-level leftovers. The report still matches a plausible gap in the updater.

## Evidence

- **Rename cleanup only targets directories under package root:** `cleanupGlobalRenameDirs` scans the supplied `globalRoot`, removes entries matching `.${packageName}-*`, and skips anything that is not a directory. That would not repair or remove a hidden npm bin symlink left in `<prefix>/bin/.openclaw-*`.
  - file: [src/infra/update-global.ts:588](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/infra/update-global.ts#L588)

  - file: [src/cli/update-cli/update-command.ts:372](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/cli/update-cli/update-command.ts#L372)
