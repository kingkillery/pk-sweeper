## Summary

Current `main` only partially covers this area: it normalizes provider-prefixed tool names during live/replay dispatch, but it does not normalize mangled tool-result IDs like `functions exec:0` or strip `functions.`/`functions ` prefixes from persisted tool-result names during transcript repair. The requested behavior is therefore not implemented on `main`, and this PR also has unresolved correctness issues from review, so there is not strong evidence to close it.

## Evidence

- **`extractToolResultId()` still returns raw IDs unchanged:** On `main` (`d3dd61f2c5bd6cea175544f3a76a68ec11fa17c2`), `extractToolResultId()` returns `toolCallId` / `toolUseId` verbatim with no normalization step. A mangled result ID such as `functions exec:0` would remain mangled.
  - file: [src/agents/tool-call-id.ts:90](https://github.com/openclaw/openclaw/blob/d3dd61f2c5bd6cea175544f3a76a68ec11fa17c2/src/agents/tool-call-id.ts#L90)
  - command: `nl -ba src/agents/tool-call-id.ts | sed -n '50,120p'`
  - sha: [d3dd61f2c5bd](https://github.com/openclaw/openclaw/commit/d3dd61f2c5bd6cea175544f3a76a68ec11fa17c2)
