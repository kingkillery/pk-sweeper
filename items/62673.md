## Summary

Keep open. Current `main` does not implement the cloud-provider env passthrough this PR proposes, so `implemented_on_main` is not supported. There is also no clean `cannot_reproduce` path here: `main` still builds the gateway service env from a narrow whitelist, while the PR discussion surfaces a real remaining gap for cloud-backed installs. At the same time, current `main` already contains a partial Bedrock mitigation (`#61194`) that weakens the PR's broadest AWS regression claim, and the PR discussion leaves an unresolved safety concern about persisting short-lived AWS secrets into the installed service environment. That combination argues for keeping the PR open for maintainer review rather than conservatively closing it.
