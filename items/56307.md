## Summary

Current `main` no longer matches the issue’s claimed hardcoded-channel architecture, but it also does not implement the requested automatic `thread: true` / `mode: "session"` injection for `sessions_spawn`. This is still an unimplemented feature request, so conservative cleanup should keep it open.

## Evidence

- **`sessions_spawn` still requires explicit thread/session inputs:** `sessions_spawn` reads `params.thread === true` directly and forwards that boolean; there is no channel-driven auto-injection path in the tool entrypoint.
  - file: [src/agents/tools/sessions-spawn-tool.ts:217](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/agents/tools/sessions-spawn-tool.ts#L217)
  - command: `nl -ba src/agents/tools/sessions-spawn-tool.ts | sed -n '60,280p'`
- **Core still errors if `mode:"session"` is used without `thread:true`:** Subagent spawn resolves `spawnMode`, then rejects `mode="session"` unless `requestThreadBinding` is true, which is the opposite of the requested automatic injection behavior.
  - file: [src/agents/subagent-spawn.ts:571](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/agents/subagent-spawn.ts#L571)
  - command: `nl -ba src/agents/subagent-spawn.ts | sed -n '470,620p'`
- **Core thread-binding policy is already generic, not hardcoded to Feishu:** The policy layer derives placement from `getChannelPlugin(...).conversationBindings.defaultTopLevelPlacement` and channel/account config keys like `threadBindings.spawnSubagentSessions`; this means the issue’s cited `agentChannel === "feishu"` style core check does not match current `main`. A repo-wide search for that exact hardcoded pattern found no core matches.
  - file: [src/channels/thread-bindings-policy.ts:62](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/channels/thread-bindings-policy.ts#L62)
  - command: `rg -n 'agentChannel === "feishu"|agentChannel==="feishu"|feishu.*sessions_spawn|thread.*feishu|subagent.*feishu' src extensions docs -g '!**/dist/**' && nl -ba src/channels/thread-bindings-policy.ts | sed -n '1,260p'`
- **Channel-specific spawn behavior already lives in plugin-owned hooks:** Feishu subagent binding is handled inside the Feishu extension’s `subagent_spawning` hook, not via a core channel-name branch.
  - file: [extensions/feishu/src/subagent-hooks.ts:288](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/extensions/feishu/src/subagent-hooks.ts#L288)
  - command: `nl -ba extensions/feishu/src/subagent-hooks.ts | sed -n '284,360p'`
- **Docs still document explicit `thread: true` rather than auto-injection:** The subagents docs say `thread` defaults to false, that `mode:"session"` requires `thread:true`, and the quick flow tells users to spawn with `thread: true`. Channel config docs also describe `spawnSubagentSessions` as an opt-in for `sessions_spawn({ thread: true })`, not as implicit parameter injection.
  - file: [docs/tools/subagents.md:96](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/docs/tools/subagents.md#L96)
  - command: `nl -ba docs/tools/subagents.md | sed -n '90,130p' && nl -ba docs/gateway/config-channels.md | sed -n '272,330p'`
