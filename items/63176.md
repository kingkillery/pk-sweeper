## Summary

Keep open. Current `main` still does not ship the PR's core feature: there is no `api.runtime.acp` surface, no `plugins.allowAcpSpawn` config key, and the public plugin-runtime docs still document `subagent` as the available orchestration helper. One ancillary Discord thread-binding normalization from the PR's last commit is already present on `main`, but that does not implement the requested plugin ACP runtime. This also does not fit a `clawhub` close: `VISION.md` explicitly says plugin API expansion belongs in core when a useful feature cannot yet be built as a plugin.
