## Summary

Current main no longer installs the Windows gateway as a PowerShell-script Scheduled Task. The shipped Windows service path writes a `gateway.cmd` wrapper, registers the Scheduled Task against that file, uses hidden detached fallback launch on Windows, and documents the native-Windows install behavior. That makes this PowerShell-specific flashing-window report obsolete on current main.

## Close Comment

这次 Codex review 检查了 current `main` 和已发布版本。现在 OpenClaw 的 Windows 网关安装路径已经不是“计划任务直接跑 PowerShell 脚本”了：`src/daemon/schtasks.ts:242` 会生成 `gateway.cmd`，`src/daemon/schtasks.ts:827` 用该 `.cmd` 作为 Scheduled Task 的 `/TR`，而回退直启路径也用了 `windowsHide: true`（`src/daemon/schtasks.ts:316`）。相关原生 Windows 行为也已经写入文档：`docs/platforms/windows.md:40` 和 `docs/cli/onboard.md:116`。这套实现已随 `v2026.4.22`（`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`）发布，所以这个“PowerShell 脚本闪现控制台窗口”的报错在 current main 上已经不再对应现状，按已实现关闭。若你在 `v2026.4.22` 或更新版本里仍能复现，请带上实际生成的任务动作、脚本内容和复现录屏再开新 issue。
