## Summary

Current main still matches the reported gap. The incomplete-turn terminal cooldown writes in runEmbeddedPiAgent call maybeMarkAuthProfileFailure without modelId, while auth-profile cooldown scoping depends on markAuthProfileFailure receiving modelId to set cooldownModel. The linked PR #67662 appears to be the right focused fix path, but it is still open/unmerged in the provided GitHub context, so the issue should remain open until that fix or an equivalent lands.

## Best Possible Solution

Keep #67661 open and land PR #67662 or an equivalent fix that passes the active modelId through both incomplete-turn auth-profile failure writes, with regression coverage in src/agents/pi-embedded-runner/run.incomplete-turn.test.ts proving cooldownModel is recorded for reasoning-only exhaustion and generic incomplete-turn cooldown persistence.
