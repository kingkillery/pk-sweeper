## Summary

Current main already routes Discord reply traffic through the configured proxy in both the normal REST client and the webhook-based final-reply path, and the changelog says this shipped in v2026.4.5.

## Evidence

## Close Comment

Closing from a Codex review because this looks implemented on current `main` and already documented as shipped in `v2026.4.5`. The Discord final-reply path now attempts webhook delivery through the resolved account proxy (`extensions/discord/src/monitor/reply-delivery.ts:303`, `extensions/discord/src/send.outbound.ts:372`, `extensions/discord/src/send.outbound.ts:380`), and the standard REST client also injects the same proxy-aware fetch into Discord sends (`extensions/discord/src/client.ts:53`, `extensions/discord/src/client.ts:88`). There is regression coverage for both webhook and REST proxy sends (`extensions/discord/src/send.webhook.proxy.test.ts:53`, `extensions/discord/src/client.proxy.test.ts:28`). The changelog entry for `2026.4.5` explicitly says Discord now keeps REST, webhook, and monitor traffic on the configured proxy (`CHANGELOG.md:1214`).
