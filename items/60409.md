## Summary

Keep open. Current main does not implement PR #60409's reset-archive fallback: `readSessionMessages` still reads only the first existing primary transcript candidate and returns `[]` when none exists, so a missing primary transcript with a valid `.reset.<timestamp>` archive still appears empty. The related #68765 PR is explicitly a different compaction-checkpoint-chain fix, not a superseding reset-archive fallback.

## Evidence
