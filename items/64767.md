## Summary

Keep open. Current `main` has partial transcript-scan mitigations, but the hang class described in #64767 is not conservatively fixed: several live gateway paths still synchronously read entire session `.jsonl` files into memory and split them with a regex or otherwise materialize the full string.

## Risks / Open Questions

- Closing as `implemented_on_main` would hide that current gateway code still contains the same eager full-transcript read pattern on live paths.
- The specific abort/retry bloat scenario from the report is still represented in current code via `persistAbortedPartials` -> `appendAssistantTranscriptMessage` -> `transcriptHasIdempotencyKey`.
- Some related PR/work may have reduced impact on listing/usage surfaces, but there is not high-confidence evidence that current `main` eliminates the gateway hang class end-to-end.
