## Summary

Current main does not implement the requested Kimi/Moonshot usage and balance display. The provider-usage allowlist and type union exclude `kimi`, `kimi-coding`, and `moonshot`; the bundled Kimi and Moonshot provider plugins register auth/catalog/runtime behavior but no usage auth or usage snapshot hooks; and the docs still list the current usage-window providers without Kimi. The linked PR #20343 is useful prior art from the issue body, but the provided context says it was closed unmerged, so it is not shipped behavior.

## Best Possible Solution

Keep the issue open. Implement Kimi/Moonshot usage as provider-owned work in the bundled Kimi/Moonshot plugin using the existing `resolveUsageAuth` and `fetchUsageSnapshot` hooks, add the necessary `UsageProviderId`/label/provider-list entries, support the relevant Kimi/Moonshot credential aliases, add focused fetch/auth/parser tests, and update the usage/status docs. Treat PR #20343 as prior art, but fix the provider aliasing concerns before reusing its UI approach.
