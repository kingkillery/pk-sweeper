## Summary

Keep open. The request is a coherent, recent feature request for a broader Control UI workspace file tree/editor. Current evidence shows core has agent workspace file RPCs and a Control UI files surface for bootstrap files, but that does not clearly satisfy the requested full workspace browser/editor with arbitrary text-file navigation, pinned shortcuts, and read-only/edit modes.

## Evidence

- **Issue is recent and actionable:** Issue #59088 was opened 2026-04-01, has only the enhancement label, no protected labels, no comments, and provides a concrete proposed Control UI feature.
- **Existing API is narrower than requested:** Gateway protocol documents agents.files.list, agents.files.get, and agents.files.set as managing bootstrap workspace files exposed for an agent, which is related but not the full file tree/editor requested for ~/.openclaw/workspace and memory/*.md.
  - file: [docs/gateway/protocol.md:438](https://github.com/openclaw/openclaw/blob/2e0ae56b1a580103f71b9b7f79b3d638e3c3698d/docs/gateway/protocol.md#L438)
  - sha: [2e0ae56b1a58](https://github.com/openclaw/openclaw/commit/2e0ae56b1a580103f71b9b7f79b3d638e3c3698d)
- **Related Control UI surface exists but does not prove completion:** Related issue #56637 describes an Agent -> Agents Files tab and agents.files.list returning core files such as AGENTS.md, SOUL.md, TOOLS.md, IDENTITY.md, USER.md, and HEARTBEAT.md. That supports partial overlap, not enough evidence that the requested file browser/editor is implemented.

I therefore avoided mutation and relied on provided GitHub context plus public repo/docs inspection.
