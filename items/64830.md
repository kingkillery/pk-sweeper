## Summary

Current `main` does not implement the requested Web UI default-resolution behavior. New-session thinking defaults still come from provider/model fallback returned by `sessions.list`, not `agents.defaults.thinkingDefault`, and `agents.defaults.reasoningDefault` is not present in the current defaults schema. The issue also has an open linked PR (#66098), so there is active in-flight work rather than a shipped fix.

## Evidence

- **Gateway session defaults still use provider/model thinking fallback:** `getSessionDefaults()` builds the `sessions.list` defaults payload from the resolved provider/model and sets `thinkingDefault` via `resolveThinkingDefaultForModel(...)`; it does not read `cfg.agents?.defaults?.thinkingDefault` here.
  - file: [src/gateway/session-utils.ts:1074](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/gateway/session-utils.ts#L1074)

- **Web UI new-session selector consumes that gateway/default payload:** The chat thinking selector labels its default as `activeRow?.thinkingDefault ?? state.sessionsResult?.defaults?.thinkingDefault ?? resolveThinkingDefaultForModel(...)`, so current UI behavior follows the model-derived gateway defaults instead of a separate config-default path.
  - file: [ui/src/ui/chat/session-controls.ts:208](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/ui/src/ui/chat/session-controls.ts#L208)
