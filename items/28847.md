## Summary

Keep open. Current main now covers a substantial part of the report with auth-profile rotation, per-profile cooldowns, rate-limit classification, and model fallback, but the specific Gemini/project bucket grouping requested in #28847 is still missing. The linked aashari fork/comment is useful prior art, but it also notes bucket/project grouping is not native there, and it is not evidence that OpenClaw main has the requested bucket-aware behavior.

## Evidence

- **Main implements related profile rotation/cooldowns:** The model failover docs describe auth-profile rotation before model fallback, ordered auth profiles, cooldowned profiles, and rate-limit cooldown behavior. This partially addresses retry storms, but the described unit of cooldown is an auth profile/model, not a quota bucket/project group.
  - file: [docs/concepts/model-failover.md:10](https://github.com/openclaw/openclaw/blob/bd796d1c85c7dec0dcb89ed11502e4271ded1314/docs/concepts/model-failover.md#L10)

  - sha: [bd796d1c85c7](https://github.com/openclaw/openclaw/commit/bd796d1c85c7dec0dcb89ed11502e4271ded1314)
- **Public auth config has no bucket/quota-group field:** AuthProfileConfig exposes provider, credential mode, email, and displayName; AuthConfig exposes profiles, order, and cooldown knobs. There is no first-class bucket/project/quota-group field for grouping multiple keys that share a Gemini project quota.
  - file: [src/config/types.auth.ts:1](https://github.com/openclaw/openclaw/blob/bd796d1c85c7dec0dcb89ed11502e4271ded1314/src/config/types.auth.ts#L1)

It does not model shared project buckets or fan out cooldowns across profiles with shared quota metadata.
