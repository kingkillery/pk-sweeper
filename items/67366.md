## Summary

Keep open. Current main still has the Telegram plaintext replacement path call `prompter.text(...)` and then `.trim()` the returned value without a nullish guard, which matches the reported crash when the prompt returns `undefined`. The issue discussion also points to the same root cause, with #67374 still open and #67733 closed unmerged rather than shipped.

## Best Possible Solution

Keep #67366 open until the credential prompt helper guards `prompter.text` results before trimming and adds a focused regression test for `undefined` text input on the token replacement path. The likely code change belongs in `src/channels/plugins/setup-wizard-helpers.ts`, with coverage in `src/channels/plugins/setup-wizard-helpers.test.ts` and/or the channel onboarding e2e tests.
