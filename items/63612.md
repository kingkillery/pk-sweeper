## Summary

Current `main` has a targeted replay-history mitigation for malformed assistant string content, but the compaction path still calls upstream `estimateTokens()` directly without local null-guard wrapping. That is not strong enough to conservatively close #63612 as implemented or obsolete.

## Risks / Open Questions

- Malformed replay blocks other than raw assistant string content may still reach upstream `estimateTokens()` and reproduce the crash.
- There is no current main proof that undefined `block.text`, `block.thinking`, `block.name`, or missing `toolResult/custom.content` are treated as zero-length instead of throwing.
- The linked discussion suggests this bug family may also affect non-main lanes/channels, so closing this item could hide an unresolved shared compaction defect.
