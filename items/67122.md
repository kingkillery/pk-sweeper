## Summary

Current main does not support the issue’s stated `workspace-*` blacklist root cause: `assertLocalMediaAllowed` only applies that blacklist when `localRoots` is omitted, and `openclaw-tools` threads an inferred session workspace into both `image` and `pdf`. But `pdf` still lacks the workspace-relative path resolution that `image` has, so a real sub-agent workspace failure remains plausible. That ambiguity is too high for a conservative auto-close.

## Evidence

- **Blacklist is only active for implicit roots:** `assertLocalMediaAllowed` now wraps the `workspace-*` rejection in `if (localRoots === undefined)`, so callers that pass explicit roots do not hit the reported early blacklist path.
