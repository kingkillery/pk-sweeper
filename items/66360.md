## Summary

Keep open. Current `main` still documents and implements `session.maintenance.rotateBytes` as `sessions.json` rotation only, with no per-transcript `.jsonl` size/line cap in the `session.maintenance` schema. The only current mitigation is whole-directory disk-budget cleanup, which can delete old/orphan transcripts or non-active sessions but does not rotate an active oversized transcript. The issue also already has active follow-up work in open PR #66546, which remains unmerged and off `main`.
