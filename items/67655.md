## Summary

Keep #67655 open. Current main still has the Windows allowlist policy scoped to `cmdInvocation`, while the surrounding command-resolution code recognizes PowerShell and `pwsh` as shell wrappers and strips `powershell -Command` for Windows allowlist analysis. The PR is a focused fix for a real remaining gap, though the submitted diff needs the stale message assertions from the review discussion addressed before it can merge cleanly.

## Best Possible Solution

Keep the PR open and ask for a small follow-up on the same change: broaden `windowsShellWrapperBlocked` to all Windows inline shell wrappers, add the PowerShell `cmdInvocation: false` regression, update stale assertions and docs/guidance to mention `powershell -Command`, and either remove or document the now-non-policy `cmdInvocation` parameter.
