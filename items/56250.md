## Summary

Current `main` does not implement this PR's headline behavior change. Bundled/internal hook discovery is still explicitly opt-in on `main`: the loader returns early unless hooks are configured, the helper treats missing `hooks.internal` as disabled, docs say startup discovery is skipped until configured, and tests assert empty/default configs load zero hooks. `main` did independently land the non-activating plugin-hook registration guard, so the PR is only partially superseded.
