## Summary

Keep #67703 open. Current main still has the reported product gap: `gateway.auth.mode="none"` accepts the auth layer, but a first-time unauthenticated loopback probe is sent without device identity and the WebSocket handshake then rejects non-Control-UI/device-less operator clients with `1008: device identity required`. The linked #69066 RFC does cover the broader service-identity direction and explicitly lists #67703, but it is still an open RFC rather than a shipped fix or clearly canonical maintainer-owned replacement for this concrete bug report.

## Best Possible Solution

Keep this issue open as a concrete bug until main either ships a targeted safe path for internal/read probes under `gateway.auth.mode="none"` or the broader #69066 service-identity design lands. The implementation should add a gateway auth/handshake regression test for a first-time loopback probe with no token/password and no device identity, plus an exec-approval/internal backend coverage case if that pipeline is affected.
