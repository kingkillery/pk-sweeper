## Summary

Keep open. Current `main` still routes PDF fallback extraction through `extensions/document-extract/document-extractor.ts` without passing `standardFontDataUrl` to PDF.js, and the existing tests/docs show that this fallback renderer is still active for non-native PDF handling. The provided issue timeline also points to an open follow-up PR (`#70936`), which is consistent with the fix not yet being implemented on `main`.

## Evidence

- **Fallback renderer still omits the PDF.js font path:** `extractPdfContent()` calls `pdfJsModule.getDocument({ data: new Uint8Array(request.buffer), disableWorker: true })` with no `standardFontDataUrl` option in the active fallback path.
