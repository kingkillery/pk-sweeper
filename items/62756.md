## Summary

Keep open. This is a broad reliability/meta issue, not a clean close candidate: several cited details no longer match current `main`, but the overall stability concern is still understandable and there is not enough concrete evidence to justify `implemented_on_main`, `cannot_reproduce`, or `incoherent` with high confidence.

## Evidence

- **Umbrella product-feedback issue, not one scoped repro:** The issue body bundles five different subsystems (`system event`, cron delivery, `sync_daemon`, session crashes, ACPX session handling) into a release-priority request to 'stop adding new features' rather than one independently verifiable bug. The only follow-up comment reiterates the same umbrella complaint and does not add fresh reproduction data.
- **Current main still has explicit system-event and cron wake/queue support:** Current docs and code describe `openclaw system event --mode now` as a supported path that enqueues a system event and can wake heartbeat immediately, and cron manual runs now return an `{ ok: true, enqueued: true, runId }` ack. That means at least part of the issue body is talking about behavior that has moved since the report's framing, but code inspection alone does not prove all real-world failures are gone.
