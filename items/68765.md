## Summary

Keep open. Current `main` still reads only a single transcript file for `readSessionMessages`, so the reported `chat.history` truncation across compaction checkpoint chains is not implemented on `main`. The PR also has unresolved correctness questions around reset/restore epochs and checkpoint overlap handling, so there is not strong evidence for an auto-close.
