## Summary

PR #60848 should stay open. Current main at a5f5608d still has the Feishu behavior the PR is trying to fix: topic-session inbound context only persists ctx.rootId as MessageThreadId, and outbound Feishu fallback-to-threadId sends do not consistently propagate replyInThread to text/card/media paths. The PR discussion also shows reviewers found media and local-image gaps, and the author later addressed them on the PR branch, so the PR still contains useful unmerged Feishu-channel bug-fix work.
