## Summary

Keep open. Current `main` still hardcodes UTF-8 decoding in the affected child-process paths, the docs still describe garbled Chinese output on native Windows as a live workaround issue, and the PR itself has unresolved review concerns about possible Windows regressions. That is not strong enough evidence for any close reason.
