## Summary

Keep open. Current main still hard-codes the Control UI chat message group width, and the config schema/bootstrap contract/docs do not expose a chat max-width or custom CSS option. The provided related ClawSweeper matches are about other Control UI chat behaviors and do not supersede this concrete layout/config request.

## Best Possible Solution

Keep the issue open and implement a narrow Control UI/WebChat layout option rather than closing it. A good path is to add a typed config field such as `gateway.controlUi.chatMessageMaxWidth`, validate it conservatively, include it in generated config metadata/docs, expose it through `ControlUiBootstrapConfig`, apply it as a CSS custom property on the Control UI shell/root, and update `ui/src/styles/chat/grouped.css` to use `var(--chat-message-max-width, min(900px, 68%))` with focused schema/bootstrap/UI tests.
