## Summary

Current `main` does not implement this Mattermost recovery path yet, and the PR still appears to need follow-up review before it is ready to land. The underlying bug report matches the shipped code: repeated `getBotUpdateAt()` failures are only logged today, so a zombie WebSocket would stay up until something else closes it.
