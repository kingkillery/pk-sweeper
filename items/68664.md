## Summary

Keep open. Current `main` partially addresses the report by applying known runtime legacy migrations before validation, but it does not repair or strip obsolete flat `memorySearch` keys such as `chunkSize`, `chunkOverlap`, and `maxResults` on the gateway load path. Those keys are still outside the strict schema, while unknown-key cleanup still lives in `doctor --fix`, so the reported startup failure remains plausible for already-nested legacy configs.
