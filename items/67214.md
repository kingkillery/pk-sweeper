## Summary

Keep open. I did not find strong evidence that current `main` fixes or obsoletes this regression. The current cron delivery path still accepts `delivery.channel: "telegram"` with no explicit `to` and forwards that unresolved shape into outbound target resolution, there is no targeted regression test for the reported `agent:main:main` + `announce:telegram` case, and GitHub search did not surface a linked fix PR for #67214. The extra user comment also corroborates that this was not a one-off report.
