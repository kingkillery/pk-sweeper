## Summary

Keep PR #67766 open. Current main still does not implement the explicit provider-config fallback requested by the PR: `resolveGatewayModelSupportsImages()` only consults the gateway catalog plus hardcoded Foundry/claude-cli shims, then returns `false` on catalog miss. The linked Qwen path remains relevant because the bundled Qwen plugin intentionally omits `qwen3.6-plus` from Coding Plan catalogs, and the current regression tests do not cover catalog miss plus explicit `models.providers.*.models[].input: ["text", "image"]`. The related PR #67768 is overlapping but proposes a Qwen pattern heuristic; #67766 is still a useful, more general candidate because it respects explicit user config and its alias-normalization review finding was addressed in the PR head commit.

## Best Possible Solution

Keep #67766 open for maintainer review or adaptation. The best path is an alias-aware explicit-provider fallback, with tests for catalog hit precedence, catalog miss plus explicit image input, text-only input, missing config, and provider aliases. Coordinate with #67759 and the overlapping #67768; #67766 appears to be the more general implementation direction than hardcoding Qwen vision model patterns.
