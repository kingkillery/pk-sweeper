## Summary

Current main does not ship Chatterbox, but the request fits OpenClaw’s documented plugin boundary: vendor-specific TTS integrations belong as speech-provider plugins published via ClawHub/npm rather than added to core by default.

## Evidence

- **Vision prefers plugins for optional capability:** The project vision says core stays lean, optional capability should usually ship as plugins, and the bar for adding optional plugins to core is intentionally high.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/VISION.md#L54)
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **Community plugin path is the canonical discovery surface:** Docs say community plugins extend providers/capabilities, are published on ClawHub or npm, and ClawHub is the canonical discovery surface.
  - file: [docs/plugins/community.md:9](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/docs/plugins/community.md#L9)
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)

## Risks / Open Questions

- Maintainers could still choose to bundle a local TTS provider later if product priorities change, but current repo vision and plugin docs point this request to the external plugin path.
- A ClawHub/npm Chatterbox speech-provider plugin may not exist yet; this decision is about scope ownership, not availability.

## Close Comment

Closing this as a ClawHub/plugin-scope request after Codex review of current `main` (`56fe2aa`). OpenClaw’s current direction is to keep optional vendor capability in plugins rather than core: `VISION.md:54-61` says optional capability should usually ship as plugins, `docs/plugins/community.md:9-22` points community extensions to ClawHub/npm, and `docs/plugins/sdk-provider-plugins.md:486-507` documents `api.registerSpeechProvider(...)` as the supported way to add a TTS provider. The manifest contract in `docs/plugins/manifest.md:404-410` also treats speech-provider ownership as plugin-owned. Current bundled TTS docs (`docs/tools/tts.md:10-20`) list only ElevenLabs, Google Gemini, Microsoft, MiniMax, OpenAI, and xAI, and there is no Chatterbox implementation on `main`. If someone wants Chatterbox support, the right path is an external speech-provider plugin published through ClawHub/npm rather than expanding OpenClaw core.
