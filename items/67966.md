## Summary

Keep open. Current `main` still enables persistent Playwright only for `remote-cdp`, not `local-managed`, and the local-managed fallback path still hard-blocks strict SSRF navigations that need redirect-hop inspection. This feature request is concrete, recent, and not implemented on `fa976e5b93b3819f9e137fe08635764edee38290`.

## Evidence

- **Local-managed still does not use persistent Playwright:** `getBrowserProfileCapabilities()` returns `usesPersistentPlaywright: true` only for non-loopback `remote-cdp`; the `local-managed` branch still returns `false`.

## Risks / Open Questions

- I did not run a live browser reproduction because this review is read-only and limited to inspection commands, so the conclusion is based on current code and docs rather than an interactive runtime trace.
- A fix is likely non-trivial because local-managed profiles also advertise capabilities not shared with `remote-cdp` (`supportsPerTabWs`, `supportsReset`, `supportsManagedTabLimit`), so enabling persistent Playwright there may need careful behavior reconciliation rather than a one-line flag flip.
