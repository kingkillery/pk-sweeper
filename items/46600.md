## Close Comment

Closing this as implemented on current `main` after a Codex review. The cron isolated-run path now resolves fallbacks through `src/cron/isolated-agent/run-fallback-policy.ts:5` and forwards them into `runWithModelFallback` from `src/cron/isolated-agent/run-executor.ts:96`, so cron jobs inherit `agents.defaults.model.fallbacks` when a cron payload overrides only the primary model. Current regression coverage in `src/cron/isolated-agent/run.cron-model-override-forwarding.test.ts:150` specifically asserts that cron model overrides preserve the configured default fallback chain. There is also a matching changelog entry in `CHANGELOG.md:4724` documenting this behavior. If you are still seeing a problem on a current build, please open a new issue with fresh run logs and the exact cron payload/config, especially if the remaining problem is in heartbeat or delivery rather than cron run failover itself.
