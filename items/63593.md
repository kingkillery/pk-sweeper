## Summary

Keep open. Current `main` still has the false-positive downgrade path for unresolved explicit registry tags: `resolveTargetVersion()` can return `null` for `--tag latest`, and `updateCommand()` still treats `targetVersion == null` as downgrade risk in `src/cli/update-cli/shared.ts:78` and `src/cli/update-cli/update-command.ts:1080`. The current test file also lacks the proposed unresolved-`latest` regression coverage, while the PR discussion shows the contributor already narrowed the fix to preserve the guard for unresolved non-`latest` tags.

## Evidence

- **Current main still flags unknown target versions as downgrade risk:** In `updateCommand()`, `downgradeRisk` is still set when `currentVersion != null` and `(targetVersion == null || (cmp != null && cmp > 0))`. That means an explicit `--tag latest` lookup that fails to resolve still enters the downgrade-confirmation path on current `main`.
