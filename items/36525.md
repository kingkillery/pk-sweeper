## Summary

Keep open. Current `main` still uses the old `workspaceAccess` semantics the issue asks to change: `rw` switches `/workspace` to the agent workspace, while `none`/`ro` keep `/workspace` on the sandbox workspace and block writes. Docs and tests on `main` explicitly describe and assert that behavior, so this feature request is not implemented on current `main`.
