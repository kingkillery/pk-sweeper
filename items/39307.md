## Summary

Current main has partial work for compaction visibility and recovery, but it does not fully satisfy issue #39307. OpenClaw now shows Control UI compaction toasts and can emit channel notices when `agents.defaults.compaction.notifyUser` is enabled, and it reinjects post-compaction startup guidance for the agent. However, docs still say compaction is silent by default, and the post-compaction prompt does not tell the assistant to explicitly disclose the reset to the user or offer reloading prior docs/context.
