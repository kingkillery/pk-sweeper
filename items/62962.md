## Summary

Keep open. Current `main` does not implement the proposed per-tool security classification properties. Tool metadata still exposes only `ownerOnly`/`displaySummary`, approval and mutation handling are still driven by hardcoded name/action heuristics, and the conformance snapshot does not track classification metadata. The issue is also only 16 days old, so `stale_insufficient_info` does not apply.
