## Summary

Keep #58755 open. It is a recent, plausible Feishu plugin feature request for a first-party configurable probe TTL. Existing ClawHub workarounds are useful evidence that users are affected, but they patch installed/source files and are not a durable substitute for an official config option.

## Evidence

- **Issue is recent and unprotected:** #58755 was opened 2026-04-01 by a non-maintainer, has no protected labels, and is well-scoped: add/increase configurable Feishu probe success TTL for `/open-apis/bot/v3/info`. It is not eligible for stale-insufficient-info because it is under 60 days old and includes current behavior, expected behavior, impact, and workaround.
- **Public Feishu docs expose quota flags, not probe TTL:** The current Feishu docs include quota optimization flags for `typingIndicator` and `resolveSenderNames`, and a search for `probeTTL` on the same page returned no match. That suggests the requested knob is not already documented as implemented.
  - file: [docs/channels/feishu.md:348](https://github.com/openclaw/openclaw/blob/44114328b4e73074ac72126f1230ddbf9c1ea8b5/docs/channels/feishu.md#L348)
  - command: `web find docs.openclaw.ai/channels/feishu for probeTTL and quota flags`
  - sha: [44114328b4e7](https://github.com/openclaw/openclaw/commit/44114328b4e73074ac72126f1230ddbf9c1ea8b5)
- **Related prior report supports the quota concern:** Issue #13264 reported the same Feishu bot-info probing pressure, described a local 24-hour cache in `extensions/feishu/src/probe.ts`, and requested an official solution for `@openclaw/feishu`. That prior report is evidence the request is coherent and user-impacting, even though it was closed separately.
  - command: `web open github issue #13264`
- **ClawHub workarounds exist but are not equivalent to core/plugin support:** ClawHub has Feishu probe/cache workaround skills, including `https://clawhub.ai/bobbycats/feishu-probe-optimize` and `https://clawhub.ai/skills/feishu-cache-guardian`. They are useful community contributions, but both describe patching `extensions/feishu/src/probe.ts` or installed OpenClaw plugin files after updates, which reinforces the issue's requested durable config surface rather than making it obsolete.
  - command: `web search ClawHub Feishu probe optimize`
