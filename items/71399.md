## Summary

Keep open. Current main does not implement an `agent:turn:end` internal hook, and the PR is the current focused implementation attempt for open issue #37833 rather than obsolete cleanup. It still needs maintainer/code review follow-up because the advertised failure `errorCode` is not in the proposed payload and the PR uses unbounded fire-and-forget hook dispatch despite an existing bounded helper.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to finish this PR or a successor against current main by emitting `agent:turn:end` from `AcpSessionManager.recordTurnCompletion()`, including `sessionKey`, `success`, `durationMs`, and optional `errorCode`, using bounded non-blocking hook dispatch or documenting why this hook is intentionally different, then covering success/failure manager emission plus the type guard in tests and updating the internal hooks docs if the event becomes public operator-facing behavior.
