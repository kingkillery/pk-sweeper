## Summary

Current `main` still builds the Telegram menu as native -> plugin -> custom, and the budget helper trims from the tail when the `setMyCommands` payload is too large. That matches the PR's reported bug, so this is not already implemented on `main` and should stay open for maintainer review rather than cleanup-closing.

## Evidence

- **Current main still appends custom commands last:** `registerTelegramNativeCommands` currently assembles `allCommandsFull` as native commands, then plugin commands, then `customCommands`, which is the exact ordering the PR identifies as causing operator-defined commands to be dropped first under menu-budget pressure.
