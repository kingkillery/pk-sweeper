## Summary

Keep open. Current `main` still shows live task detail from `progressSummary`, but the cron runtime only creates a running task row and later writes terminal state/`terminalSummary`; there is no current-main path for managed/isolated cron runs to publish intermediate `progressSummary` updates into the task ledger.

## Evidence

- **Live task surfaces still depend on `progressSummary` while running:** `formatTaskStatusDetail` returns only `task.progressSummary` for `running`/`queued` tasks, so `/tasks` and related status views remain blank unless producers populate that field.
