## Summary

Keep open. Current `main` still sends heartbeat ticks into the session as verbatim user messages, and the Control UI history path only suppresses assistant-side silent/repair noise. Existing webchat heartbeat suppression covers assistant `HEARTBEAT_OK` chatter, not the user heartbeat prompt itself, so the requested Control UI filtering is not implemented on `main`.
