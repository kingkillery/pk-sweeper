## Summary

Keep open. Current `main` still ties OpenAI `prompt_cache_key` to the run `sessionId`, and isolated cron explicitly creates a fresh session for every run. That means the requested stable cross-run cache affinity for recurring isolated cron jobs is not implemented on `main`. A custom `session:...` cron target can persist context across runs, but that is a behavioral workaround, not the same as preserving isolated-run semantics with stable first-turn cache affinity.
