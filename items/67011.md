## Summary

Keep open. Current `main` still has the false-positive behavior the PR is trying to fix: diagnostic payload traversal uses a global `WeakSet` for all visited objects, so repeated shared references are still treated as `"[Circular]"`. The proposed active-path-plus-memo approach is not present on `main`, and the regression coverage from the PR is also absent.

## Evidence

- **Current main still uses global seen-set traversal:** `visitDiagnosticPayload` initializes `const seen = new WeakSet<object>()`, returns `"[Circular]"` when `seen.has(input)`, and never removes objects after finishing a branch. That still misclassifies acyclic shared references encountered from sibling branches.
