## Summary

Current main does not show the downgrade confirmation for known upgrades: the warning path is still gated by downgradeRisk, which excludes cmp < 0. The PR's current approach would instead add a confirmation gate for normal upgrades, matching the bot review concern and risking non-interactive upgrade regressions.

## Best Possible Solution

Close this PR. Keep the current downgrade-only confirmation gate; if maintainers want to improve the wording for unknown registry targets like 'latest (unknown)', that should be a new narrow copy-only PR that preserves non-interactive upgrade behavior and adds focused regression coverage.
