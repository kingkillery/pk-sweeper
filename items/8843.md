## Summary

Keep open. Current `main` has partial coverage: Telegram inbound media is auto-downloaded into local `MediaPath`/`MediaPaths`, and captioned no-download fallbacks now prepend `[file_id:...]` for the primary media tag. But the exact request is not implemented generally for Telegram audio/document attachments: there is no generic `file_id` exposure for plain `<media:audio>` / `<media:document>` placeholders, and Telegram does not expose a `download-file` message action.

- Successful downloads already provide `MediaPath`, so some of the original use case may now work; closing would still be premature because the requested generic `file_id` exposure / Telegram download action is not fully present.
