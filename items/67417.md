## Summary

Keep #67417 open. Current main still archives backup sources through `tar.c` without an ENOENT skip path, while session maintenance can remove transcript files during normal store saves. PR #67472 is a directly related open fix, but it is not merged or shipped, so the underlying core backup race remains actionable in this repo.

## Best Possible Solution

Keep the issue open until a core fix lands on main. The likely path is to finish PR #67472 or an equivalent change in `src/infra/backup-create.ts`, add focused regression coverage for a file disappearing during archive creation, and decide whether skipped files must be represented in the in-archive manifest as well as the CLI summary.
