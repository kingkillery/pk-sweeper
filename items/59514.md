## Summary

Keep open. The discussion shows a related Mantle/OpenAI-compatible provider path was added, but the specific requested behavior, Bedrock Converse-format gateway proxy with plain Bearer auth and no AWS SDK SigV4 signing, is still described as unsupported. This is a plausible feature request, not a cleanup-close candidate.

## Evidence

- **Issue request is specific to Bedrock Converse bearer auth:** The issue asks for `api: "bedrock-converse-stream"` plus `auth: "api-key"` to send `Authorization: Bearer <api-key>` and skip SigV4 signing for Bedrock gateway proxies.
  - sha: [44114328b4e7](https://github.com/openclaw/openclaw/commit/44114328b4e73074ac72126f1230ddbf9c1ea8b5)
- **Prior discussion says Mantle is related but not equivalent:** Comment 4192076077 says Mantle covers the official AWS gateway path and OpenAI-compatible proxies, but explicitly says a Converse-format proxy with Bearer auth remains unsupported and would require a separate HTTP transport path or upstream SDK support.
  - sha: [44114328b4e7](https://github.com/openclaw/openclaw/commit/44114328b4e73074ac72126f1230ddbf9c1ea8b5)
- **Public docs describe Mantle as OpenAI-compatible:** The public Bedrock Mantle docs describe `amazon-bedrock-mantle` as using OpenAI-compatible APIs, which supports the conclusion that Mantle is related but does not directly implement the requested Bedrock Converse proxy mode.
  - file: [docs/providers/bedrock-mantle.md](https://github.com/openclaw/openclaw/blob/44114328b4e73074ac72126f1230ddbf9c1ea8b5/docs/providers/bedrock-mantle.md)
