## Summary

Keep open. Current `main` still hardcodes the embedding retry/concurrency knobs, so the feature is not implemented on `main`, and the PR as described does not line up with the current config contract: memory search resolves from `agents.defaults` / `agents.list[]`, while the validated schema and docs expose no `memorySearch.embedding.*` section. The review discussion shows one runtime bug was fixed on the PR branch, but the schema/resolution/docs gaps still make this unsuitable for conservative auto-close.
