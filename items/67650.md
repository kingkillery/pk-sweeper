## Summary

Keep #67650 open. Current main still documents and generates a Slack manifest with `assistant:write` but without `assistant_view` or `assistant_thread_started`, while the Slack runtime currently registers only `message` and `app_mention` events. The PR is focused, but a maintainer already said OpenClaw is holding off on this Slack Assistant surface for now, so this needs product/API alignment rather than automated closure.

## Best Possible Solution

Keep this PR open for maintainer judgment. If OpenClaw wants Slack Agents & AI Apps enabled by default, update both `docs/channels/slack.md` and `extensions/slack/src/setup-shared.ts`, and either implement/verify `assistant_thread_started` handling or omit that event until supported. If OpenClaw is intentionally holding off, move `assistant:write`/Assistant setup into an optional section or remove it from the default manifest so Slack warnings and OpenClaw defaults stay aligned.
