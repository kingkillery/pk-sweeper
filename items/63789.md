## Summary

Keep open. Current `main` has core agent-to-agent messaging, but it does not implement the PR’s proposed handoff contract (`handoff_id`, delivered/accepted/queued/rejected acknowledgment states, or an append-only handoff ledger). The PR is a docs-only draft that still describes a real gap in the existing core `sessions_send` path, and the discussion usefully narrows scope against adjacent higher-level meeting/lifecycle work rather than duplicating it.

## Evidence

- **Current core A2A response shape is narrower than the proposal:** `sessions_send` currently returns `accepted` for async starts and `ok` for waited replies, with `delivery` only marked `pending` or `skipped`. There is no `handoff_id`, no structured `delivered/accepted/queued/rejected` envelope, and no ledger result surfaced here.
