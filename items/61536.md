## Summary

Keep open. Current main still matches the reported native Discord voice output-contract gap: Discord VC enters the agent as the Discord message provider, the shared policy only hides the `tts` tool for the `voice` provider, and the Discord voice playback path still requires returned reply text before it can synthesize/play audio. The latest maintainer comment also explicitly says the issue remains valid on current main.
