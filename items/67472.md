## Summary

Keep PR #67472 open. Current main still has the backup-create race: it plans existing paths once, then hands those paths to `tar.c` without an archive-time ENOENT/ENOTDIR skip path. The PR is not implemented or superseded on main, though the submitted patch likely needs revision for manifest/verification behavior and regression-test coverage before merge.

## Best Possible Solution

Keep #67472 open and request a focused revision: implement archive-time ENOENT/ENOTDIR handling in `src/infra/backup-create.ts`, record/report skipped paths without making `--verify` produce an internally inconsistent archive, and add a regression test whose tar mock distinguishes Pack file errors from output stream errors. The linked bug #67417 should remain the canonical user problem until that fix lands.
