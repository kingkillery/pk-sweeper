## Summary

Keep open. Current `main` does not implement the PR’s Google/Gemini inline audio `@file` ref rewriting, so there is no strong maintainer-cleanup basis to close it as already fixed or obsolete.

## Evidence

- **Current wrapper chain has no audio-ref inlining step:** `applyPostPluginStreamWrappers` only installs the Google thinking wrapper, OpenAI context/string wrappers, MiniMax thinking wrapper, and optional `parallel_tool_calls` injection. There is no payload pass that scans text parts for local audio refs and rewrites them to Gemini `inlineData`.

It does not import `node:fs`, `node:path`, workspace path-policy helpers, or MIME helpers, which matches the absence of any local-audio-file inlining implementation in this module.
