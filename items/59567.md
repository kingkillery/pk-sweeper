## Summary

Keep open. This is a recent, coherent feature request from a non-maintainer with no protected labels, and current main does not appear to expose an idle-only heartbeat mode. Existing behavior skips heartbeats when the main queue is busy and offers isolated/light-context modes, but that is not the same as delaying fixed-interval heartbeats until the session has been silent for a configured idle period.

## Evidence

- **Issue asks for idle-only heartbeat scheduling:** The issue specifically requests `idleOnly: true` or `triggerWhen: "idle"` so heartbeat prompts run only after the session has been silent, and it explains why isolated cron jobs are not equivalent because they lack main-session context. ([github.com](https://github.com/openclaw/openclaw/issues/59567))
- **Current heartbeat config lacks idle-only fields:** Current main's `AgentDefaultsConfig.heartbeat` type exposes fields such as `every`, `activeHours`, `session`, `target`, `lightContext`, `isolatedSession`, and `includeReasoning`; searches within that opened source page found no `idleOnly` or `triggerWhen` field.
