## Summary

Current `main` does not make `openclaw logs --follow` use local time by default. The CLI still treats local rendering as opt-in via `--local-time` (or a valid `TZ` env var), docs still describe it that way, and tests explicitly assert UTC output by default. This remains a plausible enhancement request, so it should stay open.

## Evidence

- **CLI still keeps local-time rendering opt-in:** `openclaw logs` still defines `--local-time` as an explicit option, and the runtime only enables local formatting when that flag is set or `process.env.TZ` is a valid timezone; otherwise the default formatter path remains UTC.
