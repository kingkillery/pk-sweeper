## Summary

Keep PR #60507 open. Current main has exec approvals, safe-bin hardening, and docs for allowlist/on-miss behavior, but it does not include an equivalent destructive-command catalog or command-verb HIL policy. The PR is also not safe to merge as-is because the proposed command-text patterns do not match current allowlist semantics, matching the existing review discussion. That is a request-for-changes situation, not an allowed conservative auto-close reason.
