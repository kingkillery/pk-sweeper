## Summary

Current `main` only implements the native Codex web-search prompt in `openclaw configure --section web`; the interactive onboarding path still goes straight into the managed-provider search flow. That means the PR's stated goal of covering both onboarding and configure is not implemented on `main`, so this should stay open rather than be conservatively closed.

## Evidence

- **Configure wizard has the native Codex prompt on main:** `promptWebToolsConfig` checks `isCodexNativeWebSearchRelevant`, prompts `Enable native Codex web search for Codex-capable models?`, lets the user choose cached/live mode, and optionally skips managed-provider setup.
