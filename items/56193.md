## Summary

Keep open. Current `main` still defines `streamTo`/`resumeSessionId` on the shared `sessions_spawn` schema, documents them as ACP-only, and explicitly errors when either reaches `runtime="subagent"`. I did not find any current bridge/normalization code that strips those ACP-only fields before execution, so the reported failure mode is not strongly disproven and does not look implemented on `main`.
