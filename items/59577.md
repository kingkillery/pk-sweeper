## Summary

Keep open. Current main still routes changed session files through the full `indexFile(... source: "sessions" ...)` path, and I found no `indexSessionFileIncremental` / `chunkMarkdownWithOffset` implementation on main. The PR contains potentially useful memory performance work, but it also has unresolved review concerns about mutation ordering, duplicate chunk insertion, stale FTS metadata, and offset handling, so it needs maintainer review rather than cleanup closure.
