## Summary

Keep open. Current `main` still lacks the asymmetric memory-embedding config/request plumbing and the Ollama `/api/embed` batch path described in this PR, so the linked work is not implemented on `main` and the PR discussion remains actionable.

## Evidence

- **Memory config surface still lacks the new fields:** `MemorySearchConfig` on current `main` includes `provider`, `fallback`, `model`, and `outputDimensionality`, but no `inputType`, `queryInputType`, or `documentInputType`, so users cannot configure asymmetric query/document labels yet.
