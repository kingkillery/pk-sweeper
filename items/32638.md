## Summary

Keep open. Current `main` has a narrow unreleased workaround for Groq `qwen/qwen3-32b`, but the code still lacks a general way to preserve provider-native reasoning vocabularies (`default`, `on/off`, etc.). Recent issue comments match that gap, so this is not strong close evidence.

## Evidence
