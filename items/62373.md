## Summary

Keep open. Current main documents `openai-codex/*` as the PI/Codex OAuth route and the native Codex app-server as a separate `embeddedHarness.runtime: "codex"` path, but I did not find the requested config or implementation to decouple Codex OAuth billing from tool execution. The provider still normalizes `openai-codex` models onto the `openai-codex-responses` transport and Codex backend URL, so this feature request is not clearly implemented on main.

They do not document a third mode that keeps Codex OAuth billing while switching tool execution to a separate local-gateway path.
