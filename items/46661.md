## Summary

Keep open. Current `main` still hardwires macOS voice wake/talk capture to Apple speech recognition and only exposes mic/language preferences for that path; there is no custom ASR server/provider configuration for the mac voice-input flow described in #46661.

## Evidence

- **macOS talk mode still uses Apple Speech directly:** `TalkModeRuntime.startRecognition()` constructs `SFSpeechRecognizer` and `SFSpeechAudioBufferRecognitionRequest` directly for live talk capture, with no provider abstraction or HTTP transcription hook in that path.
  - file: [apps/macos/Sources/OpenClaw/TalkModeRuntime.swift:183](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/apps/macos/Sources/OpenClaw/TalkModeRuntime.swift#L183)
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **macOS voice wake still uses Apple Speech directly:** `VoiceWakeRuntime.configureSession()` instantiates `SFSpeechRecognizer` for wake-word recognition; this runtime remains speech-framework based rather than configurable to a custom endpoint.
  - file: [apps/macos/Sources/OpenClaw/VoiceWakeRuntime.swift:274](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/apps/macos/Sources/OpenClaw/VoiceWakeRuntime.swift#L274)
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **macOS settings only persist mic and locale choices:** App state stores `voiceWakeMicID`, `voiceWakeMicName`, `voiceWakeLocaleID`, and `voiceWakeAdditionalLocaleIDs`; there are no persisted voice-wake ASR provider, base URL, API key, or model settings on `main`.
  - file: [apps/macos/Sources/OpenClaw/AppState.swift:121](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/apps/macos/Sources/OpenClaw/AppState.swift#L121)
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **Voice Wake settings UI has language/mic controls, not ASR endpoint controls:** The macOS Voice Wake settings expose a recognition-language picker and additional-language list, which matches the current built-in recognizer flow. I found no Voice Wake UI/config matches for endpoint/API-key/model fields.
  - file: [apps/macos/Sources/OpenClaw/VoiceWakeSettings.swift:432](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/apps/macos/Sources/OpenClaw/VoiceWakeSettings.swift#L432)
