## Summary

Current `main` does not implement configurable reasoning/thinking block formatting per channel. Telegram still routes reasoning through a shared formatter that hardcodes italic `_..._` output, and the surfaced reasoning controls are visibility-only (`on|off|stream`), not format selection.

## Evidence

- **Hardcoded reasoning formatting:** `formatReasoningMessage()` still trims the text, wraps each non-empty line in underscores, and returns a fixed `Reasoning:` prefix. The inline comment explicitly calls out Telegram multiline italics handling rather than a configurable format.
  - file: [src/agents/pi-embedded-utils.ts:164](https://github.com/openclaw/openclaw/blob/c0a7b6a510758645c5267734b984f4885a605b67/src/agents/pi-embedded-utils.ts#L164)
  - sha: [c0a7b6a51075](https://github.com/openclaw/openclaw/commit/c0a7b6a510758645c5267734b984f4885a605b67)
- **Telegram path uses shared formatter directly:** Telegram reasoning splitting still calls `formatReasoningMessage(taggedReasoning)` with no channel-specific format option or config lookup before delivery.
