## Summary

Keep open. Current `main` still styles multiple Control UI text inputs below the 16px iOS Safari focus threshold, including the active chat composer textarea and shared `.field` inputs, so the report remains credible and not implemented on `main` or in the latest shipped release inspected locally.

## Evidence

- **Active chat textarea is still below 16px on main:** The live Control UI chat view renders a `<textarea>` inside `.agent-chat__input` (`ui/src/ui/views/chat.ts:1249`), and the current stylesheet still sets `.agent-chat__input > textarea { font-size: 0.92rem; }`. `body` is still `font: 400 14px/1.55 ...`, so the computed textarea font stays below 16px on iOS.
  - file: [ui/src/styles/chat/layout.css:561](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/ui/src/styles/chat/layout.css#L561)
