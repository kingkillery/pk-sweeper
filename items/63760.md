## Summary

Current `main` still handles subagent and parent-owned one-shot ACP completion delivery as a normal agent turn on the requester session. I found no announce-specific model override or lightweight/isolated announce-context option on `main`; the only subagent announce config knob is timeout, and ACP explicitly rejects `lightContext`.

## Evidence

- **Announce delivery still re-enters the requester session with a plain `agent` call:** `sendSubagentAnnounceDirectly()` calls gateway `agent` on `canonicalRequesterSessionKey` and sends the completion payload there. That call includes delivery metadata and `internalEvents`, but no `model`, `thinking`, `bootstrapContextMode`, or isolated-session setting for the announce turn.
  - file: [src/agents/subagent-announce-delivery.ts:574](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/agents/subagent-announce-delivery.ts#L574)
  - command: `nl -ba src/agents/subagent-announce-delivery.ts | sed -n '481,680p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Gateway model override is optional, and announce code does not provide one:** The gateway `agent` handler only uses `request.provider` / `request.model` when present. Session model resolution otherwise falls back to the persisted/default session model, so the announce turn inherits the requester session's model selection.
  - file: [src/gateway/session-utils.ts:1082](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/gateway/session-utils.ts#L1082)

- **No announce-model or announce-context config exists in subagent defaults:** `agents.defaults.subagents` exposes `model`, `thinking`, `runTimeoutSeconds`, and `announceTimeoutMs`, but no announce-specific `model`, `lightContext`, or isolated-session field.
  - file: [src/config/types.agent-defaults.ts:388](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/config/types.agent-defaults.ts#L388)

  - file: [src/agents/subagent-spawn.ts:885](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/agents/subagent-spawn.ts#L885)

The `sessions_spawn` tool rejects `lightContext` for `runtime: "acp"`, so ACP does not currently have the requested cost-control knob either.
