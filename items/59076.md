## Summary

Keep open. The report is concrete, recent, non-maintainer-authored, and has no protected labels, but current main does not appear to contain the proposed sanitizer. The linked fix PR #59118 is still open, so there is not enough evidence to close as implemented.

## Evidence

- **Issue has actionable reproduction and impact:** #59076 describes a specific regression in 2026.3.31 with Discord, Anthropic streaming tool calls, long CJK text, exact leaked JSON parse errors, and later clarifies the session can enter failed/error state. The reporter also documented a partial workaround: prefer Write over Edit for substantial CJK content.
- **Related fix exists but is not landed:** PR #59118 is linked from the issue and proposes adding an isStreamingJsonParseError detector plus tests, but GitHub shows it as open and targeting main rather than merged.
  - command: `GitHub issue/PR context and web view of openclaw/openclaw#59118`
  - sha: [082dba2343b8](https://github.com/openclaw/openclaw/commit/082dba2343b878f43d38d037fa54f56cbb918c02)
- **Current main still falls back to raw error text:** On current main, formatAssistantErrorText handles many known error classes but still ends by returning the raw error text when it is 600 characters or shorter. The reported JSON parse error is shorter than that, and searches of current main did not find the proposed isStreamingJsonParseError detector or friendly replacement message.
  - file: [src/agents/pi-embedded-helpers/errors.ts](https://github.com/openclaw/openclaw/blob/2e0ae56b1a580103f71b9b7f79b3d638e3c3698d/src/agents/pi-embedded-helpers/errors.ts)
