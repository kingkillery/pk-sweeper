## Summary

Keep open. Current `main` has plugin SDK lifecycle hooks (`gateway_start` / `gateway_stop`) and restart shutdown metadata, but the internal hook/event system requested in #6711 still only exposes `gateway:startup`. I found no runtime emission of `gateway:shutdown` or `gateway:pre-restart` internal events on current `main`.

## Evidence

- **Internal hooks docs only list startup:** The internal hooks reference enumerates supported events and includes `gateway:startup`, but not `gateway:shutdown` or any pre-restart event.
  - file: [docs/automation/hooks.md:34](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/docs/automation/hooks.md#L34)
  - command: `nl -ba docs/automation/hooks.md | sed -n '34,50p'`
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Gateway startup is the only internal gateway hook emitted:** Gateway sidecar startup explicitly creates and triggers only `createInternalHookEvent("gateway", "startup", "gateway:startup", ...)`.
  - file: [src/gateway/server-startup-post-attach.ts:268](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/gateway/server-startup-post-attach.ts#L268)
  - command: `nl -ba src/gateway/server-startup-post-attach.ts | sed -n '262,286p'`
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **No shutdown/pre-restart internal event emitter found:** Repo-wide search for `gateway:shutdown` / `gateway:pre-restart` found no runtime emitter; the only `gateway:shutdown` hit is a negative test case.
  - command: `rg -n "gateway:shutdown|gateway:pre-restart|gateway:restart|pre-restart" src docs -g '!node_modules'`
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Tests confirm shutdown is not treated as a supported gateway internal hook type:** `internal-hooks.test.ts` checks that `gateway:shutdown` is a non-startup gateway event and does not define a corresponding supported event path.
  - file: [src/hooks/internal-hooks.test.ts:227](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/hooks/internal-hooks.test.ts#L227)

  - file: [src/plugins/hook-types.ts:81](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/plugins/hook-types.ts#L81)

## Risks / Open Questions

- The issue body uses internal-hook naming (`command:*`, `gateway:boot`, `session:start`), but current main also has similarly named plugin lifecycle hooks; someone could misread that partial overlap as a full implementation.
- The follow-up comment about exposing `activeTurns` in `sessions.list` or status responses also does not appear fully implemented; current main exposes `activeTurns` in ACP doctor output, not in `sessions.list`.
