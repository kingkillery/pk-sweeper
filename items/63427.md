## Summary

Keep open. The issue body's original single-client 'no backoff' framing is outdated, but the discussion narrowed it to repeated fresh CLI probe sessions hitting `device-required`, and current `main` still does not implement that short-circuit/cache fix or equivalent caller-side handling.

## Evidence

- **Single-client reconnects already pause on device-required:** `GatewayClient.shouldPauseReconnectAfterAuthFailure()` returns `true` for `DEVICE_IDENTITY_REQUIRED` and `CONTROL_UI_DEVICE_IDENTITY_REQUIRED`, which matches the reporter's corrected RCA that a single client instance is not the tight reconnect loop.
  - file: [src/gateway/client.ts:608](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/gateway/client.ts#L608)
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Probe path still opens a fresh WebSocket per call:** `probeGateway()` still disables device identity for unauthenticated loopback probes and then constructs a new `GatewayClient` for every call. There is no per-URL device-required cache or pre-connect short-circuit in the current file.
  - file: [src/gateway/probe.ts:155](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/gateway/probe.ts#L155)
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Daemon health path still uses the affected probe flow:** `confirmGatewayReachable()` still calls `probeGateway()` against `ws://127.0.0.1:<port>`, and `looksLikeAuthClose()` only treats reasons containing `auth`, `token`, `password`, `scope`, or `role` as reachable. `device identity required` is not covered there.
  - file: [src/cli/daemon-cli/restart-health.ts:62](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/cli/daemon-cli/restart-health.ts#L62)
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **No landed regression coverage for repeated device-required probes:** Current probe tests cover an immediate close failure (`pairing required`) but do not include the repeated `device identity required` cache/short-circuit scenarios discussed in the issue and PR.
  - file: [src/gateway/probe.test.ts:241](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/gateway/probe.test.ts#L241)
