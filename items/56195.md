## Summary

Current `main` already preserves operator scopes for the supported direct-local `gateway-client`/`backend` shared-auth path without device pairing, and it intentionally still clears unbound scopes for other device-less shared-auth sessions. The regression described by this PR does not match current `main` behavior.

## Close Comment

Closing after a Codex review against current `main`. The reported backend shared-auth regression does not reproduce on the current codepath: direct-local `gateway-client` / `backend` connections already keep requested scopes without device pairing (`src/gateway/server/ws-connection/handshake-auth-helpers.ts:249`, `src/gateway/server/ws-connection/message-handler.ts:636`), and the behavior is covered by compatibility tests for both token and password auth (`src/gateway/server.auth.compat-baseline.test.ts:127` and `src/gateway/server.auth.compat-baseline.test.ts:300`). Current `main` also has explicit tests that still clear unbound scopes for other device-less shared-auth sessions (`src/gateway/server/ws-connection/connect-policy.test.ts:305`), so the blanket guard restored by this PR no longer matches the intended behavior contract.
