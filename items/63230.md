## Summary

Keep open. Current `main` does not implement per-channel Slack `thread.requireExplicitMention`, and the PR discussion already surfaced a real schema/runtime mismatch that still needs follow-up before this is ready to land.

## Evidence

- **Current schema still lacks channel-level thread override:** On current `main`, `SlackChannelSchema` has `enabled`, `requireMention`, `tools`, `allowBots`, `users`, `skills`, and `systemPrompt`, but no `thread` field. `SlackAccountSchema` still carries the only `thread: SlackThreadSchema.optional()` entry.
