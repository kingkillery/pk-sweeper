## Summary

Keep open. Current `main` still wires `pluginToolsMcpBridge` to a stdio `plugin-tools-serve` subprocess, and the only ACPX runtime special-casing I found is for `openclaw acp` bridge sessions, not for Claude Code ACP sessions or `sessions_send`/session rebind lifecycle. I did not find a repo-side fix or tests covering the reported disconnect/orphan path.
