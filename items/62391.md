## Summary

Keep open. Current `main` still drops Slack structured error data during delivery-queue recovery: `recoverPendingDeliveries` formats failures via `formatErrorMessage(err)`, and that helper only uses `Error.message` plus `.cause`, not Slack `err.data.needed` or `err.data.response_metadata.*`. `missing_scope` is also still absent from the delivery queue permanent-error allowlist, so these failures continue retrying instead of being moved directly to `failed/`.
