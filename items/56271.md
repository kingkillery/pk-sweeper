## Summary

Current `main` has partial browser cleanup for session teardown and detached task runtimes, but it does not implement the requested Control UI feature to auto-close browser windows/frames after a normal task completes, and there is no `browser.autoClose` config surface.

## Evidence

- **No `browser.autoClose` config surface:** `src/config/types.browser.ts:29` defines the browser config shape (`enabled`, `evaluateEnabled`, `cdpUrl`, `headless`, `attachOnly`, `profiles`, `snapshotDefaults`, `ssrfPolicy`, `extraArgs`, etc.) and does not include any `autoClose` or task-completion close option.
  - file: [src/config/types.browser.ts:29](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/config/types.browser.ts#L29)

## Risks / Open Questions

- There may be user confusion because recent browser cleanup work exists, but it only covers session teardown and detached runtimes, not ordinary interactive Control UI runs.
- The issue proposes auto-closing a visible UI surface after task completion; current code inspection found no matching UI/config contract, so closing this as implemented would be premature.
