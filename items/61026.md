## Summary

Keep open. Current `main` partially addresses the report: assistant-side external aborts are intentionally prevented from rotating profiles or switching fallback models, and there is test coverage for that. But prompt-side external aborts still log `surface_error` and rethrow the raw prompt error, while abort-shaped provider messages are still classified as `timeout`; the outer model-fallback loop only short-circuits plain non-timeout `AbortError`s. That leaves a credible path for the misleading `reason=timeout` log and fallback cascade described in #61026.
