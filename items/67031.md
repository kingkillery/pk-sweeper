## Summary

Keep open. Current main still leaves key image limits hardcoded in core: tool-result sanitization still falls back to a 5 MB byte cap, the image pipeline still rejects inputs above 25,000,000 pixels before resize, and the generic media layer still uses a fixed 6 MB image cap. The issue is partially outdated, though: OpenAI-compatible chat-completions image limits and the `agents.defaults.imageMaxDimensionPx` knob are already configurable and documented on main.
