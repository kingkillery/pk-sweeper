## Summary

Keep open: current `main` still drops outbound session identity from the `message_sending` hook context on the gateway send path and still collapses hook-cancelled sends to a generic `No delivery result`, so this PR is not already implemented on `main`.

## Evidence

- **Gateway send still collapses empty delivery to a generic error:** Current `main` builds an outbound session and passes it into `deliverOutboundPayloads`, but if that call returns no delivery result the handler still throws `No delivery result` instead of surfacing a guarded `message_sending` cancellation.
  - file: [src/gateway/server-methods/send.ts:524](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/gateway/server-methods/send.ts#L524)
  - command: `nl -ba src/gateway/server-methods/send.ts | sed -n '524,557p'`
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **`message_sending` still receives only minimal context:** `applyMessageSendingHook` currently calls `runMessageSending` with a context object containing only `channelId`, `accountId`, and `conversationId`; it does not forward outbound session identity from `params.session`.
  - file: [src/infra/outbound/deliver.ts:590](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/infra/outbound/deliver.ts#L590)
  - command: `nl -ba src/infra/outbound/deliver.ts | sed -n '589,607p'`
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Public hook context still lacks `agentId`:** The public `PluginHookMessageContext` on `main` includes `sessionKey`, `runId`, and `messageId`, but no `agentId`, which matches the PR's proposed additive contract change rather than an already-shipped implementation.
  - file: [src/plugins/hook-message.types.ts:4](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/plugins/hook-message.types.ts#L4)
  - command: `nl -ba src/plugins/hook-message.types.ts | sed -n '1,20p'`
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Message context mapper has no outbound override path:** `toPluginMessageContext` on `main` only copies fields from the canonical context. There is no override parameter to inject outbound `agentId` or an alternate `sessionKey` for the `message_sending` path.
  - file: [src/hooks/message-hook-mappers.ts:220](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/hooks/message-hook-mappers.ts#L220)

That supports the reported behavior as a real gap, not an abstract cleanup.
