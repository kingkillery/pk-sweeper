## Summary

Current `main` still drops `ReplyToId`/`ReplyToBody` before the `inbound_claim` plugin event, while the underlying inbound context and agent metadata already carry those fields. The PR is small, additive, and still addresses a real gap in a core plugin seam, so there is not strong evidence to close it.
