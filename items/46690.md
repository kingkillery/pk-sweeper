## Summary

Keep open. Current main does not implement orphan-skill detection in `openclaw skills list/check`, and the bundled `skill-creator` workflow still allows creating skills at arbitrary output paths. The issue body overstates `workspace/skills` as the only canonical location, but there is still no strong basis to close this as fixed or obsolete.

## Evidence

- **Docs define multiple supported skill roots:** Current docs say OpenClaw loads skills from `skills.load.extraDirs`, bundled skills, `~/.openclaw/skills`, `~/.agents/skills`, `<workspace>/.agents/skills`, and `<workspace>/skills`, so the report's `workspace/skills`-only premise is incomplete.
  - file: [docs/tools/skills.md:13](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/docs/tools/skills.md#L13)

  - file: [src/agents/skills/workspace.ts:533](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/agents/skills/workspace.ts#L533)
