## Summary

Keep open. Current `main` still routes browser screenshots through Sharp-backed normalization code, and I did not find a source or history change proving the Windows packaged-install missing-`@img/colour` failure was fixed after the report. The lockfile declares `@img/colour`, but that only shows intended dependency resolution, not that the Windows app now ships it correctly.
