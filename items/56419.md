## Summary

Keep open. Current `main` still hardcodes a 30,000 ms timeout in the shared `image` tool path and still aborts image-analysis requests with an `AbortController` when that timeout elapses. I did not find evidence in the checked-out `main` or latest release tag that this was already shipped or made configurable.

## Evidence

- **Image tool still injects fixed 30s timeout on current main:** `runImagePrompt()` passes `timeoutMs: 30_000` in all shared image-analysis call sites.
  - file: [src/agents/tools/image-tool.ts:221](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/agents/tools/image-tool.ts#L221)
