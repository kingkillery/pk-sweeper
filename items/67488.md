## Summary

Keep open. Current main and the latest release still use ASCII-only JSON parsing/repair for streamed tool-call arguments and do not implement the structural Unicode smart-quote normalization described in the linked PR #67551. Cron state persists runner status separately from delivery status, so the reported combination of `lastRunStatus: "error"` with `lastDeliveryStatus: "delivered"` remains plausible when the isolated runner reports a post-delivery parse error. The linked PR is open/unmerged, so it is a useful implementation path, not shipped evidence for closing #67488.

## Best Possible Solution

Keep #67488 open until #67551 or an equivalent patch lands. The best fix is to add structural Unicode quote normalization to the tool-call argument repair path, update balanced JSON extraction to respect those delimiters without rewriting content-level smart quotes, and add regression coverage for edit-tool arguments containing CJK text, Markdown bold syntax, and smart quotes in an isolated cron announce run.
