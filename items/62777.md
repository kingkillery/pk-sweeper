## Summary

Keep open. Current `main` shows cron `failureAlert` wiring is implemented and documented, but the Discord explicit-target path still strips the normalized prefix in `parseDiscordExplicitTarget`, and `resolveSessionDeliveryTarget` feeds that bare id back into outbound normalization, which can still reinterpret explicit Discord targets instead of preserving them end-to-end.
