## Summary

Keep open. Current main has partial implementation: the generic message-tool plumbing and Discord plugin/runtime support `channel-create`, but the issue's Slack-focused use case is still unsupported and the CLI still exposes only `channel info` and `channel list`.

## Evidence

- **Generic message tool already has channel-create plumbing:** The shared message action enum includes `channel-create`, and the message tool schema includes shared channel-management parameters like `name`, `type`, `parentId`, `topic`, and `position`. This shows the core/tool contract exists, but does not prove provider coverage.
  - file: [src/channels/plugins/message-action-names.ts:39](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/channels/plugins/message-action-names.ts#L39)
  - command: `nl -ba src/channels/plugins/message-action-names.ts | sed -n '35,45p'`
- **Discord exposes channel-create in tool discovery:** Discord adds `channel-create` when the `channels` action gate is enabled, so agents can discover and invoke it on Discord.
  - file: [extensions/discord/src/channel-actions.ts:132](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/extensions/discord/src/channel-actions.ts#L132)
  - command: `nl -ba extensions/discord/src/channel-actions.ts | sed -n '128,138p'`
- **Discord runtime actually handles channel-create:** Discord guild-admin dispatch maps `channel-create` to `channelCreate`, and the guild runtime calls `createChannelDiscord`; tests cover successful creation and gate enforcement.
  - file: [extensions/discord/src/actions/handle-action.guild-admin.ts:156](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/extensions/discord/src/actions/handle-action.guild-admin.ts#L156)
  - command: `nl -ba extensions/discord/src/actions/handle-action.guild-admin.ts | sed -n '146,168p'`
- **Slack still lacks channel-create support:** Slack message-action discovery only exposes `send`, reactions, read/edit/delete, file upload/download, pins, `member-info`, and `emoji-list`. There is no `channel-create` action in the Slack plugin.
  - file: [extensions/slack/src/message-actions.ts:30](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/extensions/slack/src/message-actions.ts#L30)
  - command: `nl -ba extensions/slack/src/message-actions.ts | sed -n '1,80p'`
- **CLI still matches the issue's reported gap:** `openclaw message channel` currently registers only `info` and `list`, and the CLI docs list only `channel info` and `channel list` under channel actions. The requested `openclaw message channel create ...` command does not exist.
  - file: [src/cli/program/message/register.discord-admin.ts:40](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/cli/program/message/register.discord-admin.ts#L40)
  - command: `nl -ba src/cli/program/message/register.discord-admin.ts | sed -n '1,80p'`

## Risks / Open Questions

- The issue mixes two scopes: generic message-tool support is partially done, but the reporter's concrete Slack workflow is still missing. Closing as implemented would hide that remaining gap.
- Because the CLI surface still lacks `openclaw message channel create`, users following the issue's proposed interface still cannot do what the issue asks for even on providers that have backend support.
