## Summary

Keep open. Current `main` still only supports whole-instance backup creation/verification, with no agent-scoped `--agent` backup path, no restore CLI, and no built-in clone flow.

## Evidence

- **Backup CLI surface has no agent-scoped option:** `openclaw backup` registers only `create` and `verify`. `create` accepts `--output`, `--json`, `--dry-run`, `--verify`, `--only-config`, and `--no-include-workspace`; there is no `--agent` or `--as` option.
  - file: [src/cli/program/register.backup.ts:20](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/cli/program/register.backup.ts#L20)

- **User-facing docs still describe only instance-level backup:** The backup docs describe a local archive for OpenClaw state/config/credentials/sessions/workspaces, list only whole-instance examples, and document `--only-config` / `--no-include-workspace` as the scope controls.
  - file: [docs/cli/backup.md:11](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/docs/cli/backup.md#L11)
