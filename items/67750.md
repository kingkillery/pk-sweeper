## Summary

Current main partially addresses the report by forwarding non-empty external-run failure details instead of always returning the old generic `/new` text. However, the reported post-compaction timeout chain is not fully implemented: the error path still does not surface that auto-compaction had just succeeded unless optional compaction notices are enabled, and there is no regression test for successful compaction followed by timeout/fallback exhaustion.

## Best Possible Solution

Keep the issue open but narrow it to the remaining gap: in `runAgentTurnWithFallback`, use the already tracked compaction state when fallback exhaustion follows a successful compaction, and produce copy that says compaction succeeded, the retried model timed out, and fallback candidates were unavailable. Add a regression test that combines a compaction event/count with a timeout/billing `FallbackSummaryError` on an external chat channel.
