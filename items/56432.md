## Summary

Current `main` does not support a fifth `ctx` argument on tool `execute()`, and the runtime config context is intentionally delivered through the registered tool factory instead. The plugin API still receives `pluginConfig` at registration time, so the issue report does not match the current implementation.

## Risks / Open Questions

## Close Comment

Closing after a Codex review of current `main` (`7d1891e`). The current tool contract does not pass a fifth `ctx` argument into `execute()`; `AnyAgentTool.execute(...)` is four args only (`src/agents/tools/common.ts:21`, `src/agents/pi-tool-definition-adapter.ts:226`). Runtime tool context is instead supplied to the tool factory via `api.registerTool((ctx) => ...)`, where `ctx` includes `config`/`runtimeConfig` (`src/plugins/tool-types.ts:8`, `src/agents/openclaw-tools.plugin-context.ts:49`, `src/plugins/tools.ts:136`). Separately, `api.pluginConfig` is still injected when the plugin API is created (`src/plugins/loader.ts:2711`, `src/plugins/api-builder.ts:134`). Based on the current code, I could not reproduce a core bug matching this report. If this is still happening on a plugin built against current `main`, please open a new issue with a minimal reproducer that shows `api.pluginConfig` is missing after registration.
