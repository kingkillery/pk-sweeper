## Summary

Keep open. Current `main` hardens `exec-approvals.json` writes, but it still stores `lastUsedAt`/`lastUsedCommand` inside allowlist entries and has no `exec-runtime.json` split, so this feature request is not implemented on `main`.

## Evidence

- **Allowlist entry type still includes mutable runtime fields:** `ExecAllowlistEntry` still defines `lastUsedAt`, `lastUsedCommand`, and `lastResolvedPath` on the allowlist object itself.
  - file: [src/infra/exec-approvals.types.ts:1](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/src/infra/exec-approvals.types.ts#L1)
  - sha: [7875092f4ddc](https://github.com/openclaw/openclaw/commit/7875092f4ddc6f2dbe218bea8506c9ac1da87c39)
- **Runtime usage is still written back into exec-approvals.json:** `recordAllowlistUse(...)` updates `lastUsedAt` and `lastUsedCommand` on matching allowlist entries, then calls `saveExecApprovals(approvals)`, so runtime tracking remains coupled to the approvals file.
  - file: [src/infra/exec-approvals.ts:843](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/src/infra/exec-approvals.ts#L843)
  - sha: [7875092f4ddc](https://github.com/openclaw/openclaw/commit/7875092f4ddc6f2dbe218bea8506c9ac1da87c39)
- **Protocol still models runtime fields on allowlist entries:** Gateway protocol schema for exec approvals still exposes `lastUsedAt`, `lastUsedCommand`, and `lastResolvedPath` directly on each allowlist entry.
  - file: [src/gateway/protocol/schema/exec-approvals.ts:4](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/src/gateway/protocol/schema/exec-approvals.ts#L4)
  - sha: [7875092f4ddc](https://github.com/openclaw/openclaw/commit/7875092f4ddc6f2dbe218bea8506c9ac1da87c39)
- **User docs still describe a single exec-approvals.json file carrying runtime metadata:** The public exec approvals docs show `~/.openclaw/exec-approvals.json` as the storage file and the example schema places `lastUsedAt`/`lastUsedCommand` inside allowlist entries.
  - file: [docs/tools/exec-approvals.md:70](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/docs/tools/exec-approvals.md#L70)
  - sha: [7875092f4ddc](https://github.com/openclaw/openclaw/commit/7875092f4ddc6f2dbe218bea8506c9ac1da87c39)
- **No exec-runtime.json implementation exists on main:** A repo search for `exec-runtime.json` returned no matches, so the requested two-file split is absent from code, tests, and docs.
  - command: `rg -n "exec-runtime\.json" src docs test packages extensions`
- **Main already includes atomic-write mitigation for the existing file:** `saveExecApprovals` writes to a temp file and renames it into place, and store tests cover atomic replacement instead of mutating linked inodes. This addresses the truncation/corruption concern, but not the requested file split.
  - file: [src/infra/exec-approvals.ts:481](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/src/infra/exec-approvals.ts#L481)
