## Summary

Keep open. Current `main` partially bridges managed `HOOK.md` hooks into message lifecycle (`message:received`, `message:sent`), but `before_dispatch` and `agent_end` still run only through the plugin hook runner. Docs also still present those two as plugin-hook surfaces, and there is no maintainer discussion here establishing that this request should be closed as out-of-scope or already handled.
