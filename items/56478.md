## Summary

Keep open. Current `main` does not show a maintainer-safe fix for #56478. The cron path still treats a matching `message`-tool send as delivered based on target matching, and the tests explicitly expect `delivered: true` in that case. At the same time, the WeChat docs now state that WeChat-specific outbound behavior and `context tokens` are owned by the external `@tencent-weixin/openclaw-weixin` plugin, so the report remains plausible as either an external-plugin bug or a core false-positive delivery-status bug. There is not strong evidence for `implemented_on_main` or `cannot_reproduce`.

## Evidence
