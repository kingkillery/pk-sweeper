## Summary

Keep open. Current `main` does not implement the PR's `before_tool_call` additions: the public event type still exposes only `toolName`, `params`, `runId`, and `toolCallId`; the hook runner still forwards only those fields; docs still document only those fields; and the execution pipeline still does not plumb `extensionContext` into `toToolDefinitions`/abort-wrapper execution for deriving assistant-message context.
