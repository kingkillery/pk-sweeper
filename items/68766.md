## Summary

Keep open. Current `main` still has the single-shot Feishu WebSocket startup path described in the report, with no OpenClaw-owned reconnect/backoff loop and no transport-liveness status reporting to let the generic channel health monitor or status surface detect a dead socket reliably.

## Evidence

- **Single-shot WebSocket startup with no app-layer reconnect:** `monitorWebSocket()` creates one WS client, stores it in `wsClients`, calls `start()`, and then only resolves on abort/cleanup. There is no retry loop, backoff scheduler, close handler, or error handler that recreates the client after startup failure or later socket death.
  - file: [extensions/feishu/src/monitor.transport.ts:85](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/extensions/feishu/src/monitor.transport.ts#L85)
  - command: `nl -ba extensions/feishu/src/monitor.transport.ts | sed -n '85,140p'`
  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)
- **WS client factory does not add recovery behavior:** `createFeishuWSClient()` only instantiates the upstream SDK `WSClient` with credentials and optional proxy agent. Recovery is therefore dependent on the SDK's internal behavior unless OpenClaw wraps it, and the current wrapper does not.
  - file: [extensions/feishu/src/client.ts:222](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/extensions/feishu/src/client.ts#L222)
  - command: `nl -ba extensions/feishu/src/client.ts | sed -n '219,237p'`
  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)
- **Status/doctor path probes credentials, not live WebSocket state:** Feishu status uses `probeFeishu()` from the channel status adapter, and `probeFeishu()` only calls the bot ping endpoint. That verifies credentials/app reachability, but not whether the WebSocket client in `wsClients` is actually alive.
  - file: [extensions/feishu/src/channel.ts:1182](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/extensions/feishu/src/channel.ts#L1182)

- **Feishu plugin does not publish transport-liveness patches:** The generic health monitor only treats a connected channel as a stale socket when `lastTransportActivityAt` is reported. In the Feishu plugin, the only direct status patch in startup is `ctx.setStatus({ accountId, port })`; there are no `createConnectedChannelStatusPatch`, `createTransportActivityStatusPatch`, or `lastTransportActivityAt` uses in `extensions/feishu/src`.
