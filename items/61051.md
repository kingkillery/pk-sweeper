## Summary

Keep open. Current `main` has a few supporting pieces from this PR lineage, but the core admin-terminal routing behavior is still not implemented: gateway approval requests still broadcast before forwarding, turn-source routing still lacks `turnSourceTo`/`sessionKey` context, the forwarder still reports suppressed native-target cases as unhandled, and Telegram target recipients still count as authorized exec approvers.
