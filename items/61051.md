## Summary

Keep PR #61051 open. Current main does not implement the core routing behavior described by the PR: approval requests are still broadcast to operator clients before forwarding is evaluated, turn-source route detection still lacks approval kind/source target context, explicit target forwarding can still be treated as unhandled when native fallback suppression filters every target, and Telegram exec authorization still includes target recipients rather than only configured approvers.
