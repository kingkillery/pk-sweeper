## Summary

Keep open. Current `main` does not already implement the requested zh-TW control-UI glossary fix, so `implemented_on_main` does not apply. But the PR as submitted targets the wrong pipeline: `scripts/control-ui-i18n.ts` reads glossaries from `ui/src/i18n/.i18n/`, while this PR adds `docs/.i18n/glossary.zh-TW.json`, and the docs i18n README does not list any zh-TW docs translation flow. Main still shows mixed Taiwan/mainland terminology in `ui/src/i18n/locales/zh-TW.ts`, so this is not `cannot_reproduce` either. The discussion also raises a separate maintainer-policy concern about the 284-entry bulk glossary, which argues for maintainer review/rework rather than auto-closing cleanup.
