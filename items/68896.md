## Summary

Keep open. Current `main` still implements only a single-toggle browser dictation flow in Control UI: speech recognition starts/stops on click, final transcripts append to the draft, and the user still sends manually. The requested short-click auto-send, 3-second hold-to-enter continuous mode, separate continuous-mode state, and matching tooltip/status behavior are not present.

## Evidence

- **STT runtime has no one-shot vs continuous mode switch:** `startStt()` always creates a browser `SpeechRecognition` instance with `continuous = true`, and `onend` only clears the active instance and forwards the callback. There is no API surface for the proposed short-click auto-send mode or restart-on-end continuous conversation mode.
