## Summary

Keep #67331 open. Current `main` has not implemented the standardized `permission_denied` helper or write/edit payload hardening, and the PR is a plausible focused core-agent hardening change. It should not be closed as obsolete, but it also should not land as-is because the provided review comments identify real semantic gaps around missing-file reads and mount-boundary classification.

## Best Possible Solution

Keep the PR open for author/maintainer follow-up. The best path is to rework the PR so only real boundary/alias/read-only failures become standardized `permission_denied` errors, ordinary missing sandbox reads keep ENOENT/not-found semantics, the classifier recognizes `escapes allowed mounts` as `workspace_boundary`, and focused tests cover host apply-patch, host workspace tools, sandbox bridge, remote bridge, and write/edit payload size/NUL guards.
