## Summary

Current `main` still does not echo mid-run queued user messages into the conversation thread. When chat is busy, the Control UI clears the draft and stores the follow-up in `chatQueue`; the transcript update path (`chatMessages`) is only used once that queued item is actually sent later. The UI renders those queued items in a separate `Queued` panel, not inline in the thread, so the issue's requested behavior is not implemented on `main`.
