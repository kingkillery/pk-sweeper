## Summary

Keep open. Current main has partial ACP cleanup, including background-task tracking, `/acp status`, and turn timeout handling, and PR #31044 addressed the missing permission docs. But the issue’s core permission-default/preflight concern is still present: ACPX still defaults to `approve-reads` plus `fail`, and the docs still warn that non-interactive writes/exec can abort with the reported `AcpRuntimeError`. The docs also still describe stalled ACP sessions as requiring manual process monitoring and killing, so the requested stall detection/cleanup is not fully implemented.
