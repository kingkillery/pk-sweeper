## Summary

Keep open. Current `main` still does not provide a per-agent selector in the Control UI Dreaming tab, and the Dreaming data path is still hard-wired to unscoped/default-agent gateway methods rather than an agent-specific context switch.

## Evidence

- **Dreaming tab header has no agent switcher:** The `dreams` branch in the Control UI header renders only `dreaming-header-controls` with a refresh button and the Dreaming on/off toggle. There is no agent dropdown, selector, or `agentsList` wiring in that branch.
  - file: [ui/src/ui/app-render.ts:1422](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/ui/src/ui/app-render.ts#L1422)
  - command: `nl -ba ui/src/ui/app-render.ts | sed -n '1380,1465p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Dreaming UI requests are not agent-scoped:** The Dreaming controller calls `doctor.memory.status` and `doctor.memory.dreamDiary` with empty payloads (`{}`), so the browser cannot ask for another agent's Dreaming state or diary.
  - file: [ui/src/ui/controllers/dreaming.ts:725](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/ui/src/ui/controllers/dreaming.ts#L725)
  - command: `nl -ba ui/src/ui/controllers/dreaming.ts | sed -n '718,748p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Gateway handlers still resolve the default agent:** `doctor.memory.status` and `doctor.memory.dreamDiary` both call `resolveDefaultAgentId(cfg)`, and the diary handler reads `resolveAgentWorkspaceDir(cfg, agentId)` for that default agent. There is no request parameter or selector path for an alternate agent context.
  - file: [src/gateway/server-methods/doctor.ts:785](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/gateway/server-methods/doctor.ts#L785)
  - command: `nl -ba src/gateway/server-methods/doctor.ts | sed -n '780,888p'`

- **Control UI docs still describe Dreams as status/toggle/reader only:** The user-facing Control UI docs list Dreams as dreaming status, enable/disable toggle, and Dream Diary reader. They do not describe any per-agent selector or per-agent Dreaming inspection flow.
  - file: [docs/web/control-ui.md:117](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/docs/web/control-ui.md#L117)
