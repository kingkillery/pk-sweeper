## Summary

Keep open. Current `main` still lets Discord startup continue with an undefined `botUserId` when `fetchUser('@me')` fails, and the guild mention-gating path still short-circuits on `botId`, so the bot-identity-loss behavior discussed in this PR is not implemented on `main`. The PR discussion also raises unresolved concerns about reconnect suppression and a timing-dependent test, so there is not strong evidence to close it as already fixed or obsolete.
