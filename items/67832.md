## Summary

Keep #67832 open. Current main already has the direct HTTP tool-invoke surface the reporter found, and it can reach `sessions_spawn` only when policy explicitly allows it, but the updated primary request is for a discoverable CLI path. Current main still has no `openclaw sessions spawn`, no generic `openclaw tools invoke`, and the `sessions` CLI remains list/cleanup only.

## Best Possible Solution

Keep the issue open and implement a narrow CLI surface. The most reusable path is likely `openclaw tools invoke --tool <name> --args-json ... --session-key main --json`, with a `sessions spawn` convenience wrapper only if maintainers want first-class UX. It should call the existing `/tools/invoke` or shared resolver, require the same `gateway.tools.allow` and agent tool policy for `sessions_spawn`, add CLI registration/tests, and update `docs/cli` plus the tools-invoke docs.
