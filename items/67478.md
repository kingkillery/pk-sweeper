## Summary

Keep #67478 open. Current main still has no early root guard or `OPENCLAW_ALLOW_ROOT` escape hatch in the CLI bootstrap, and the state plus systemd paths still derive from the invoking user's home. The linked fix PR #67509 is open/unmerged in the provided context, so the issue should remain the canonical bug until a fix lands.

## Best Possible Solution

Keep #67478 open as the canonical bug. Maintainers should choose and ship the actual product boundary: either an image-specific `/usr/local/bin/openclaw` guard/MOTD for the DigitalOcean 1-Click packaging, or a broader core CLI root guard with explicit escape hatches plus updates to root-based VPS/container docs and regression tests.
