## Summary

Keep open. I did not find strong evidence that current `main` fixes this report. The current embedded-agent lifecycle logging path still writes the derived error text straight into the `assistant_error` log metadata and `consoleMessage` without applying the repo's control-character sanitizer, and the relevant files are unchanged from the latest released commit `a9797214` (`v2026.4.23`). There is also no targeted regression test for control characters inside `errorMessage` itself.
