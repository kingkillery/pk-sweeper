## Summary

Keep open. Current `main` still sends provider-configured vision models through `resolveImageRuntime()` as qualified `provider/model` refs, but `resolveImageRuntime()` still normalizes `params.model` without stripping an already-qualified provider prefix first. That matches the Bailian failure described in the PR, and the PR discussion shows the author already moved to the safer consumption-point fix after review feedback.

## Evidence
