## Summary

Current `main` already contains targeted Control UI fixes and regression tests for the transcript-loss race described here, and the changelog shows the dashboard/chat UI fix shipped in release `2026.3.13`.

## Evidence

- **Active-run session.message reloads are now deferred:** `handleSessionMessageGatewayEvent` explicitly skips `chat.history` reloads while `host.chatRunId` is active because mid-run reloads race with the optimistic user-message update and can delay or hide the user message card.
  - file: [ui/src/ui/app-gateway.ts:490](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/ui/src/ui/app-gateway.ts#L490)
  - command: `nl -ba ui/src/ui/app-gateway.ts | sed -n '481,500p'`
- **Deferred reload is replayed only after the run ends:** After terminal chat states, deferred `session.message` reloads are replayed only when the active run has cleared, preventing the mid-run transcript replacement that the issue reports.
  - file: [ui/src/ui/app-gateway.ts:458](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/ui/src/ui/app-gateway.ts#L458)
  - command: `nl -ba ui/src/ui/app-gateway.ts | sed -n '440,478p'`
- **Per-tool-event history reloads are covered by regression tests:** Regression tests assert that live tool result events do not reload chat history on every event, and that history reload happens once after the final chat event when tool output was used.
  - file: [ui/src/ui/app-gateway.node.test.ts:622](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/ui/src/ui/app-gateway.node.test.ts#L622)
  - command: `nl -ba ui/src/ui/app-gateway.node.test.ts | sed -n '622,878p'`
- **Cross-run final handling no longer drops visible messages:** `handleChatEvent` keeps the active run intact and appends valid final assistant messages from other runs instead of blindly discarding non-matching run activity.
  - file: [ui/src/ui/controllers/chat.ts:437](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/ui/src/ui/controllers/chat.ts#L437)
  - command: `nl -ba ui/src/ui/controllers/chat.ts | sed -n '429,499p'`
- **Fix was shipped in a released version:** The changelog entry for release `2026.3.13` says: `Dashboard/chat UI: stop reloading full chat history on every live tool result in dashboard v2 so tool-heavy runs no longer trigger UI freeze/re-render storms while the final event still refreshes persisted history. (#45541)`
  - file: [CHANGELOG.md:2503](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/CHANGELOG.md#L2503)
  - command: `nl -ba CHANGELOG.md | sed -n '2487,2504p'`

## Close Comment

Closing as implemented on `main` after a Codex review. Current Control UI code now defers `session.message` history reloads while a chat run is active because those reloads raced with the optimistic user-message update and could hide/delay the visible transcript (`ui/src/ui/app-gateway.ts:490`). It then replays a single deferred reload only after the terminal chat event clears the active run (`ui/src/ui/app-gateway.ts:458`), and regression coverage now asserts that live tool-result events do not reload history per event while the final event performs the one needed refresh (`ui/src/ui/app-gateway.node.test.ts:622`, `ui/src/ui/app-gateway.node.test.ts:860`). The shipped changelog entry for release `2026.3.13` also matches this fix directly (`CHANGELOG.md:2503`). If you still see transcript disappearance on a newer build, please open a fresh issue with the exact OpenClaw version and a current repro, because that would likely be a different remaining Control UI bug.
