## Summary

Keep open. Current `main` still shows the Homebrew recommendation for any non-Windows platform when a selected skill has a brew-backed install and `brew` is missing, so the reported FreeBSD/WSL behavior is still present in source and tests.

## Evidence

- **Current onboard logic still prompts on non-win32:** `setupSkills` sets `needsBrewPrompt` when `process.platform !== "win32"`, any selected skill has an install option with kind `brew`, and `brew` is missing. That condition still includes FreeBSD and WSL/Linux.

There is no current test asserting suppression on FreeBSD or other non-macOS/non-Linux platforms.
