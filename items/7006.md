## Summary

Current `main` does not implement this feature request. OpenClaw still carries the requested model id through the OpenAI-compatible transport, persists that same runtime model into session metadata, and renders the per-reply usage footer from usage/cost only. For `openrouter/auto`, the bundled catalog cost is still a zero placeholder, so OpenClaw cannot currently show the resolved OpenRouter model name plus its real per-reply cost in the footer.

## Evidence

- **OpenAI-family transport keeps the requested model id:** Both OpenAI-compatible stream paths initialize `output.model` from `model.id` (`openrouter/auto` for auto mode) and hand the requested model into the stream processors. In the inspected parser code, there is no update from a returned `response.model`/`chunk.model` field before the result is finalized.
