## Summary

Keep open. Current main still lacks a first-class `openclaw gateway preflight` command and does not provide the requested shared pre-bind validation path. Existing config validation and startup checks cover part of the request, but context-engine resolution still happens during agent execution and gateway startup still binds before post-attach sidecars/channels run. The related safe-update flow also has no stop -> install -> gateway preflight -> rollback/start path.

## Best Possible Solution

Keep the issue open and implement this as core gateway/update work: factor the existing startup path into a no-listen preflight mode that reads and validates the config snapshot, resolves auth/bind/port, builds the plugin registry/runtime config, resolves the configured context engine without touching sessions, probes port availability with existing port helpers, and emits structured `--json` checks. Then wire a safe update path to stop, install, run that preflight, and start or roll back. Avoid a parallel validator that can drift from real startup.
