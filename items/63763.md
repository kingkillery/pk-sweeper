## Summary

Keep open. Current main documents and tests the plugin CLI contract clearly, but the evidence in this repo does not prove whether `lossless-claw` itself declares the required root-CLI metadata or only runtime slash-command aliases. Because `lossless-claw` is an external community plugin and the issue has no follow-up reproduction on current main, there is not strong enough evidence to auto-close this as implemented, cannot-reproduce, or incoherent.

## Evidence

- **Lossless Claw is external, not bundled in core:** The repo documents `lossless-claw` as a community plugin installed from npm/GitHub, which means its actual CLI registration metadata lives outside this checkout.
  - file: [docs/plugins/community.md:64](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/docs/plugins/community.md#L64)

  - file: [docs/plugins/sdk-overview.md:161](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/docs/plugins/sdk-overview.md#L161)

  - file: [docs/plugins/manifest.md:193](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/docs/plugins/manifest.md#L193)
