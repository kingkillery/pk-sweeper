## Summary

Keep open. Current main partially improves the drain UX by returning a friendlier restart message for GatewayDrainingError in the agent runner, but it still rejects new command enqueues during gateway drain, so the original inbound message is not queued or replayed after restart. The related #51917 is open and overlaps the auto-retry/resume work, but it does not fully supersede this issue's drain-message UX scope. The Kafka-producer PR #57090 was closed unmerged and no Kafka plugin is present on main.

## Best Possible Solution

Keep the item open or explicitly supersede it with #51917 after maintainer review. The product fix belongs in the core gateway/reply pipeline: during the restart drain window, preserve inbound work in FIFO order per channel/session, replay it after resetAllLanes/startup recovery, and keep the friendly restart copy for any path that still cannot be queued.
