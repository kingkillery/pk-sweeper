## Summary

Keep open. Current `main` still appears to create default ACP bridge sessions as isolated `acp:<uuid>` keys without initializing persisted ACP metadata, while the ACP session manager still treats ACP-shaped keys with missing metadata as stale and fails turns with `ACP_SESSION_INIT_FAILED`. The docs still advertise that default isolated mapping, so the reported bridge failure mode does not look resolved on `main`.

## Evidence

- **ACP bridge still defaults new sessions to ACP-shaped fallback keys:** `AcpGatewayAgent.newSession()` resolves the session key with fallback `acp:${sessionId}` and only creates the in-memory ACP session mapping; there is no ACP metadata initialization in this path.
