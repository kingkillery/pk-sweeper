## Summary

Keep open. Current `main` still intentionally emits a `## Current Date & Time` section whose only content is `Time zone: ...`, with separate guidance to use `session_status` for actual current date/time. The behavior the issue reports is therefore still present, not implemented away, and the docs are internally inconsistent enough that the report remains actionable.
