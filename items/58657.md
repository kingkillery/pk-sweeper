## Summary

Keep open. The issue is recent, concrete, contributor-authored, has no protected labels that would change handling, and current main does not show a clear landed fix for the reported acpx/codex-acp durable-session self-ingestion failure. I found related ACP bounds for prompt/replay/projection, but not enough evidence that large `tool_call_update` raw outputs are capped before acpx durable persistence or that poisoned `~/.acpx/sessions` replay is repaired.

## Evidence

- **Issue is actionable and not stale:** The provided issue body includes a concrete failure pattern, observed OOM logs, affected OpenClaw version, command pattern, oversized `~/.acpx/sessions` artifacts, and likely fix areas. It was opened 2026-04-01, less than 60 days before this review date, so `stale_insufficient_info` does not apply.
  - command: `provided GitHub context for issue #58657`
- **Current ACP bridge still forwards raw tool outputs:** Current main's ACP translator sends `tool_call_update` events with `rawOutput: partialResult` for streaming tool updates and `rawOutput: data.result` for final tool results. I did not find evidence in this path that raw tool output is size-capped before being emitted as ACP session updates.
  - file: [src/acp/translator.ts](https://github.com/openclaw/openclaw/blob/44114328b4e73074ac72126f1230ddbf9c1ea8b5/src/acp/translator.ts)
  - command: `web inspection of current main source for `rawOutput: partialResult` and `rawOutput: data.result``
  - sha: [44114328b4e7](https://github.com/openclaw/openclaw/commit/44114328b4e73074ac72126f1230ddbf9c1ea8b5)
- **Existing limits are adjacent, not dispositive:** Current main has a 2 MB ACP prompt limit and an `ACP_LOAD_SESSION_REPLAY_LIMIT = 1_000_000` constant, and docs expose `acp.stream.maxOutputChars`/`maxSessionUpdateChars` for projected stream output. These do not prove the reported acpx durable `tool_call_update` self-ingestion and replay amplification is fixed.
  - file: [src/acp/translator.ts](https://github.com/openclaw/openclaw/blob/44114328b4e73074ac72126f1230ddbf9c1ea8b5/src/acp/translator.ts)
