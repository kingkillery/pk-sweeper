## Summary

Keep open. On current `main` (`7d1891e6e63dc35dd9dcb688ba308049e68fe695`), the documented `openclaw gateway status --json` path still routes through the daemon-status implementation, and that JSON payload has no `version` or `runtimeVersion` field. There is a separate gateway-status JSON envelope that can expose `targets[].self.version` from system presence, but the user-facing `openclaw gateway status` command is not wired to that surface.
