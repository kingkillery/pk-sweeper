## Summary

Keep open. On current `main` (`86f8c826e20a01d1bba67c0ba6e73182fbf0455f`), the OpenAI transport still constructs `OpenAI`/`AzureOpenAI` clients without passing any HTTP timeout, while the transport fetch helper only honors a timeout when callers supply one. That leaves the reported SDK-level timeout path unaddressed in the inspected code.
