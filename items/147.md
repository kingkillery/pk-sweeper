## Summary

Keep open. Current main has building blocks for external voice nodes (`voice.transcript`, global wake-word sync, `/v1/chat/completions` SSE, cross-platform headless node transport), but it does not ship or document a first-class Raspberry Pi/Brabble distributed voice-wake node. The existing headless node host is scoped to `system.run`/`system.which`, and the voice-wake docs only cover macOS/iOS plus manual Android behavior.

## Evidence
