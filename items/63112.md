## Summary

Keep open. Current `main` still accepts `--system-event` payloads for main-session cron jobs without any shell-command warning, and the docs still describe main-session cron as enqueuing notifications rather than executing commands. The PR remains a plausible user-facing fix rather than something already implemented or obsolete.
