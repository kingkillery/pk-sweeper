## Summary

Keep open. The PR still appears to contain useful unmerged behavior: current main only formats assistant errors for `stopReason === "error"`, does not treat aborted assistant turns as error surfaces, and still allows reasoning text to be emitted on aborted turns when reasoning is enabled. The PR discussion is also relevant because Greptile identified the reasoning leak, and the current PR head includes a test and code change addressing that gap.
