## Summary

Keep open. Current `main` does not implement this PR's poll-stripping behavior: send requests with non-zero/true poll fields are still treated as poll intent and rejected, while only zero-valued numeric defaults were relaxed. The `components` part of the PR is partly stale because current send flows use `presentation`/`interactive` normalization rather than the older raw `components` path, so this would need a rebase/rethink rather than closure.
