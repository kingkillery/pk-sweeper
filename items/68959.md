## Summary

Current `main` still does not expose a Control UI chat-side agent switcher/new-session-by-agent flow. The chat header selector is session-based and populated from existing session rows, while agent selection exists separately in the `/agents` management tab. The chat view also has agent-switch props wired through state, but it does not render a control that uses them.

## Evidence

- **Chat header selector switches sessions, not agents:** `renderChatSessionSelect()` binds the header `<select>` to `state.sessionKey` and switches by `nextSessionKey`; there is no agent picker rendered here.
  - file: [ui/src/ui/chat/session-controls.ts:23](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/ui/src/ui/chat/session-controls.ts#L23)
  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)
- **Session options come from existing session rows only:** `resolveSessionOptionGroups()` builds options from `sessions?.sessions ?? []`, adds each `row.key`, and finally re-adds only the current `sessionKey`. It does not read configured agents directly, so agents without an existing session cannot appear in the chat selector.
  - file: [ui/src/ui/chat/session-controls.ts:478](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/ui/src/ui/chat/session-controls.ts#L478)
  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)
- **Agent dropdown exists only on the separate Agents page:** `renderAgents()` renders an agent `<select>` in the `/agents` tab toolbar for inspection/config management, which confirms agent selection exists elsewhere but not in chat.
  - file: [ui/src/ui/views/agents.ts:147](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/ui/src/ui/views/agents.ts#L147)
  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)
- **Chat view defines agent-switch props but does not render them:** The chat props include `agentsList`, `currentAgentId`, and `onAgentChange`, and the app container passes a handler that would switch to `buildAgentMainSessionKey({ agentId })`; however, `rg -n "currentAgentId|onAgentChange|onNavigateToAgent|agentsList" ui/src/ui/views/chat.ts ui/src/ui/app-render.ts` shows these are only declared/passed, not rendered into a visible chat control.
  - file: [ui/src/ui/app-render.ts:2311](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/ui/src/ui/app-render.ts#L2311)
  - command: `rg -n "currentAgentId|onAgentChange|onNavigateToAgent|agentsList" ui/src/ui/views/chat.ts ui/src/ui/app-render.ts`
  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)
- **Current Control UI docs still describe sessions, not an agent picker:** The user-facing Control UI docs list session browsing and per-session overrides under current capabilities, but do not document any agent switcher or start-chat-with-agent flow.
  - file: [docs/web/control-ui.md:107](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/docs/web/control-ui.md#L107)
