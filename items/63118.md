## Summary

Keep open. Current `main` has partial session-stop plumbing (`sessions.abort`, `/sessions/:sessionKey/kill`, chat stop controls), but the user-facing feature request in #63118 is still not implemented: `openclaw sessions` exposes listing and cleanup only, the Sessions Control UI exposes refresh/filter/delete but no stop/pause/resume actions, and I found no pause/resume or per-session resource-limit surface.

## Evidence

- **CLI docs only advertise list and cleanup:** The public `openclaw sessions` docs show listing flags plus the `cleanup` subcommand; there is no documented `stop`, `kill`, `pause`, or `resume` CLI command.
  - file: [docs/cli/sessions.md:8](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/docs/cli/sessions.md#L8)
