## Summary

Current `main` still leaves Chat Completions `tool_choice` unset unless a caller explicitly provides it, and the existing vLLM/local-backend docs only offer compat downgrades like `requiresStringContent`, `localModelLean`, or disabling tools entirely. That makes this report still plausible and actionable on `main`, so it is not a conservative auto-close candidate.

## Evidence
