## Summary

Keep open. Current `main` still routes explicit `memorySearch.provider: "ollama"` through the generic API-key-required branch in `noteMemorySearchHealth`, while the Ollama embedding runtime still works without any API key and docs still describe Ollama memory search as not needing one.

## Evidence

- **Doctor still warns for any explicit non-local/non-lmstudio provider without a key:** In the explicit-provider path, only `local` and `lmstudio` are special-cased. Everything else falls through to `hasApiKeyForProvider(...)`, then emits `"Memory search provider is set to \"${resolved.provider}\" but the API key was not found in the CLI environment."` when the gateway probe is ready.
