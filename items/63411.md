## Summary

Keep open. Current `main` still implements LLM idle timeout as a no-chunk watchdog, existing circuit breakers target tool-call loops rather than streaming text repetition, and transcript hygiene only drops/moves tool-result artifacts around aborted turns instead of rolling back the looping assistant content described in the issue.

## Evidence

- **Idle timeout is still chunk-activity based:** `streamWithIdleTimeout(...)` races `streamIterator.next()` against a timer and throws only when no next chunk arrives before the timeout. The public docs say the same thing: `agents.defaults.llm.idleTimeoutSeconds` aborts when "no response chunks arrive before the idle window." That means a model that keeps streaming repetitive tokens would still reset the watchdog.

I found no analogous detector on streamed assistant text, n-grams, or `<think>` repetition.
