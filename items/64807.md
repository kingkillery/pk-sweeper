## Summary

Keep open. Current `main` still has the reported divergence path: the CLI checks the live registry, while the gateway snapshot/UI hydrates `updateAvailable` from persisted `update-check.json` and returns early during the throttle window. If the channel-tag resolve returns no version, the gateway writes a fresh `lastCheckedAt` and exits without repairing or refreshing availability, so a stale or empty cache can suppress the Overview banner until the next interval.

## Evidence

That matches the reported CLI/UI divergence when the persisted cache is stale.
