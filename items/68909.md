## Summary

Keep open. Current `main` still lacks the system-bus runtime fallback, cgroup-aware stale-PID dedup, and hint suppression described in the PR, so the reported headless Linux/system-level systemd gap is not implemented. The PR discussion also contains an unresolved correctness finding about selecting a node service as the primary gateway unit, so this is not a safe conservative close.

## Evidence

- **Current runtime probe is still user-bus only:** `readSystemdServiceRuntime()` returns `status: "unknown"` immediately when `assertSystemdAvailable()` fails, then otherwise calls `execSystemctlUser(..., ["show", unitName, ...])`. There is no system-bus fallback on current `main`.
  - file: [src/daemon/systemd.ts:670](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/src/daemon/systemd.ts#L670)
  - command: `nl -ba src/daemon/systemd.ts | sed -n '670,705p'`
  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)
- **Current stale-PID logic is not cgroup-aware:** `inspectGatewayRestart()` still marks any non-owned gateway listener PID as stale while runtime is running. The current code has no `/proc/<pid>/cgroup` read or service-cgroup comparison to exempt sibling system services.
  - file: [src/cli/daemon-cli/restart-health.ts:186](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/src/cli/daemon-cli/restart-health.ts#L186)
