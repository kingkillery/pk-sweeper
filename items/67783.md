## Summary

Keep open. This PR is a focused Feishu/Lark bug fix with a concrete reproduction, screenshots, regression-test plan, and a supportive Greptile review. Current `main` still routes the Feishu typing reaction to `replyToMessageId`; in topic or `replyInThread` mode that value is intentionally the thread/root message, so the reported UX bug is still present in source. The related bug report #70209 being closed does not supersede this PR because I did not find the fix on current `main`.

## Best Possible Solution

Keep the PR open for maintainer review or merge a current-main equivalent: add an optional inbound message id to the Feishu reply dispatcher, pass `ctx.messageId` from both `handleFeishuMessage` dispatcher call sites, target typing reactions at that inbound id with the existing `replyToMessageId` fallback, and keep focused regression coverage for topic/replyInThread plus fallback behavior.
