## Summary

Keep open. Current `main` still does not treat manifest `commandAliases` like `wiki` as valid `plugins.allow` / `plugins.entries` refs in the stale-plugin doctor scan, so the bug this PR targets appears unresolved and the proposed patch is still not sufficient.

## Evidence

- **Doctor stale-plugin scan only checks canonical plugin ids:** `collectPluginRegistryState` builds `knownIds` from `registry.plugins.map((plugin) => plugin.id)`, and `scanStalePluginConfigWithState` only suppresses hits when `knownIds.has(normalizePluginId(...))` is true. There is no `commandAliases` set in this path on current `main`.
  - file: [src/commands/doctor/shared/stale-plugin-config.ts:21](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/commands/doctor/shared/stale-plugin-config.ts#L21)
  - command: `nl -ba src/commands/doctor/shared/stale-plugin-config.ts | sed -n '1,220p'`
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **Plugin id normalization still ignores manifest command aliases:** `normalizePluginId` populates its lookup from bundled plugin ids, provider ids, legacy plugin ids, and a few hard-coded fallbacks. It does not read `plugin.commandAliases`, so `wiki` does not normalize to `memory-wiki`.
  - file: [src/plugins/config-state.ts:49](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/plugins/config-state.ts#L49)

- **Bundled memory-wiki plugin now declares the alias from the PR discussion:** The bundled manifest already declares `"commandAliases": [{ "name": "wiki" }]`, so current core behavior still misses a real manifest contract already present on `main`.
  - file: [extensions/memory-wiki/openclaw.plugin.json:177](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/memory-wiki/openclaw.plugin.json#L177)

  - file: [docs/plugins/manifest.md:193](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/docs/plugins/manifest.md#L193)

Current `main` still reflects that gap in the doctor scan path.
