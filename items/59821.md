## Summary

Keep open. The PR is a small, plausible bug fix for documented bundled-skill allowlist semantics, has a targeted regression test, and the only review comment agrees it is safe. I did not find high-confidence evidence that current main already implements the empty-array distinction, so conservative cleanup should not close it.

## Evidence

- **PR reports a concrete config bug and proposes a narrow fix:** The PR body identifies that `normalizeAllowlist()` collapses an explicit empty array to `undefined`, and `isBundledSkillAllowed()` then treats `undefined` and `[]` as unrestricted. The patch preserves empty arrays and makes `allowlist === undefined` the only unrestricted state.
  - file: [src/agents/skills/config.ts:44](https://github.com/openclaw/openclaw/blob/60e452b20b64a80a5a5e3ff9c3076eb5b4c4aa0f/src/agents/skills/config.ts#L44)
  - sha: [60e452b20b64](https://github.com/openclaw/openclaw/commit/60e452b20b64a80a5a5e3ff9c3076eb5b4c4aa0f)
- **Regression coverage included in PR:** The PR adds `empty allowBundled blocks all bundled skills`, asserting a bundled `healthcheck` skill is excluded while a workspace skill remains visible under `config: { skills: { allowBundled: [] } }`.
  - file: [src/agents/skills.build-workspace-skills-prompt.applies-bundled-allowlist-without-affecting-workspace-skills.test.ts:37](https://github.com/openclaw/openclaw/blob/60e452b20b64a80a5a5e3ff9c3076eb5b4c4aa0f/src/agents/skills.build-workspace-skills-prompt.applies-bundled-allowlist-without-affecting-workspace-skills.test.ts#L37)
  - sha: [60e452b20b64](https://github.com/openclaw/openclaw/commit/60e452b20b64a80a5a5e3ff9c3076eb5b4c4aa0f)
- **Existing review supports the fix:** The Greptile bot review says the PR correctly distinguishes `undefined` (no restriction) from `[]` (block all), preserves workspace skills, and found no P0/P1 issues.
- **Public docs support the requested behavior:** Current docs describe `allowBundled` as an optional allowlist where, when set, only listed bundled skills are eligible while managed/workspace skills are unaffected. That contract makes an explicitly set empty allowlist a meaningful block-all configuration.
