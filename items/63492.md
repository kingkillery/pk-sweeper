## Summary

Keep open. Current main still implements session persistence as filesystem-backed `sessions.json` plus per-session `.jsonl` transcripts; configuration only supports choosing a file path, and docs still say horizontal scaling needs sticky sessions or external state management. That means the requested Redis/Mongo-style pluggable backend is not implemented on `main`, and this is core scope rather than a ClawHub/plugin close.
