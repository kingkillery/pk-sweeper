## Summary

Keep open. Current `main` still applies `agents.defaults.memorySearch.extraPaths` to each agent independently and still defaults memory search storage to per-agent SQLite files, so the reported cross-agent duplication risk has not been implemented away on `main`. I found no `sharedPaths` or `_shared.sqlite` implementation in the checkout.

## Evidence

- **Defaults extraPaths are merged into each agent config:** `mergeConfig()` concatenates `defaults?.extraPaths` and `overrides?.extraPaths`, deduplicates them, and returns the merged list as the resolved config for the requested agent.
  - file: [src/agents/memory-search.ts:202](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/agents/memory-search.ts#L202)
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **Default memory store remains agent-scoped:** `resolveStorePath()` still falls back to `memory/<agentId>.sqlite`, which means separate stores per agent by default rather than a shared index.
  - file: [src/agents/memory-search.ts:135](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/agents/memory-search.ts#L135)
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **QMD backend also merges defaults extraPaths per agent:** `resolveMemoryBackendConfig()` builds `mergedExtraPaths` from `agents.defaults.memorySearch.extraPaths` plus the selected agent's overrides, then turns those into that agent's collections.
  - file: [src/memory-host-sdk/host/backend-config.ts:368](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/memory-host-sdk/host/backend-config.ts#L368)
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **Tests still assert per-agent merged extraPaths behavior:** The current test suite expects defaults and per-agent `memorySearch.extraPaths` to merge for one agent, matching the issue's described topology rather than a shared index.
  - file: [src/agents/memory-search.test.ts:281](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/agents/memory-search.test.ts#L281)
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **Docs still describe extraPaths as added to the memory index and default store path as per-agent SQLite:** The config docs describe `extraPaths` as additional paths for the memory index, and the schema help still documents the default store path as `~/.openclaw/memory/{agentId}.sqlite`.
  - file: [docs/reference/memory-config.md:274](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/docs/reference/memory-config.md#L274)
