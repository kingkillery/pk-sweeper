## Summary

Keep open. Current `main` still falls back into hook-pack validation after ordinary plugin install failures and still formats the combined `Also not a valid hook pack` error. The only current escape hatch is the special case for `--dangerously-force-unsafe-install`, which matches the lone issue comment/workaround but does not resolve the issue’s broader reported behavior.

## Evidence

- **Plugin install still falls back to hook-pack validation on normal failures:** For local-path installs, failed plugin installs call `tryInstallHookPackFromLocalPath(...)` and then format the combined plugin + hook-pack error unless a narrow forced-unsafe guard triggers first.
  - file: [src/cli/plugins-install-command.ts:428](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/cli/plugins-install-command.ts#L428)

  - file: [src/cli/plugins-install-command.ts:577](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/cli/plugins-install-command.ts#L577)
