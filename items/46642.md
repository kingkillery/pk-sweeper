## Summary

Current `main` does not appear to implement this PR’s core behavior. The embedded runner result types still expose no `finalAuthProfileId`, the attempt execution path only passes the existing session `authProfileOverride` into the run, and the post-run session-store update persists model/token metadata but does not write back a winning auth profile. The runner still only marks the final profile as `lastGood`/`lastUsed` in auth state.
