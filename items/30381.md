## Summary

Keep open. Current main still rejects a generic request body model such as gpt-4 even when x-openclaw-agent-id selected the agent, so the requested behavior is not implemented.

## Evidence

- **Header can select the agent, but model validation is separate:** resolveAgentIdFromHeader reads x-openclaw-agent-id, while resolveOpenAiCompatModelOverride immediately rejects any request model that does not parse as openclaw, openclaw/default, openclaw/<agentId>, or agent:<agentId>. That means a header-selected agent plus model: "gpt-4" still returns a 400.
