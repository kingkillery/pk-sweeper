## Summary

Current `main` still intentionally adds Claude CLI `--permission-mode bypassPermissions` when OpenClaw's effective exec policy is YOLO, and both docs and tests assert that behavior. This PR is proposing a real behavior change, not fixing something already implemented or obsolete, and the existing review discussion raises a plausible non-interactive timeout regression if the bypass is removed.
