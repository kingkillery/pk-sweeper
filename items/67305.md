## Summary

Keep open. The request is coherent, recent, and still unimplemented on current `main`: OpenClaw only exposes timer-based queue collection today, and the same debounce-based behavior is used for busy-session announce/completion delivery. This looks like a legitimate core queueing feature request rather than a ClawHub/plugin-only item.

## Evidence

- **Issue is recent and specific, not stale/incoherent:** Issue #67305 was opened on 2026-04-15 with concrete desired behavior, acceptance criteria, and core implementation targets (`until_ready`, session-level aggregate buffer, starvation protection). There are no comments or timeline items changing that reading.
- **Docs describe only timer-based queue collection:** Current queue docs list `steer`, `followup`, `collect`, `steer-backlog`, `interrupt`, and `queue`; `collect` is documented as debounce-based, with `debounceMs` described as 'wait for quiet before starting a followup turn.' No readiness-aware flush mode is documented.
