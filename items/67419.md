## Summary

Keep open. Current main still defaults `agents.defaults.contextInjection` to `"always"`, so the central reported behavior, bootstrap context being injected on every normal follow-up unless users opt into `continuation-skip`, is still present. The related fix PR #67447 was closed unmerged in the provided GitHub context, and the maintainer umbrella #69208 explicitly says #67419 should stay as a concrete repro unless proven to be a strict duplicate.

## Best Possible Solution

Land a scoped fix that changes the default context-injection mode to `continuation-skip`, updates config docs/help/generated schema and regression tests, and keeps `"always"` as an explicit compatibility override. Track the remaining MEMORY.md stub/cache-boundary and duplicate transcript work through the existing narrower issues or #69208 after confirming which symptoms remain.
