## Summary

Keep open. The issue is a plausible, concrete feature request for per-candidate/provider-model retry tuning, and current main evidence only shows global auth cooldown/profile-rotation knobs plus model-scoped cooldown behavior, not the requested per-fallback-candidate retry budget. The linked #55118/#58707 discussion is useful and partially related, but the reporter explicitly narrowed the remaining gap to different retry budgets for pooled proxy vs direct fallback candidates.
