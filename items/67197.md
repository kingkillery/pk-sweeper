## Summary

Keep #67197 open. Current main still implements fallback state tracking, /status visibility, lifecycle events, and verbose-mode fallback notices, but it does not provide an independent opt-in config for normal non-verbose replies. The linked PR #69399 appears to be the focused implementation path, but it is open/unmerged in the provided GitHub context and the checked-out main still explicitly tests that verbose-off replies do not include fallback notices.

## Best Possible Solution

Keep #67197 open until PR #69399 or an equivalent maintainer-approved implementation lands. The likely code path is to add a default-false `agents.defaults.notifyOnModelFallback`-style config field, wire it into `runReplyAgent` so fallback and fallback-cleared notices can be prepended when either verbose mode or that opt-in is enabled, regenerate config docs/schema artifacts, and update the existing e2e coverage for non-verbose enabled/disabled behavior.
