## Summary

Keep open. Current `main` still has the reported exec-completion relay gap for heartbeat runs that switch into an isolated relay session: the exec result is queued on the original session, but the relay prompt still claims the result is "shown in the system messages above" while the reply pipeline drains system events from the new run session instead.

## Evidence

- **Exec completion still queues output on the original session event queue:** Background exec completion builds a summary like `Exec completed (... ) :: <output>`, enqueues it as an untrusted system event on `session.sessionKey`, and immediately requests a heartbeat wake for reason `exec-event`.
  - file: [src/agents/bash-tools.exec-runtime.ts:279](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/src/agents/bash-tools.exec-runtime.ts#L279)
  - command: `nl -ba src/agents/bash-tools.exec-runtime.ts | sed -n '279,307p'`
  - sha: [6f72b74cecf8](https://github.com/openclaw/openclaw/commit/6f72b74cecf8d1fec5c147ae932161bf0f37a68b)
- **Heartbeat preflight still inspects pending exec events from the base session:** For `exec-event` wakes, preflight peeks `pendingEventEntries` from `session.sessionKey` and unconditionally sets `shouldInspectPendingEvents` true. There is no matching guard here for the later isolated-session handoff.
  - file: [src/infra/heartbeat-runner.ts:550](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/src/infra/heartbeat-runner.ts#L550)
  - command: `nl -ba src/infra/heartbeat-runner.ts | sed -n '550,590p'`

- **Heartbeat run can switch to a fresh `:heartbeat` session before reply generation:** When `heartbeat.isolatedSession === true`, `runHeartbeatOnce` replaces `runSessionKey` with a fresh isolated session key before calling `getReplyFromConfig`, while `ctx.SessionKey` is set to that new run session.
  - file: [src/infra/heartbeat-runner.ts:867](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/src/infra/heartbeat-runner.ts#L867)

- **The exec relay prompt still assumes missing context is already present:** `buildExecEventPrompt()` still emits the exact prompt shape described in the issue: "The result is shown in the system messages above" plus "Please relay the command output to the user...". Unlike cron prompts, it does not embed the event text into the prompt body.
  - file: [src/infra/heartbeat-events-filter.ts:41](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/src/infra/heartbeat-events-filter.ts#L41)

In the isolated heartbeat path, that means the fresh relay session is drained, not the original session that held the exec completion event.
