## Summary

Current `main` does not implement an operator-configurable browser URL/domain denylist or warnlist. The browser config surface only exposes `browser.ssrfPolicy` host allowlist/private-network controls, and the tab open/navigation path enforces only those SSRF/protocol guards. This remains a plausible unimplemented feature request, so it should stay open.

## Evidence

- **Browser config type has no denylist field:** `BrowserConfig` defines `enabled`, profile/CDP settings, `snapshotDefaults`, `ssrfPolicy`, and `extraArgs`, but nothing like `browser.denylist`, `browser.blockedUrls`, or a path-pattern block list.

- **Published browser docs only document SSRF policy allowlist knobs:** The browser config example and SSRF section document `browser.ssrfPolicy.dangerouslyAllowPrivateNetwork`, `hostnameAllowlist`, and `allowedHostnames`; there is no documented blocked-URL/domain denylist option.
  - file: [docs/tools/browser.md:95](https://github.com/openclaw/openclaw/blob/84fc00afdacbbb67e39fc1341242eedad09243df/docs/tools/browser.md#L95)
