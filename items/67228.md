## Summary

Keep open. Current main has new env-proxy/NO_PROXY support for trusted HTTP fetches and Feishu's ambient WebSocket agent, but it still does not expose the requested per-provider/per-channel proxy configuration for Brave or Feishu, and WeChat is owned by an external plugin rather than OpenClaw core.

## Evidence

- **Brave web-search config has no proxy field:** The public Brave config example only exposes `apiKey` and `mode`, and the bundled Brave plugin manifest schema likewise only defines `webSearch.apiKey` and `webSearch.mode`. There is no per-provider proxy setting on main.
  - file: [docs/tools/brave-search.md:21](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/docs/tools/brave-search.md#L21)
  - sha: [6f72b74cecf8](https://github.com/openclaw/openclaw/commit/6f72b74cecf8d1fec5c147ae932161bf0f37a68b)
- **Brave manifest schema confirms missing proxy surface:** Bundled Brave plugin config schema defines only `apiKey` and `mode` under `webSearch`; no proxy option is available.
  - file: [extensions/brave/openclaw.plugin.json:24](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/extensions/brave/openclaw.plugin.json#L24)
  - sha: [6f72b74cecf8](https://github.com/openclaw/openclaw/commit/6f72b74cecf8d1fec5c147ae932161bf0f37a68b)
- **Feishu docs still expose no channel proxy setting:** The Feishu config table lists channel settings such as `connectionMode`, webhook fields, and account credentials, but no `channels.feishu.proxy` or account-level proxy field.
  - file: [docs/channels/feishu.md:375](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/docs/channels/feishu.md#L375)
  - sha: [6f72b74cecf8](https://github.com/openclaw/openclaw/commit/6f72b74cecf8d1fec5c147ae932161bf0f37a68b)
- **Feishu runtime still uses ambient env proxy only:** Feishu WebSocket setup calls `resolveAmbientNodeProxyAgent()` and passes the resulting agent into the SDK client; there is no Feishu-specific proxy config path in the runtime.
  - file: [extensions/feishu/src/client.ts:85](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/extensions/feishu/src/client.ts#L85)
  - sha: [6f72b74cecf8](https://github.com/openclaw/openclaw/commit/6f72b74cecf8d1fec5c147ae932161bf0f37a68b)
- **Main now supports NO_PROXY for trusted HTTP fetches:** Core proxy fetch helpers explicitly state that env-proxy fetch respects `NO_PROXY`/`no_proxy`, and `matchesNoProxy()` reimplements Undici NO_PROXY matching semantics for trusted env-proxy HTTP helpers.
  - file: [src/infra/net/proxy-fetch.ts:49](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/src/infra/net/proxy-fetch.ts#L49)
  - sha: [6f72b74cecf8](https://github.com/openclaw/openclaw/commit/6f72b74cecf8d1fec5c147ae932161bf0f37a68b)
- **WeChat path is external-plugin-owned, not core-owned:** The WeChat docs state that the WeChat runtime does not live in the OpenClaw core repo and that monitoring/API behavior is owned by the external `@tencent-weixin/openclaw-weixin` plugin, so this repo cannot prove per-channel proxy support there.
  - file: [docs/channels/wechat.md:27](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/docs/channels/wechat.md#L27)
  - sha: [6f72b74cecf8](https://github.com/openclaw/openclaw/commit/6f72b74cecf8d1fec5c147ae932161bf0f37a68b)
- **Other channels already have explicit proxy knobs:** Current docs already document explicit per-channel proxy settings for Telegram, Matrix, and Discord, which shows the requested surface exists in principle but has not been added for Brave/Feishu here.
  - file: [docs/channels/discord.md:749](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/docs/channels/discord.md#L749)
  - sha: [6f72b74cecf8](https://github.com/openclaw/openclaw/commit/6f72b74cecf8d1fec5c147ae932161bf0f37a68b)
- **Feishu ambient proxy dependency supports NO_PROXY:** `package.json` pins `proxy-agent`, Feishu's ambient proxy helper constructs `new ProxyAgent()`, and `npm view proxy-agent@8.0.1 readme` states that it uses proxy env vars including `NO_PROXY` via `proxy-from-env`. That helps the minimum env-based workaround, but it is still not the per-provider/per-channel config requested in this issue.
  - file: [package.json:1617](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/package.json#L1617)
  - command: `npm view proxy-agent@8.0.1 version readme --json`
  - sha: [6f72b74cecf8](https://github.com/openclaw/openclaw/commit/6f72b74cecf8d1fec5c147ae932161bf0f37a68b)

## Risks / Open Questions

- The new NO_PROXY-capable env path may reduce the original pain for Brave+Feishu setups, but it does not provide the explicit per-provider/per-channel config surface requested by the issue title/body.
- WeChat behavior cannot be verified from this repo because the runtime is external-plugin-owned; closing from core-only evidence would be premature.
