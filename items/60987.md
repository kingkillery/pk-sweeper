## Summary

Keep open. Current `main` still writes `models: []` into the MiniMax OAuth config patch, and the only shipped follow-up I found addressed `api`/`authHeader`, not the empty-models behavior. There is merge logic that may preserve the implicit MiniMax portal catalog despite an empty explicit list, but there is no issue-specific regression test or end-to-end proof in-tree that the OAuth onboard path now resolves `minimax-portal/*` models correctly.
