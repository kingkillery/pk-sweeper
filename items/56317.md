## Summary

Current main already fixes the reported OpenRouter auto-compaction failure by prechecking prompt size locally and triggering compaction before the provider call, so it no longer depends on OpenRouter returning a context-overflow error or exposing a context-compression toggle.

## Close Comment

Closing from a Codex review because current OpenClaw already fixes the reported failure mode on `main` and in the latest release. Auto-compaction no longer depends solely on an upstream `context overflow` response: `src/agents/pi-embedded-runner/run/preemptive-compaction.ts:1` estimates prompt size locally, and `src/agents/pi-embedded-runner/run/attempt.ts:2278` routes oversized prompts into precheck compaction/truncation before the provider call. The user docs were updated to match this behavior in `docs/concepts/compaction.md:97`, and the precheck path is covered by regression tests in `src/agents/pi-embedded-runner/run/preemptive-compaction.test.ts:68` and `src/agents/pi-embedded-runner/run.overflow-compaction.loop.test.ts:237`. I also verified the shipped `v2026.4.22` release commit (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`) contains that implementation. This means the specific OpenRouter context-compression toggle proposed here is no longer required to make auto-compaction work for small-context routes.
