## Summary

Current `main` does not include this PR's substantive behavior changes. The overflow recovery loop still retries tool-result truncation whenever oversized tool output exists, without the PR's reduction-potential gate; attempt-local overflow estimates are still not threaded out of `run/attempt.ts`; and the auto-reply layer still resets sessions on plain `context_overflow` payloads. Main has adjacent preemptive-compaction work, but not this PR's run-loop and fallback changes, so there is not strong evidence to close it.
