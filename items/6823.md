## Summary

Current main already provides the requested safety model: host `exec` is guarded by built-in exec approvals/allowlists/ask-fallback, and plugins can intercept any tool call with `before_tool_call` to either block it or require human approval. The exact YAML sketch from the issue was not added, but the underlying capability is present and shipped.

## Close Comment

Closing from a Codex review because current `main` already has this capability, even though it does not use the exact YAML shape proposed here. Host `exec` is already gated by built-in exec approvals, allowlists, and ask-fallback policy (`docs/tools/exec-approvals.md`, `src/agents/bash-tools.exec-host-shared.ts`), and generic tool interception now exists through plugin `before_tool_call` hooks that can either block a tool call or require human approval (`src/plugins/hook-types.ts`, `src/agents/pi-tools.before-tool-call.ts`, `src/gateway/server-methods/plugin-approval.ts`). The approval UX is documented for the exec overlay, Telegram/Discord interactions, and `/approve`, and the behavior is covered by end-to-end tests for allow, deny, timeout, and fail-closed cases (`docs/plugins/building-plugins.md`, `src/agents/pi-tools.before-tool-call.e2e.test.ts`). I also confirmed these surfaces are already present in release `v2026.4.22`.
