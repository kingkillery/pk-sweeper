## Summary

Current `main` does not implement the requested behavior. OpenClaw still limits Responses reasoning-compat payload shaping to built-in native OpenAI/Codex/Azure routes, and the tests/docs explicitly preserve the opposite behavior for proxy-style custom providers, so there is not strong evidence to close this issue.

## Evidence

- **Current guard excludes custom providers from OpenAI reasoning-compat payload shaping:** `supportsOpenAIReasoningCompatPayload` is true only for `openai`, `openai-codex`, `azure-openai`, and `azure-openai-responses`, and only when the route is not proxy-like.
