## Summary

Keep open. Current `main` still enforces a strict Feishu config schema, but the Feishu bundled channel does not expose any doctor compatibility normalizer or stale-config cleaner. That means the upgrade break described here remains plausible on current `main`, especially given the issue discussion expanded the failing legacy keys beyond the original `installs`/`plugins` report.

## Evidence

- **Feishu schema is strict and only allows the current key set:** `FeishuGroupSchema`, `FeishuAccountConfigSchema`, and `FeishuConfigSchema` are all `.strict()`. The accepted shared/top-level fields shown there do not include the legacy keys discussed on the issue (`ackReaction`, `threadSession`, `mediaLocalRoots`, `installs`, `plugins`, or per-account `botName`).
