## Summary

Keep open. Current main already has optional batch embedding and an exact-hash embedding cache, but the issue's requested behavior is only partially implemented. The remaining asks in the issue body, especially fuzzy cache reuse, cache hit/miss/latency stats, and model-switch migration instead of full reindex, are not present on current main.

## Evidence

- **Batch embedding already exists:** Memory indexing can batch missing chunks either through provider-native batch jobs or through in-process batching. The public config docs also expose `remote.batch.*` and state batch indexing is available for `openai`, `gemini`, and `voyage`. That means the issue's first task is already at least partially implemented, so the item cannot be closed as a pure duplicate of missing batch support.
