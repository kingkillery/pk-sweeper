## Summary

Current main already implements the requested suppression option for noisy exec lifecycle system-event chat messages via `tools.exec.notifyOnExit`, and it is covered by targeted tests.

## Evidence

- **Runtime suppression for node exec lifecycle events:** The node event handler explicitly checks `cfg.tools?.exec?.notifyOnExit !== false` and returns early before enqueueing system events when the option is disabled.
  - file: [src/gateway/server-node-events.ts:632](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/gateway/server-node-events.ts#L632)

- **Regression coverage for started/finished/denied:** Tests verify that `exec.started`, `exec.finished`, and `exec.denied` do not enqueue system events or wake heartbeat when `tools.exec.notifyOnExit` is false.

## Risks / Open Questions

- `docs/tools/exec.md:95` still describes `tools.exec.notifyOnExit` only in terms of backgrounded exec exits, so discoverability for the node-exec suppression path is a bit uneven.

## Close Comment

Closing as implemented on current `main`. Codex review on `cec34821754ad935c780805f3116f99449d46ac2` found that the requested suppression option already exists as `tools.exec.notifyOnExit`: `src/gateway/server-node-events.ts:632` skips enqueueing node `exec.started`, `exec.finished`, and `exec.denied` system events when that setting is `false`, and `src/gateway/server-node-events.test.ts:341`, `src/gateway/server-node-events.test.ts:363`, and `src/gateway/server-node-events.test.ts:386` cover those cases. The option is also part of the config surface in `src/config/zod-schema.agent-runtime.ts:469` and described in `src/config/schema.help.ts:581`. I could not determine a released tag containing this from the available checkout, so `fixedRelease` is left unset and this is being closed as implemented on `main`.
