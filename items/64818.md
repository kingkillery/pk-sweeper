## Summary

Keep open. Current `main` still has the same basic failure mode the issue describes: dev-update preflight walks candidate commits with plain `git checkout --detach` and no reset/clean between failed candidates, `scripts/bundle-a2ui.mjs` still rewrites `src/canvas-host/a2ui/.bundle.hash`, and that hash file is still tracked on both `main` and the latest release even though `.gitignore` treats `.bundle.hash` as generated. The documented walkback behavior therefore is not clearly fixed on current code.

## Evidence

- **Preflight walkback still uses plain checkout with no intervening cleanup:** The candidate loop in `runUpdate` still does `git -C <worktree> checkout --detach <sha>` for each candidate and simply `continue`s on any failed checkout/build/lint step. There is no `git reset --hard`, `git clean`, or stash step before moving to the next candidate.
