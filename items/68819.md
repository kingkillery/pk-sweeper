## Summary

Keep open. Current `main` still routes the bundled Anthropic CLI backend through the supervisor with bare `claude`, but the Windows shim handling in that path only knows about `npm`/`pnpm`/`yarn`/`npx` and has no `.cmd`-to-underlying-target rewrite. The PR is therefore addressing a real gap, not something already implemented on `main`.

## Evidence

- **Bundled Anthropic CLI backend still defaults to `claude`:** The shipped Anthropic CLI backend config uses `command: "claude"`, so Windows runs still depend on the supervisor resolving that bare command correctly.
  - file: [extensions/anthropic/cli-backend.ts:30](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/extensions/anthropic/cli-backend.ts#L30)
