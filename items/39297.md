## Summary

Current `main` has partial built-in usage surfaces: chat `/usage cost`, a `openclaw gateway usage-cost` CLI, and a Control UI usage tab backed by `usage.cost` and `sessions.usage`. But the shipped data model only exposes daily totals plus aggregates by model/provider/agent/channel. I did not find the requested built-in by-source or per-cron cost attribution, nor a top-level `openclaw usage` command matching the issue’s proposed scope, so the evidence is not strong enough to close as implemented.
