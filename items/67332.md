## Summary

Keep open. Current main does not implement the PR's observable surface: there is no `task` or `team` core tool, no `task-tool.ts` or `team-tool.ts`, and owner-only core tools are still limited to `cron`, `gateway`, and `nodes`. The existing task registry, CLI task ledger, `/tasks`, and `sessions_spawn` surfaces are related but do not replace the proposed model-facing task/team management tools with owner-only authorization.

## Best Possible Solution

Keep the PR open for maintainer/product review. The likely path is a narrower, current-main-ready implementation that wires first-class `task` and `team` tools only if maintainers still want them in core, marks them owner-only through the existing owner-only tool policy, uses existing task-registry contracts, adds or justifies a team registry, updates tool groups/docs as needed, and resolves the existing review findings before merge.
