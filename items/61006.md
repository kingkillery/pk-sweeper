## Summary

Current `main` still rewrites fallback-retry prompts for sessions with history. `resolveFallbackRetryPrompt` prepends a synthetic retry banner instead of preserving the original body verbatim, and `runAgentAttempt` passes that rewritten prompt into both CLI and embedded runners, so the reported transcript/prompt-mutation issue is not fully implemented away.
