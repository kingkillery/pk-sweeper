## Summary

Keep open. Current main still promotes the rehydrated candidate snippet directly into `MEMORY.md`; scoring, rehydration, and contamination filters decide whether a candidate is promoted, but there is no distillation/summarization/write-policy step and no config option for distilled promotion. The related metadata-contamination PR #67601 is not a canonical fix for this issue because its own scope says it does not redesign durable-memory distillation.

## Best Possible Solution

Keep the issue open for maintainer product judgment and implementation. The likely fix belongs in `memory-core` deep promotion: add a promotion write policy that turns eligible snippets into concise durable memory entries while preserving provenance, plus tests for raw-log suppression, source metadata retention, and docs/config updates. If raw promotion is intentional, document that explicitly and consider a `distilled` promotion mode or equivalent configurable path.
