## Summary

Keep #67404 open. Current main still does not refresh QMD session exports from transcript-update events: QmdMemoryManager exports sessions only during qmd sync/update paths, skips session collections in its filesystem watcher, and does not subscribe to onSessionTranscriptUpdate. The related listener now exists for the builtin memory index path, which supports the PR direction, but it does not cover the QMD manager. The PR also has an unresolved P1 review finding about dropping transcript events during an in-flight QMD update, so it should be revised rather than closed as implemented.

## Best Possible Solution

Keep the PR open and ask for a refreshed patch that wires QmdMemoryManager to transcript updates through an appropriate public SDK boundary, debounces per-agent session changes, uses a forced or queued update path so in-flight qmd updates do not drop later transcript deltas, cleans up the listener/timer on close, and adds qmd-manager coverage for debounce, agent filtering, close cleanup, and in-flight update queuing.
