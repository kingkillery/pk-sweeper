## Summary

Current main already supports the requested Docker/headless-node workflow: run the node in the foreground with `openclaw node run`, then approve its device pairing with `openclaw devices approve`. `openclaw node install` remains intentionally service-manager-only.

## Close Comment

Closing from a Codex review: current main already supports the containerized/headless node flow, but the supported path is `openclaw node run`, not `openclaw node install`. The CLI separates those explicitly: `node run` is the foreground path, while `node install` is the service-manager path (`src/cli/node-cli/register.ts:41`, `src/cli/node-cli/register.ts:75`, `src/cli/node-cli/daemon.ts:89`). The node host connects as a normal paired device with `role: "node"` (`src/node-host/runner.ts:179`), so `pairing required` on first connect is expected until you approve it with `openclaw devices list` and `openclaw devices approve <requestId>` (`docs/cli/node.md:109`, `docs/nodes/index.md:23`, `docs/cli/devices.md:51`). This same behavior is already present in release `v2026.4.22`, so there is not an open core feature gap left to track here.
