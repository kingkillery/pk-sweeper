## Summary

Keep open. Current `main` does not implement exec-approvals denylist support: the exec security model still only accepts `deny`, `allowlist`, and `full`; the approvals file schema still only exposes `allowlist`; and approval prompting logic only treats `on-miss` as an allowlist miss. There is partial argument-pattern support via `argPattern`, but that is attached to allowlist entries and is currently enforced only for Windows path matches, so it does not satisfy the issue’s requested cross-platform `allow everything except X` behavior.
