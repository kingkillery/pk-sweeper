## Summary

Keep open. Current `main` has a fresh partial mitigation for the daemon/gateway status path, but the top-level `openclaw status` and `openclaw health` flows still read config through the normal best-effort config path and still do additional status/plugin work, so there is not strong evidence that issue #67114 is already implemented on `main` or obsolete.

## Evidence
