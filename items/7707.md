## Summary

Keep open. Current `main` has partial untrusted-content safeguards, but it does not implement source-based trust tagging or trust-policy enforcement for memory entries. The memory index and search runtime only distinguish coarse sources (`memory` vs `sessions`), and ranking is limited to vector/BM25 merge, temporal decay, and MMR.

## Risks / Open Questions

- Durable memory still lacks provenance/trust state for recalled content from different origins, so retrieval cannot downweight or isolate low-trust material.
- Existing protections are localized to prompt assembly and transcript sanitation; they do not implement the requested source-tagged memory lifecycle policies.
