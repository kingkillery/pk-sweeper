## Summary

Keep open. Current `main` still lacks `--session-key` on `openclaw agent`, while the underlying session-routing stack already supports explicit session keys. The PR addresses a real CLI parity gap and the discussion adds current downstream demand, so there is not strong evidence for any auto-close reason.
