## Summary

Keep open. Current `main` has broad lifecycle coverage through plugin hooks plus internal `message:received`/`message:sent`, but it does not expose the specific internal `agent:thinking:*`, `agent:response:*`, `agent:tool:*`, `agent:error`, `session:start`, and `session:end` events requested in #7724.

## Evidence

- **Internal hook docs still omit the requested agent-loop events:** The internal hooks page lists `agent:bootstrap`, `gateway:startup`, `message:received`, `message:transcribed`, `message:preprocessed`, and `message:sent`, but not `agent:thinking:*`, `agent:response:*`, `agent:tool:*`, `agent:error`, `session:start`, or `session:end`.
  - file: [docs/automation/hooks.md:36](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/docs/automation/hooks.md#L36)
  - command: `nl -ba docs/automation/hooks.md | sed -n '36,60p'`
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Internal hook implementation only defines bootstrap plus message lifecycle types:** `src/hooks/internal-hooks.ts` defines typed internal hook contexts for `agent:bootstrap` and message lifecycle events; there is no typed internal emitter surface for thinking/response/tool/error lifecycle boundaries.
  - file: [src/hooks/internal-hooks.ts:23](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/hooks/internal-hooks.ts#L23)

## Risks / Open Questions

- Closing as `implemented_on_main` would overstate parity between today's plugin-hook surface and the issue's requested internal `agent:*` event API.
- Some user goals are probably achievable already via plugin hooks, but the exact internal hook names and semantics requested here are still absent from current docs and emitters.
