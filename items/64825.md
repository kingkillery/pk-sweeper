## Summary

Current `main` at `fd65caf4b0bad1b0ac27a82d349c22e92167e7bc` still has the stale-final path described in #64825. `handleChatEvent()` derives `wasActiveRun` from `state.activeChatRunId === evt.runId`, and `finalizeRun()` / `terminateRun()` still only transition the footer when that flag is true. If the active run id is cleared before a `final`/`error`/`aborted` event, the current code can clear the watchdog without restoring `idle`, so the issue is not safely closable as fixed.
