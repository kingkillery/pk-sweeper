## Summary

Keep open. Current `main` exposes some dreaming counters and next-run timing, but the Dreams surface still does not provide the requested run-observability fields like last successful run, explicit run result/status, or workspace/diary update visibility.

## Evidence

- **Dreaming status payload lacks run outcome metadata:** `DoctorMemoryDreamingPayload` includes counts, phase config, and optional `lastPromotedAt`, but no last-run timestamp, success/partial/failure result, failed-count, or diary-write/workspace status fields.

  - sha: [6f72b74cecf8](https://github.com/openclaw/openclaw/commit/6f72b74cecf8d1fec5c147ae932161bf0f37a68b)
- **Gateway status handler only merges counts plus next scheduled run:** `doctor.memory.status` builds the Dreams payload from config, store stats, and managed cron `nextRunAtMs`; it does not attach any last-run/result metadata for completed dreaming cycles.
