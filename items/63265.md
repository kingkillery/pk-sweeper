## Summary

Keep open. Current `main` still resolves doctor config reads/writes from the invoking shell environment and does not pin doctor writes to the previously detected config path, so the reported `/root/.openclaw/openclaw.json` drift remains plausible. The issue is also recent and already has a linked open PR (#63268) in the provided timeline.

## Evidence

- **Doctor preflight still targets the caller home/config path:** `maybeMigrateLegacyConfig()` derives `~/.openclaw/openclaw.json` from `resolveHomeDir()`, and `runDoctorConfigPreflight()` then reads the active snapshot with `readConfigFileSnapshot()` without any running-gateway/service-config divergence guard.
  - file: [src/commands/doctor-config-preflight.ts:12](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/commands/doctor-config-preflight.ts#L12)
  - command: `nl -ba src/commands/doctor-config-preflight.ts | sed -n '1,120p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Config resolution still depends on current env/home:** `resolveCanonicalConfigPath()` and `resolveConfigPathCandidate()` still choose the config file from `OPENCLAW_CONFIG_PATH` or the current env-derived state dir/home. There is no code here that consults a running gateway or service-owned config path before falling back to the caller's default home.
  - file: [src/config/paths.ts:106](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/config/paths.ts#L106)
  - command: `nl -ba src/config/paths.ts | sed -n '90,170p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Doctor records a path but the write path is not pinned:** `doctor-health.ts` stores `configResult.path` on `ctx.configPath`, but `runWriteConfigHealth()` writes with plain `writeConfigFile(ctx.cfg)`. The exported `writeConfigFile()` then creates a fresh config IO from the current environment and only accepts `expectedConfigPath` as metadata for env-ref restoration, not as an override for where to write.
  - file: [src/flows/doctor-health-contributions.ts:464](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/flows/doctor-health-contributions.ts#L464)

That supports keeping the bug open rather than closing it as obsolete.
