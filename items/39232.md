## Summary

Current `main` does not provide a public plugin hook that fires before channel allowlist/sender/mention filtering. Existing public hooks (`message_received`, `inbound_claim`, `before_dispatch`) sit inside the post-routing reply pipeline, while at least WhatsApp and Telegram still drop filtered inbound messages earlier. Telegram/Signal have narrow internal silent-ingest paths for mention-skipped messages, but those are internal HOOK.md/config behavior, not the requested cross-channel plugin SDK seam.
