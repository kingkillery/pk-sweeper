## Summary

Keep open. Current main still forwards wizard select and multiselect labels unchanged, and the existing wizard test file does not cover blank-label fallback. The related onboarding crash PRs in the discussion cover adjacent metadata and text-input `.trim()` paths, but they do not supersede this adapter-level select/multiselect hardening.

## Best Possible Solution

Keep the PR open. The best path is to land a reviewed adapter-level normalization in `src/wizard/clack-prompter.ts` for select, autocomplete select, multiselect, and autocomplete multiselect, with focused tests in `src/wizard/clack-prompter.test.ts`; alternatively, explicitly supersede it with a canonical wizard-hardening PR that covers the same blank-label behavior.
