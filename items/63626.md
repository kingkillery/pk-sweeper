## Summary

Keep open. Current `main` still allows `qmd embed` to run even when `memory.qmd.update.embedInterval` is `0` or very large: `shouldRunEmbed()` bypasses the interval when `lastEmbedAt` is still `null`, and session-start syncs still call into that path. The public config help also says `set 0 to disable periodic embeds`, so current behavior remains at least a docs/behavior mismatch, and the latest shipped release still contains the same logic.
