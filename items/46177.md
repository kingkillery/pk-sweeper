## Summary

Keep open. Current `main` still has the cron-timezone mismatch the PR is trying to fix: runtime falls back to the gateway host timezone when `tz` is omitted, but the agent-tool description and CLI help still do not clearly say that. The PR remains useful, and the existing review comment raises a valid wording issue (`system local timezone` is ambiguous for remote gateways), so this is not a conservative cleanup close.

## Evidence

  - file: [src/cron/schedule.ts:9](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/src/cron/schedule.ts#L9)
