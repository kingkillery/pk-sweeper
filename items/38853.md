## Summary

Keep open. Current main has partial mitigation: channel crash loops now use exponential restart backoff and Discord allowlist resolution falls back to configured entries when guild/user lookup fails. But the report is not fully implemented away: Discord still lacks explicit Cloudflare 1015 handling, still preserves raw non-JSON error bodies, and still hard-fails startup when `/oauth2/applications/@me` returns a non-OK response.

## Evidence
