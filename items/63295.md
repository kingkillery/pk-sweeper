## Summary

Current `main` still appears to drop the inbound sender/conversation envelope for bare `/new` and `/reset` turns in the channel reply path, so #63295 should remain open.

## Evidence

- **Bare reset branch still omits inbound envelope:** `get-reply-run.ts` builds `inboundUserContext`, but `baseBodyForPrompt` only prepends it in the non-bare-reset branch. The bare-reset branch joins `startupContextPrelude`, `baseBodyFinal`, and optional `softResetTail`, so sender/conversation metadata is still excluded on bare session resets.
  - file: [src/auto-reply/reply/get-reply-run.ts:436](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/auto-reply/reply/get-reply-run.ts#L436)
  - command: `nl -ba src/auto-reply/reply/get-reply-run.ts | sed -n '388,490p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Inbound envelope still contains sender metadata when included:** `buildInboundUserContextPrefix(...)` still produces `Conversation info (untrusted metadata)` with `sender_id`/`sender` and a separate `Sender (untrusted metadata)` block from `SenderName`, `SenderId`, and `SenderUsername`. That means the metadata exists in the builder; it is just not attached on the bare-reset branch.
  - file: [src/auto-reply/reply/inbound-meta.ts:178](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/auto-reply/reply/inbound-meta.ts#L178)
