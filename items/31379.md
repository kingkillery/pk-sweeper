## Summary

Keep #31379 open. Current main still has the reported Control UI behavior: `resolveChannelOrder` returns only backend `channelMeta` or `channelOrder` when either is present, and the backend still builds those arrays from loaded channel plugins. Bundled channel manifests for the listed channels are not default-enabled, so a custom-only loaded channel list can still produce custom-only cards. The related PR #44165 is open and unmerged, while #31350 was closed unmerged, so there is not yet a landed fix or a better canonical issue to close this against.

## Evidence

A custom-only snapshot therefore still renders only the custom channel.
