## Summary

Keep open. Current main still does not implement PR #46303's central behavior: inbound debouncer buffers are closure-local, there is no global flush/drain hook, and SIGUSR1 restart only waits for active tasks/runs before closing the server. The PR is not superseded by the related restart/orphan work because the current restart deferral counts active operations, not pending debounce buffers or followup queue state.

## Best Possible Solution

Keep #46303 open for maintainer review. The best path is a scoped restart-drain fix that flushes pending inbound debounce buffers before teardown, waits for followup/inbound-worker work that can actually drain, preserves public plugin SDK compatibility, and has regression tests around SIGUSR1/config.patch timing and busy-session followups.
