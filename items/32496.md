## Summary

Keep open: current `main` still does not implement first-class `frequency_penalty`/`presence_penalty` support in the generic extra-params wrapper or the OpenAI transport payload builders, and the linked implementation attempts in the issue discussion were closed without merging.

## Evidence

- **Generic extra-param wrapper still only forwards a small allowlist:** `createStreamFnWithExtraParams()` copies `temperature`, `maxTokens`, transport, warmup, and cache-related values into stream options. There is no handling for `frequency_penalty`, `presence_penalty`, `top_p`, `top_k`, or `repetition_penalty` in this wrapper on current `main`.
