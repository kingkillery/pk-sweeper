## Summary

Keep open. Current `main` has agent-scoped media-root helpers, but the CLI `message send` path still always seeds outbound actions with the default agent, not the agent implied by `--account`, and there is no `--agent` flag. That means non-default agent workspaces like `workspace-business` still are not reachable for CLI media sends unless they belong to the default agent.

## Evidence

- **CLI message command always uses the default agent:** `messageCommand()` passes `agentId: resolveDefaultAgentId(cfg)` into `runMessageAction()` for CLI sends, so the active agent context is fixed to the default agent unless a session key is involved.
  - file: [src/commands/message.ts:74](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/commands/message.ts#L74)
  - command: `nl -ba src/commands/message.ts | sed -n '68,80p'`
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **CLI surface exposes `--account` but not `--agent`:** The shared message CLI helper registers `--account <id>` and no agent-selection flag, so operators cannot select the workspace-owning agent for `message send --media`.
  - file: [src/cli/program/message/helpers.ts:40](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/cli/program/message/helpers.ts#L40)
  - command: `nl -ba src/cli/program/message/helpers.ts | sed -n '38,44p'`
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **Agent-scoped roots only add the specified agent workspace:** `getAgentScopedMediaLocalRoots()` starts from default roots and appends exactly the resolved workspace for the provided `agentId`; without the correct agent id, other `workspace-*` directories are not added.
  - file: [src/media/local-roots.ts:54](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/media/local-roots.ts#L54)
  - command: `nl -ba src/media/local-roots.ts | sed -n '54,70p'`
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **Outbound media access trusts the passed agent id:** Plugin/direct outbound media handling calls `resolveAgentScopedOutboundMediaAccess()` with `params.ctx.agentId`, so the wrong/default agent context yields the wrong allowed local roots.
  - file: [src/infra/outbound/outbound-send-service.ts:79](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/infra/outbound/outbound-send-service.ts#L79)
  - command: `nl -ba src/infra/outbound/outbound-send-service.ts | sed -n '79,95p'`
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **Current account binding lookup is agent -> account, not account -> agent:** `buildChannelAccountBindings()` builds a `channel -> agent -> [accountIds]` map used to infer an account from a chosen agent. There is no reverse lookup here that would recover the correct agent from `--account`.
  - file: [src/routing/bindings.ts:48](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/routing/bindings.ts#L48)
