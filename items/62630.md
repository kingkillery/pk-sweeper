## Summary

Keep open. Current `main` still preserves leaked absolute `sessionFile` paths when the current state dir/home differs from the stored path, and transcript creation still does `mkdir(path.dirname(sessionFile))`. For a migrated Docker path like `/home/node/.openclaw/...`, that makes the reported `EACCES: permission denied, mkdir '/home/node'` failure still plausible on current code.

## Evidence

- **Cross-root absolute session paths are intentionally preserved:** `resolveSessionFilePath` keeps canonical `.../agents/<agentId>/sessions/<file>` absolute paths even when they are outside the current sessions dir via `structuralFallback`, instead of remapping them into the active state dir.
  - file: [src/config/sessions/paths.ts:190](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/config/sessions/paths.ts#L190)
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **Tests codify the stale-path behavior:** Current tests explicitly expect a stored absolute session path from a different state dir to be returned unchanged when `OPENCLAW_STATE_DIR` differs.
  - file: [src/config/sessions.test.ts:618](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/config/sessions.test.ts#L618)
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **Resolved session path is persisted back to the session store:** Once resolved, the chosen `sessionFile` is written back into `sessions.json`; if the preserved path is stale, OpenClaw persists the stale absolute path instead of repairing it.
  - file: [src/config/sessions/session-file.ts:30](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/config/sessions/session-file.ts#L30)
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **Write path still mkdirs the leaked parent directory:** When the transcript file does not exist, `ensureSessionHeader` calls `fs.promises.mkdir(path.dirname(params.sessionFile), { recursive: true })` before writing the JSONL header. With a leaked `/home/node/...` session path, this is the path that would trigger the reported `mkdir '/home/node'` failure.
  - file: [src/config/sessions/transcript.ts:18](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/config/sessions/transcript.ts#L18)
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **Docs mention layout migrations, not cross-home absolute-path repair:** Doctor docs cover legacy layout migration and integrity checks, but the documented migration surface does not include rewriting persisted absolute home paths from one environment to another.
  - file: [docs/gateway/doctor.md:259](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/docs/gateway/doctor.md#L259)
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
