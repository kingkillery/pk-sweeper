## Summary

Keep #29442 open. Current main has meaningful prompt-injection hardening, external-content wrapping, tool policy, sandboxing, and plugin hooks, and the ClawMoat link in the discussion is useful defense-in-depth. But the requested cron-integrated dual-LLM privilege-separation/sanitization path is not implemented on main, and the current plugin API does not expose the async post-external-content-wrap sanitizer seam proposed by the closed/unmerged #30046. This issue remains the canonical place for maintainer judgment on whether to add a core plugin API seam or a first-class workflow.
