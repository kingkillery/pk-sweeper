## Summary

Keep open. Current `main` still does not implement generic outbound agent-attestation or sender/agent header propagation for external HTTP requests. The shared fetch/runtime surfaces remain thin wrappers, the plugin hook catalog has no pre-request mutation hook such as `before_llm_request`, and the embedded runner only exposes observational `llm_input`/`llm_output` hooks after setup. The only adjacent implementation attempt I found was PR #11124, which was closed unmerged as stale.

## Evidence

- **Shared fetch wrapper does not add identity headers:** `src/infra/fetch.ts:15` through `src/infra/fetch.ts:108` only handles duplex and AbortSignal wrapping before delegating to `fetchImpl`; there is no version, agent-id, attestation, or allowlist logic.
  - file: [src/infra/fetch.ts:15](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/infra/fetch.ts#L15)

  - file: [src/plugin-sdk/fetch-runtime.ts:1](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/plugin-sdk/fetch-runtime.ts#L1)
