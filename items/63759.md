## Summary

Current main has partial evidence of a fix: Telegram uses the shared reply typing pipeline, instant typing is now signaled at run start before preflight compaction, and docs/changelog say DMs and mentioned groups start immediately. But the default behavior still intentionally delays typing for unmentioned group chats (`typingMode: "message"`), and the issue report does not say whether the affected Telegram turn was a DM, a mentioned group turn, or an unmentioned group turn. That makes auto-closing too risky.
