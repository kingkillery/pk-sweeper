## Summary

Keep open. Current `main` still treats the configured `provider/model[:suffix]` string as the actual model id, carries that id through runtime param resolution, and sends `model.id` directly to provider requests. I found no inspected post-issue change that adds suffix stripping or another first-class replacement for this use case.
