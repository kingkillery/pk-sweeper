## Summary

Keep open. Current main still renders `openclaw memory status` from the deep-only short-term promotion dreaming config, so a configuration with global dreaming enabled, light enabled, and deep disabled still collapses to `Dreaming: off`. The linked fixing PRs are not merged: #67882 is closed/unmerged, while #67910 and #67923 remain open.

## Best Possible Solution

Keep #67868 open until main changes `extensions/memory-core/src/cli.runtime.ts` to derive `Dreaming:` from the unified dreaming config or explicit light/REM/deep helpers, renders phase-level state, and adds a focused `extensions/memory-core/src/cli.test.ts` regression for global dreaming enabled with light on and deep off. Then close via the merged PR, resolving the duplicate open fix PRs as appropriate.
