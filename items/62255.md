## Summary

Keep open. Current `main` does not expose or document a global cross-agent model-call rate limiter; existing controls are limited to auth-profile failover/backoff, cron concurrency, and unrelated gateway/webhook/session-creation limiters.

## Evidence

- **Model provider config has no rate-limit field:** `ModelProviderConfig` only exposes `baseUrl`, `apiKey`, `auth`, `api`, `injectNumCtxForOpenAICompat`, `headers`, `authHeader`, `request`, and `models`. There is no per-provider or global model-call rate-limit setting in the typed config surface.
  - file: [src/config/types.models.ts:115](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/config/types.models.ts#L115)
  - command: `nl -ba src/config/types.models.ts | sed -n '75,135p'`
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Documented rate-limit handling is failover-oriented, not queuing/throttling:** The model-failover docs say rate limits exhaust profile rotation and then advance to fallback; they explicitly say OpenClaw switches to the next configured fallback 'without waiting'. That is the opposite of a global queue/wait budget across agents.
  - file: [docs/concepts/model-failover.md:215](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/docs/concepts/model-failover.md#L215)
  - command: `nl -ba docs/concepts/model-failover.md | sed -n '214,236p'`
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Gateway config docs only expose auth-profile rate-limit knobs:** The public configuration reference documents `overloadedProfileRotations`, `overloadedBackoffMs`, and `rateLimitedProfileRotations` under `auth.cooldowns`. Those settings govern retry/fallback behavior after provider errors; there is no global `models.*RateLimit` or provider request budget knob documented there.
  - file: [docs/gateway/configuration-reference.md:730](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/docs/gateway/configuration-reference.md#L730)

- **Runtime defaults confirm only per-run rate-limit rotation handling:** The runner helper defines `DEFAULT_MAX_RATE_LIMIT_PROFILE_ROTATIONS = 1` and resolves only `auth.cooldowns.rateLimitedProfileRotations`; there is no corresponding global model-call pacing or shared token-bucket state in this path.
