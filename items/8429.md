## Summary

Current `main` does not implement the requested browser coordinate-clicking or browser action record/replay flow. The browser tool remains ref-driven, while existing trace support is only for debugging and saving Playwright trace zips. The issue is coherent and still actionable, so conservative cleanup should leave it open.

## Evidence

- **Browser act contract has no coordinate-click shape:** `BrowserActRequest` defines click/type/hover/drag/select/fill/wait/evaluate around `ref`, `selector`, `targetId`, and other action-specific fields only. There is no `x`/`y` or coordinate-targeting variant.
  - file: [extensions/browser/src/browser/client-actions.types.ts:7](https://github.com/openclaw/openclaw/blob/c0a7b6a510758645c5267734b984f4885a605b67/extensions/browser/src/browser/client-actions.types.ts#L7)
  - command: `nl -ba extensions/browser/src/browser/client-actions.types.ts | sed -n '1,140p'`
  - sha: [c0a7b6a51075](https://github.com/openclaw/openclaw/commit/c0a7b6a510758645c5267734b984f4885a605b67)
- **Tool schema exposes no coord or recording action:** The browser tool schema only allows action kinds `click`, `type`, `press`, `hover`, `drag`, `select`, `fill`, `resize`, `wait`, `evaluate`, and `close`, and the top-level tool actions stop at `act` plus debug helpers like `console`/`pdf`/`upload`/`dialog`. No coordinate fields or record/replay actions are present.
  - file: [extensions/browser/src/browser-tool.schema.ts:4](https://github.com/openclaw/openclaw/blob/c0a7b6a510758645c5267734b984f4885a605b67/extensions/browser/src/browser-tool.schema.ts#L4)
  - command: `nl -ba extensions/browser/src/browser-tool.schema.ts | sed -n '1,120p'`
  - sha: [c0a7b6a51075](https://github.com/openclaw/openclaw/commit/c0a7b6a510758645c5267734b984f4885a605b67)
- **Docs still describe ref-based automation only:** Current browser docs explicitly say `click`/`type`/etc require a `ref` from `snapshot`, and the CLI docs call this `ref-based UI automation` with examples like `openclaw browser click <ref>`. That matches the current code and does not cover raw page coordinates.
  - file: [docs/tools/browser.md:716](https://github.com/openclaw/openclaw/blob/c0a7b6a510758645c5267734b984f4885a605b67/docs/tools/browser.md#L716)

The docs frame it as deep debugging, not recording interactions for later replay or modification.

## Risks / Open Questions

- A replacement browser plugin could plausibly implement specialized canvas automation, but this issue also requests changes to the shipped browser tool contract, so closing as `clawhub` would be too aggressive.
- Playwright trace support partially overlaps the word "recording", but it is not a user-facing action record/replay feature and should not be treated as implementing the request.
