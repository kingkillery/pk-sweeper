## Summary

Keep open. Current `main` still has the inline-provider API inheritance gap described by the PR, so this is not implemented or obsolete; however, the PR head also bundles an unrelated Telegram retry change with a reviewed P1 duplicate-send risk, so it is not a safe cleanup close either.
