## Summary

Keep #33975 open. Current main partially covers model visibility through `messages.responsePrefix` templates such as `{modelFull}`, but the issue also requests an opt-in approval gate before model fallback. I found no config/schema field or fallback-loop approval hook for that remaining behavior, so closing as implemented would lose a meaningful requested capability.
