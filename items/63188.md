## Summary

Keep open. Current `main` has partial incomplete-turn recovery for planning-only, reasoning-only, and empty-response cases, but it does not implement the requested per-agent `onLengthTruncation` / `maxContinuations` runtime controls or a general `stopReason: "length"` catch-and-continue/fail path for `mode:"run"` sessions. The issue discussion adds credible cross-provider evidence (Gemini/OpenRouter and lemonade OpenAI-compatible Qwen), so this still looks like a real gap rather than stale noise.

## Evidence
