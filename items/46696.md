## Summary

Keep open. Current `main` has partial outbound suppression for exact silent tokens and reasoning payloads, but it does not cover mixed/internal text like the reported `NO_REPLY to=functions.message commentary ...` leak. Discord still allows the `requireMention=false` and bot-authored handling path, so the reported scenario remains plausible on current `main`.

## Evidence

- **Silent suppression is exact-token only:** `isSilentReplyText` matches only the exact `NO_REPLY` token, and `isSilentReplyEnvelopeText` matches only a one-field JSON envelope with `{"action":"NO_REPLY"}`.
