## Summary

Keep open. Current `main` still does not preserve an existing `dmPolicy` during the channel-setup wizard, and the PR discussion already identified that this PR's proposed `initialValue: current` change was broken by an out-of-scope variable. This is not implemented on `main`, and it is not a safe cleanup close.

## Evidence

- **Current main still omits an initial DM policy selection:** `maybeConfigureDmPolicies` calls `prompter.select` for the DM policy with four options, but does not pass `initialValue`, so the wizard has no way to preselect the account's current `dmPolicy`.
  - file: [src/flows/channel-setup.prompts.ts:134](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/flows/channel-setup.prompts.ts#L134)
  - command: `git show 86f8c826e20a01d1bba67c0ba6e73182fbf0455f:src/flows/channel-setup.prompts.ts | nl -ba | sed -n '130,148p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Current value is only read after prompting:** `const current = policy.getCurrent(cfg, accountId);` happens after the select call, so the existing value is available for comparison but is not used to seed the prompt.
  - file: [src/flows/channel-setup.prompts.ts:143](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/flows/channel-setup.prompts.ts#L143)
