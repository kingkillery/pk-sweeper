## Summary

Current `main` still treats Control UI/WebChat composer attachments as image-only. Generic file/document upload is not implemented, and unsupported files are silently skipped, so this item should remain open.

## Evidence

- **Composer file picker is image-only:** The Control UI attachment accept list is hard-coded to `image/*`, and supported MIME checks only allow `image/` types.
  - file: [ui/src/ui/chat/attachment-support.ts:1](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/ui/src/ui/chat/attachment-support.ts#L1)
  - command: `nl -ba ui/src/ui/chat/attachment-support.ts | sed -n '1,20p'`
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Unsupported uploads are ignored in the chat view:** `handleFileSelect` and drag/drop both `continue` when `isSupportedChatAttachmentMimeType(file.type)` fails, with no error/reporting path.
  - file: [ui/src/ui/views/chat.ts:242](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/ui/src/ui/views/chat.ts#L242)
