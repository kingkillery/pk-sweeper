## Summary

Keep open. Current `main` does not implement a Control UI filter for hiding dreaming/system sessions in the Sessions tab, and the protocol/backend only expose global/unknown/search filters rather than a dedicated system-session filter. The issue is also recent and has an open linked PR (#67155), so it is still active rather than stale.

## Evidence

- **Sessions tab filters are limited:** The Sessions view exposes only `Active`, `Limit`, `Global`, `Unknown`, and a free-text search. There is no toggle or chip for hiding system/dreaming sessions in the Control UI Sessions tab.

  - file: [src/gateway/protocol/schema/sessions.ts:38](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/src/gateway/protocol/schema/sessions.ts#L38)
