## Summary

Keep open. Current `main` still emits `message_received` only inside `dispatchReplyFromConfig`, while active-session inbound messages are enqueued and later drained through the follow-up runner path that goes straight to `runEmbeddedPiAgent`. The PR author’s own later comment also says the proposed patch still missed substantial real inbound traffic, so there is not high-confidence evidence that this PR is the right cleanup close candidate.
