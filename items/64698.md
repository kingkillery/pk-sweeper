## Summary

Keep open. Current `main` explicitly implements `requireMention: false` for Telegram groups and has tests/docs that expect unmentioned group messages to pass, but I did not find a post-`v2026.4.9` fix tied to this report, and `pnpm` is unavailable in this checkout so I could not run a live reproduction against current `main`. Given the issue is recent and the reported behavior is still plausible in a multi-account setup, the evidence is not strong enough for an auto-close.
