## Summary

Keep open. Current `main` and the latest release still preserve raw prior-session `user:`/`assistant:` turns into `memory/YYYY-MM-DD-*.md` on `/reset`, and the tests explicitly lock that behavior in. There is a mitigation on the read path because startup context now injects daily memory as quoted untrusted notes, and config can exclude `/reset` from that preload, but that does not implement the requested change in the session-memory hook itself or strongly prove the report obsolete.
