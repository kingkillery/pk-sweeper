## Summary

Current `main` clearly improved subagent usage aggregation and the user-visible subagent stats path, but I did not find strong proof that the raw assistant `usage` object written into subagent session JSONL is now populated. The available source and tests support a partial fix for session-store/announce stats, not a conservative close of this issue’s transcript-specific report.

## Evidence

- **Subagent runs now persist token totals into the session store after each run:** `agentCommandInternal` calls `updateSessionStoreAfterAgentRun(...)` once a run finishes, and that helper writes `inputTokens`, `outputTokens`, `totalTokens`, `cacheRead`, `cacheWrite`, and `estimatedCostUsd` from `result.meta.agentMeta?.usage` into the session entry.
