## Summary

Keep open. Current `main` still routes bundled channel entry loads through `jiti(modulePath)` without Windows path normalization, and the existing regression coverage in `src/plugin-sdk/channel-entry-contract.test.ts` does not lock in the `file://` conversion this PR adds. The PR remains relevant and unshipped on `main` and in the latest release tag `v2026.4.23`.

## Evidence

- **Current main still passes raw modulePath to Jiti in bundled entry loader:** `loadBundledEntryModuleSync` resolves `modulePath` and then calls `jiti(modulePath)` in both the node-require fallback and direct-Jiti branches. There is no `toSafeImportPath` or `pathToFileURL` step in this file on current `main`.
