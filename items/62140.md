## Summary

Keep open. Current main still rebuilds gateway service metadata from durable state-dir `.env` values, and the macOS launchd path serializes that environment directly into the LaunchAgent plist. That means a `.env`-managed `OPENCLAW_GATEWAY_TOKEN` can still be re-embedded into launchd metadata after `gateway install` or doctor-driven reinstall, which matches the reported drift surface rather than fixing it.

## Risks / Open Questions

- Docs and warnings now say SecretRef-managed installs do not persist resolved tokens, but the macOS launchd path still persists durable `.env` values into plist metadata.
- Doctor can detect embedded-token drift, but its reinstall path still rebuilds service env from the same durable `.env` source, so the reported macOS drift can recur.
