## Summary

Keep open. The issue is a recent feature request from a non-maintainer with no protected labels, and current main appears to expose session persistence, session list/send/create, and checkpoint branch/restore surfaces, but not the requested general `forkSession(id)`, `resume: sessionId`, or `continue: true` API behavior. This is core Gateway/session state territory rather than an obvious ClawHub-only plugin request.

## Evidence
