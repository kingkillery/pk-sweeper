## Summary

Keep open. Current `main` still generates the macOS gateway LaunchAgent plist without a `ProcessType` key, and `openclaw gateway install` rewrites that plist from the same template, so the requested fix is not implemented.

## Evidence

- **LaunchAgent plist template omits ProcessType:** `buildLaunchAgentPlist()` renders `Label`, `RunAtLoad`, `KeepAlive`, `ThrottleInterval`, `Umask`, `ProgramArguments`, `WorkingDirectory`, `StandardOutPath`, `StandardErrorPath`, and optional `EnvironmentVariables`. There is no `ProcessType` entry in the generated plist.
  - file: [src/daemon/launchd-plist.ts:89](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/src/daemon/launchd-plist.ts#L89)
  - command: `nl -ba src/daemon/launchd-plist.ts | sed -n '80,140p'`
  - sha: [7875092f4ddc](https://github.com/openclaw/openclaw/commit/7875092f4ddc6f2dbe218bea8506c9ac1da87c39)
- **Install path rewrites the plist from that template:** `writeLaunchAgentPlist()` calls `buildLaunchAgentPlist(...)` and writes the result to `~/Library/LaunchAgents/...plist`; `installLaunchAgent()` uses the same path before activating the LaunchAgent. Any manual plist edit would be replaced on reinstall/force-install.
