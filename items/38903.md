## Summary

Closing as cannot_reproduce. Current main does not match the reported root cause: Bedrock is routed through the native pi-ai Bedrock serializer, and that serializer rewrites replayed message blocks into Bedrock Converse payload objects like `{ text: ... }`, `{ toolUse: ... }`, and `{ toolResult: ... }` rather than forwarding Anthropic-style `{ type: "text", ... }` blocks unchanged.

## Evidence

- **Bedrock does not use the Anthropic/OpenAI transport wrappers:** `src/agents/provider-transport-stream.ts` only installs custom transport wrappers for OpenAI, Azure OpenAI, Anthropic, and Google APIs. `bedrock-converse-stream` is not in `SUPPORTED_TRANSPORT_APIS`, so Bedrock requests go through the native provider serializer instead of the Anthropic transport path described in the report.
  - file: [src/agents/provider-transport-stream.ts:13](https://github.com/openclaw/openclaw/blob/ed6094b3014c56882da15a7942130b00ea410c6e/src/agents/provider-transport-stream.ts#L13)
  - sha: [ed6094b3014c](https://github.com/openclaw/openclaw/commit/ed6094b3014c56882da15a7942130b00ea410c6e)
- **Current main and latest shipped release both pin pi-ai versions with the Bedrock serializer path:** Current main pins `@mariozechner/pi-ai` `0.70.0`, and the latest release `v2026.4.22` pins `0.69.0`.
  - file: [package.json:1579](https://github.com/openclaw/openclaw/blob/ed6094b3014c56882da15a7942130b00ea410c6e/package.json#L1579)
  - command: `git show v2026.4.22:package.json | nl -ba | sed -n '1542,1551p'`
  - sha: [ed6094b3014c](https://github.com/openclaw/openclaw/commit/ed6094b3014c56882da15a7942130b00ea410c6e)
- **Bedrock payload expectations in repo tests already omit `type` keys:** `extensions/amazon-bedrock/index.test.ts` asserts Bedrock payloads shaped as `messages: [{ role: "user", content: [{ text: "Hello" }] }]` and keeps that shape after wrapper processing, which is incompatible with the reported malformed `{ type: "text", text: ... }` payload root cause.
  - file: [extensions/amazon-bedrock/index.test.ts:748](https://github.com/openclaw/openclaw/blob/ed6094b3014c56882da15a7942130b00ea410c6e/extensions/amazon-bedrock/index.test.ts#L748)
  - sha: [ed6094b3014c](https://github.com/openclaw/openclaw/commit/ed6094b3014c56882da15a7942130b00ea410c6e)
- **Pinned pi-ai Bedrock serializer strips Anthropic-style content block tags:** Read-only inspection of the pinned pi-ai tarballs (`0.69.0` and `0.70.0`) shows `dist/providers/amazon-bedrock.js` calling `transformMessages(...)` and then converting user/assistant/toolResult blocks to Bedrock Converse shapes like `{ text: ... }`, `{ toolUse: ... }`, and `{ toolResult: ... }`; it does not forward `{ type: "text", ... }` objects into `commandInput.messages`.
