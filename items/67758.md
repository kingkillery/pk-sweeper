## Summary

Keep #67758 open. Current main still maps the bundled `google-gemini-cli` `flash` model alias to `gemini-3.1-flash-preview`, while the CLI default and user-facing docs already point flash users at `gemini-3-flash-preview`. The existing Google provider normalizer handles deprecated API model refs, but the CLI backend alias path is separate and passes the alias result directly to the Gemini CLI, so current main does not make this PR obsolete.

## Best Possible Solution

Keep the PR open or replace it with an equivalent maintainer patch that updates the bundled Google CLI backend `flash` alias to `gemini-3-flash-preview` and adds a focused regression assertion for `resolveCliBackendConfig("google-gemini-cli")` or `normalizeCliModel("flash", ...)`. Before landing, verify the remaining `pro` and `flash-lite` IDs against the current Gemini CLI/Google model catalog so the one-line fix does not leave adjacent aliases accidentally stale.
