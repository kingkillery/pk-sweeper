## Summary

Keep open. Current `main` still lets auth-profile bookkeeping save failures propagate from `markAuthProfileGood`, `markAuthProfileUsed`, and `markAuthProfileFailure`, so this PR is not implemented on `main` and is not obsolete.

## Evidence

- **`markAuthProfileGood` still propagates fallback save failures on current `main`:** On `main`, `markAuthProfileGood` falls back to mutating `store.lastGood` and then directly calls `saveAuthProfileStore(store, agentDir)` with no local `try/catch`. A persistence failure in that fallback path would still escape the function.

That matches the failure mode described in the PR.
