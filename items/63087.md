## Summary

Keep open. Current `main` still sends memory-index embedding HTTP through the guarded fetch path in its default strict mode, and that mode intentionally ignores `HTTP_PROXY`/`HTTPS_PROXY` unless a caller explicitly opts into trusted env-proxy handling. The memory embedding code does not opt in, so the reported proxy bug remains plausible on `main` for Gemini and the shared remote embedding providers.

## Evidence

- **Guarded fetch only honors env proxy in explicit trusted mode:** `fetchWithSsrFGuard` creates an `EnvHttpProxyAgent` only when `mode === TRUSTED_ENV_PROXY`; otherwise it builds direct/pinned dispatchers. That means env proxy vars are not used on the default path.
  - file: [src/infra/net/fetch-guard.ts:357](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/infra/net/fetch-guard.ts#L357)
  - command: `nl -ba src/infra/net/fetch-guard.ts | sed -n '330,410p'`
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Test suite documents current default behavior:** The SSRF fetch test explicitly asserts that strict mode 'ignores env proxy by default' and only trusted proxy mode routes through env proxy.
  - file: [src/infra/net/fetch-guard.ssrf.test.ts:1007](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/infra/net/fetch-guard.ssrf.test.ts#L1007)
  - command: `nl -ba src/infra/net/fetch-guard.ssrf.test.ts | sed -n '1000,1035p'`
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Memory remote HTTP helper does not opt into env-proxy mode:** `withRemoteHttpResponse` calls `fetchWithSsrFGuard` with only `url`, `fetchImpl`, `init`, and `policy`; it does not set a trusted env-proxy mode.
  - file: [src/memory-host-sdk/host/remote-http.ts:14](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/memory-host-sdk/host/remote-http.ts#L14)

- **Gemini memory embeddings use the default helper without proxy override:** Gemini embedding requests call `withRemoteHttpResponse` directly and do not pass a proxy-aware `fetchImpl`, so `openclaw memory index` keeps the default guarded transport that ignores env proxy vars.
  - file: [extensions/google/embedding-provider.ts:183](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/extensions/google/embedding-provider.ts#L183)
