## Summary

Keep open. Current `main` still routes Claude CLI runs through the child supervisor with `argv: [backend.command, ...args]`, Anthropic’s bundled CLI backend still sets `command: "claude"`, and the Windows shim allowlist in `src/process/supervisor/adapters/child.ts` still excludes `claude`, so the reported `spawn claude ENOENT` path remains plausible on Windows. The PR is also part of an open duplicate group with #68819 and #68849, so conservative cleanup should leave maintainer choice open rather than auto-close it.
