## Summary

Current `main` does not show a generic hook-level primitive for session-scoped next-turn suppression. `before_agent_reply` is still single-turn, and the only built-in follow-up rerouting path found is plugin conversation binding, which is exposed to plugin commands rather than hook handlers.

## Evidence

- **`before_agent_reply` is documented as single-turn:** The public hooks doc still describes `before_agent_reply` only as short-circuiting the current model turn with a synthetic reply or silence. It does not document session-scoped pending ownership or next-turn suppression.
  - file: [docs/plugins/hooks.md:66](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/docs/plugins/hooks.md#L66)
  - command: `nl -ba docs/plugins/hooks.md | sed -n '58,92p'`
  - sha: [7875092f4ddc](https://github.com/openclaw/openclaw/commit/7875092f4ddc6f2dbe218bea8506c9ac1da87c39)
- **Hook result has no pending-state field:** `PluginHookBeforeAgentReplyResult` still only exposes `handled`, optional `reply`, and optional `reason`, with no field to persist pending workflow ownership or suppress later turns.
  - file: [src/plugins/hook-types.ts:169](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/src/plugins/hook-types.ts#L169)
  - command: `nl -ba src/plugins/hook-types.ts | sed -n '166,186p'`
  - sha: [7875092f4ddc](https://github.com/openclaw/openclaw/commit/7875092f4ddc6f2dbe218bea8506c9ac1da87c39)
- **Conversation binding is command-scoped API:** The only generic API that can bind a conversation for follow-up routing is on `PluginCommandContext` (`requestConversationBinding`, `detachConversationBinding`, `getCurrentConversationBinding`). Those methods are not present on the hook contexts used by managed media flows.
  - file: [src/plugins/types.ts:1819](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/src/plugins/types.ts#L1819)
  - command: `nl -ba src/plugins/types.ts | sed -n '1806,1828p'`
  - sha: [7875092f4ddc](https://github.com/openclaw/openclaw/commit/7875092f4ddc6f2dbe218bea8506c9ac1da87c39)
- **Core only wires binding requests during command execution:** `executePluginCommand(...)` injects `requestConversationBinding(...)` while running a plugin command, reinforcing that conversation binding is currently available through commands rather than hook handlers.
  - file: [src/plugins/commands.ts:232](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/src/plugins/commands.ts#L232)
  - command: `nl -ba src/plugins/commands.ts | sed -n '228,244p'`
  - sha: [7875092f4ddc](https://github.com/openclaw/openclaw/commit/7875092f4ddc6f2dbe218bea8506c9ac1da87c39)
- **Inbound rerouting requires an existing plugin-owned binding:** Normal dispatch routes follow-up inbound messages through `runInboundClaimForPluginOutcome(...)` only when `pluginOwnedBinding` already exists; otherwise processing falls through to the normal agent path.
  - file: [src/auto-reply/reply/dispatch-from-config.ts:542](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/src/auto-reply/reply/dispatch-from-config.ts#L542)
