## Summary

Keep open. Current main partially covers the `<available_skills>` concern through existing per-agent skill allowlists, but the central request for user-facing per-section prompt gates and a `systemPromptOverride` cookbook is still not implemented.

## Best Possible Solution

Keep the issue open as a real prompt-size/product request. A maintainer-quality path would add typed default-true prompt-section gates to agent defaults and per-agent config, wire them through schema/help/docs/tests into `buildAgentSystemPrompt`, document `agents.defaults.skills` as the current allowlist solution for `<available_skills>`, and add the requested `systemPromptOverride` required-vs-optional cookbook table on the public system-prompt/config docs.
