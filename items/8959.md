## Summary

Current `main` does not implement the requested cross-platform multi-select capability in the shared `message` tool. The shared message/presentation contract only exposes a single `select` block with `placeholder` and `options`, while Slack renders that as `static_select`, Telegram degrades selects into one-button-per-option callbacks, and only Discord's provider-specific `components` schema exposes `minValues`/`maxValues` for multi-select behavior.
