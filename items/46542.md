## Summary

Keep open. Current `main` does not implement this PR’s shared builtin memory store, CLI flags, config surface, or result/status schema. The closest current alternative is QMD `extraCollections` for cross-agent search, which is different from the proposed `_shared` builtin store.

## Evidence

- **Resolved builtin memory config still only supports agent-local extra paths:** `ResolvedMemorySearchConfig` includes `extraPaths` only, and `mergeConfig()` only merges `defaults.extraPaths` + `overrides.extraPaths`. There is no `sharedPaths`, shared store path, or `_shared` manager wiring on current main.
  - file: [src/agents/memory-search.ts:15](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/agents/memory-search.ts#L15)
  - command: `nl -ba src/agents/memory-search.ts | sed -n '1,260p'`
- **Runtime schema has no shared-memory config or reserved `_shared` agent id:** `MemorySearchSchema` exposes `extraPaths` and `qmd.extraCollections`, but not `sharedPaths`. `AgentEntrySchema` still accepts a plain `id: z.string()` with no reserved `_shared` validation.
  - file: [src/config/zod-schema.agent-runtime.ts:624](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/config/zod-schema.agent-runtime.ts#L624)
  - command: `nl -ba src/config/zod-schema.agent-runtime.ts | sed -n '620,860p'`
- **Current memory CLI has no `--shared` path:** The bundled memory plugin registers `memory status`, `memory index`, and `memory search` without any `--shared` option. Current options are limited to flags like `--agent`, `--json`, `--deep`, `--index`, `--fix`, `--force`, and `--verbose`.
  - file: [extensions/memory-core/src/cli.ts:116](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/extensions/memory-core/src/cli.ts#L116)
  - command: `nl -ba extensions/memory-core/src/cli.ts | sed -n '100,190p'`
- **Current docs do not advertise shared store commands:** The CLI docs list examples and options for `memory status`, `memory index`, and `memory search`, but there is no `--shared` example or option. The current multi-agent docs instead point cross-agent memory search users to QMD `extraCollections`.
  - file: [docs/cli/memory.md:24](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/docs/cli/memory.md#L24)

## Risks / Open Questions

- The memory implementation has moved into the bundled `extensions/memory-core` plugin on current main, so this PR will need a substantial rebase/port even if the feature is still desired.
- Current docs already advertise QMD `extraCollections` and Honcho for cross-agent memory use cases, so maintainers may want explicit product-direction review before accepting a second builtin cross-agent model.
