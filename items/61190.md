## Summary

Keep PR #61190 open. Current main still has the reported resolver behavior: directory matches return the initially inferred target kind instead of the matched directory entry kind, and the PR's regression tests are not present on main. The PR discussion also supports that the fix is targeted and that the author addressed the bot's follow-up note about channel-kind normalization.
