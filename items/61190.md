## Summary

Keep open. Current `main` still uses the initially guessed target kind for directory matches, so the PR is addressing a real gap, including the DingTalk scenario described in the thread. But the review discussion also surfaced a plausible regression for mixed `channel`/`group` plugins, so this is not strong close territory; it needs maintainer judgment, not cleanup closure.

## Evidence

- **Main still returns the guessed kind for directory matches:** `resolveMessagingTarget()` computes `kind` once via `detectTargetKind()` and then writes that same variable into both the single-match and ambiguous-match directory success paths instead of consulting the matched directory entry kind.
  - file: [src/infra/outbound/target-resolver.ts:354](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/infra/outbound/target-resolver.ts#L354)
  - command: `nl -ba src/infra/outbound/target-resolver.ts | sed -n '340,420p'`
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Resolved kind changes downstream routing:** `inferPeerKind()` treats `resolvedTarget.kind === "channel"` differently from `"group"`; that choice affects the route peer kind, session key shape, and chat type.
  - file: [src/infra/outbound/outbound-session.ts:56](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/infra/outbound/outbound-session.ts#L56)

- **Mixed channel/group plugins exist on main:** Mattermost advertises both `channel` and `group` chat types, so collapsing one into the other is not generically safe across plugins.
  - file: [extensions/mattermost/src/channel.ts:271](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/extensions/mattermost/src/channel.ts#L271)

## Risks / Open Questions

- Closing this PR would leave current `main` with the still-unfixed directory-kind propagation bug in `src/infra/outbound/target-resolver.ts`.
- Merging the PR exactly as written may regress mixed `channel`/`group` plugins by downgrading channel-preferring lookups to `group` when directory entries are returned as `kind: "group"`.
