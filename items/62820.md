## Summary

Keep open. Current `main` still sends outbound/channel text through `sanitizeUserFacingText()`, and that sanitizer still only strips `<final>` tags in `stripFinalTagsFromText()`. The existing XML stripper lives on the separate assistant-visible-text path and is not wired into this messaging path. The linked fix attempt in PR #62883 / commit `02be4990bd74d06d2265ce2935abc7f7e599979a` matches the reported root cause, but it is not present on current `main`.
