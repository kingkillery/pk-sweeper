## Summary

Keep open: current `main` now documents a strict split between `openai-codex/*` (PI Codex OAuth) and native Codex harness runs (`openai/*` plus `embeddedHarness.runtime: "codex"`), but there is still no `openai-codex/gpt-5.4-nano` implementation on `main`. `gpt-5.4-nano` exists only on the `openai/*` route, so the reported Codex-runtime nano mismatch is not strongly proven fixed.
