## Summary

Current `main` does not implement provider-429 auto-pause/auto-resume. It handles rate limits by cooldowning credentials, rotating profiles/keys, and falling back across models, then still returns a terminal error or incomplete-turn payload that asks the user to retry later.

## Evidence

- **Rate-limit exhaustion still throws a terminal error:** When same-provider rate-limit profile rotations exceed the configured cap, the embedded runner throws `FailoverError("The AI service is temporarily rate-limited. Please try again in a moment.")` instead of scheduling a future resume.

  - command: `nl -ba src/agents/pi-embedded-runner/run.ts | sed -n '620,650p'`
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)

  - file: [src/agents/provider-transport-fetch.ts:13](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/agents/provider-transport-fetch.ts#L13)
