## Summary

Current main already covers the two PR goals with a broader managed runtime-deps implementation: bundled runtime npm installs use an OpenClaw-owned cache under the install execution root instead of inherited home npm cache, and doctor/health scans respect explicit plugin/channel disablement while still including configured non-disabled channels. The PR's channel-change direction is also superseded by main because the review comment correctly identified that dropping the explicit-disable fallback would regress default-enabled channel plugins.

## Best Possible Solution

Close PR #71614 and keep current main's managed runtime-deps stage/cache implementation plus explicit-disable semantics. Further changes should be tracked as narrower follow-ups against bundled runtime-deps behavior rather than merging this patch, because the PR's channel-disable edit would regress default-enabled channel suppression.
