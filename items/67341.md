## Summary

Keep PR #67341 open. Current main still lacks the requested `amazon-bedrock-mantle` `config.discovery.enabled` gate, while the linked bug #67288 and maintainer discussion describe a real, reproducible log-spam path. The closed sibling PR #67311 was not merged, so it does not supersede this open PR.

## Best Possible Solution

Keep the PR open and use it as the active implementation path for #67288, but ask for Mantle regression tests and consider moving the opt-out into `resolveImplicitMantleProvider` or otherwise matching the live-config semantics used by `amazon-bedrock`. The final fix should declare `plugins.entries.amazon-bedrock-mantle.config.discovery.enabled` in the plugin manifest, skip IAM token generation when explicitly false, preserve default auto-detection when unset, and update user-facing docs/config help if the new option is accepted.
