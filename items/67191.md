## Summary

Keep open. Current `main` still runs `BOOT.md` through a silent `agentCommand(..., deliver: false)` path, and tests explicitly lock that behavior in. The docs/template also still tell users to use the message tool plus `NO_REPLY`, while the message-tool pipeline requires a concrete target/current chat context for `action=send`. That means the reported root cause is not fully proven, but the underlying startup-messaging behavior is still unresolved enough that conservative cleanup should not close this item.
