## Summary

Current main already supersedes this PR: the old `python-exec-encoded` regex path is gone, host exec no longer uses heuristic command-obfuscation gating, and inline interpreter eval is handled by explicit `strictInlineEval` policy. The PR's unrelated `systemPromptReport` and browser `~` expansion changes are also already present on main.

## Best Possible Solution

Close this stale PR and keep the current main implementation: policy/allowlist/`strictInlineEval` handling for exec, existing CLI `systemPromptReport`, and existing browser executable-path tilde expansion. Any future exec false-positive or inline-eval gap should be filed as a new focused issue/PR against the current `exec-inline-eval` and approval-policy code, not against the removed regex detector.
