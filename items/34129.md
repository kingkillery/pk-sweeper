## Summary

Keep #34129 open. Current main still preserves a blank tool-call name when no usable toolCallId can infer a real tool, then assigns a call_auto_* fallback id. The exact remaining fix is tracked by open PR #56391, but it is not merged into main, so the reported user-visible bug is not implemented-on-main yet.
