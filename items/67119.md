## Summary

Current `main` does not contain this fix. The Google video provider still defaults to a preview Veo model while constructing the GenAI client without any API-version override, and the current test suite explicitly expects no `apiVersion` for that preview path. The upstream `@google/genai` contract supports `apiVersion`, so the PR is addressing a still-plausible bug rather than duplicated work.
