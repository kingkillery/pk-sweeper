## Summary

Keep open. Current `main` already has plugin-level pre-delivery hooks and newer empty-response handling, so part of the report is outdated, but there is still no documented or typed path that lets a hook reject a post-LLM reply and hand that rejection into the built-in model failover chain.

## Evidence

- **Plugin pre-delivery hooks now exist:** The plugin hook docs list `message_sending` (rewrite/cancel outbound content), `before_dispatch` (inspect/rewrite before channel handoff), and `reply_dispatch` (final reply-dispatch pipeline). That directly contradicts the issue's claim that there is no hook between model output and delivery, at least for plugin hooks on current `main`.
  - file: [docs/plugins/hooks.md:81](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/docs/plugins/hooks.md#L81)
  - command: `nl -ba docs/plugins/hooks.md | sed -n '50,110p'`
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **Internal hooks still stop at post-delivery `message:sent`:** The internal-hooks docs still list `message:sent` as the only outbound message event; there is no internal `message:before_send` event on current `main`.
  - file: [docs/automation/hooks.md:36](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/docs/automation/hooks.md#L36)

The typed surfaces do not provide a way to reject output and route that decision into the existing failover machinery.

  - file: [src/infra/outbound/deliver.ts:846](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/infra/outbound/deliver.ts#L846)
