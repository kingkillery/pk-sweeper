## Summary

Keep open. Current `main` still lacks the selector cleanup this PR is aiming for, but the proposed synthetic-name filter would also hide explicit session `label` values, and those labels are user-owned metadata in the current contract. The open review comment identifies a real regression risk, so this PR should be updated/rebased rather than auto-closed.

## Evidence

- **Current UI still treats only `main` and `agent:main:main` as main-session fallbacks and otherwise shows `label`/`displayName` verbatim:** `parseSessionKey` only special-cases `main` and `agent:main:main`; `resolveSessionDisplayName` then prefers `row.label` and falls back to `row.displayName`; `resolveSessionScopedOptionLabel` delegates to that display-name resolver whenever either field is present. That means the underlying UX issue is still present on current `main`, but it also means filtering `label` changes authoritative UI metadata, not just transport noise.
