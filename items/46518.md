## Summary

Current `main` has related WhatsApp login hardening, but the evidence is not strong enough to say issue #46518 is fixed. The repo still pins `@whiskeysockets/baileys@7.0.0-rc.9`, the login wait path still depends on Baileys emitting `connection.update` with `open` or `close`, and the Control UI still treats QR generation and post-scan waiting as separate manual actions. If the reported failure mode is a post-scan Baileys stall with no follow-up event, current code does not clearly close that gap.
