## Summary

Keep open. Current `main` contains a partial Ollama routing fix: the plugin resolves `providerBaseUrl` per provider and the main run/compaction paths use the returned provider stream function directly. But the shared custom API registry is still keyed only by `api`, and simple-completion callers still short-circuit once any `ollama` API provider is registered. That leaves a plausible multi-provider `api: "ollama"` collision unresolved, so there is not strong enough evidence to conservatively close this issue.

## Evidence

- **Custom API registry is still keyed only by `api`:** `ensureCustomApiRegistered` exits early if any provider is already registered for the API string. There is no provider-id or baseUrl dimension, so multiple configured providers that all use `api: "ollama"` still collapse onto one registry slot.
