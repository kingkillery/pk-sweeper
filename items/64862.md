## Summary

Keep open. Current `main` shipped a broader TUI streaming watchdog, but the exact race this PR targets is not clearly covered by the current `finalizeRun`/`terminateRun` logic, and the PR already has an unresolved review note about regressing local `/btw` runs.

## Evidence

If a run final/error arrives after `activeChatRunId` was cleared earlier, current main clears the watchdog for that run but does not transition the status here.
