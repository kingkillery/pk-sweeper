## Summary

Current main already supports suppressing sub-agent completion announce delivery well enough for `sessions_spawn`. The tool reads an `expectsCompletionMessage` flag, the announce dispatcher has a separate non-completion path, and the lifecycle test verifies completion handoff stays internal (`deliver: false`) with no direct external send.

## Evidence

## Risks / Open Questions

- The suppression control is implemented as `expectsCompletionMessage: false`, not the `delivery.mode = "none"` or `announce: false` shape proposed in the issue.
- Docs still describe the legacy/default announce model in places, so discoverability may lag the current behavior.
