## Summary

Keep open. Current `main` still retains the gateway `error` listener after `dispose()`, and current tests now document that this is intentional because late gateway `error` events must still be suppressed after disposal. The proposed PR would remove that sink and regress current behavior by allowing post-dispose `error` emissions to become uncaught EventEmitter errors.

## Evidence

- **Current dispose path does not detach the error listener:** `createDiscordGatewaySupervisor()` registers `emitter.on("error", onGatewayError)` and `dispose()` only clears lifecycle state, sets `phase = "disposed"`, and clears `pending`; it does not call `off()`. That means current `main` still expects `onGatewayError` to handle late errors in the disposed phase.
