## Summary

Keep open. Current `main` does not implement this PR's partial-mode coalescing behavior, and recent Telegram streaming changes on `main` explicitly preserve the opposite boundary rotation behavior. The PR is therefore not stale-closeable under the allowed reasons; it needs rebase/alignment if it is to proceed.
