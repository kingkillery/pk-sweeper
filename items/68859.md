## Summary

Current `main` does not inject `OPENCLAW_SESSION_KEY` into exec child environments or document that contract, so this PR is not implemented on `main` and is not a safe cleanup close. The request is coherent and discussion improved it by catching the inherited-env edge case, but the PR still contains an unrelated UI cleanup and lacks landed test coverage for the new env var.
