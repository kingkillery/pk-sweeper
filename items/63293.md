## Summary

Keep open. Current `main` does not show an iMessage-specific retry/deduplication fix, and the relevant iMessage delivery files are unchanged from the latest release `v2026.4.23`. The report is recent and specific enough that it is not a stale-data close, but I could not conservatively prove the bug is already fixed or obsolete from the checked-out code alone.

  - file: [extensions/imessage/src/monitor/deliver.ts:19](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/extensions/imessage/src/monitor/deliver.ts#L19)

  - file: [extensions/imessage/src/send.ts:97](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/extensions/imessage/src/send.ts#L97)
