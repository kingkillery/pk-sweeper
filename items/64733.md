## Summary

Keep open. Current `main` still has the unbounded `await Promise.allSettled(pendingToolTasks)` drain, so this PR is not implemented there; related mitigations landed for adjacent failure modes, but the linked deadlock issue remains open and this PR’s hard 30s timeout still has an unresolved ordering/regression concern from review.

## Risks / Open Questions

- The original deadlock described by the PR remains plausible on current `main` because unresolved `pendingToolTasks` still block the reply pipeline indefinitely.
- Closing now would discard an open contributor PR while the related bug issue (#53889) is still open and no end-to-end replacement fix is present on `main`.
- Merging this PR as-is may introduce message-ordering regressions on slow but healthy tool-result delivery paths because tool results are intentionally serialized on current `main`.
