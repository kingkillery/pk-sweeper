## Summary

Keep open. Current `main` has native OpenAI strict-tool compatibility work, but #63398 is about LiteLLM-routed OpenAI requests. LiteLLM is still treated as a proxy-style OpenAI-compatible route, and the current code/tests explicitly skip the native OpenAI tool-schema shaping on proxy routes, so there is not strong evidence that the reported MCP schema failure is fixed.

## Evidence
