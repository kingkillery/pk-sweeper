## Summary

Keep #67511 open. Current main has nearby session, compaction, status, and subagent metadata, but it does not persist a per-session workingContext capsule or expose cwd/repo/branch/temp-clone state in /status. The linked draft PR #71135 is relevant and positive, but it is explicitly only a persistence primitive and remains open/draft with no runtime wiring.

## Best Possible Solution

Keep the issue open as the umbrella product/core work. A good implementation path is to land a versioned session working-context capsule, populate it from exec/tool runtime and subagent/new/compact boundaries, expose it in /status, and update the compaction/workspace docs to distinguish semantic transcript continuity from engineering working-state continuity. PR #71135 can be used as a starting point, but the issue should remain open until the runtime wiring and user-facing surfaces exist.
