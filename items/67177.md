## Summary

Keep open. Current `main` still leaves the `application/vnd.microsoft.teams.file.download.info` path on the raw `downloadUrl`, while only the separate `contentUrl` path rewrites SharePoint/OneDrive shared links through the Graph `shares` endpoint. The silent-swallowing part of the report appears partly outdated because current code now logs warnings on attachment download failures, but the core DM file-download regression described in the issue is still plausible and not covered by the existing shared-link tests.
