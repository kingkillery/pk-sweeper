## Summary

Keep open. Current `main` partially handles `/new` and `/reset` auto-override clearing, but it does not implement this PR's defense-in-depth fix for general new-session auth-profile resolution, and it also lacks the PR's legacy-auth preservation logic/tests.

## Evidence

- **New-session resolver still rotates from the existing override:** `resolveSessionAuthProfileOverride()` on current `main` still treats any existing override as the starting point for `isNewSession`, using `pickNextAvailable(current)` instead of discarding auto-selected state and re-picking from the best available profile. That is the core behavior this PR says it fixes.
