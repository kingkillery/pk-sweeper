## Summary

Keep open. Current main still emits the exact context-limit reset messages that point only to `agents.defaults.compaction.reserveTokensFloor`, with no heartbeat-model or active-model context. The linked PR #67381 directly tracks this issue but is open/unmerged and not present on main, so the report remains actionable rather than closeable as implemented.

## Best Possible Solution

Keep this issue open until the user-facing overflow reset guidance is fixed or a canonical merged PR supersedes it. The best path is a shared context-overflow message helper used by both reset paths in `src/auto-reply/reply/agent-runner-execution.ts`, backed by tests, that can include the active model/context window and suggest heartbeat isolation/light-context only when the evidence points to heartbeat model override bleed. Coordinate with PR #67381 and the broader reserveTokensFloor guidance cleanup in #65839.
