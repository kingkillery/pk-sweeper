## Summary

Keep #67621 open. Current main still matches `$[A-Z_]...` directly against full Python/JS file content without any string/comment awareness, so `$SIM` in Python docstrings or argparse help can still hit the reported false-positive path. The linked fix PR #67644 is closed unmerged and PR #67924 is still open, so the issue is not implemented on main or safely superseded by shipped code.

## Best Possible Solution

Keep the issue open and fix the source-file preflight path in `src/agents/bash-tools.exec.ts` so `$VAR` detection ignores Python/JS string and comment literals while still rejecting actual raw shell-variable syntax in executable code. Review or replace PR #67924 with a narrow patch plus regression tests covering `$SIM` in docstrings/help text and dollar-formatted f-strings such as `f"${value:.2f}"`.
