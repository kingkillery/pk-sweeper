## Summary

Keep open. Current `main` still lacks wizard-wide back navigation for `openclaw onboard`; the interactive flow remains forward-only aside from cancel, with one limited `Back` option only inside plugin configuration that returns to that section menu.

## Evidence

- **Wizard prompter contract has no back action:** `WizardPrompter` exposes `intro`, `outro`, `note`, `select`, `multiselect`, `text`, `confirm`, and `progress` only. There is no `back`, `previous`, or step-rewind API in the onboarding prompt contract.
  - file: [src/wizard/prompts.ts:38](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/wizard/prompts.ts#L38)
  - command: `nl -ba src/wizard/prompts.ts | sed -n '1,120p'`
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **CLI prompter only handles accept or cancel:** `createClackPrompter()` wraps `@clack/prompts` and `guardCancel()` converts cancel into `WizardCancelledError`. The select/text/confirm wrappers pass values through directly and do not inject a back option or previous-step handler.
  - file: [src/wizard/clack-prompter.ts:24](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/wizard/clack-prompter.ts#L24)
  - command: `nl -ba src/wizard/clack-prompter.ts | sed -n '1,220p'`
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Wizard session API is next/answer/cancel only:** `WizardSession` supports `next()`, `answer()`, and `cancel()`. It tracks only the current pending step and has no mechanism to move to a prior step or replay preserved answers.
  - file: [src/wizard/session.ts:176](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/wizard/session.ts#L176)
  - command: `nl -ba src/wizard/session.ts | sed -n '1,280p'`
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Onboarding flow is still sequential:** `runSetupWizard()` awaits prompts in order for setup mode, config handling, reset scope, mode selection, workspace, auth, and later sections. The control flow moves forward through awaited prompts; there is no state machine or previous-step branch for general onboarding navigation.
  - file: [src/wizard/setup.ts:201](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/wizard/setup.ts#L201)
  - command: `nl -ba src/wizard/setup.ts | sed -n '120,260p' && nl -ba src/wizard/setup.ts | sed -n '430,560p'`
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Only a narrow submenu-level Back exists:** The only `Back` label found in onboarding code is in plugin configuration, where `value: "__skip__"` is explicitly labeled `Back` with hint `Return to section menu`. That is not wizard-wide back navigation across onboarding steps.
  - file: [src/wizard/setup.plugin-config.ts:382](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/wizard/setup.plugin-config.ts#L382)
