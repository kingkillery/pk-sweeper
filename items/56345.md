## Summary

Current `main` no longer uses the older OpenClaw-owned ACPX queue-owner compatibility path described in the report. The bundled ACPX plugin now ships `acpx` `0.5.3`, delegates runtime/session behavior to `acpx/runtime`, and explicitly marks `queueOwnerTtlSeconds` as a legacy compatibility field that is accepted but ignored. That makes the reported `2026.3.24` + `acpx 0.3.1` Windows queue-owner TTL failure path obsolete on current `main` and in the `v2026.4.22` release.
