## Summary

Current `main` still treats `memory.backend` as a root-level/global setting, rejects `agents.list[].memory`, and resolves memory backend selection only from `cfg.memory`. The request is still live and is already linked to an open follow-up PR (#64647), so this is not a conservative close.

## Evidence

- **Agent config surface has no per-agent `memory` field:** `AgentConfig` includes `memorySearch` but no `memory` override, so the typed agent config still lacks the requested per-agent backend selection.
  - file: [src/config/types.agents.ts:70](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/config/types.agents.ts#L70)
