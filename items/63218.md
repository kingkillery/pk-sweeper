## Summary

Keep open. Current `main` still shows the shared-API Ollama routing shape this PR is trying to fix, and the PR also has an unresolved correctness concern around dedicated-lane fallback matching that needs maintainer review rather than conservative auto-close.

## Evidence

- **Current main still registers one stream per API:** `registerProviderStreamForModel()` resolves a provider-specific stream function, then registers it under `params.model.api`. For shared `api: "ollama"` providers, that means the first resolved stream can become the global handler for the API.
  - file: [src/agents/provider-stream.ts:15](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/agents/provider-stream.ts#L15)
  - command: `nl -ba src/agents/provider-stream.ts | sed -n '15,40p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Custom API registry is first-wins:** `ensureCustomApiRegistered()` bails out once an API provider already exists, so later Ollama providers sharing the same API cannot replace the earlier registered stream.
  - file: [src/agents/custom-api-registry.ts:15](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/agents/custom-api-registry.ts#L15)
  - command: `nl -ba src/agents/custom-api-registry.ts | sed -n '15,35p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Ollama provider stream closes over provider baseUrl:** The bundled Ollama plugin builds its stream function from the selected provider config's `baseUrl`, so registering that stream globally by API can misroute later runs that select a different Ollama provider id.
  - file: [extensions/ollama/index.ts:161](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/extensions/ollama/index.ts#L161)
  - command: `nl -ba extensions/ollama/index.ts | sed -n '161,167p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Current main still excludes Ollama from boundary-aware fallback transport:** `createBoundaryAwareStreamFnForModel()` only returns a handler for transport-aware supported APIs, and the existing test suite explicitly expects `createBoundaryAwareStreamFnForModel(model)` to be `undefined` for an Ollama model.
  - file: [src/agents/provider-transport-stream.test.ts:131](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/agents/provider-transport-stream.test.ts#L131)
