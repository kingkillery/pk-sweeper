## Summary

Keep open. Current `main` partially reflects the closed cross-reference by routing both direct and gateway sends through shared outbound/session plumbing, and the shared delivery layer does run `message_sending`. But the core acceptance gap from #63011 is still present: cancelled sends do not produce one canonical blocked/denied result, direct sends can surface `result: undefined`, and gateway sends still collapse empty delivery into `UNAVAILABLE: No delivery result` instead of an explicit guarded outcome.

## Risks / Open Questions

- Closing as implemented would overstate the current state: hook wiring and route/session correlation improved, but guarded denial semantics are still not normalized.
- The direct and gateway paths still diverge on blocked-send surface area (`undefined` result vs `UNAVAILABLE: No delivery result`), which is the ambiguity this issue was opened to remove.
