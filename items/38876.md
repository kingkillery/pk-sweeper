## Summary

Keep open. Current `main` still only extracts tool-result text when `item.text` or `item.content` is a plain string, so the array-shaped tool-result case described by PR #38876 is not implemented on `main` and there is no strong basis to close it as obsolete.

## Evidence

- **Current extractor still ignores array-shaped tool text:** `extractToolText` returns only `item.text` or `item.content` when they are strings; there is no `Array.isArray(...)` handling for either field.
  - file: [ui/src/ui/chat/tool-cards.ts:44](https://github.com/openclaw/openclaw/blob/4d7c4b3298920c0e6e2c5cc8bd7a1546e8122abe/ui/src/ui/chat/tool-cards.ts#L44)
