## Summary

Keep open. Current `main` still has no stale-activity recovery path in the macOS `WorkActivityStore`, so the reported menubar-stuck symptom remains plausible when the gateway never emits a terminal job event. The PR also still has review-noted correctness questions, so there is not strong evidence to close it as already implemented or obsolete.

## Evidence

- **Current main only clears work activity on explicit terminal events:** `handleJob` treats `started`/`streaming` as active and clears state only in the `else` branch for terminal states. There is no watchdog, eviction, or disconnect cleanup in this file on `main`. If a provider/gateway path drops the terminal event, the stored activity can remain indefinitely.
