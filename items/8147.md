## Summary

Keep open. Current main already avoids IME-triggered accidental sends in both WebChat surfaces, but the requested configurable send hotkey is not implemented on main. Browser Control UI/WebChat still advertises and uses plain Enter-to-send, and the exposed WebChat config/schema only includes `gateway.webchat.chatHistoryMaxChars`, not a send-hotkey option.
