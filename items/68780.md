## Summary

Keep open. Current `main` still emits the same open-allowlist and missing-provenance warnings for auto-discovered non-bundled plugins, and loader tests explicitly assert that behavior. One subclaim in the report looks partially outdated on current `main` because bundled Feishu now uses `feishu` for both plugin and channel id, but the broader UX complaint remains unresolved.

- **Loader still emits the reported warnings:** `warnWhenAllowlistIsOpen` still warns whenever `plugins.allow` is empty and any workspace/global plugin is discoverable, and `warnAboutUntrackedLoadedPlugins` still warns for loaded non-bundled plugins without install/load-path provenance.
