## Summary

Keep open. Current `main` still refreshes the managed gateway service by rerunning `gateway install --force` with the updater process environment, and the service environment builder still intentionally persists proxy variables into systemd/launchd env. The same behavior is present in the latest release commit `a9797214338ba31c52c796adbb75afb16e0684a9`, so this is neither fixed on `main` nor obsolete.

## Evidence

- **Updater refresh reuses current process environment:** `refreshGatewayServiceEnv` runs the updated CLI entrypoint with `env: resolveServiceRefreshEnv(process.env, ...)`, and that helper only normalizes path-like vars; it does not strip proxy vars before invoking `gateway install --force`.
  - file: [src/cli/update-cli/update-command.ts:256](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/cli/update-cli/update-command.ts#L256)
  - command: `nl -ba src/cli/update-cli/update-command.ts | sed -n '174,282p'`
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **Post-update restart path enables service env refresh:** After a successful update, if the gateway service is loaded, `update-command` sets `refreshGatewayServiceEnv = true` and calls `maybeRestartService(...)`, which triggers the refresh path before restart.
  - file: [src/cli/update-cli/update-command.ts:1390](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/cli/update-cli/update-command.ts#L1390)
  - command: `nl -ba src/cli/update-cli/update-command.ts | sed -n '1390,1445p'`
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **Managed service environment still persists proxy vars from host env:** `buildCommonServiceEnvironment` spreads `sharedEnv.proxyEnv` into the service environment, and `resolveSharedServiceEnvironmentFields` populates `proxyEnv` from `readServiceProxyEnvironment(env)`, which reads `HTTP_PROXY`, `HTTPS_PROXY`, `NO_PROXY`, and related lowercase variants from the caller environment.
  - file: [src/daemon/service-env.ts:321](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/daemon/service-env.ts#L321)

- **Tests still assert proxy forwarding as intended behavior:** The daemon service env test explicitly expects launchd/systemd runtime env to forward `HTTP_PROXY`, `HTTPS_PROXY`, `NO_PROXY`, `http_proxy`, and `all_proxy`.
  - file: [src/daemon/service-env.test.ts:328](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/daemon/service-env.test.ts#L328)
