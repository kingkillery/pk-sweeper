## Summary

Keep open. Current `main` still appears to route `openclaw secrets audit` through the resolved runtime config, so `${VAR}` config entries can become plaintext before audit classification and still hit `PLAINTEXT_FOUND` in `openclaw.json`.

## Evidence

- **Audit reads resolved runtime config:** `runSecretsAudit()` calls `readConfigFileSnapshot()` and then uses `snapshot.config` for both defaults and the config under audit. `ConfigFileSnapshot.config` is the runtime-shaped config, not the raw authored file.
  - file: [src/secrets/audit.ts:648](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/secrets/audit.ts#L648)
  - command: `nl -ba src/secrets/audit.ts | sed -n '644,668p'`
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Snapshot config is after ${ENV} substitution:** `ConfigFileSnapshot` documents `sourceConfig`/`resolved` as being after `${ENV}` substitution, and `config` is the runtime config field. `createConfigFileSnapshot()` also annotates `sourceConfig` as built from data after `${ENV}` substitution.
  - file: [src/config/types.openclaw.ts:166](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/config/types.openclaw.ts#L166)

- **Snapshot builder comment confirms substitution timing:** `createConfigFileSnapshot()` comments that `resolvedConfigRaw` is used after `$include` and `${ENV}` substitution, which matches the audit path consuming substituted config data rather than raw placeholders.
  - file: [src/config/io.ts:1459](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/config/io.ts#L1459)

- **Plaintext finding path only skips entries that still parse as refs:** `collectConfigSecrets()` first calls `resolveSecretInputRef()` and only skips the finding when a ref survives to that point; otherwise any non-empty secret-like value falls through to `PLAINTEXT_FOUND`. Once `${OPENAI_API_KEY}` has already been substituted to the actual key, this branch will flag it as plaintext.
  - file: [src/secrets/audit.ts:220](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/secrets/audit.ts#L220)

- **Supported config syntax is exactly ${VAR_NAME}:** The public config docs explicitly state that any config string value can reference env vars with `${VAR_NAME}`, including provider `apiKey` fields. That is the syntax reported in this issue body, so treating it as plaintext would be a real mismatch with documented behavior.
  - file: [docs/gateway/configuration.md:623](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/docs/gateway/configuration.md#L623)
