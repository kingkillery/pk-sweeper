## Summary

Current `main` already contains ACPX stale-session recovery for the exact class of failure described here: fresh-session preparation, retry without a stale persisted ACP session id, and regression coverage for `NO_SESSION`/resource-not-found recovery. The bundled ACPX runtime also suppresses reopening stale backend-owned state after reset/close, which is the load-bearing fix for intermittent empty `payloads: []` results on ACP-backed agent runs.

## Evidence
