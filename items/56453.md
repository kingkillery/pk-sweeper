## Summary

Keep open. Current `main` still hardcodes implicit `openclaw agent --to ...` session derivation to `main` instead of the configured default agent, so the PR's reported bug is still present and not implemented on `main`. The PR may need a rebase because `src/agents/command/session.ts` changed later for session-id routing, but the requested default-agent fix is still missing.
