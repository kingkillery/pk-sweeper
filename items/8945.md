## Summary

Current main already supports persistent per-channel model defaults via `channels.modelByChannel`, with explicit session `/model` overrides taking precedence. The feature is documented, wired into model resolution, covered by tests, and present in the latest release evidence.

## Evidence

## Risks / Open Questions

- The implemented config shape is `channels.modelByChannel` rather than the issue's proposed `channels.<channel>.model`, so the syntax differs even though the capability is present.

## Close Comment

Closing after Codex review: current `main` already implements this. OpenClaw now supports persistent channel-specific model defaults through `channels.modelByChannel`, documented in `docs/gateway/configuration-reference.md:56`, accepted by the config schema in `src/config/types.channels.ts:69` and `src/config/zod-schema.providers.ts:143`, and applied at runtime in `src/auto-reply/reply/get-reply.ts:315` only when a session does not already have an explicit `/model` override. The matcher in `src/channels/model-overrides.ts:157` also handles channel/topic/thread-style keys, and the changelog shows this shipped in `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).

One syntax note: the shipped config uses `channels.modelByChannel` instead of the exact `channels.<channel>.model` shape proposed here, but the underlying capability requested by this issue is present.
