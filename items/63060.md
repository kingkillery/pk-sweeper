## Summary

Keep open. Current main still models Z.AI as an OpenAI-compatible chat-completions provider, but that same routing/setup was already present in `v2026.4.5`, so I did not find strong evidence that the reported `/v4/v1/messages` regression has been fixed on main. I also could not do a current-main runtime repro in this checkout because `pnpm` is unavailable here.

## Evidence

- **Current main still configures Z.AI as `openai-completions`:** The bundled Z.AI onboarding path still writes provider config with `api: "openai-completions"` and a Z.AI base URL, which is the expected transport for GLM on coding/general endpoints.
  - file: [extensions/zai/onboard.ts:43](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/extensions/zai/onboard.ts#L43)
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Current main still probes Z.AI coding endpoints via `/chat/completions`:** Endpoint detection in current main sends probe requests to `${baseUrl}/chat/completions`, not `/v1/messages`, and the tests cover both general and coding-plan endpoints.
  - file: [src/plugins/provider-zai-endpoint.ts:29](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/plugins/provider-zai-endpoint.ts#L29)
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Current tests assert Z.AI provider config remains `openai-completions`:** The Z.AI onboarding test on current main still expects `models.providers.zai.api` to be `openai-completions`.
  - file: [extensions/zai/onboard.test.ts:8](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/extensions/zai/onboard.test.ts#L8)
