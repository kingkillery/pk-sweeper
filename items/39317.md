## Summary

Keep open. Current `main` still appears to treat fresh `codex exec --json` runs as successful whenever the process exits 0, because the CLI runner only throws on non-zero exit/timeout and the JSONL parser only harvests Claude result events or `item.text` message records. There is no current Codex-specific structured sandbox-denial detection on the success path.

## Evidence
