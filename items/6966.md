## Summary

Current `main` already does rate-limit-triggered model fallback inside a single run, but it does not implement the issue's requested sustained dynamic default-model switching. Auto failover selections are explicitly treated as transient and are cleared before the next turn, so OpenClaw still retries the configured primary model on the next message instead of staying downgraded until conditions recover.
