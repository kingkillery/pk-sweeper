## Summary

Keep open. Current `main` still scans fenced-code content when extracting memory-wiki links, so the reported false-positive lint path remains plausible on both `main` and the latest release. The PR is directionally useful, but its latest reviewed head still has unresolved regression feedback about strip order, so this is not a conservative auto-close.
