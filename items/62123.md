## Summary

Keep open: current `main` still does not expose a public async plugin hook that can mutate or suppress a tool result before transcript persistence. The inspected surface is still `after_tool_call` (observe-only, async void/fire-and-forget), `tool_result_persist` (sync-only), and `before_message_write` (sync-only); the only async `tool_result` middleware on `main` is a separate bundled Codex extension seam, not the public plugin hook API requested here.
