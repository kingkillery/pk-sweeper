## Summary

Keep open. Current `main` learns Telegram forum topic names during inbound handling, but it does not feed those names into session metadata or the session dropdown label path. The picker still resolves from `label`/`displayName` or falls back to raw scoped key text, so the requested `Telegram : GroupName : TopicName` behavior is not implemented on current `main`.

## Evidence

- **Telegram topic names are captured, but only in inbound context:** Forum topic names are read from `forum_topic_created`/`forum_topic_edited` service messages, cached, and assigned to a local `topicName` value during Telegram inbound processing.
  - file: [extensions/telegram/src/bot-message-context.ts:157](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/extensions/telegram/src/bot-message-context.ts#L157)
  - command: `nl -ba extensions/telegram/src/bot-message-context.ts | sed -n '157,208p'`
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Telegram session context keeps chat title and topic name separate:** When the inbound context is finalized, `GroupSubject` is set from `msg.chat.title`, while `TopicName` is emitted as a separate field. There is no code here that promotes `TopicName` into the session display fields.
  - file: [extensions/telegram/src/bot-message-context.session.ts:333](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/extensions/telegram/src/bot-message-context.session.ts#L333)
  - command: `nl -ba extensions/telegram/src/bot-message-context.session.ts | sed -n '255,385p'`
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Session metadata derivation ignores `TopicName`:** `deriveGroupSessionPatch()` builds the group/session patch from `GroupSubject`, `GroupSpace`, and `GroupChannel`, then computes `displayName` from those values. `TopicName` is not read anywhere in this path.
  - file: [src/config/sessions/metadata.ts:122](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/config/sessions/metadata.ts#L122)
  - command: `nl -ba src/config/sessions/metadata.ts | sed -n '108,190p'`
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Group display names have no topic-name slot:** `buildGroupDisplayName()` can only format `provider`, `subject`, `groupChannel`, `space`, `id`, and `key`, producing tokens like `telegram:g-...`. It cannot produce a `Telegram : GroupName : TopicName` label.
  - file: [src/config/sessions/group.ts:45](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/config/sessions/group.ts#L45)
  - command: `nl -ba src/config/sessions/group.ts | sed -n '38,75p'`
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Session dropdown uses only `label`/`displayName`, otherwise raw scoped text:** The UI picker resolves names from `row.label` or `row.displayName`; if neither exists, it falls back to the parsed key/rest (`telegram:group:...:topic:...`). Since session metadata never injects `TopicName`, the dropdown cannot show the requested human-readable topic label.
  - file: [ui/src/ui/chat/session-controls.ts:390](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/ui/src/ui/chat/session-controls.ts#L390)
