## Summary

Keep open. Current `main` does not already implement this PR, and the proposed regex expansion in PR #38846 still appears unsafe by inspection: it would route normal Windows batch invocations through `validateWindowsArgument()` and reject common legitimate inputs such as backslash paths, scoped package names with `/`, and flags containing `=`.

## Evidence

- **Current main keeps the narrow cmd.exe blocklist:** `src/process/exec.ts` on `main` still uses `const WINDOWS_UNSAFE_CMD_CHARS_RE = /[&|<>^%\r\n]/;` and has no exported `validateWindowsCommandArgument()`, so this PR has not already landed separately.
  - file: [src/process/exec.ts:15](https://github.com/openclaw/openclaw/blob/c971c58fc7592344539ac3cf87c7c4229be9978e/src/process/exec.ts#L15)
  - command: `nl -ba src/process/exec.ts | sed -n '1,70p'`
  - sha: [c971c58fc759](https://github.com/openclaw/openclaw/commit/c971c58fc7592344539ac3cf87c7c4229be9978e)
- **Batch wrapper validates the resolved command path itself:** `buildCmdExeCommandLine()` applies `escapeForCmdExe(resolvedCommand)` and `resolveChildProcessInvocation()` uses that path for `.cmd`/`.bat` commands. If the PR's expanded regex blocks `\\`, any normal Windows absolute batch path would throw before spawn.
  - file: [src/process/exec.ts:40](https://github.com/openclaw/openclaw/blob/c971c58fc7592344539ac3cf87c7c4229be9978e/src/process/exec.ts#L40)
  - command: `nl -ba src/process/exec.ts | sed -n '1,120p'`
  - sha: [c971c58fc759](https://github.com/openclaw/openclaw/commit/c971c58fc7592344539ac3cf87c7c4229be9978e)
- **PR patch expands the blocked character set too far:** The PR patch at commit `73a73bbf30ed14b3ab8323ab34caaf5c12077fc8` changes the regex to `/[&|<>\^%\r\n;`$(){}[\]=+'\\/*?!~]/` and makes `escapeForCmdExe()` call `validateWindowsArgument(arg, "escapeForCmdExe")`, which is the source of the path/argument regressions called out in review.
  - file: [src/process/exec.ts](https://github.com/openclaw/openclaw/blob/73a73bbf30ed14b3ab8323ab34caaf5c12077fc8/src/process/exec.ts)
  - command: `curl -L --silent https://patch-diff.githubusercontent.com/raw/openclaw/openclaw/pull/38846.patch | sed -n '1,220p'`

- **Current main has legitimate slash/equals arguments that the PR would reject:** `buildNodeInstallCommand()` passes package names like `@openclaw/matrix` through the process launcher, and global update commands include `--loglevel=error` and `--omit=optional`. Those are normal arguments on current `main`; the PR's added `/` and `=` bans would reject them.
