## Summary

Keep open. Current main includes mitigations around memory indexing, but the reported problem is not strongly closed out: batch indexing is now opt-in and rate-limit retries exist, yet batch mode still disables itself after repeated failures, full reindexes still roll back atomically on failure, and the macOS/network-family follow-up is not clearly fixed in the memory path.

## Evidence

- **Batch indexing is mitigated, not removed:** The current config docs show `remote.batch.enabled` defaults to `false` and `remote.batch.concurrency` defaults to `2`, so large-workspace async batch indexing is no longer the default path, but it still exists when enabled.
