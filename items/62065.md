## Summary

Keep open. Current `main` has a plugin/API path for memory embedding providers, but the bundled MiniMax plugin does not implement or declare one yet. The issue body mixes a separate session-memory-loss complaint into the feature request, so it likely needs clarification or splitting, but there is still a real actionable request here and it is too recent for a stale close.

## Evidence

- **Memory embedding providers are a supported plugin contract:** The plugin SDK explicitly supports `api.registerMemoryEmbeddingProvider(adapter)`, and `agents.defaults.memorySearch.provider` resolves against registered adapter ids. That makes a MiniMax memory embedding adapter a valid current architecture path rather than an out-of-scope core rewrite.
