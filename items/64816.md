## Summary

Keep open. Current `main` has the generic code paths that should surface explicitly enabled bundled plugin skills, and docs now explain that skill inventories are session snapshots. But the post-release regression coverage I found only proves the default-enabled bundled-plugin case (`browser`), not the reporter's explicitly enabled `memory-wiki` case, and the original report never confirmed a fresh-session repro because that path was blocked.

## Evidence

- **memory-wiki still declares bundled skills:** The bundled `memory-wiki` manifest still exports `skills: ["./skills"]`, so this issue is still about a real plugin-owned skill path rather than a removed surface.
  - file: [extensions/memory-wiki/openclaw.plugin.json:2](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/memory-wiki/openclaw.plugin.json#L2)

- **wiki-maintainer has no frontmatter gate:** `wiki-maintainer` only declares `name` and `description`; there is no `disable-model-invocation` or metadata gate in the skill file itself that would obviously suppress it from `<available_skills>`.
  - file: [extensions/memory-wiki/skills/wiki-maintainer/SKILL.md:1](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/memory-wiki/skills/wiki-maintainer/SKILL.md#L1)
