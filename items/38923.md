## Summary

Keep open. Current `main` does not clearly implement this PR’s `IDENTITY.md` avatar fixes: `parseIdentityMarkdown` still preserves backtick-wrapped avatar values, there is no regression test for that case on `main`, and the avatar resolver still uses the pre-PR relative-path resolution logic. I do not have strong evidence that #38439 is already fixed on `main` or in `v2026.4.22`.
