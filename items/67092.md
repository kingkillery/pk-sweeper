## Summary

Current `main` still preserves text that appears before an orphan trailing `</think>` and appends the trailing answer after it, so the reported reasoning leak remains plausible for both user-visible UI text and persisted history paths.

## Evidence

- **Current reasoning-tag stripper keeps both sides of an orphan closing tag:** `stripReasoningTagsFromText` appends `cleaned.slice(lastIndex, idx)` whenever it sees a tag while `inThinking === false`, and for a closing tag it never enters thinking mode before later appending the suffix too. For input shaped like `[reasoning] </think> [answer]`, that logic preserves both the leaked reasoning prefix and the final answer.

  - command: `nl -ba src/shared/text/reasoning-tags.ts | sed -n '20,100p'`
  - sha: [c7f18a6b9d76](https://github.com/openclaw/openclaw/commit/c7f18a6b9d76a31f4daed5096c0a25b3794c76ac)
- **Local reproduction of the current algorithm matches the reported leak shape:** Replaying the current loop with `internal reasoning </think> final answer` produced `internal reasoning  final answer`, which confirms the orphan closing tag path does not discard the leaked prefix on current `main`.
