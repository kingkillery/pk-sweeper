## Summary

Keep open. Current `main` still drives dashboard and status context percentages from `totalTokens`, and `totalTokens` is still defined as prompt-side tokens that include `cacheRead`/`cacheWrite`, so the issue's reported UI-accounting behavior is not clearly fixed. `/new` does reset persisted counters, but the code/docs also show that a fresh first turn can legitimately be heavy because startup/bootstrap context is injected again.
