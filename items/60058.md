## Summary

Keep PR #60058 open. Current main has not implemented the PR’s handler change: `handleToolExecutionStart` still manually checks only `path` and `file_path`. The shared `resolvePathArg` helper does cover `filePath`, but not the `file` alias named in linked issue #60008, and the existing handler tests only cover `file_path` and the missing-path warning. This is useful work, but no conservative close reason applies.
