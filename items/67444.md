## Summary

Keep open. Current `main` still suppresses `ANNOUNCE_SKIP` before the completion-delivery path, and the existing tests explicitly assert that suppression. The PR targets a real remaining behavior difference, but the submitted branch is not ready as-is because review comments identify an unaddressed correctness issue: falling through with the literal `ANNOUNCE_SKIP` token would forward that control token as completion content.

## Best Possible Solution

Keep the PR open, but require a follow-up patch before merge: allow `ANNOUNCE_SKIP` to skip only the soft announce step when `expectsCompletionMessage` is true, clear the control token so delivery receives `(no output)` or real child completion findings, preserve `NO_REPLY` suppression, and add tests that positively verify the completion-delivery agent call plus cleanup/dependency reset behavior.
