## Summary

Current `main` does not implement Discord role-mention wake triggers. Discord guild wake gating still relies on explicit bot user mentions, configured mention patterns, reply-to-bot implicit mentions, and `@everyone`; Discord role IDs are used for sender allowlists/routing, not inbound wake triggers.

## Evidence

- **Explicit Discord mention detection only checks bot user mentions:** `explicitlyMentioned` is derived from `message.mentionedUsers?.some((user) => user.id === botId)`. `message.mentionedRoles` only feeds `hasAnyMention`/`hasUserOrRoleMention`, not the explicit wake path.
  - file: [extensions/discord/src/monitor/message-handler.preflight.ts:695](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/extensions/discord/src/monitor/message-handler.preflight.ts#L695)
  - command: `nl -ba extensions/discord/src/monitor/message-handler.preflight.ts | sed -n '688,709p'`
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Wake decision logic has no role-mention branch:** `resolveDiscordMentionState` marks a message as mentioned via `@everyone`, explicit mention matching, and reply-to-bot implicit mention. There is no path that treats a configured role ID mention as `wasMentioned`.
  - file: [extensions/discord/src/monitor/message-handler.preflight.ts:195](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/extensions/discord/src/monitor/message-handler.preflight.ts#L195)

  - file: [docs/channels/discord.md:453](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/docs/channels/discord.md#L453)
