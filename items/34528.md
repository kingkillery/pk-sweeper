## Summary

Current main still appears to pass Feishu synthetic reaction message IDs through to Feishu API-facing reply, typing, media, and reaction paths without stripping the `:reaction:<emoji>:<uuid>` suffix. The linked PRs are useful evidence and contain proposed fixes, but the observable fix is not implemented on current main, so the issue should remain open.
