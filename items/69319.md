## Summary

Keep open. Current `main` still does not implement `skills.uninstall`, so issue #69299 remains unresolved, and this PR still has two concrete correctness gaps: it can delete a non-ClawHub-managed `skills/<slug>` directory, and it maps malformed uninstall slugs to `UNAVAILABLE` instead of `INVALID_REQUEST`.

## Risks / Open Questions

- Closing this PR would leave issue #69299 unresolved even though current `main` still lacks any scope-gated gateway uninstall path for ClawHub skills.
- Merging the PR as written would introduce a destructive path where `skills.uninstall` can remove a non-ClawHub-managed `skills/<slug>` directory, and clients may retry malformed requests because they are reported as `UNAVAILABLE`.
