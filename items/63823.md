## Summary

Keep open. Current `main` still does not contain the requested MiniMax manifest auth-alias mapping, and the new auth-alias resolver only reads manifest-owned aliases/deprecated choice ids, not runtime `hookAliases`. MiniMax still publishes `minimax-cn`/`minimax-portal-cn` only as `hookAliases`, so there is not strong evidence that #63823 is fixed on `main`. The current onboarding flow does store CN API-key setup under provider `minimax`, which narrows the exact reproduction path, but that is not enough to safely auto-close the issue.
