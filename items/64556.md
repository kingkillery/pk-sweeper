## Summary

Keep open: current `main` still hard-codes wake hooks to the main session, the mapping pipeline still drops `agentId`/`sessionKey` for `action: "wake"`, and the existing tests explicitly encode that behavior, so this is not implemented on `main` yet.

## Evidence

- **Wake dispatch still hard-codes the main session:** `dispatchWakeHook` resolves `resolveMainSessionKeyFromConfig()` and enqueues there unconditionally; it does not accept or derive a target `agentId` or `sessionKey`.
  - file: [src/gateway/server/hooks.ts:35](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/gateway/server/hooks.ts#L35)
  - command: `nl -ba src/gateway/server/hooks.ts | sed -n '28,46p'`
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **Wake mappings still drop routing fields:** `buildActionFromMapping()` returns `{ kind: "wake", text, mode }` only for wake mappings. By contrast, the agent branch carries `agentId` and `sessionKey`, so wake mappings still discard those fields before dispatch.
  - file: [src/gateway/hooks-mapping.ts:249](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/gateway/hooks-mapping.ts#L249)
