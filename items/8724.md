## Summary

Keep open. Current `main` has a global LLM idle timeout and a global agent runtime timeout, but it does not have a per-model generation-timeout setting, and the existing idle watchdog only fires when no chunks arrive.

## Evidence

- **Per-model model entries lack timeout fields:** `AgentModelEntryConfig` only allows `alias`, provider `params`, and `streaming`, and `agents.defaults.models` uses that same shape. There is no first-class per-model generation-timeout override in the config schema.
  - file: [src/config/types.agent-defaults.ts:31](https://github.com/openclaw/openclaw/blob/c0a7b6a510758645c5267734b984f4885a605b67/src/config/types.agent-defaults.ts#L31)

  - sha: [c0a7b6a51075](https://github.com/openclaw/openclaw/commit/c0a7b6a510758645c5267734b984f4885a605b67)
- **Only global LLM idle timeout exists:** `resolveLlmIdleTimeoutMs()` reads only `cfg.agents.defaults.llm.idleTimeoutSeconds`, `runTimeoutMs`, and `cfg.agents.defaults.timeoutSeconds`; it does not consult the selected model or `agents.defaults.models[...]`.
  - file: [src/agents/pi-embedded-runner/run/llm-idle-timeout.ts:22](https://github.com/openclaw/openclaw/blob/c0a7b6a510758645c5267734b984f4885a605b67/src/agents/pi-embedded-runner/run/llm-idle-timeout.ts#L22)
