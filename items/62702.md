## Summary

Current `main` does not implement the requested non-blocking async hook execution model. Prompt/model hook results are still awaited on the turn path, the shared modifying-hook runner is explicitly sequential, and plugin config/docs do not expose per-hook `timeoutMs`, fallback, or `critical` execution controls.

## Evidence

- **Sequential modifying-hook runner:** `runModifyingHook(...)` is documented as sequential and uses `await` inside a `for` loop over hook handlers, so modifying hooks still block on each handler in priority order.
  - file: [src/plugins/hooks.ts:343](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/plugins/hooks.ts#L343)
  - command: `nl -ba src/plugins/hooks.ts | sed -n '343,366p'`

- **Prompt-build path awaits hook completion:** `resolvePromptBuildHookResult(...)` awaits `runBeforePromptBuild(...)` and legacy `runBeforeAgentStart(...)` before continuing prompt assembly, so these hooks remain on the synchronous turn path.
  - file: [src/agents/pi-embedded-runner/run/attempt.prompt-helpers.ts:42](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/agents/pi-embedded-runner/run/attempt.prompt-helpers.ts#L42)
