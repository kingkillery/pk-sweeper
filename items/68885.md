## Summary

Current `main` still uses trusted system `ffmpeg`/`ffprobe` and the bundled-binary approach from PR #68885 is not implemented. The PR also still hits real load-bearing seams called out in review: skill sandboxes strip `node_modules`, `ffmpeg-exec` is re-exported through the public media runtime barrel, and existing tests do not cover the new binary-resolution path. No conservative auto-close reason fits, so this should stay open for maintainer judgment.

## Risks / Open Questions

- This PR changes `src/media/ffmpeg-exec.ts`, which is exported through `openclaw/plugin-sdk/media-runtime`; import-time regressions there can affect unrelated extensions.
- Skill sandboxes explicitly exclude `node_modules`, so a skill script that imports repo-only packages can fail after sync even if it works in-repo.
- Current test coverage does not exercise binary-resolution fallback or the skill CLI behavior, so merging without additional validation would be high-risk.
