## Summary

Keep open. Current `main` still uses a batched voice-call response pipeline: the voice response generator returns one final `text` value, the webhook waits for that full result before calling `speak`, Twilio stream playback waits for a full telephony audio `Buffer` before chunking, and Telnyx still uses a one-shot `speak` action. That does not satisfy the sentence-level LLM→TTS→audio streaming request in #8355.
