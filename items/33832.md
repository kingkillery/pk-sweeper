## Summary

Current main partially covers the timing/telemetry side via diagnostic model-call events, plugin hooks, and OTEL duration metrics, but it does not implement the requested Gateway log lines with request/response body sizes or time-to-first-byte. The linked implementation PR #35513 was closed unmerged in the provided GitHub context, and its exact `wrapStreamFnWithModelApiLogging` implementation is not present on main.

## Evidence

There is no adjacent model API logging wrapper for request/response body size or first-byte logging.
