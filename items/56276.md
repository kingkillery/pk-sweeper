## Summary

Current `main` does not implement showing the hostname in the TUI status/footer bar. The TUI header includes the connection URL, but the footer/status content still omits any host/hostname field, so this issue should remain open.

## Evidence

- **Footer docs omit hostname:** The TUI docs describe the footer as `connection state + agent + session + model + think/fast/verbose/trace/reasoning + token counts + deliver`, with no hostname field.
  - file: [docs/web/tui.md:56](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/docs/web/tui.md#L56)

  - file: [src/tui/tui.ts:816](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/tui/tui.ts#L816)
