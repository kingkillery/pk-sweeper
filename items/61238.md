## Summary

Keep open. Current `main` still treats daily 4:00 AM rollover as the default session policy and silently creates a new `sessionId` once a session is stale. The codebase now archives the prior transcript instead of orphaning/deleting it, so the report overstates irreversible data loss, but the core UX complaint about an implicit daily reset without a user-facing warning/cancel flow is still not resolved on `main`.

## Evidence
