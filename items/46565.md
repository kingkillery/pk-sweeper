## Summary

Keep open. Current `main` already fixes the reported HEARTBEAT/subagent bleed-through in part by filtering bootstrap files for subagent/cron sessions and by supporting lightweight heartbeat/cron bootstrap modes, but the broader request is not fully implemented: bootstrap files do not have native `!command`-style preprocessing, and live current time is still intentionally omitted from the system prompt.

## Evidence

- **Subagent/cron bootstrap filtering exists on main:** `filterBootstrapFilesForSession(...)` now special-cases subagent and cron session keys and only keeps a minimal allowlist instead of injecting the full bootstrap set.
  - file: [src/agents/workspace.ts:552](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/agents/workspace.ts#L552)

  - file: [src/agents/bootstrap-files.ts:165](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/agents/bootstrap-files.ts#L165)
