## Summary

Keep open. Current `main` at `c0a7b6a510758645c5267734b984f4885a605b67` still does not implement general retry/backoff for OAuth refresh failures. The refresh path makes a single `getOAuthApiKey(...)` call, and the only retry in the manager is a narrow second attempt for `refresh_token_reused` race handling, not for transient network/timeouts/API blips described in #8673.

## Evidence

- **Single refresh call only:** `refreshOAuthCredential()` directly awaits one `getOAuthApiKey(...)` call and returns/throws from that result; there is no loop, backoff, or retry helper around the OAuth API call.
