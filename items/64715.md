## Summary

Current `main` still exposes Telegram interactive output as inline keyboards only. The shared presentation contract, Telegram outbound adapter, Telegram send surface, and user docs do not implement native Telegram reply keyboards or reply-keyboard removal flags, so issue #64715 remains a valid feature gap and should stay open.

## Evidence

- **Telegram send surface only declares inline buttons:** `TelegramSendOpts` exposes `buttons?: TelegramInlineButtons` and documents them as `Inline keyboard buttons (reply markup)`; there is no reply-keyboard or remove-keyboard option on the outbound send surface.
  - file: [extensions/telegram/src/send.ts:100](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/telegram/src/send.ts#L100)
  - command: `nl -ba extensions/telegram/src/send.ts | sed -n '90,140p'`

- **Telegram markup builder emits only `InlineKeyboardMarkup`:** The Telegram helper imports `InlineKeyboardMarkup` from `@grammyjs/types` and returns `{ inline_keyboard: rows }` only. No `ReplyKeyboardMarkup`, `ReplyKeyboardRemove`, `resize_keyboard`, or `one_time_keyboard` handling exists here.
  - file: [extensions/telegram/src/inline-keyboard.ts:1](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/telegram/src/inline-keyboard.ts#L1)
