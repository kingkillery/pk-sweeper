## Summary

Current `main` still rejects extra top-level fields on Gateway `agent` requests, and `AgentParamsSchema` still does not define a `paperclip` field. A nearby Paperclip-related fix landed only for `wake` payloads, not for `agent`, so this issue is not implemented on `main` yet.

## Evidence
