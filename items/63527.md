## Summary

Keep open. Current `main` has targeted WhatsApp auth-store hardening for atomic `creds.json` writes and queued-save draining before reconnect/bootstrap, but the self-chat presence-on-connect behavior still exists and I found no strong evidence that issue #63527 is fully resolved on current `main`. The reporter's own follow-up also weakens the original regression theory without disproving the underlying bug.

## Evidence

- **Reporter follow-up contradicts the claimed regression window:** The issue comment says the same `creds.json` corruption cycle was also observed on v3.28 on April 9, 2026, so the report no longer cleanly supports a `2026.4.2`-introduced regression. That weakens the suspected-cause theory, but it does not prove the bug is gone.
  - command: `GitHub issue context comment 4211563900`
- **Current main still sends presence on connect in self-chat mode:** `attachWebInboxToSocket()` still computes `presence = options.selfChatMode ? "unavailable" : "available"` and immediately calls `sock.sendPresenceUpdate(presence)`. The requested disable flag is not present, so the suspected behavior was not removed.
  - file: [extensions/whatsapp/src/inbound/monitor.ts:133](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/extensions/whatsapp/src/inbound/monitor.ts#L133)
