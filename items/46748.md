## Summary

Current `main` does not implement the requested default behavior. `openclaw logs` still defaults to UTC formatting and only shows local time when `--local-time` is passed or a valid `TZ` env var is set. The issue’s suggested root cause is outdated, though: the parser already prefers the log entry’s `time` field, but the CLI formatter converts that parsed timestamp back to UTC by default.
