## Summary

Keep open. Current `main` still does not implement Feishu-native exec approval buttons/cards for approval resolution. The Feishu plugin only wires approver authorization, not a native approval adapter or interactive outbound path, while approval prompts are generated as generic interactive replies that Feishu does not render through its current outbound delegates. Docs for native chat exec approvals list Discord, Slack, and Telegram, but not Feishu.

## Evidence

- **Feishu approval capability is auth-only:** The Feishu plugin exports `feishuApprovalAuth` via `createResolvedApproverActionAuthAdapter`, resolving approvers from `allowFrom` and normalizing sender IDs. There is no native approval delivery/runtime or button-rendering logic in this capability.
  - file: [extensions/feishu/src/approval-auth.ts:15](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/extensions/feishu/src/approval-auth.ts#L15)
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Feishu channel wires auth, then text/media outbound only:** `feishuPlugin` sets `approvalCapability: feishuApprovalAuth`. Its outbound adapter later uses `createRuntimeOutboundDelegates` with only `sendText` and `sendMedia`, so the channel does not expose a `sendPayload`/interactive delivery path for approval buttons.
  - file: [extensions/feishu/src/channel.ts:666](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/extensions/feishu/src/channel.ts#L666)
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Runtime outbound delegates do not include interactive payload support:** `createRuntimeOutboundDelegates` only forwards `sendText`, `sendMedia`, and `sendPoll`. There is no forwarded `sendPayload` hook here, which is the path other channels use for richer interactive delivery.
  - file: [src/channels/plugins/runtime-forwarders.ts:70](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/channels/plugins/runtime-forwarders.ts#L70)
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Approval prompts are built with interactive buttons upstream:** Shared approval payloads include `interactive: buildApprovalInteractiveReply(...)`, so the product-side approval UX is generated as interactive buttons before channel delivery.
  - file: [src/plugin-sdk/approval-renderers.ts:16](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/plugin-sdk/approval-renderers.ts#L16)
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Feishu outbound text/media normalization ignores interactive blocks:** `resolveSendableOutboundReplyParts` only considers `text`, `mediaUrls`, and `mediaUrl`. Without a channel `sendPayload` implementation, Feishu falls back to text/media delivery instead of rendering approval buttons.
  - file: [src/plugin-sdk/reply-payload.ts:145](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/plugin-sdk/reply-payload.ts#L145)
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Docs still scope native exec approval clients to other channels:** The exec-approvals guide says `channels.<channel>.execApprovals` controls native approval clients, then documents only Discord, Slack, and Telegram configs. Feishu is not listed there, and the Feishu channel page still documents commands as plain text messages.
  - file: [docs/tools/exec-approvals-advanced.md:269](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/docs/tools/exec-approvals-advanced.md#L269)
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
