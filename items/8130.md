## Summary

Keep open. Current `main` and release `v2026.4.22` added fallback visibility in verbose replies, `/status`, and WebUI indicators, but they do not implement default proactive active-channel notifications for failover/errors when verbose mode is off.

## Evidence

- **Reply notice is explicitly gated by verbose mode:** The reply runner always emits fallback lifecycle events, but it only prepends a user-visible `Model Fallback` / `Model Fallback cleared` message inside `if (verboseEnabled)`. With default verbose off, the active channel still gets no textual fallback notice.
  - file: [src/auto-reply/reply/agent-runner.ts:1509](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/auto-reply/reply/agent-runner.ts#L1509)
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **E2E test locks in silent default-channel behavior:** Test `announces model fallback only when verbose mode is enabled` expects the fallback notice in payload text when verbose is `on`, and explicitly expects no `Model Fallback:` text when verbose is `off`, while lifecycle `fallback` events still fire.
  - file: [src/auto-reply/reply/agent-runner.runreplyagent.e2e.test.ts:622](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/auto-reply/reply/agent-runner.runreplyagent.e2e.test.ts#L622)
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Current non-verbose visibility is WebUI lifecycle state, not channel delivery:** Fallback lifecycle events are consumed into transient `fallbackStatus` UI state for the web chat, which is then rendered as a temporary indicator/toast. That improves UI visibility but is narrower than proactive notification sent to the user's active channel.
  - file: [ui/src/ui/app-tool-stream.ts:388](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/ui/src/ui/app-tool-stream.ts#L388)
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Shipped scope was narrower than the issue request:** The changelog entry for the shipped work says it added `model fallback lifecycle visibility in verbose logs, /status active-model context with fallback reason, and cohesive WebUI fallback indicators`, which matches the code and does not claim default channel-level failover/error notifications.
  - file: [CHANGELOG.md:4353](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/CHANGELOG.md#L4353)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Requested config knobs are absent:** A repo-wide search found no implementation of the proposed knobs `failoverNotify` or `retryNotifyAfterMs`, and no equivalent config surfaced in schema/docs.
  - command: `rg -n "failoverNotify|retryNotifyAfterMs" src docs ui`
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
