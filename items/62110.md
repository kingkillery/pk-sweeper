## Summary

Keep open: current `main` does not implement `agents.defaults.skipOptionalBootstrapFiles`, and the PR still has unresolved correctness concerns around workspace-bootstrap call-site coverage and legacy setup detection.

## Risks / Open Questions

- Merging the PR as shown in the provided discussion could leave `skipOptionalBootstrapFiles` ineffective in common reply/setup/gateway flows because not all bootstrap entrypoints receive the new option.
- The template-fallback approach discussed on the PR can re-seed `BOOTSTRAP.md` for existing customized workspaces, which conflicts with the current legacy-completion behavior covered in `src/agents/workspace.test.ts`.
