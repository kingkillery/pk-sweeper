## Summary

Current `main` contains a loader-side fix and regression coverage that preserve the registered memory `publicArtifacts` capability across `activate:false` snapshot plugin loads, which matches the reported failure mode behind `wiki.bridge.import` returning zero artifacts.

## Evidence

- **Loader now preserves memory capability during snapshot loads:** `loadOpenClawPlugins()` snapshots the previous memory capability and restores it when `shouldActivate` is false, preventing read-only/plugin-snapshot loads from wiping the active memory capability registration.
