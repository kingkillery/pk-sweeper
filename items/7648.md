## Summary

Keep open. Current `main` has a partial mitigation for manual scroll-up, but it does not implement the requested disable-auto-scroll option or a scroll-to-top control for Control UI/WebChat.

## Evidence

- **Send path still force-scrolls chat responses:** `sendChatMessageNow()` resets chat scroll state, then explicitly force-scrolls after send so the incoming response stays at the bottom: `ui/src/ui/app-chat.ts:166-189`.
  - file: [ui/src/ui/app-chat.ts:166](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/ui/src/ui/app-chat.ts#L166)
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Scroll helper still sticks to bottom on initial load:** `scheduleChatScroll()` treats `force=true` as an initial-load override and scrolls to `target.scrollHeight`, which matches the reported behavior for long responses: `ui/src/ui/app-scroll.ts:22-99`.
  - file: [ui/src/ui/app-scroll.ts:22](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/ui/src/ui/app-scroll.ts#L22)
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Tests lock in current bottom-follow behavior:** Current tests assert that `force=true` scrolls to the bottom on initial load, and only suppresses auto-scroll after the user has already scrolled up: `ui/src/ui/app-scroll.test.ts:155-202`.
  - file: [ui/src/ui/app-scroll.test.ts:155](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/ui/src/ui/app-scroll.test.ts#L155)
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Only user-facing affordance is a bottom-follow button:** The chat UI renders `New messages` wired to `scrollToBottom()`. There is no scroll-to-top button in the shared chat surface: `ui/src/ui/views/chat.ts:1158-1162`, `ui/src/ui/app-render.ts:2293-2294`, `ui/src/ui/app.ts:626-632`.
  - file: [ui/src/ui/views/chat.ts:1158](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/ui/src/ui/views/chat.ts#L1158)
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **No config/docs surface for an auto-scroll toggle:** A repo search for `gateway.(controlUi|webchat).*scroll|autoScroll|scrollToTop|scroll-top|Scroll to Top` under `src/config`, `docs`, and `ui/src/ui` returned no matches, so there is no existing config or documented toggle implementing the request.
  - command: `rg -n "gateway\.(controlUi|webchat).*scroll|autoScroll|scrollToTop|scroll-top|Scroll to Top" src/config docs ui/src/ui ui/src/styles`
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
