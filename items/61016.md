## Summary

Keep open. Current `main` already handles mixed-case and parameterized MIME before `sanitizeMimeType`, so part of the PR rationale is obsolete, but the stricter end-anchored/RFC-style validation proposed here is still not implemented on `main`. That means there is still unlanded behavior in this PR, so there is no strong close reason.
