## Summary

Keep open. Current `main` partially supersedes the docs portion with broader outbound `MEDIA:` policy docs, but it does not implement the PR's Feishu runtime behavior: blocked local-media send failures still only log and stop, with no Feishu-specific hint and no one-time plain-text fallback.

## Evidence

- **PR scope from the discussion:** The PR body and Greptile summary both describe three intended changes: a Feishu-specific allowed-root hint, a one-time plain-text fallback for final replies that fail on disallowed local `MEDIA:` paths, and docs updates. That means a close-as-implemented decision needs both docs and runtime behavior on `main`, not docs alone.
- **Feishu reply dispatcher still only logs delivery errors:** `extensions/feishu/src/reply-dispatcher.ts` logs `feishu[account] <kind> reply failed` and then closes streaming / idles typing. There is no local-media error detection, no hint text, and no fallback `sendMessageFeishu` path in the current handler.
  - file: [extensions/feishu/src/reply-dispatcher.ts:515](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/extensions/feishu/src/reply-dispatcher.ts#L515)
  - command: `nl -ba extensions/feishu/src/reply-dispatcher.ts | sed -n '430,560p'`
  - sha: [ec8dbc459558](https://github.com/openclaw/openclaw/commit/ec8dbc4595583816f71af2568df61dbe7c090e1f)
- **Dispatcher contract would route media-send failures into that Feishu onError path:** `createReplyDispatcher` awaits channel delivery and catches thrown errors at the dispatcher layer, invoking the channel `onError` callback. For a `path-not-allowed` media send, current Feishu behavior therefore terminates at the log-only handler above.
