## Summary

Keep open. Current `main` has conservative auto-recovery for invalid `openclaw.json` and runtime secret snapshots, but the requested broader rollback for risky self-modification of other critical startup artifacts is not implemented strongly enough to justify closure.

## Evidence

- **Startup config recovery exists, but it is config-specific:** Gateway startup restores `openclaw.json` from a last-known-good backup when the config snapshot is invalid, then reloads that config snapshot.
  - file: [src/gateway/server-startup-config.ts:60](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/gateway/server-startup-config.ts#L60)
  - command: `nl -ba src/gateway/server-startup-config.ts | sed -n '1,220p'`
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Docs scope the rollback mechanism to openclaw.json:** User-facing docs describe last-known-good restoration for `openclaw.json`, including `.clobbered.*` recovery and rejection of invalid external edits, but do not claim equivalent rollback for manifests, registries, or other startup artifacts.
  - file: [docs/gateway/configuration.md:87](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/docs/gateway/configuration.md#L87)
  - command: `nl -ba docs/gateway/configuration.md | sed -n '80,120p;498,516p'`
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Hot-reload tests cover invalid config recovery only:** Current tests verify restoring last-known-good on invalid external config edits and reloading the recovered config; this is evidence of a narrow config safeguard, not a general self-modification rollback layer.
  - file: [src/gateway/config-reload.test.ts:651](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/gateway/config-reload.test.ts#L651)
  - command: `nl -ba src/gateway/config-reload.test.ts | sed -n '640,714p'`
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Plugin registry restore is in-memory loader state, not persisted artifact rollback:** There is a plugin registry snapshot/restore helper, but it restores in-memory registry arrays and handler maps during plugin loading rather than snapshotting and rolling back on-disk critical startup artifacts after failed self-modification.
  - file: [src/plugins/loader.ts:340](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/plugins/loader.ts#L340)

## Risks / Open Questions

- The existing last-known-good mechanism may already mitigate part of the reported pain for config-only failures, so any future scope decision should clarify whether the request is for broader artifact coverage beyond `openclaw.json`.
