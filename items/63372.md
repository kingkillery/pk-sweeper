## Summary

Current `main` does not expose a supported user-configurable dreaming model override. `extensions/memory-core/openclaw.plugin.json` still limits public dreaming config to `enabled`, `frequency`, `timezone`, `verboseLogging`, `storage`, and phase thresholds, and `src/plugins/loader.ts` enforces plugin config against that manifest schema. `src/memory-host-sdk/dreaming.ts` contains internal `execution.defaults.model` parsing, but that path is not part of the shipped `memory-core` config contract or docs, so the requested feature is not implemented for users on current `main`.

## Evidence

- **Public memory-core schema omits any dreaming model override:** `extensions/memory-core/openclaw.plugin.json:21` defines the plugin `configSchema`. The `dreaming` object at `extensions/memory-core/openclaw.plugin.json:25` includes `enabled`, `frequency`, `timezone`, `verboseLogging`, `storage`, and `phases`, but no `model` field and no `execution` block, with `additionalProperties: false` on the dreaming object and nested phase objects.
  - file: [extensions/memory-core/openclaw.plugin.json:21](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/extensions/memory-core/openclaw.plugin.json#L21)
  - command: `nl -ba extensions/memory-core/openclaw.plugin.json | sed -n '1,220p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Plugin loader rejects config outside the manifest schema:** `src/plugins/loader.ts:2382` and `src/plugins/loader.ts:3034` call `validatePluginConfig` with `manifestRecord.configSchema` and abort plugin loading on invalid config. That means a user-supplied `plugins.entries.memory-core.config.dreaming.model` or `dreaming.execution.defaults.model` is not a supported public config path unless the manifest schema is updated too.

## Risks / Open Questions

- The internal `src/memory-host-sdk/dreaming.ts` support for `execution.defaults.model` makes this easy to misread as already implemented, but the bundled `memory-core` manifest and docs still block it as a public feature.
