## Summary

Keep open. Current `main` has a stale-credential retry path for control-UI bootstrap, but the same recovery is still missing for avatar and `__openclaw__/assistant-media` HTTP reads, so the reported 401s remain plausible even when the WebSocket reconnect succeeds.

## Evidence

- **Auth helper documents stale-credential retry as a separate opt-in path:** `resolveControlUiAuthToken` still picks a single credential in priority order (`hello.auth.deviceToken`, then `settings.token`, then `password`). The same file says callers that need resilience to stale credentials must instead use `resolveControlUiAuthCandidates` and retry on 401.
