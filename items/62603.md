## Summary

Keep open. Current `main` still routes Windows host exec through `PowerShell -Command`, the exec tool schema has no `shell` selector or file-based PowerShell mode, and I found no regression test covering `$_`/pipeline-variable preservation. Without a Windows repro host I cannot prove the bug still happens, but there is not strong evidence to close it.

## Evidence

- **Windows host exec still uses PowerShell `-Command`:** `getShellConfig()` selects `resolvePowerShellPath()` on Windows and passes `-NoProfile -NonInteractive -Command`, so current main still sends host exec commands through inline PowerShell command text rather than a `.ps1` file path.
  - file: [src/agents/shell-utils.ts:54](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/agents/shell-utils.ts#L54)

  - file: [src/agents/bash-tools.exec-runtime.ts:669](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/agents/bash-tools.exec-runtime.ts#L669)
