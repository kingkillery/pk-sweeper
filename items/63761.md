## Summary

Current `main` still leaves HTTP 412 unclassified and does not recognize the reported Fireworks-style "account suspended due to spending limit" message as billing/auth-permanent, so the issue remains actionable rather than safe to close.

## Evidence

- **HTTP status classifier still has no 412 branch:** `classifyFailoverClassificationFromHttpStatus()` handles 402, 429, 401/403, 408, 410, 404, 503, 499, 500/502/504, 529, and 400/422, then returns `null`. There is no case for 412.
  - file: [src/agents/pi-embedded-helpers/errors.ts:591](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/agents/pi-embedded-helpers/errors.ts#L591)
  - command: `nl -ba src/agents/pi-embedded-helpers/errors.ts | sed -n '560,690p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Bare suspended/spending-limit text is not matched as billing:** `ERROR_PATTERNS.billing` matches 402/payment/credits/balance-style text, and `BILLING_ERROR_HEAD_RE` only admits billing/credit/payment/HTTP 402 prefixes. The reported Fireworks wording with `suspended` and `spending limit` is not covered here.
  - file: [src/agents/pi-embedded-helpers/failover-matches.ts:143](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/agents/pi-embedded-helpers/failover-matches.ts#L143)
  - command: `nl -ba src/agents/pi-embedded-helpers/failover-matches.ts | sed -n '1,250p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **When status classification is null, fallback depends on message classification:** `classifyFailoverSignal()` first tries the HTTP-status classifier, but if that returns null it only falls back to `messageClassification`. For a 412 response, the reported message therefore still needs to match a billing/auth classifier to trigger a reasoned failover.
  - file: [src/agents/pi-embedded-helpers/errors.ts:829](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/agents/pi-embedded-helpers/errors.ts#L829)

- **Current tests cover 402 monthly-spend cases, not 412 suspended-account cases:** Tests explicitly assert monthly spending-limit messages as billing under HTTP 402 or `402 Payment Required` wrappers, which matches recent 402 work but does not cover the reported 412/account-suspended path.
  - file: [src/agents/pi-embedded-helpers.isbillingerrormessage.test.ts:901](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/agents/pi-embedded-helpers.isbillingerrormessage.test.ts#L901)

Otherwise the decision path returns `surface_error`, which is consistent with the reporter's "bot simply stops responding" description for an unclassified 412 error.
