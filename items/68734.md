## Summary

Keep open. Current `main` still lacks hook-driven tool/skill narrowing, and the PR discussion on the latest revision includes unresolved correctness concerns, so there is no conservative basis to close this item.

## Evidence

- **Hook types on main do not expose dynamic tool/skill narrowing:** `PluginHookBeforePromptBuildEvent` only includes `prompt` and `messages`, and `PluginHookBeforePromptBuildResult` only includes prompt-mutation fields. There is no `availableTools`, `availableSkills`, `toolsAllow`, or `skillsAllow` on current `main`.
  - file: [src/plugins/hook-before-agent-start.types.ts:22](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/src/plugins/hook-before-agent-start.types.ts#L22)

  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)
- **Prompt-build helper does not propagate allowlists:** `resolvePromptBuildHookResult` calls `runBeforePromptBuild` with only `{ prompt, messages }` and returns only `systemPrompt`, `prependContext`, `prependSystemContext`, and `appendSystemContext`. Main therefore has no hook result path for dynamic tool or skill filtering.
  - file: [src/agents/pi-embedded-runner/run/attempt.prompt-helpers.ts:24](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/src/agents/pi-embedded-runner/run/attempt.prompt-helpers.ts#L24)
