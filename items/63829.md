## Summary

Keep open. Current main still treats `memory-wiki` as a single top-level plugin config, not a per-agent plugin config. `AgentConfig` has per-agent `memorySearch` but no per-agent `plugins` field, the loader passes one `pluginConfig` into plugin registration, `extensions/memory-wiki/index.ts` resolves that config once at register time, and wiki search/apply continue to use the shared `config.vault.path` rather than an agent-scoped vault.

## Evidence

- **Per-agent agent config has `memorySearch` but no `plugins` field:** `AgentConfig` includes `workspace`, `agentDir`, `skills`, `memorySearch`, `tools`, and `runtime`, but there is no per-agent `plugins` property that could carry `agents.list[].plugins.entries["memory-wiki"]` overrides.

  - file: [extensions/memory-wiki/README.md:17](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/extensions/memory-wiki/README.md#L17)
