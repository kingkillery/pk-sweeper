## Summary

Keep open. The report is recent and plausible, and I did not find strong evidence that current `main` already fixed or disproved the session-disappearance behavior across Control UI and TUI. Current code still depends on reconnect-triggered history reload paths in both surfaces, and the visible regression coverage is narrower than the issue being reported.

## Evidence

- **Control UI reconnect still clears active run state and rebuilds from reload/subscriptions:** On reconnect hello, the Control UI clears `chatRunId`, stream state, and tool stream state, then resubscribes and refreshes tabs. That matches the issue's reconnect-like failure mode and is not itself evidence of a shipped fix for disappearing sessions.

That is complex current behavior, not a clear proof the reported continuity problem is gone.
