## Summary

Keep open. Current main still builds heartbeat outbound session context with the base `sessionKey` even after isolated heartbeat runs set `runSessionKey` to `<base>:heartbeat`, so the PR's central one-line behavior change is not implemented on main. Existing tests cover the agent run `SessionKey` stability but do not cover the outbound delivery session context that this PR adds.

## Best Possible Solution

Keep the PR open and either merge or recreate the narrow fix on current main: pass `sessionKey: runSessionKey` to `buildOutboundSessionContext` in `src/infra/heartbeat-runner.ts`, keep the regression coverage for fresh base-key and wake re-entry cases, and clarify any separate explicit transcript-mirror behavior if needed.
