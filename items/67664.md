## Summary

Keep #67664 open. Current main still has the reported session-store behavior: unresolved context-window lookup is called with DEFAULT_CONTEXT_TOKENS and the result is always written back to the session entry. The PR is a focused fix for a real current code path and includes regression coverage for unresolved lookup, stale snapshot, and model-change cases, so there is no conservative close reason.

## Best Possible Solution

Keep the PR open and review it as a real bug fix. The best path is to rebase/scope-clean the branch if needed, then land the session-store change with regression tests that avoid persisting DEFAULT_CONTEXT_TOKENS while preserving resolved values unless the authoritative runtime model changed.
