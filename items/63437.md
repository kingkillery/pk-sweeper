## Summary

Keep open. This issue is too recent for stale closure, and current `main` does not provide strong evidence that the reported Feishu WebSocket failure is fixed or no longer reproducible. The current transport still just starts the SDK WebSocket client and waits for external abort, with no OpenClaw-side close/error/health handling for a silent post-first-message stall; the latest shipped release also carries the same transport code.

## Evidence

- **Current websocket monitor still lacks health/error handling:** `monitorWebSocket` logs startup, creates the SDK client, calls `wsClient.start({ eventDispatcher })`, and then only resolves on external abort. There is no OpenClaw-side listener for close/error/reconnect or any watchdog/health check that would prove the silent-stop failure is fixed.
