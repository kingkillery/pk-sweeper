## Summary

Keep PR #59535 open. The requested behavior is still not implemented on current main: parseBooleanValue lowercases the input but still builds custom truthy/falsy lookup sets directly from the caller-provided arrays, so uppercase custom entries remain case-sensitive. The PR discussion also shows useful review feedback that the contributor addressed in later commits, and the related #61316 cross-reference is another open PR rather than a landed fix.
