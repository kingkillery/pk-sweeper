## Summary

Current `main` already contains and ships the Anthropic replay safeguards this issue asks for: signed thinking blocks are preserved for Anthropic replay, unsafe sibling tool-call mutations are avoided by dropping the whole assistant turn instead of rewriting it, and regression tests cover both plain replay and immutable replay cases. The same implementation is present in release `v2026.4.22`.

## Evidence

- **Anthropic replay path enables provider-owned thinking preservation:** `shouldAllowProviderOwnedThinkingReplay` only turns on for Anthropic/Bedrock when signatures are preserved and thinking blocks are not dropped; `sanitizeSessionHistory` then skips tool-id sanitization on that path and passes `preserveSignatures: true` into session-history sanitization.
  - file: [src/agents/transcript-policy.ts:32](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/agents/transcript-policy.ts#L32)

  - file: [src/agents/pi-embedded-helpers/images.ts:126](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/agents/pi-embedded-helpers/images.ts#L126)
