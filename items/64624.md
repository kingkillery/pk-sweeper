## Summary

Keep open. Current `main` still injects WhatsApp connect/disconnect system events into the agent main session without any throttle, verbosity, or surface-aware filter, and the documented config only covers health-monitor restarts rather than notification suppression.

## Evidence

- **WhatsApp connect event is still injected unconditionally:** The WhatsApp monitor resolves a route and immediately enqueues `WhatsApp gateway connected...` to `connectRoute.sessionKey` on each successful connection.
  - file: [extensions/whatsapp/src/auto-reply/monitor.ts:388](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/whatsapp/src/auto-reply/monitor.ts#L388)
  - command: `nl -ba extensions/whatsapp/src/auto-reply/monitor.ts | sed -n '380,455p'`
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **WhatsApp disconnect event is still injected unconditionally:** On close, the same monitor enqueues `WhatsApp gateway disconnected...` to the same resolved session key before deciding whether to stop or reconnect.
  - file: [extensions/whatsapp/src/auto-reply/monitor.ts:445](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/whatsapp/src/auto-reply/monitor.ts#L445)
  - command: `nl -ba extensions/whatsapp/src/auto-reply/monitor.ts | sed -n '380,455p'`
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **Those events resolve to the main session when no peer is present:** `resolveAgentRoute` uses `buildAgentSessionKey(...)`, and `buildAgentPeerSessionKey(...)` falls back to `buildAgentMainSessionKey(...)` for direct routes without a peer id under the default `dmScope: "main"`. The WhatsApp connect/disconnect path passes only channel/account, not a peer, so the status event is not surface-aware.
  - file: [src/routing/session-key.ts:169](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/routing/session-key.ts#L169)
  - command: `nl -ba src/routing/session-key.ts | sed -n '130,173p'; nl -ba src/routing/resolve-route.ts | sed -n '623,680p'`
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **System-event queue has no throttle or duration gate:** `enqueueSystemEvent` only trims text, stores the session key/context, and skips exact consecutive duplicates. There is no built-in delay threshold, verbosity mode, or per-surface filtering here.
  - file: [src/infra/system-events.ts:90](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/infra/system-events.ts#L90)
