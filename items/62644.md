## Summary

Keep open. Current main still enforces `tools.sessions.visibility` defaulting to `tree`, and the exact `sessions_send` forbidden message reported in the issue is still present. The change is now documented in config/changelog, so the "silent" part is weaker than stated, but there is still no clear doctor/startup migration warning or cron-specific compatibility check for existing cross-agent isolated cron workflows.
