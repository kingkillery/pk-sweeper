## Summary

Keep open. Current main still has the file-based memory instructions in workspace templates, while memory-core's prompt section only emits recall guidance. The PR addresses a real core/template boundary issue for memory plugins and is not implemented, superseded, stale, or a ClawHub-only item.

## Best Possible Solution

Keep the PR open for maintainer review. The best product direction is to let memory-core own file-backed memory persistence prompt text through its `promptBuilder`, keep generic workspace templates plugin-agnostic, and decide whether to add a small regression test or follow-up doctor/docs guidance for existing workspaces.
