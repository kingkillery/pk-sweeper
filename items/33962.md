## Summary

Keep open. Current main only partially mitigates this family by letting slug generation inherit `agents.defaults.timeoutSeconds`; it still selects the default agent's effective primary model for slug generation and has no dedicated lightweight `slugModel`/`slugGenerator.model` config, no 3s heuristic fallback, and no debounce. The related discussion is useful: #34644/#66237 addressed timeout behavior, #65492 was an unmerged timeout-only PR, and #71709 remains a separate open slug-generator failure-classification issue.
