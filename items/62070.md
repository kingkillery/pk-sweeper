## Summary

Keep open. Current `main` still misses the image-sanitization early-return fix, and the OpenAI-compatible streaming path still finalizes immediately on lifecycle `error` events without emitting error content for that specific path. The PR remains materially useful, though its gateway hunk now overlaps with newer `main` changes and should be rebased/revalidated before merge.
