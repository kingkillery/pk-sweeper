## Summary

Keep open: current `main` at `cec3482` still limits `/usage` footers to token/cache/cost text plus optional session id, while provider quota windows like GitHub Copilot Premium are fetched for `/status`, not the per-reply footer.

## Evidence

- **Reply footer formatter only renders token usage, cache, and optional estimated cost:** `src/auto-reply/reply/agent-runner-usage-line.ts` builds the footer from `input`, `output`, `cacheRead`, `cacheWrite`, and optional cost, then returns `Usage: ...`; there is no provider quota or premium-percentage field in the formatter.
  - file: [src/auto-reply/reply/agent-runner-usage-line.ts:9](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/auto-reply/reply/agent-runner-usage-line.ts#L9)
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Reply execution path appends only the formatted usage line and optional session key:** `src/auto-reply/reply/agent-runner.ts` resolves `responseUsage`, calls `formatResponseUsageLine(...)`, and for `full` mode only appends ``· session `<sessionKey>` ``. This path does not call `loadProviderUsageSummary` or otherwise attach provider quota windows to reply footers.
  - file: [src/auto-reply/reply/agent-runner.ts:1449](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/auto-reply/reply/agent-runner.ts#L1449)
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Provider quota windows are still loaded for `/status`:** `src/status/status-text.ts` loads provider usage via `loadProviderUsageSummary(...)` and formats it as `📊 Usage: ...`, which matches the issue report that `/status` already shows premium usage while `/usage` does not.
  - file: [src/status/status-text.ts:192](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/status/status-text.ts#L192)

  - file: [docs/concepts/usage-tracking.md:23](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/docs/concepts/usage-tracking.md#L23)

That matches the current implementation rather than the requested feature.
