## Summary

Keep open. Current `main` hardens cron store loading so jobs should arrive with a `state` object before startup reads `job.state.runningAtMs`, which makes the exact crash in the body look obsolete. But the title’s claimed trigger path is still not clearly fixed: plugin schema compilation still calls AJV `compile()` without a local guard, and I did not find a regression test tying schema-compilation failure to gateway cron startup. That is not strong enough evidence for a conservative auto-close.
