## Summary

Keep open. Current `main` still has the channel-scope bugs this PR targets: `openclaw status`/`health` still use `scope: "channels"`, `resolveChannelPluginIds()` still includes disabled channel-capable plugins, empty `channels` scopes still widen to unscoped loads, and warm superset channel registries still satisfy narrower or empty channel scopes. The latest release `v2026.4.23` still ships the same runtime-loader behavior. The PR discussion also shows the author addressed earlier review findings with follow-up commits, so this remains useful active work rather than stale noise.
