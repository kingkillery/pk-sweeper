## Summary

Keep open. Current `main` shows Telegram inbound media is supposed to be saved before the path is handed to the agent, but the reviewed code/tests do not prove the reporter's immediate-read failure is obsolete. The path is still passed through as a host filesystem path unless sandbox staging rewrites it, and there is no end-to-end coverage here that verifies an agent can immediately open a Telegram audio file after dispatch.

## Evidence

- **Telegram handler awaits media resolution before dispatch:** `extensions/telegram/src/bot-handlers.runtime.ts:1111` awaits `resolveMedia(...)`, then at `extensions/telegram/src/bot-handlers.runtime.ts:1160` builds `allMedia` from `media.path` and enqueues the message. Intended contract: only dispatch after media save completes.
