## Summary

Keep open. Current `main` still emits the raw `Usage: /btw <side question>` string when `/btw` is sent without arguments, and the existing test suite codifies that exact output. Slack specifically now escapes unknown angle-bracket tokens, so the issue body's Slack example may be outdated, but the core BTW usage text is still unescaped and remains risky for markdown/HTML-capable channels that do not preserve literal `<...>` placeholders.
