## Summary

Keep open. Current `main` still strips LM Studio model refs at the first `@` in `src/plugins/providers.ts`, and the shared auth-profile parser on `main` still only preserves `@q*`/`@4bit`-style suffixes, not `@iq*`. The PR is a coherent, targeted fix for a real gap rather than something already implemented or safely auto-closable.

## Evidence

- **Current main still truncates any `@` suffix during provider ownership resolution:** `stripModelProfileSuffix()` trims the value, finds the first `@`, and slices everything after it off. `splitExplicitModelRef()` uses that helper before model-support matching, so `lmstudio/qwen3.6-27b@iq3_xxs` would be reduced to `qwen3.6-27b` on the current branch.
  - file: [src/plugins/providers.ts:292](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/src/plugins/providers.ts#L292)
