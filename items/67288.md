## Summary

Current `main` still lacks the reported `amazon-bedrock-mantle` discovery gate. The mantle plugin manifest exposes no `discovery.enabled` config, the runtime never reads `api.pluginConfig`, and `resolveImplicitMantleProvider` still falls through to IAM token generation without any disable check. The issue is recent and well-specified, and the discussion points to PR #67341 as the intended fix, but that change is not present in the checked-out `main` at `7875092f4ddc6f2dbe218bea8506c9ac1da87c39`.

## Evidence

- **Mantle manifest still lacks discovery config:** `extensions/amazon-bedrock-mantle/openclaw.plugin.json` on current `main` has `configSchema.properties` empty and no `uiHints` or schema entry for `discovery.enabled`, unlike the Bedrock plugin.
