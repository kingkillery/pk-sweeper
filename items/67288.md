## Summary

Keep #67288 open. Current main still lacks an `amazon-bedrock-mantle` `config.discovery.enabled` contract: the Mantle manifest rejects arbitrary config keys, registration does not read or pass `api.pluginConfig`, and implicit discovery still attempts IAM bearer-token generation whenever no explicit bearer token is present. The related PR #67341 is useful prior work and is referenced in the discussion as resolving the issue once merged, but it is open/unmerged in the provided GitHub context, so current main does not yet solve the bug.

## Best Possible Solution

Keep the issue open and land PR #67341 or an equivalent narrow fix on main: add a Mantle `discovery.enabled` config schema entry, read and pass plugin config from `register.sync.runtime.ts`, return null before IAM token generation when the gate is false, add disabled-discovery tests, and update Bedrock Mantle provider docs if the new user-facing config is shipped.
