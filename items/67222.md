## Summary

Keep open. Current `main` still has the mismatch described in the issue: the Discord security audit clears `channels.discord.commands.native.no_allowlists` only from legacy `channels.discord.dm.allowFrom` (plus store entries), while the remediation text, runtime command authorization, config schema, and docs all point users toward `channels.discord.allowFrom` as the preferred path. The same behavior is still present in the latest release `v2026.4.23`, so this is not implemented on main and not stale.
