## Summary

Keep open. The issue discussion already identified a targeted fix in PR #61132, but current `main` does not contain that guard or its regression tests. The embedded reply pipeline on `main` still streams and emits assistant text directly without tying exec-looking fenced shell replies to actual tool activity, so there is not strong evidence that #60955 is fixed.

## Evidence

- **Current streaming path still emits assistant text without an exec-intent/tool-activity guard:** `handleMessageUpdate()` parses assistant text and assigns `cleanedText = parsedFull.text`, then emits it through `assistant` / `onPartialReply`. There is no check here for exec-looking shell fences or for whether a real tool call happened in the turn.
