## Summary

Keep open. Current `main` still synthesizes a fallback/default account and always emits a channel status line when `listAccountIds()` is empty, so the reported phantom-status path is not implemented away. The PR discussion also surfaces real regressions in the proposed fix: suppressing `buildChannelSummary` or the status line on empty account lists would hide adapter-provided diagnostics for channels that intentionally report `[]` until credentials resolve.

## Evidence

- **Current main still takes the phantom-status path:** `buildChannelSummary` falls back from an empty `accountIds` array to `[defaultAccountId]`, builds `entries` for that synthetic account, calls `plugin.status.buildChannelSummary` when present, and unconditionally appends the rendered status line. There is no guard on `main` that suppresses the no-real-accounts case.
