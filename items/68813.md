## Summary

Keep open. Current `main` does not implement this PR's `providerConfigSchema` routing for `models.providers.<id>`. Main now documents Comfy under `plugins.entries.comfy.config`, which reduces the original docs mismatch, but the runtime still preserves legacy `models.providers.comfy` fallback and the PR thread itself ends with an acknowledged unresolved provider-ownership hazard that needs maintainer judgment.
