## Summary

Keep open. Current `main` still hardcodes zsh/fish profile locations and still checks bash completion in `~/.bashrc` while install can write to `~/.bash_profile`, so the reported behavior is still present and actionable.

## Evidence

- **Completion detection still ignores ZDOTDIR/XDG_CONFIG_HOME:** `getShellProfilePath()` on current `main` returns `~/.zshrc`, `~/.bashrc`, and `~/.config/fish/config.fish` directly, with no `ZDOTDIR` or `XDG_CONFIG_HOME` handling. `isCompletionInstalled()` and `usesSlowDynamicCompletion()` both rely on that helper, so detection still points at the wrong files for affected users.
  - file: [src/cli/completion-runtime.ts:128](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/cli/completion-runtime.ts#L128)
  - command: `nl -ba src/cli/completion-runtime.ts | sed -n '120,250p'`
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Install path still mismatches bash detection and hardcodes zsh/fish targets:** `installCompletion()` still writes zsh to `~/.zshrc`, fish to `~/.config/fish/config.fish`, and bash to `~/.bashrc` with a fallback to `~/.bash_profile`. That preserves the exact bash check/install mismatch described in the issue and still ignores `ZDOTDIR` and `XDG_CONFIG_HOME` during install.
  - file: [src/cli/completion-runtime.ts:194](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/cli/completion-runtime.ts#L194)
  - command: `nl -ba src/cli/completion-runtime.ts | sed -n '120,250p'`
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **User-facing setup and doctor flows still surface the same hardcoded paths:** The setup wizard reload hint still resolves zsh as `~/.zshrc`, bash as `~/.bashrc` or `~/.bash_profile` based only on file existence under `$HOME`, and fish as `~/.config/fish/config.fish`. The doctor command also tells users to `source ~/.zshrc`, `~/.bashrc`, or `~/.config/fish/config.fish`, so the wrong-path behavior is still reflected in current UX.
  - file: [src/wizard/setup.completion.ts:21](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/wizard/setup.completion.ts#L21)
