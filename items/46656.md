## Summary

Keep open. Current `main` still lacks the requested WebChat/Control UI inline-button path: the gateway chat schemas do not accept or emit button payloads, there is no `chat.callback` gateway method, the WebChat config surface has no `inlineButtons` capability, and the Control UI chat renderer only handles text/tool/attachment/canvas content.

## Evidence

- **Gateway chat schema has no buttons field and no callback RPC:** `ChatSendParamsSchema` only accepts `sessionKey`, `message`, routing metadata, attachments, timeout, provenance, and `idempotencyKey`. `ChatEventSchema` only carries `runId`, `sessionKey`, `seq`, `state`, `message`, error fields, usage, and `stopReason`. There is no `buttons` payload and no `chat.callback` schema here.
