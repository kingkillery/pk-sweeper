## Summary

Keep open. Current `main` still has the gap this PR addresses: inter-session `task_completion` wakes are sent through the gateway `agent` path without rebuilding the normal session-derived `extraSystemPrompt` context, and current tests do not lock that behavior in.

## Evidence

- **Gateway agent handler still forwards only caller-provided prompt text:** In current `main`, the gateway `agent` handler passes `request.extraSystemPrompt` straight through to `dispatchAgentRunFromGateway` and does not synthesize inbound/group prompt context from `sessionEntry`, `internalEvents`, or `inputProvenance` before dispatch.
