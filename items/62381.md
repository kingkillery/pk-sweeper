## Summary

Keep open. Current `main` still caches each subsystem's `fileLogger` in a closure and only creates it once, while the parent logger can rebuild onto a new date-rolled file path. That matches the reported stale-log-file behavior, and there is no landed regression test or code change on current `main` showing this was fixed.

## Evidence

- **Subsystem logger still caches the child file logger once:** `createSubsystemLogger()` keeps `fileLogger` in a closure, initializes it only when null, and reuses it for later writes. A long-lived subsystem logger therefore does not refresh its child logger after the parent logger changes.
  - file: [src/logging/subsystem.ts:327](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/logging/subsystem.ts#L327)

  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
