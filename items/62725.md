## Summary

Keep open. Current `main` has partial runtime support for agent-level typing settings, but `agents.list[]` still does not accept `typingMode` or `typingIntervalSeconds` in config validation, and the docs only describe defaults plus session overrides.

## Evidence

- **`agents.list[]` schema is strict and omits typing fields:** `AgentEntrySchema` for `agents.list[]` enumerates allowed per-agent keys, does not include `typingMode` or `typingIntervalSeconds`, and ends with `.strict()`, so unknown keys are rejected.
  - file: [src/config/zod-schema.agent-runtime.ts:812](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/config/zod-schema.agent-runtime.ts#L812)
  - command: `nl -ba src/config/zod-schema.agent-runtime.ts | sed -n '800,875p'`
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **Global defaults do support typing fields:** `AgentDefaultsSchema` includes both `typingIntervalSeconds` and `typingMode`, matching the issue report that global defaults work.
  - file: [src/config/zod-schema.agent-defaults.ts:249](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/config/zod-schema.agent-defaults.ts#L249)
  - command: `nl -ba src/config/zod-schema.agent-defaults.ts | sed -n '236,255p'`
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **Session overrides do support typing fields:** `SessionSchema` accepts `session.typingIntervalSeconds` and `session.typingMode`, matching the documented workaround in the issue.
  - file: [src/config/zod-schema.session.ts:54](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/config/zod-schema.session.ts#L54)
  - command: `nl -ba src/config/zod-schema.session.ts | sed -n '48,60p'`
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **Reply runtime already checks agent-level typing settings:** The reply path resolves typing mode from `sessionCfg?.typingMode ?? agentCfg?.typingMode`, and typing cadence from `agentCfg?.typingIntervalSeconds ?? sessionCfg?.typingIntervalSeconds`, so runtime code appears prepared for per-agent values once schema/config support exists.
  - file: [src/auto-reply/reply/get-reply-run.ts:308](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/auto-reply/reply/get-reply-run.ts#L308)

- **Current docs only document defaults and session overrides:** The config docs show typing settings under `agents.defaults` and mention only per-session overrides; they do not document `agents.list[].typingMode` or `agents.list[].typingIntervalSeconds`.
