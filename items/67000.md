## Summary

Keep open. Current `main` still creates a fresh active-memory sub-agent session for each recall pass, hands it to `runEmbeddedPiAgent`, and deletes the temporary transcript afterward unless transcript persistence is enabled. The documented config exposes identical-query result caching (`cacheTtlMs`) and query-size tuning, but nothing for warm standby, pre-initialization, or persistent embedded-session reuse. The issue is recent and the discussion includes multiple corroborating reports, so there is not strong evidence to close it conservatively.
