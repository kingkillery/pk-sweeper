## Summary

Keep open. Current `main` at `d3dd61f2c5bd6cea175544f3a76a68ec11fa17c2` does not implement this PR's core behavior: the reply pipeline still does not auto-switch the main agent run to `agents.defaults.imageModel.primary` when inbound images arrive on a text-only primary model. On `main`, `imageModel` is still wired into media-understanding/image-inspection paths instead.

## Evidence

- **Main reply model selection has no imageModel auto-switch:** `getReplyFromConfig` resolves the reply model from defaults, heartbeat override, stored session override, and channel override, then passes that provider/model into the reply runner. There is no `imageModel`-based branch in this function on current `main`.
  - file: [src/auto-reply/reply/get-reply.ts:193](https://github.com/openclaw/openclaw/blob/d3dd61f2c5bd6cea175544f3a76a68ec11fa17c2/src/auto-reply/reply/get-reply.ts#L193)
  - sha: [d3dd61f2c5bd](https://github.com/openclaw/openclaw/commit/d3dd61f2c5bd6cea175544f3a76a68ec11fa17c2)
- **Inbound media is handled as preprocessing, not a main-model swap:** `applyMediaUnderstandingIfNeeded` runs before the later session/channel override handling and only invokes media understanding; it does not replace the main reply model with `agents.defaults.imageModel.primary`.
  - file: [src/auto-reply/reply/get-reply.ts:131](https://github.com/openclaw/openclaw/blob/d3dd61f2c5bd6cea175544f3a76a68ec11fa17c2/src/auto-reply/reply/get-reply.ts#L131)

  - file: [src/media-understanding/runner.ts:503](https://github.com/openclaw/openclaw/blob/d3dd61f2c5bd6cea175544f3a76a68ec11fa17c2/src/media-understanding/runner.ts#L503)
