## Summary

Keep open. Current `main` at `56fe2aab9c87596515f2280c8060dc740ea64a88` does not already contain this schema-hardening work, so this PR is not `implemented_on_main`. It also still needs maintainer review because some proposed caps in the PR would narrow behavior below current runtime/back-end limits.

- The hardening idea appears useful, but the unresolved cap mismatches mean conservative maintainer cleanup should keep the item open for revision rather than close it.
