## Summary

Keep open. Current `main` still has the underlying bug path: primary CLI scoping only loads manifest-matched command owners, `wiki` is owned by `memory-wiki`, and wiki bridge status still computes `bridgePublicArtifactCount` from the active memory capability and warns when that count is zero. The proposed fix is not present on `main`, and the PR discussion includes an independent reproduction on OpenClaw 2026.4.15 plus related open duplicate PRs rather than a shipped replacement.

## Evidence
