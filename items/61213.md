## Summary

Keep open. Current `main` still surfaces the raw session key as the primary Control UI label, direct-session fallbacks still include the raw identifier, and the gateway display-name pipeline does not implement the requested `binding.comment -> agent.name -> technical identifier` priority.

## Evidence

- **Sessions table still foregrounds the raw session key:** The Control UI sessions view renders `row.key` as the main linked text and only shows `displayName` as a muted secondary line, so a Feishu/Lark `ou_xxx` key remains the primary visible identifier.
  - file: [ui/src/ui/views/sessions.ts:487](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/ui/src/ui/views/sessions.ts#L487)
  - command: `nl -ba ui/src/ui/views/sessions.ts | sed -n '450,525p'`
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Direct-session fallback still uses the raw identifier:** `parseSessionKey()` formats direct sessions as `<Channel> · <identifier>`, so when there is no better label the UI still falls back to the underlying DM identifier rather than a friendlier binding or agent name.
  - file: [ui/src/ui/chat/session-controls.ts:362](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/ui/src/ui/chat/session-controls.ts#L362)
  - command: `nl -ba ui/src/ui/chat/session-controls.ts | sed -n '340,420p'`
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)

There is no path here for `binding.comment` or `agent.name` to become the session display label.
