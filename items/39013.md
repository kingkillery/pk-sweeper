## Summary

Do not close. Current `main` still has the `/context list` naming bug for hook-injected bootstrap files that provide `path` without `name`, and the bootstrap cache still exposes a mutable per-session array to hooks, leaving a plausible duplicate-accumulation path. The session-memory part is less supported on current `main` because transcript indexing only ingests `user`/`assistant` message records, not system/bootstrap records, but the bundled issue still contains an unresolved maintainer-actionable bug.

## Evidence

- **`/context list` still renders the recorded bootstrap-file `name` directly:** The context report formats each injected workspace file as `- ${f.name}: ...` with no fallback to `path` or basename, so a hook-provided virtual file lacking `name` will still show as `undefined`.
  - file: [src/auto-reply/reply/commands-context-report.ts:124](https://github.com/openclaw/openclaw/blob/a7c1376b207e337cd7c3058b3a163fac1235fd7d/src/auto-reply/reply/commands-context-report.ts#L124)
  - sha: [a7c1376b207e](https://github.com/openclaw/openclaw/commit/a7c1376b207e337cd7c3058b3a163fac1235fd7d)
- **Bootstrap stats still preserve an undefined `name`:** `buildBootstrapInjectionStats()` copies `name: file.name` into the report and only falls back for `path`, not for `name`. That matches the issue’s `undefined` label symptom for hook objects shaped like `{ path, content, virtual: true }`.
  - file: [src/agents/bootstrap-budget.ts:145](https://github.com/openclaw/openclaw/blob/a7c1376b207e337cd7c3058b3a163fac1235fd7d/src/agents/bootstrap-budget.ts#L145)
  - sha: [a7c1376b207e](https://github.com/openclaw/openclaw/commit/a7c1376b207e337cd7c3058b3a163fac1235fd7d)
- **Hook overrides receive a mutable cached bootstrap array:** `getOrLoadBootstrapFiles()` caches and returns the same `WorkspaceBootstrapFile[]` per `sessionKey`, and `applyBootstrapHookOverrides()` passes that array straight into `event.context.bootstrapFiles`. A hook that does `bootstrapFiles.push(...)` can therefore mutate the cached snapshot and create duplicate injected files on later runs in the same session.
  - file: [src/agents/bootstrap-cache.ts:3](https://github.com/openclaw/openclaw/blob/a7c1376b207e337cd7c3058b3a163fac1235fd7d/src/agents/bootstrap-cache.ts#L3)
  - sha: [a7c1376b207e](https://github.com/openclaw/openclaw/commit/a7c1376b207e337cd7c3058b3a163fac1235fd7d)
- **Hook context uses the cached array directly:** The bootstrap hook context is created with `bootstrapFiles: params.files`, and the updated value is returned after hook execution. There is no defensive clone before hook code runs.
  - file: [src/agents/bootstrap-hooks.ts:19](https://github.com/openclaw/openclaw/blob/a7c1376b207e337cd7c3058b3a163fac1235fd7d/src/agents/bootstrap-hooks.ts#L19)
  - sha: [a7c1376b207e](https://github.com/openclaw/openclaw/commit/a7c1376b207e337cd7c3058b3a163fac1235fd7d)
- **Current session-memory ingestion does not index bootstrap/system records directly:** `buildSessionEntry()` skips all non-`message` JSONL records and only keeps `message.role === "user"` or `"assistant"`. That weakens the specific claim that Project Context itself is being indexed as transcript history on current `main`.
  - file: [src/memory-host-sdk/host/session-files.ts:352](https://github.com/openclaw/openclaw/blob/a7c1376b207e337cd7c3058b3a163fac1235fd7d/src/memory-host-sdk/host/session-files.ts#L352)
  - sha: [a7c1376b207e](https://github.com/openclaw/openclaw/commit/a7c1376b207e337cd7c3058b3a163fac1235fd7d)
- **Session rollover now clears the bootstrap snapshot:** `initSessionState()` calls `clearBootstrapSnapshotOnSessionRollover(...)` before reusing or rotating sessions, so the reporter’s cross-`/new` accumulation claim is not strong enough to close as already-fixed, but it also is not enough to dismiss the still-open naming bug.
  - file: [src/auto-reply/reply/session.ts:491](https://github.com/openclaw/openclaw/blob/a7c1376b207e337cd7c3058b3a163fac1235fd7d/src/auto-reply/reply/session.ts#L491)
  - sha: [a7c1376b207e](https://github.com/openclaw/openclaw/commit/a7c1376b207e337cd7c3058b3a163fac1235fd7d)
