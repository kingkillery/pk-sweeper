## Summary

Keep open. The report is recent, unprotected but not stale, and the current documented behavior still supports the core complaint: OpenClaw has prompt-size reduction knobs, but the default context injection mode is still full bootstrap reinjection rather than the requested lazy/lightweight default. A related open issue and open PR are actively tracking this same root cause, so this is not high-confidence closable as implemented.

## Evidence

- **Issue metadata:** #59427 was opened 2026-04-02 by a non-maintainer, has no protected labels, no comments, and is only 23 days old as of 2026-04-25. It is not eligible for stale_insufficient_info, and there is no maintainer-authored/protected-label reason to force-close or force-keep beyond normal evidence review.
- **Current token-use docs:** The public token-use docs say OpenClaw assembles its own system prompt on every run and includes tool list, skills metadata, workspace/bootstrap files, time, reply tags, and runtime metadata. That matches the class of baseline prompt/context overhead described by the issue.
