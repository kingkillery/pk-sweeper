## Summary

Current `main` no longer routes `codex-cli` through helper/embedded model resolution. `codex-cli` is handled as a CLI backend, while embedded/native Codex flows now use the separate `codex` provider and harness, so this PR's normalization target is obsolete against today's architecture.
