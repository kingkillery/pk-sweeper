## Summary

Keep open. Current `main` still suppresses tool failure reasons for normal chat payloads unless verbose mode is on, and successful retries only clear `lastToolError` state without emitting any follow-up success message.

## Evidence

- **Failure reasons are gated behind verbose mode:** `shouldIncludeToolErrorDetails()` only returns true for `verboseLevel === "on" | "full"`, plus a narrow exec-timeout cron case, so ordinary chat payloads still omit the tool error reason by default.
  - file: [src/agents/pi-embedded-runner/run/payloads.ts:68](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/agents/pi-embedded-runner/run/payloads.ts#L68)
  - command: `nl -ba src/agents/pi-embedded-runner/run/payloads.ts | sed -n '50,120p'`
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **User-facing warning still formats as generic 'failed' without reason by default:** The emitted warning is `⚠️ ${toolSummary} failed${errorSuffix}`, and `errorSuffix` is only appended when `warningPolicy.includeDetails` is true.
  - file: [src/agents/pi-embedded-runner/run/payloads.ts:339](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/agents/pi-embedded-runner/run/payloads.ts#L339)
  - command: `nl -ba src/agents/pi-embedded-runner/run/payloads.ts | sed -n '330,380p'`
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **Current tests explicitly expect mutating tool failures to hide the reason at normal verbosity:** `keeps non-exec mutating tool failures visible` asserts `absentDetail: "permission denied"` for a `write` tool failure with `verboseLevel: "off"`, which matches the issue's reported bad behavior.
  - file: [src/agents/pi-embedded-runner/run/payloads.test.ts:132](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/agents/pi-embedded-runner/run/payloads.test.ts#L132)
  - command: `nl -ba src/agents/pi-embedded-runner/run/payloads.test.ts | sed -n '128,175p'`
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **Retry success is silent:** When the same mutating action later succeeds, `handleToolExecutionEnd()` only clears `ctx.state.lastToolError = undefined`; there is no code here to enqueue or emit a success/retry-confirmation chat message.
