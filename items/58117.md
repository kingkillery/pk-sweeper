## Summary

Keep open. Current `main` does not implement Mistral/Voxtral as a TTS speech provider. The bundled Mistral plugin only wires model, embedding, media-understanding, and realtime transcription support. The issue's silent-fallback complaint is at least partly outdated on current `main`, because unsupported TTS providers now produce an explicit `no provider registered` failure.

## Evidence

- **Mistral plugin does not register a speech provider:** The bundled Mistral plugin registers memory embedding, media understanding, and realtime transcription only. There is no `api.registerSpeechProvider(...)` call.
  - file: [extensions/mistral/index.ts:56](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/extensions/mistral/index.ts#L56)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Mistral manifest has no speechProviders contract:** The Mistral plugin manifest declares `memoryEmbeddingProviders`, `mediaUnderstandingProviders`, and `realtimeTranscriptionProviders`, but no `speechProviders` entry.
  - file: [extensions/mistral/openclaw.plugin.json:23](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/extensions/mistral/openclaw.plugin.json#L23)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Current TTS docs do not list Mistral as supported:** The user-facing TTS docs say OpenClaw supports ElevenLabs, Google Gemini, Microsoft, MiniMax, OpenAI, and xAI for TTS; Mistral is absent from both the overview sentence and supported-services list.
  - file: [docs/tools/tts.md:10](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/docs/tools/tts.md#L10)

  - file: [docs/providers/mistral.md:10](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/docs/providers/mistral.md#L10)
