## Summary

Keep open. Current `main` still implements Control UI chat uploads as image-only on both the UI and gateway paths, so generic file upload support is not implemented despite the issue comment pointing to PR #57707.

## Evidence

- **Control UI accept filter is image-only:** `CHAT_ATTACHMENT_ACCEPT` is still `"image/*"`, and the support check only returns true for MIME types starting with `image/`.
  - file: [ui/src/ui/chat/attachment-support.ts:1](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/ui/src/ui/chat/attachment-support.ts#L1)
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Chat file picker and drag/drop skip non-image files:** The chat input uses `accept=${CHAT_ATTACHMENT_ACCEPT}` and both `handleFileSelect` and `handleDrop` immediately continue past files whose MIME type is not supported by the image-only helper.
  - file: [ui/src/ui/views/chat.ts:250](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/ui/src/ui/views/chat.ts#L250)
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Outgoing chat attachments are serialized only as images:** `buildApiAttachments()` maps every attachment to `{ type: "image", mimeType, content }`, so the Control UI send path does not emit generic file/document attachment types.
  - file: [ui/src/ui/controllers/chat.ts:213](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/ui/src/ui/controllers/chat.ts#L213)
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Gateway chat attachment parsing still rejects or drops non-images:** `normalizeAttachment()` throws `only image/* supported` when image MIME is required, and `parseMessageWithAttachments()` drops attachments whose sniffed or provided MIME type is not an image.
  - file: [src/gateway/chat-attachments.ts:229](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/gateway/chat-attachments.ts#L229)
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **No evidence on current main that PR #57707 shipped this feature:** Read-only inspection of current `main` shows the feature is absent in the live code paths above, so the issue cannot be conservatively closed as implemented.
  - command: `git ls-remote https://github.com/openclaw/openclaw.git 'refs/pull/57707/*'`
