## Summary

Keep open. Current `main` still requires both operator and node connectivity before Android onboarding can finish, while the app elsewhere explicitly treats `Connected (operator offline)` as a valid connected state. The PR's proposed behavior still addresses a live mismatch in current code, and the provided discussion shows no substantive review objection beyond a small cleanup note.

## Evidence

- **Current finish gate still blocks offline-operator case:** `OnboardingFlow` on current `main` computes `canFinishOnboarding` from both `isConnected` and `isNodeConnected`, and the helper still returns `isConnected && isNodeConnected`. That means a node-connected/operator-offline state still cannot finish onboarding.
