## Summary

Keep open. Current main still has single-attempt CDP WebSocket handling: `withCdpSocket` opens once and rejects on open/close errors, so the PR targets a real missing remote-CDP resilience path. The PR should not be closed as implemented or obsolete, but it needs changes because the proposed retry loop can replay non-idempotent CDP callbacks such as `Target.createTarget` and `Runtime.evaluate`.

## Best Possible Solution

Keep the PR open and request a narrower, safer implementation: retry only connection/open or explicitly idempotent request phases, preserve close code/reason before classifying retryability, clamp jitter after applying randomization, and add regression tests for recovery, retry exhaustion, non-replay of `Target.createTarget`/`Runtime.evaluate`, and non-retryable close/rate-limit cases.
