## Summary

Keep open. Current `main` has partial session-specific bootstrap reductions for heartbeats, cron runs, and subagents, but it does not implement the requested user-configurable per-file/session-type bootstrap loading or file-level frontmatter control.

## Evidence

- **No bootstrap-per-session config surface:** `AgentDefaultsConfig` exposes only global bootstrap injection mode and size caps here; there is no config for per-file session routing such as `bootstrapFiles.<name>.sessions`.
  - file: [src/config/types.agent-defaults.ts:229](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/config/types.agent-defaults.ts#L229)
  - command: `nl -ba src/config/types.agent-defaults.ts | sed -n '224,240p'`
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Heartbeat lightweight mode is a narrow special case:** Bootstrap filtering has a dedicated lightweight branch that keeps only `HEARTBEAT.md` for heartbeat runs; it is not a general per-file/session-type mechanism.
  - file: [src/agents/bootstrap-files.ts:165](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/agents/bootstrap-files.ts#L165)
  - command: `nl -ba src/agents/bootstrap-files.ts | sed -n '165,185p'`
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Subagent/cron filtering is hardcoded:** Session-based filtering exists, but only as a fixed allowlist for subagent and cron session keys. The allowlist is compiled into code, not configurable from bootstrap file metadata or config.
  - file: [src/agents/workspace.ts:544](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/agents/workspace.ts#L544)

  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Tests confirm blanket-load for normal chats and fixed filtering for subagent/cron:** The current tests expect all bootstrap files for a normal chat session and only the hardcoded reduced set for subagent/cron sessions.
  - file: [src/agents/workspace.test.ts:295](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/agents/workspace.test.ts#L295)

  - file: [docs/reference/token-use.md:24](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/docs/reference/token-use.md#L24)

## Risks / Open Questions

- Main already includes partial token-saving behavior (`heartbeat.lightContext`, `heartbeat.isolatedSession`, and hardcoded subagent/cron filtering), so closing as implemented would overstate what shipped.
- There is a documentation/behavior mismatch around subagent bootstrap contents (`docs/tools/subagents.md` says only `AGENTS.md` + `TOOLS.md`, while `src/agents/workspace.ts` and its tests allow `SOUL.md`, `IDENTITY.md`, and `USER.md` too), which makes the current behavior less clear to audit.
