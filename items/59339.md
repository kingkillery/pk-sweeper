## Summary

The issue reports a plausible current feature/behavior gap in gateway A2A session delivery: announcement timeout handling still appears fixed at 30 seconds on current main, with no configurable announce timeout exposed through the session send tool schema or gateway config. The item is recent, externally authored, unprotected by labels, and contains concrete reproduction details, so it should remain open for maintainer/product judgment rather than be closed by cleanup.
