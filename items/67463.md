## Summary

Keep #67463 open. Current main has already implemented the task timestamp normalization portion, but the doctor SecretRef false-positive path still exists: the gateway auth health check detects a SecretRef and warns without using the resolver that can prove the token is available. The PR is not cleanly closeable as implemented or duplicate because it still addresses a real remaining bug, though it should be revised to drop the now-shipped task work and address the review feedback about gating SecretRef resolution and preserving lazy doctor imports.

## Best Possible Solution

Keep the PR open but ask for a focused rebase: drop the task timestamp changes now covered by `de5b173546d4405bdc0971222c2dc85ae9e9c5b0`, keep the doctor SecretRef fix, resolve the review feedback by only resolving the gateway token SecretRef after token auth is actually needed, preserve the dynamic doctor loading boundary, and add a regression test showing a resolvable `gateway.auth.token` SecretRef suppresses the Gateway auth warning. Link the remaining work to the existing SecretRef false-positive issue #65201 if maintainers want one canonical tracker.
