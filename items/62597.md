## Summary

Keep open. Current `main` still implements `hasIndexedContent()` as uncached SQLite probes on the hot `search()` path, so this PR is not already implemented. The PR discussion also shows the change is narrowly scoped and technically useful, but still needs dedicated cache-behavior tests before a conservative maintainer cleanup would close it.

## Evidence

- **Current main still does uncached `hasIndexedContent()` queries:** On `main`, `hasIndexedContent()` directly runs `SELECT 1 FROM chunks LIMIT 1` and, when FTS is enabled, `SELECT 1 FROM ${FTS_TABLE} LIMIT 1`, then returns the result. There is no cached field or memoized fast path in this method.
