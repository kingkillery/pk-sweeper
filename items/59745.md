## Summary

Keep PR #59745 open. Current main still implements the reported hot-mode behavior: restart-required config changes in `gateway.reload.mode = "hot"` are logged and ignored rather than falling back to restart, and there is no `fallbackToRestart` config/type/schema support on main. The PR discussion is useful: Greptile reviewed the approach positively, and the listed review blocker about the strict schema appears addressed by the PR's second commit, so this remains plausible useful work rather than cleanup-close material.
