## Summary

Keep open. Current `main` still truncates the sorted child-directory scan by `maxSkillsLoadedPerSource` before checking whether each directory actually contains `SKILL.md`, so the reported skills-discovery bug is still present. The PR also remains a mixed bundle with unrelated Telegram/TUI/dependency changes, so there is no strong basis for an auto-close.
