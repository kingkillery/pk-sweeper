## Summary

Keep open. Current main has partial coverage for the forced-reinstall stale-service-env path, but the item is not fully resolved: `channels list` still performs provider usage fetching after listing auth profiles and logs fetch failures, and the macOS LaunchAgent path still persists durable environment values inline in the plist.

## Best Possible Solution

Keep this open until maintainers either split it or fix both surfaces. The channels path should skip provider usage fetching when there are no relevant configured chat channels, or make usage fetch failures quiet/non-fatal for `channels list` while preserving an explicit `--usage`/diagnostic path. The LaunchAgent portion should be narrowed against #53742/#70752 or fixed by ensuring `gateway install --force` removes embedded `OPENCLAW_GATEWAY_TOKEN` across stale-service and durable-env sources.
