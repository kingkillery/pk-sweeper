## Summary

Keep open. Current `main` still does not repair missing global npm bin links after `openclaw update`, so the PR addresses a real gap. The review thread also surfaced follow-up correctness concerns in the proposed repair logic, so there is not strong evidence to close this conservatively.

## Evidence

- **Global npm update path has no bin-link repair:** `runGatewayUpdate` detects a global package-manager install, runs the global install, then re-resolves `packageRoot` and verifies the installed package. There is no call that reads `package.json.bin` or repairs a missing `<prefix>/bin/openclaw` link.
  - file: [src/infra/update-runner.ts:1285](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/infra/update-runner.ts#L1285)
  - command: `nl -ba src/infra/update-runner.ts | sed -n '1284,1368p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Post-install verification only checks package contents:** `collectInstalledGlobalPackageErrors` only validates the installed package version and packaged dist inventory under `packageRoot`. It does not inspect the global bin directory or executable symlink state, so a missing global command would not be repaired or even detected here.
  - file: [src/infra/update-global.ts:92](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/infra/update-global.ts#L92)

- **Main branch lacks the PR's bin-manifest helper:** `src/infra/package-json.ts` on `main` only exposes `readPackageVersion` and `readPackageName`. The PR's added `readPackageBin` helper is not present on current `main`.
  - file: [src/infra/package-json.ts:1](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/infra/package-json.ts#L1)
