## Summary

Keep open. Current main has `agents.defaults.compaction.notifyUser`, but it only emits brief notices when compaction itself starts and ends. The issue asks for a user-visible prompt before the silent pre-compaction memory flush so the user can add what matters before compaction runs; that behavior is not implemented on current `main`.

## Evidence

- **Compaction notices are only start/end status messages:** `runAgentTurnWithFallback` gates user-facing compaction notices behind `agents.defaults.compaction.notifyUser` and sends fixed messages `🧹 Compacting context...`, `🧹 Compaction complete`, or `🧹 Compaction incomplete` on compaction lifecycle events.

## Risks / Open Questions

- A maintainer could consider existing `notifyUser` close enough to the request at a high level, but it does not satisfy the issue's key requirement: notifying the human before the silent memory flush and giving them a chance to respond.
- No reproduction run was needed because this is a feature-gap review against current code and docs, and the relevant control flow is directly visible in `main`.
