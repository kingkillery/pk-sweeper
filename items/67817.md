## Summary

Keep #67817 open. Current main still does not copy an env-backed Discord channel token SecretRef into the managed gateway service environment. The daemon install plan only merges durable env sources and auth-profile env refs, while Discord setup/docs still support storing `channels.discord.token` as an env SecretRef. The related fix PR #67881 was closed unmerged, so it is not a shipped canonical fix.

## Best Possible Solution

Implement a narrow daemon install bridge that detects a safe, resolvable env-backed `channels.discord.token` default-account SecretRef and adds that env var to the managed service environment, with focused coverage in `src/commands/daemon-install-helpers.test.ts`. If maintainers prefer not to persist channel SecretRef env values, update the Discord onboarding/docs flow to clearly require `DISCORD_BOT_TOKEN` in `~/.openclaw/.env` or the service environment before daemon install/restart.
