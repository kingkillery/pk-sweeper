## Summary

Keep open. Current main still has the Feishu startup/status probe behavior this PR is trying to improve, but PR #67466 as submitted has unresolved lifecycle regressions called out in review: detaching the `accountId` path would make the gateway treat the Feishu account task as exited and restart it repeatedly.

## Best Possible Solution

Keep #67466 open for maintainer/author revision. Preserve the `accountId` path as a long-lived `monitorSingleAccount()` promise, limit any fire-and-forget behavior to safe non-gateway multi-account startup if still desired, keep account-specific error logging, and add focused regression coverage for the gateway lifecycle plus the status probe timeout. Coordinate with #63475/#63481 so the final Feishu startup fix is one coherent implementation rather than parallel conflicting PRs.
