## Summary

Keep open. Current main supports node command declaration only as string command names; it does not provide a node command schema declaration mechanism for input/output schemas, gateway validation, or API discovery. This is a core gateway/node protocol gap, not a ClawHub-only plugin cleanup item.

## Evidence

- **Connect protocol only has string command names:** The WebSocket connect schema exposes `commands` as an optional array of non-empty strings and rejects extra top-level connect fields, so a node cannot declare per-command input/output schemas through the current handshake.
  - file: [src/gateway/protocol/schema/frames.ts:37](https://github.com/openclaw/openclaw/blob/bd796d1c85c7dec0dcb89ed11502e4271ded1314/src/gateway/protocol/schema/frames.ts#L37)
  - command: `nl -ba src/gateway/protocol/schema/frames.ts | sed -n '1,220p'`
  - sha: [bd796d1c85c7](https://github.com/openclaw/openclaw/commit/bd796d1c85c7dec0dcb89ed11502e4271ded1314)
- **Pairing surface stores commands, not schemas:** `NodeDeclaredSurface` and the node pairing request model contain `commands?: string[]`; pending and approved pairing preserve only that string list.
  - file: [src/infra/node-pairing.ts:16](https://github.com/openclaw/openclaw/blob/bd796d1c85c7dec0dcb89ed11502e4271ded1314/src/infra/node-pairing.ts#L16)
  - command: `nl -ba src/infra/node-pairing.ts | sed -n '1,380p'`
  - sha: [bd796d1c85c7](https://github.com/openclaw/openclaw/commit/bd796d1c85c7dec0dcb89ed11502e4271ded1314)
- **Runtime registry routes by declared command name only:** Connected node sessions store `commands: string[]`; `node.invoke` forwards arbitrary `params` after command allowlist/declaration checks, without schema lookup or validation against node-declared input/output schemas.
  - file: [src/gateway/node-registry.ts:18](https://github.com/openclaw/openclaw/blob/bd796d1c85c7dec0dcb89ed11502e4271ded1314/src/gateway/node-registry.ts#L18)

  - sha: [bd796d1c85c7](https://github.com/openclaw/openclaw/commit/bd796d1c85c7dec0dcb89ed11502e4271ded1314)
- **Gateway invoke validation remains schema-light:** `NodeInvokeParamsSchema` allows `params` as `Type.Unknown()`, which is consistent with no per-command input schema validation at the gateway boundary.
  - file: [src/gateway/protocol/schema/nodes.ts:66](https://github.com/openclaw/openclaw/blob/bd796d1c85c7dec0dcb89ed11502e4271ded1314/src/gateway/protocol/schema/nodes.ts#L66)
