## Summary

Current `main` does not implement a TUI startup `--agent` flag. The TUI can switch agents after launch (`/agent`, picker), infer an agent from CWD, or honor a fully qualified agent-scoped session key, but the specific request in #8892 is still not implemented.

## Evidence

- **TUI CLI has no `--agent` option:** `registerTuiCli()` defines `--local`, `--url`, `--token`, `--password`, `--session`, `--deliver`, `--thinking`, `--message`, `--timeout-ms`, and `--history-limit`; there is no `--agent` flag, and the action passes only `session` through to `runTui()`.
  - file: [src/cli/tui-cli.ts:10](https://github.com/openclaw/openclaw/blob/c0a7b6a510758645c5267734b984f4885a605b67/src/cli/tui-cli.ts#L10)

  - sha: [c0a7b6a51075](https://github.com/openclaw/openclaw/commit/c0a7b6a510758645c5267734b984f4885a605b67)
- **User docs still expose agent selection only inside the TUI:** The TUI docs describe agent switching via the agent picker and `/agent <id>`, and the options list still omits any startup `--agent` flag.
  - file: [docs/web/tui.md:81](https://github.com/openclaw/openclaw/blob/c0a7b6a510758645c5267734b984f4885a605b67/docs/web/tui.md#L81)
