## Summary

Keep #67539 open. Current main has Google/Gemini TTS support and docs for Gemini expressive square-bracket tags, but the shared speech provider contract still has no provider-owned prompt-hint hook and `speech-core` only injects the generic TTS runtime hint. The requested core/plugin API work remains valid.

## Best Possible Solution

Add a backward-compatible optional speech-provider prompt hint hook, have `speech-core` append only the active provider's short hint from `buildTtsSystemPromptHint`, pass resolved provider config/model override context, and add tests for active-provider-only injection plus off-policy controls. Then implement the initial Gemini hint in the Google speech provider and update plugin SDK docs.

## Risks / Open Questions

- Adding this correctly touches the public plugin contract, so it should include SDK docs/export-baseline coverage and focused speech-core/provider tests.
- Provider hints must be derived from the resolved model, voice, target, provider config, and `modelOverrides` policy so agents do not advertise disabled or wrong-provider controls.
