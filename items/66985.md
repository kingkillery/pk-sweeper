## Summary

Current `main` still contains the raw `boundNode !== requestedNode` guard in `executeNodeHostCommand`, while the config and node-resolution code explicitly allow node names, IPs, and ID prefixes. That means the PR is still addressing a real bug on `main`, not an already-implemented or obsolete issue.
