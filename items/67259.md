## Summary

Keep open. Current `main` still suppresses empty assistant bubbles without checking `opts.isStreaming`, which matches the reported Control UI regression when tool calls are hidden. I did not find a replacement fix or a regression test covering `showToolCalls: false` while streaming.

## Evidence

- **Current main still has the unfixed guard:** `renderGroupedMessage` on current `main` returns `nothing` when there is no markdown, no visible tool cards, no images, no assistant attachments/view blocks, and no reply target. The guard does not exempt streaming messages, and the proposed PR change (`!opts.isStreaming &&`) is not present.

Because the empty-bubble guard runs afterward without checking `opts.isStreaming`, the live bubble can still be suppressed before any text arrives.
