## Summary

Keep open. Current `main` still does not implement `appliedTags` forwarding for Discord `channel-edit`, and the supplied review discussion points to a real empty-array edge case rather than an obsolete or incoherent change request.

## Evidence

- **Current send layer does not forward `applied_tags` on channel edits:** `editChannelDiscord` builds the PATCH body with `name`, `topic`, `position`, `parent_id`, `nsfw`, `rate_limit_per_user`, `archived`, `locked`, `auto_archive_duration`, and `available_tags`, but no `applied_tags` field.
  - file: [extensions/discord/src/send.channels.ts:40](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/discord/src/send.channels.ts#L40)
  - command: `nl -ba extensions/discord/src/send.channels.ts | sed -n '34,90p'`
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **Shared channel-edit parser on `main` has no `appliedTags` input:** `readDiscordChannelEditParams` returns `availableTags` but does not read or expose any `appliedTags`/`applied_tags` value, so both action paths miss the feature before the request reaches the send layer.
  - file: [extensions/discord/src/actions/runtime.shared.ts:42](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/discord/src/actions/runtime.shared.ts#L42)
  - command: `nl -ba extensions/discord/src/actions/runtime.shared.ts | sed -n '35,70p'`
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **Both Discord action entrypoints depend on the shared parser:** The guild-admin `channel-edit` handler spreads `readDiscordChannelEditParams({ ...actionParams, channelId })`, and the guild runtime `channelEdit` case passes `readDiscordChannelEditParams(params)` directly into `editChannelDiscord`, so neither path can currently forward forum post tags.
  - file: [extensions/discord/src/actions/handle-action.guild-admin.ts:170](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/discord/src/actions/handle-action.guild-admin.ts#L170)
  - command: `nl -ba extensions/discord/src/actions/handle-action.guild-admin.ts | sed -n '165,185p'; nl -ba extensions/discord/src/actions/runtime.guild.ts | sed -n '332,348p'`
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **Current type contract lacks the requested field:** `DiscordChannelEdit` on `main` has no `appliedTags?: string[]` member, which matches the missing parser and send-layer support.
  - file: [extensions/discord/src/send.types.ts:163](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/discord/src/send.types.ts#L163)
