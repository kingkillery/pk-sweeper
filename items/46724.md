## Summary

Keep open. Current `main` still models the Gateway as a single control plane with only `operator` and `node` roles, and the docs for running multiple gateways describe isolated profiles/ports rather than any gateway federation or peer-routing surface. I found no implemented `gateway` role, `gw.send`-style RPC, or federation/interconnection feature on `main`.

## Evidence
