## Summary

Keep open. Current `main` does not implement Feishu account-level TTS overrides, and the PR discussion still contains a plausible unresolved TTS gating bug that is consistent with the current code paths.

## Evidence

- **Feishu account schema on main has no `tts` override field:** `FeishuAccountConfigSchema` includes account credentials and shared channel settings, but no `tts` property. A repo grep for `tts` in the Feishu config schema files returns no matches, so the proposed `channels.feishu.accounts.<accountId>.tts` surface is not present on current `main`.
  - file: [extensions/feishu/src/config-schema.ts:191](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/extensions/feishu/src/config-schema.ts#L191)
  - command: `nl -ba extensions/feishu/src/config-schema.ts | sed -n '180,225p'; rg -n "tts" extensions/feishu/src/config-schema.ts extensions/feishu/src/config-schema.test.ts`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **No account-aware TTS resolver exists on main:** Current runtime exports include `resolveTtsConfig` only. `src/tts/tts.ts`, `extensions/speech-core/runtime-api.ts`, and `src/plugin-sdk/tts-runtime.ts` do not export `resolveTtsConfigForAccount`, and `src/plugin-sdk/tts-runtime.types.ts` has no account-aware resolver or `accountId` on `MaybeApplyTtsToPayloadParams`.
  - file: [src/tts/tts.ts:1](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/tts/tts.ts#L1)
