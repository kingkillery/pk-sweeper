## Summary

Keep open. Current main still combines tagged Gemini reasoning prompts with native Gemini thinkingConfig, which is the specific conflict reported in #69220. The merged #71362/#71074 work adds a retry mitigation for Gemini empty/reasoning-only/planning-only turns, but it does not remove the tagged/native conflict or the token-waste root cause described by multiple affected users.

## Best Possible Solution

Keep #69220 open and address it in the bundled Google provider/core reasoning contract: make Gemini reasoning output native when native `thinkingConfig` is used, or otherwise conditionally avoid injecting the `<think>/<final>` tagged directive for Gemini. Update `provider-utils`, the `google-gemini` replay-family hook/tests/docs, and add a regression that proves Gemini native thinking plus a tool pivot does not produce text-tag pollution while preserving the #71362 retry guard as a fallback.
