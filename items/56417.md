## Summary

Current `main` already implements the missing routing for configured vision-capable providers and has regression coverage for the Moonshot/MiniMax cases described in #56417.

## Evidence

- **Config providers now auto-register for image media-understanding:** `buildMediaUnderstandingRegistry()` adds fallback media-understanding providers for any `models.providers.*` entry that declares an image-capable model, which is the direct fix for the original `No media-understanding provider registered` failure on configured providers.
  - file: [src/media-understanding/provider-registry.ts:30](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/media-understanding/provider-registry.ts#L30)
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Regression test covers configured image-capable config providers:** `src/media-understanding/provider-registry.test.ts:88` asserts that a config-only provider with `input: ["text", "image"]` is auto-registered for media understanding, while a text-only provider is not.
  - file: [src/media-understanding/provider-registry.test.ts:88](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/media-understanding/provider-registry.test.ts#L88)
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Image tool no longer requires a provider-specific registration:** `runImagePrompt()` looks up a provider-specific media provider when available, but otherwise falls back to the generic `describeImageWithModel` / `describeImagesWithModel` runtime for the configured `provider/model` pair.
  - file: [src/agents/tools/image-tool.ts:193](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/agents/tools/image-tool.ts#L193)
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Moonshot regression is covered on main:** `src/agents/tools/image-tool.test.ts:823` exercises `agents.defaults.imageModel: "moonshot/kimi-k2.5"` with an image input and expects a successful `/chat/completions` request containing both text and image blocks.
  - file: [src/agents/tools/image-tool.test.ts:823](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/agents/tools/image-tool.test.ts#L823)

  - file: [src/agents/tools/image-tool.test.ts:1301](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/agents/tools/image-tool.test.ts#L1301)
