## Summary

Keep #61183 open. The PR is contributor-authored and has no protected close-blocking labels, but current main still does not contain the core bookkeeping fix proposed by the PR: embedded message bookkeeping commits pending/start-event values and checks `startData?.args`, while adjusted `before_tool_call` params are only consumed later for `after_tool_call`. The linked bug discussion and bot review remain relevant evidence, and the PR still contains potentially useful regression coverage that is absent from current main.
