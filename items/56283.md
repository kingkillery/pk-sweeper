## Summary

Current `main` does not implement this PR’s outbound markdown-stripping behavior. Plain-text channel delivery still only runs HTML sanitization, `MarkdownConfig` has no `strip` option, and there is no outbound call site wiring `stripMarkdown()` into delivery, so this should remain open.

## Evidence

- **Outbound delivery path still lacks markdown stripping:** `normalizePayloadsForChannelDelivery()` only applies `handler.sanitizeText(...)` before payload normalization. There is no `shouldStripMarkdown()` check and no `stripMarkdown()` call in the current delivery pipeline.
