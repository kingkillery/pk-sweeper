## Summary

Keep open. Current `main` only overlaps with a small subset of this PR, so it is not `implemented_on_main`, and the report is still coherent/actionable. Several advertised hardening changes are absent from `main` today: the local TUI shell still runs with `shell: true`, `web-media` still accepts `localRoots: "any"` as a traversal bypass, `memory-schema.ts` still interpolates SQL identifiers directly, `channel-bridge.ts` still iterates/mutates `pendingWaiters` without the PR's queue serialization, and multiple caches remain plain `Map`s without the proposed caps. One piece did land separately (`costUsageCache` is now bounded), which further argues this PR is only partially superseded, not closeable.
