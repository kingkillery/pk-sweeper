## Summary

Keep open. Current `main` still records `sessionTarget: "main"` + `wakeMode: "next-heartbeat"` runs as `ok` immediately after enqueueing the system event and requesting a heartbeat, and the gateway run-log path still has no ghost-run warning/flag path. The report is recent and is reinforced by the linked open PR #63111, so there is not strong evidence for an auto-close reason.

## Evidence

- **Main-session next-heartbeat path still returns success without awaiting execution:** `executeMainSessionCronJob` enqueues the system event, calls `requestHeartbeatNow(...)`, and immediately returns `{ status: "ok", summary: text }` for the `wakeMode !== "now"` path.
