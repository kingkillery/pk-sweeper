## Summary

Keep open. Current `main` still preserves the reported UX confusion: session status/control surfaces prefer the last recorded runtime model when a session has no explicit override, so changing the global default can leave an active session looking pinned to the old model. The actual reply-selection path does fall back to the configured default on the next turn, which means this is still a real UX/state-refresh gap rather than an implemented fix.

## Evidence

- **Status tool derives the displayed model from runtime identity when no explicit override exists:** `session_status` computes the card model from `runtimeProviderForCard` / `runtimeModelForCard` whenever `providerOverride` and `modelOverride` are absent, so an old runtime model remains the displayed primary model instead of the newly configured default.

  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)

That means session/control surfaces keep using the persisted runtime model.
