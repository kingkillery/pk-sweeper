## Summary

Keep open. Current `main` does not implement the PR's core change: Windows drive-letter sandbox binds are still classified as `non_absolute` before blocked-path checks run, and the security audit still downgrades that classification to a warning. The existing tests also still document Windows bind paths as intentionally rejected as non-POSIX, so this remains an unresolved product/security decision rather than a conservative cleanup-close candidate.

## Evidence

- **Current main still short-circuits Windows-style binds before blocked-path classification:** `getBlockedBindReason()` returns `{ kind: "non_absolute" }` as soon as the bind source does not start with `/`, before normalization or `getBlockedReasonForSourcePath()` runs. That means a source like `C:\\Users\\tester\\.docker\\config.json` never reaches the dangerous-path classifier on current `main`.
