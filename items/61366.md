## Summary

Keep open. Current main has part of the registry-stability work, but it still does not implement the PR's main sessions_spawn behavior change: agent channel hints are still validated strictly before dispatch, with no subagent-spawn exception for deliver=false/lane=subagent or subagent session keys. A split-brain plugin channel that is not present in the channel registry can still be rejected as unknown on the current code path, so this PR is not safe to close as implemented. The PR also needs rework before merge because the provided review context shows a duplicate import in its branch.
