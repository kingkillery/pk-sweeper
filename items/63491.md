## Summary

Current `main` has landed several relevant Windows restart and cron mitigations, but the evidence is still partial. I found code and tests covering stale Scheduled Task listeners, better restart timeout reporting, startup-fallback relaunches, and stale cron `runningAtMs` cleanup. I did not find matching evidence that the specific `code=1008 reason=connect failed` / `device-required` restart probe false negative is handled, and I did not find a direct fix or reproduction proof for the reported post-`ready` `/health` or `/` disappearance. Conservative maintainer cleanup should leave this open.
