## Summary

Keep open. Current `main` does not forward `params.serviceTier`/`service_tier` into Amazon Bedrock Converse requests, and the linked open PR (#64512) is adding exactly that missing behavior. This is still an actionable enhancement, not a stale or already-shipped fix.

## Evidence

- **Current Bedrock wrapper does not read model extra params:** On current `main`, `amazon-bedrock` registers `wrapStreamFn` with `{ modelId, config, model, streamFn }` only. The implementation applies guardrail, cache, and region handling, but there is no `extraParams` input and no payload patch for a Bedrock `serviceTier` field.
