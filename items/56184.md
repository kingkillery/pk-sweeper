## Summary

Current `main` already contains a Discord gateway cleanup/supervisor fix for late `Max reconnect attempts` errors during startup/teardown, and the changelog says the stale-socket restart crash was shipped in `v2026.3.28`.

## Evidence

- **Changelog entry for shipped fix:** `CHANGELOG.md` lists three Discord gateway shutdown fixes under `## 2026.3.28`, including: "suppress reconnect-exhausted events that were already buffered before teardown flips lifecycleStopping, so stale-socket Discord restarts no longer crash the whole gateway."
  - file: [CHANGELOG.md:2008](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/CHANGELOG.md#L2008)
  - command: `nl -ba CHANGELOG.md | sed -n '1914,2010p'`
- **Release version containing the fix:** The Discord gateway cleanup/shutdown entries are under the `## 2026.3.28` section, establishing the shipped release version.
  - file: [CHANGELOG.md:1917](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/CHANGELOG.md#L1917)
  - command: `nl -ba CHANGELOG.md | sed -n '1914,2010p'`
- **Current main suppresses late gateway errors after teardown/dispose:** `createDiscordGatewaySupervisor()` now routes late gateway `error` events in `teardown`/`disposed` phases to suppression logging instead of surfacing them as fatal lifecycle errors.
  - file: [extensions/discord/src/monitor/gateway-supervisor.ts:140](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/extensions/discord/src/monitor/gateway-supervisor.ts#L140)

## Close Comment

Closing as implemented on current `main`. Codex review found that Discord startup/teardown cleanup now suppresses late `reconnect-exhausted` gateway errors instead of letting them crash the process (`extensions/discord/src/monitor/gateway-supervisor.ts:140`, `extensions/discord/src/monitor/provider.ts:1143`). There is also regression coverage asserting that a late `Max reconnect attempts (0)` error after startup cleanup does not throw (`extensions/discord/src/monitor/provider.test.ts:281`). The changelog shows this shipped in `v2026.3.28`, including the specific stale-socket restart fix (`CHANGELOG.md:1917`, `CHANGELOG.md:2008`). If this still reproduces on a newer release than `v2026.3.28`, please open a new issue with fresh logs and the exact OpenClaw version.
