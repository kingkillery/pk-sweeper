## Summary

Keep open. Current `main` still renames the temp config into place without a post-rename `chmod`, while the Windows fallback does tighten to `0o600`. That leaves the PR aligned with the repository's existing security expectation that `openclaw.json` should be `0600`, so there is not strong evidence for cleanup closure.

## Evidence

- **Current main lacks chmod after successful rename in config hot-save path:** `writeConfigFile` writes the temp file with `mode: 0o600`, then calls `rename(tmp, configPath)` and returns to the normal flow without tightening the destination mode. Only the Windows fallback branch applies `chmod(configPath, 0o600)`.
