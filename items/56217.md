## Summary

Current `main` still fail-fast aborts gateway startup when required SecretRefs cannot resolve, and there is no built-in startup backoff/circuit-breaker for secret-provider failures. Exec SecretRefs are batched per provider invocation, but the documented 1Password integration still uses direct `op read` exec providers, so the reported launchd restart storm remains plausible and is not clearly fixed on `main`.

## Evidence
