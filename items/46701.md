## Summary

Current `main` does not implement an auto-reply for unauthorized Telegram group senders. Regular group messages that fail `groupAllowFrom` / `groupPolicy` checks are still dropped with logging only, while docs and doctor text explicitly describe silent drops. Only native command paths send an authorization reply.

## Evidence
