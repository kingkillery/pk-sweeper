## Summary

Keep PR #60500 open. Current main has not implemented PDF upload support in the web chat attachment path: the UI picker still accepts only images, the gateway parser still drops sniffed PDFs as non-images, and existing tests still assert PDF payloads are rejected. The Codex review concern also remains relevant: the native prompt attachment resolver only loads media refs when they resolve to images, so merely offloading PDFs would not by itself hydrate document content into model input.
