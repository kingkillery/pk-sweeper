## Summary

Keep open. Current main does not implement `allowedChatIds`/`deniedChatIds`, and the PR addresses a real active-memory scoping gap. It should not be auto-closed, but the current PR needs revision before merge because its proposed conversation-id parser misses supported session-key shapes and threaded sessions.

## Best Possible Solution

Keep the PR open and ask for a focused revision: parse conversation ids through existing session-key utilities or an equivalent shared helper, strip generic `:thread:<id>` suffixes before matching, support all current `dmScope` direct-key shapes plus group/channel keys, and add regression tests for per-peer, per-channel-peer, per-account-channel-peer, and threaded channel cases. Once corrected, the manifest/docs additions and tests can land as a plugin-owned active-memory feature.
