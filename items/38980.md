## Summary

Issue #38980 is already implemented on current main and in release v2026.4.22: the Feishu reply dispatcher now forces block streaming off for this channel, which directly bypasses the `agents.defaults.blockStreamingDefault = "on"` path described in the report, and release-tagged regression tests cover both the silent-drop prevention and the missing-final fallback.

## Close Comment

Closing from a Codex review because this is already implemented on current `main` and was shipped in `v2026.4.22`. Feishu now forces `disableBlockStreaming: true` in `extensions/feishu/src/reply-dispatcher.ts:494`, and `src/auto-reply/reply/get-reply-directives.ts:437` resolves that override to block streaming `off` even when `agents.defaults.blockStreamingDefault = "on"`. The release also includes targeted regressions in `extensions/feishu/src/reply-dispatcher.test.ts:278` (`prevent silent reply drops`) and `extensions/feishu/src/reply-dispatcher.test.ts:314` (`closes streaming with block text when final reply is missing`). If this still reproduces on `v2026.4.22` or newer, please open a fresh issue with the exact build SHA and current Feishu logs.
