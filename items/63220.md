## Summary

Current `main` has related hardening for late agent callbacks, but the repo does not show a direct fix for the reported `pi-agent-core` `Agent.processEvents` race in the gateway abort/fallback path. The embedded `@mariozechner/pi-agent-core` version is still `0.70.2`, generic unhandled rejections still exit the process, and I did not find a test or code path proving this specific compaction/abort corruption report is resolved end-to-end.
