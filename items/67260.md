## Summary

Keep open. Current `main` does not show a native-Ollama/session-replay fix for this report: the Ollama plugin still limits replay hooks to OpenAI-compatible Ollama routes, native `api:"ollama"` still has no replay policy, assistant failures with configured fallbacks still escalate to the fallback model, and session persistence still records the fallback winner as the session runtime model. That matches the reported long-lived Telegram-session symptom closely enough that this is not conservatively closable.

## Evidence

- **Native Ollama replay policy is still absent on main:** Current test coverage explicitly expects `provider.buildReplayPolicy` to be `undefined` for `provider: "ollama", modelApi: "ollama"`, while neighboring assertions cover replay policy only for the OpenAI-compatible Ollama routes.
  - file: [extensions/ollama/index.test.ts:461](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/extensions/ollama/index.test.ts#L461)
  - sha: [7875092f4ddc](https://github.com/openclaw/openclaw/commit/7875092f4ddc6f2dbe218bea8506c9ac1da87c39)
- **Ollama replay hooks are wired only through the OpenAI-compatible path:** The Ollama plugin spreads `OPENAI_COMPATIBLE_REPLAY_HOOKS`, and its transport gate only treats models with `api === "openai-completions"` as the compatible path. Native `api: "ollama"` is not included in that gate.
  - file: [extensions/ollama/index.ts:42](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/extensions/ollama/index.ts#L42)
  - sha: [7875092f4ddc](https://github.com/openclaw/openclaw/commit/7875092f4ddc6f2dbe218bea8506c9ac1da87c39)
- **Fallback winner is still persisted as the session runtime model:** After each run, session persistence resolves `modelUsed`/`providerUsed` from `result.meta.agentMeta` or the fallback/default model, then writes that pair with `setSessionRuntimeModel`. That means a successful fallback still becomes the model shown later by session status.
  - file: [src/agents/command/session-store.ts:69](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/src/agents/command/session-store.ts#L69)

- **Assistant-stage failures still escalate to fallback when a fallback is configured:** Current failover policy returns `action: "fallback_model"` for assistant-stage failures once rotation/retry logic is exhausted and `fallbackConfigured` is true; there is no native-Ollama replay-specific carveout here.
  - file: [src/agents/pi-embedded-runner/run/failover-policy.ts:155](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/src/agents/pi-embedded-runner/run/failover-policy.ts#L155)
