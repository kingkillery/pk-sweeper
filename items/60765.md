## Summary

Keep PR #60765 open. Current main still reads Discord modal `radio` fields through Carbon's `getStringSelect()` path, while Discord Radio Group modal submissions use a singular `value`, so the observable bug described by the PR still appears to exist on main. The related PR #47415 is closed but unmerged and was explicitly superseded by this PR, not a canonical replacement to close against.
