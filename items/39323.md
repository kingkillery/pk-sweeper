## Summary

Keep open. Current `main` does not implement this PR’s owner-notification feature for new DM pairing requests. There is no `pairing.notify` config surface on `main`, no `pairing` internal-hook event, and `upsertChannelPairingRequest()` still just persists the request without firing any notification path. The only adjacent shipped notifier is the bundled `device-pair` plugin, which covers node/device pairing and is Telegram-only, so it is not an implementation of this PR.
