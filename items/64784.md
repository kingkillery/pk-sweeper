## Summary

Keep open. Current `main` only has partial inbound LINE sticker normalization; it does not implement outbound LINE sticker sends, a `STICKER:packageId:stickerId` reply path, or the proposed `line-sticker` catalog/skill. The issue also already links an open companion PR (#65270), so there is still active implementation work rather than an obsolete request.

## Evidence

- **Inbound sticker handling is only receive-side text normalization:** `buildLineMessageContext` turns LINE sticker webhook events into plain text like `[Sent a ... sticker]`, which shows partial inbound support but not sticker send/round-trip support.
  - file: [extensions/line/src/bot-message-context.ts:213](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/line/src/bot-message-context.ts#L213)
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **Outbound LINE send path has no sticker branch:** `sendMessageLine` only constructs text, image, video, and audio messages from `text`/`mediaUrl`; there is no sticker message builder and no `packageId`/`stickerId` handling.
  - file: [extensions/line/src/send.ts:242](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/line/src/send.ts#L242)

  - file: [extensions/line/src/types.ts:131](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/line/src/types.ts#L131)
