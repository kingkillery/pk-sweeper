## Summary

Keep #67781 open. Current main still lacks the exact Vertex ADC marker stripping and still silently returns no reply for zero final payloads, so the PR is not obsolete. But the submitted patch is not safe to close/merge as-is: it overwrites the Google provider stream wrapper surface and its empty-payload guard conflicts with existing successful no-final-payload paths such as block streaming, same-target messaging-tool dedupe, direct block sends, and silent turns.

## Best Possible Solution

Keep the PR open, but require a revision. The Google side should compose a narrow google-vertex ADC marker-stripping wrapper with the existing Google thinking wrapper, ideally using the existing provider-stream composition helper and adding coverage for both Vertex ADC and non-Vertex thinking behavior. The empty-payload side should distinguish true no-content model completions from successful already-delivered cases: block streaming that didStream without aborting, direct block-key dedupe, same-target messaging-tool suppression, and silentExpected turns should remain non-errors, while a non-silent run with no delivered output and no renderable final payload can surface the friendly retry/context error.
