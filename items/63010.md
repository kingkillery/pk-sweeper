## Summary

Keep open. Current `main` still writes the resolved ClawHub version into the tracked install spec for unversioned installs, and the updater reuses that stored spec on later `plugins update` runs, so the reported behavior still appears present.

## Evidence

- **Install path pins the resolved ClawHub version:** `plugins install` calls `persistPluginInstall` with `spec: formatClawHubSpecifier({ name: result.clawhub.clawhubPackage, version: result.clawhub.version })`, so an unversioned ClawHub install is recorded as a versioned spec.
  - file: [src/cli/plugins-install-command.ts:512](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/cli/plugins-install-command.ts#L512)
  - command: `nl -ba src/cli/plugins-install-command.ts | sed -n '498,530p'`
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Current CLI test locks in the reported behavior:** The test installs `clawhub:demo` and asserts that the persisted install record contains `spec: "clawhub:demo@1.2.3"`.
  - file: [src/cli/plugins-cli.install.test.ts:416](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/cli/plugins-cli.install.test.ts#L416)
  - command: `nl -ba src/cli/plugins-cli.install.test.ts | sed -n '380,434p'`
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Update path reuses the stored spec:** For ClawHub installs, `plugins update` calls `installPluginFromClawHub` with `effectiveSpec ?? record.spec ?? \`clawhub:${record.clawhubPackage!}\`` and then re-records `spec: effectiveSpec ?? record.spec ?? ...`, so a previously pinned record stays pinned.
  - file: [src/plugins/update.ts:533](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/plugins/update.ts#L533)
  - command: `nl -ba src/plugins/update.ts | sed -n '532,619p'`
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Docs describe exact pins as a stored special case:** The CLI docs say id-based updates reuse stored specs and that stored exact pinned versions continue to be used later, which aligns with the issue's expectation that only intentionally pinned installs should stay pinned.
  - file: [docs/cli/plugins.md:263](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/docs/cli/plugins.md#L263)
