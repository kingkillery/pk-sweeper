## Summary

Keep open. Current `main` still regenerates per-agent `models.json` during normal agent startup, and merge mode only preserves limited fields from an existing file (`baseUrl` and sometimes `apiKey`). That means the reported overwrite behavior is not implemented away. At the same time, current docs are mixed: one page now explains that most `models.json` fields are regenerated, while another still tells users they can add custom providers directly in `models.json`. That is not strong enough evidence for any conservative auto-close reason.
