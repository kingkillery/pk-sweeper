## Summary

Keep open. Current main still wires provider-owned runtime hooks by provider id, not by `models.providers.<id>.api`, so a custom provider like `bernard` with `api: "ollama"` would not activate the bundled Ollama stream hooks. The codebase even has a plugin-level test showing custom Ollama-named providers can work once the plugin is reached, which supports the reporter/commenter diagnosis that the remaining gap is ownership/activation wiring rather than Ollama transport itself.

## Evidence

- **Runtime stream registration keys off the model provider id only:** `registerProviderStreamForModel` asks `resolveProviderStreamFn` for `params.model.provider` and never falls back to `model.api`, so a model resolved as provider `bernard` + api `ollama` will only work if a provider plugin is loaded for `bernard`.
  - file: [src/agents/provider-stream.ts:15](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/agents/provider-stream.ts#L15)
  - command: `nl -ba src/agents/provider-stream.ts | sed -n '1,120p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Provider hook lookup matches exact provider ids and declared aliases only:** `resolveProviderRuntimePlugin` loads plugins for `providerRefs: [params.provider]`, and `matchesProviderId` only accepts the plugin's `id`, `aliases`, or `hookAliases`. There is no fallback from custom provider id to `providerConfig.api`.
  - file: [src/plugins/provider-hook-runtime.ts:17](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/plugins/provider-hook-runtime.ts#L17)

- **Plugin ownership resolution is provider-id based:** `resolveOwningPluginIdsForProvider` matches manifest `providers` and `cliBackends` entries against the normalized provider id. It does not consider `models.providers.<id>.api`, so `bernard` will not automatically map to the Ollama plugin.
  - file: [src/plugins/providers.ts:367](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/plugins/providers.ts#L367)
