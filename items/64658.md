## Summary

Keep open. Current `main` still treats `sessions_spawn` as auto-announcing completion back to the requester channel, and there is no supported public parameter for the requested `none` or `parent` delivery modes. The prior comment usefully points out that the architecture already carries delivery-state concepts, but the exposed API/docs still do not provide the requested control, and the existing thread-bound `mode:"session"` behavior is only a narrow special case.

## Evidence
