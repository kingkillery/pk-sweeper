## Summary

Keep open. Current `main` still enqueues `sessionTarget:"main"` cron `systemEvent` payloads as plain system-event text, the main reply path consumes those events as prompt context rather than executable commands, and the CLI add/edit flows have no shell-command warning or test coverage for one. The issue timeline also points at an open follow-up PR rather than a merged fix.
