## Summary

Keep open. Current `main` still implements ack reactions as a single resolved string, not a configurable array or per-message/contextual choice. The config types, docs, tests, and channel handlers all reflect the static `ackReaction` contract, so the feature request is not implemented on `main`. It also is not a strong `clawhub` close candidate because ack reactions are handled in the built-in channel/config path before agent work, not as a standalone skill surface.
