## Summary

Keep open. The report is recent, concrete, and still had a reporter-confirmed reproduction on 2026-04-18 against OpenClaw 2026.4.15. A closely related open issue (#46494) and an attempted health-path fix discuss the same `health --json` mismatch, but the available discussion includes unresolved concerns that the fix may still read account runtime data from the wrong payload shape. I could not verify a current-main implementation locally because read-only commands failed in the sandbox, so there is not enough high-confidence evidence to close.
