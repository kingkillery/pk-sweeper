## Summary

Keep open. Current `main` still caches runtime auth snapshots without checking `auth-profiles.json` or `auth-state.json` mtimes, so the requested behavior is not already implemented, and the PR discussion raises unresolved P1 regression risks around save-time snapshot refresh and subagent fallback behavior.

## Evidence

- **Current main does not invalidate runtime snapshots on disk change:** `runtimeAuthStoreSnapshots` is still a plain `Map<string, AuthProfileStore>`, and `getRuntimeAuthProfileStoreSnapshot` / `hasRuntimeAuthProfileStoreSnapshot` only read that map. There is no mtime or file-stat check in current `main`, so the stale-snapshot problem described by the PR is still plausible.
