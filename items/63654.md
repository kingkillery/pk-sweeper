## Summary

Keep open. Current `main` still intentionally filters and suppresses `qwen3.6-plus` on Qwen Coding Plan endpoints, and both tests and docs describe that as expected behavior, so the requested support is not implemented on `main`. The report likely overlaps with a separate image-model routing bug that has since been covered by current tests, but that does not remove the provider-level Coding Plan block this issue is about.
