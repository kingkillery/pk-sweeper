## Summary

Current `main` does not implement the requested passive `context=X%` runtime metadata in the system prompt. OpenClaw now has related mitigations (`/status`, `/context`, `session_status`, automatic pre-compaction memory flush, and an optional `/new`/`/reset` session-memory hook), but those are separate mechanisms and do not fully satisfy this issue’s specific request or make it safe to close as implemented.

## Risks / Open Questions

- Maintainers could judge `/status`/`session_status` plus memory-flush behavior as a sufficient replacement, but the code on `main` still does not implement the specific passive runtime-line signal requested here.
- The issue mixes two related concerns: proactive compaction awareness and persistence across `/new` or `/reset`. Current main addresses those concerns unevenly through different features, which makes a closure decision partly product-judgment rather than purely code-evidence.
