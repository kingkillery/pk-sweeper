## Summary

Keep open. Current main does not provide strong evidence that Telegram image delivery from a plain `read` tool result is fixed. The repo-local media extraction path still drops image tool results when they have image content but no `details.media`, `MEDIA:` token, or `details.path`, and the tests explicitly model that case as yielding no outbound media.

## Evidence

- **Read wrapper does not attach outbound media metadata:** `createOpenClawReadTool` only runs adaptive paging, MIME normalization, and image sanitization before returning the read result. In the visible wrapper code it does not add `details.media` or `details.path` for image reads.
  - file: [src/agents/pi-tools.read.ts:674](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/agents/pi-tools.read.ts#L674)
  - command: `nl -ba src/agents/pi-tools.read.ts | sed -n '674,698p'`
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Media extractor documents the unfixed read-tool gap:** The extractor comment says Pi SDK `read` can return base64 image data with no file path, and in that case a different delivery path would be needed. The implementation only recognizes `details.media`, `MEDIA:` text, or `details.path`.
  - file: [src/agents/pi-embedded-subscribe.tools.ts:245](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/agents/pi-embedded-subscribe.tools.ts#L245)
  - command: `nl -ba src/agents/pi-embedded-subscribe.tools.ts | sed -n '237,340p'`
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Tool-result delivery bails out when no media artifact is extracted:** `handleToolExecutionEnd` returns early unless `extractToolResultMediaArtifact(result)` produces media URLs, so a pathless image read result never reaches channel delivery.
  - file: [src/agents/pi-embedded-subscribe.handlers.tools.ts:536](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/agents/pi-embedded-subscribe.handlers.tools.ts#L536)
