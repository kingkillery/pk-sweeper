## Summary

Keep open. Current `main` already provides partial browser-profile isolation (separate managed profile state, per-profile CDP ports, isolated user-data dirs, and explicit `profile` selection on browser calls), but it does not implement the two core requested features: first-class per-profile browser proxy configuration and agent-bound default browser-profile routing. The linked PR for this exact feature, #42806, was closed unmerged.

## Evidence

## Risks / Open Questions

- The issue overstates the isolation gap slightly: named managed browser profiles are already isolated by user-data dir and CDP port, so only the proxy support and agent-default routing remain unimplemented.
- There are workarounds such as `browser.extraArgs` for launch flags and explicit per-call `profile` selection, but they are not the first-class config and inheritance model requested here.
