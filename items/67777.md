## Summary

Keep open. Current main still does not implement the requested durable subagent completion spool/inbox: run records persist only frozen result and retry markers, completion delivery remains direct-first with conditional queue fallback, cleanup still has bounded retry/give-up paths, and orphan reconciliation can prune restored runs before any durable completion handoff.

## Best Possible Solution

Keep #67777 open as the implementation/planning thread. The likely product path is to add a durable completion spool/inbox to the subagent registry model, create the pending spool entry before cleanup retry/give-up branches, make direct delivery a fast path over that durable state, retry pending spooled completions on resume/restart, preserve pending spool entries during orphan reconciliation, align task delivery status with spool state, and add focused tests around dispatch failure, cleanup delete mode, restart recovery, retry exhaustion, and orphan preservation.
