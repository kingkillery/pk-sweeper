## Summary

Current `main` already implements the reported repair path: a platform/device-family mismatch triggers `metadata-upgrade` pairing, and approving that pending request overwrites the paired record with the refreshed `platform`/`deviceFamily`, so manual JSON editing is no longer required.

## Close Comment

Closing from a Codex review because current OpenClaw already implements the recovery path this report asks for. On a platform/device-family mismatch, the gateway now routes the reconnect through `metadata-upgrade` approval in `src/gateway/server/ws-connection/message-handler.ts:1076`, and approving that request rewrites the paired record from the pending request, including `platform` and `deviceFamily`, in `src/infra/device-pairing.ts:634`. That means the paired metadata no longer has to be hand-edited for the re-approval to stick. The latest shipped release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`) already contains this implementation, so the reported manual-edit loop is obsolete on current main/recent release.
