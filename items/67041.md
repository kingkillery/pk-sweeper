## Summary

Keep open. Current `main` still lacks the added JWT/Basic-auth/custom-header redaction patterns, so this PR is not implemented on `main`, and the existing review comments flag real correctness and test-coverage gaps in the proposed patch.

## Evidence

- **Current main redaction defaults do not include the new patterns:** `DEFAULT_REDACT_PATTERNS` on current `main` includes env/json/CLI/bearer/PEM/common token prefixes and Telegram bot tokens, but no generic JWT pattern, no `Authorization: Basic` pattern, and no custom header patterns such as `X-OpenClaw-Token` or `x-pomerium-jwt-assertion`.
  - file: [src/logging/redact.ts:16](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/src/logging/redact.ts#L16)

  - file: [src/logging/redact.test.ts:11](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/src/logging/redact.test.ts#L11)
