## Summary

Keep open. Current `main` does not show this PR's cross-handler Discord inbound dedupe fix: the replay guard is still created per `createDiscordMessageHandler(...)` instance, the underlying claimable dedupe is closure-local in memory, and the current test coverage only locks in duplicate suppression when the same handler instance sees the same message twice.
