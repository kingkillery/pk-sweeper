## Summary

Keep open. Current `main` still contains the P2P-thread behavior described in #38806: Feishu DM replies are forced down the non-thread path, which strips reply metadata and falls back to a top-level send.

## Evidence

- **DM thread replies still disabled at bot entry:** `extensions/feishu/src/bot.ts:578` sets `const replyInThread = isGroup ? (groupSession?.replyInThread ?? false) : false;`, so P2P messages cannot opt into thread replies even when the inbound event is in a thread.
  - file: [extensions/feishu/src/bot.ts:578](https://github.com/openclaw/openclaw/blob/ed6094b3014c56882da15a7942130b00ea410c6e/extensions/feishu/src/bot.ts#L578)
  - command: `nl -ba extensions/feishu/src/bot.ts | sed -n '560,620p'`
  - sha: [ed6094b3014c](https://github.com/openclaw/openclaw/commit/ed6094b3014c56882da15a7942130b00ea410c6e)
- **Reply target selection is still group-only:** `extensions/feishu/src/bot.ts:1031-1036` only treats `root_id` as the reply target for topic/config-thread cases when `isGroup` is true; otherwise replies target `ctx.messageId`, matching the issue report's regression description for P2P threads.
  - file: [extensions/feishu/src/bot.ts:1031](https://github.com/openclaw/openclaw/blob/ed6094b3014c56882da15a7942130b00ea410c6e/extensions/feishu/src/bot.ts#L1031)
  - command: `nl -ba extensions/feishu/src/bot.ts | sed -n '1018,1110p'`
  - sha: [ed6094b3014c](https://github.com/openclaw/openclaw/commit/ed6094b3014c56882da15a7942130b00ea410c6e)
- **DM send path still strips reply metadata:** `extensions/feishu/src/bot.ts:1095` and `extensions/feishu/src/bot.ts:1204` pass `skipReplyToInMessages: !isGroup` into the reply dispatcher. In `extensions/feishu/src/reply-dispatcher.ts:111`, that clears `sendReplyToMessageId`, so DM replies are sent without reply targeting.
  - file: [extensions/feishu/src/reply-dispatcher.ts:111](https://github.com/openclaw/openclaw/blob/ed6094b3014c56882da15a7942130b00ea410c6e/extensions/feishu/src/reply-dispatcher.ts#L111)

- **No reply target means top-level create, not reply:** `extensions/feishu/src/send.ts:143-145` calls the fallback direct send when `replyToMessageId` is missing. That is the exact behavior produced after the DM path clears reply metadata, so the bot reply will land as a standalone message.
  - file: [extensions/feishu/src/send.ts:143](https://github.com/openclaw/openclaw/blob/ed6094b3014c56882da15a7942130b00ea410c6e/extensions/feishu/src/send.ts#L143)
