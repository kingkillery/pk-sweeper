## Summary

Keep #60420 open. Current main still only unbinds stale conversation bindings on ACP init failures; it does not implement the PR's requested session-store scrub of persisted routing fields (`deliveryContext`, `origin`, `lastChannel`, `lastTo`, `lastAccountId`, `lastThreadId`). The related #60720 context is adjacent but closed unmerged and does not supersede this cleanup.
