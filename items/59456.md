## Summary

Keep open. The current main CLI help still describes `cron add --tz` as only applying to cron expressions, while the issue specifically asks for that help text to mention offset-less `--at` datetime strings too. The issue also has an active linked PR (#69975), so this is salvageable work rather than cleanup-close material.

## Evidence

- **Current main still has misleading help text:** `registerCronAddCommand` defines `--tz <iana>` with help text `Timezone for cron expressions (IANA)`, which does not mention offset-less `--at` values.
  - file: [src/cli/cron-cli/register.cron-add.ts:156](https://github.com/openclaw/openclaw/blob/2e0ae56b1a580103f71b9b7f79b3d638e3c3698d/src/cli/cron-cli/register.cron-add.ts#L156)
  - command: `web/open GitHub blob for current main at src/cli/cron-cli/register.cron-add.ts`

- **Current parser behavior makes the docs distinction real:** `parseAt` explicitly resolves offset-less ISO datetimes in the provided IANA timezone when `tz` is set, so the flag is not cron-only for all `--at` inputs.
  - file: [src/cli/cron-cli/shared.ts:1190](https://github.com/openclaw/openclaw/blob/2e0ae56b1a580103f71b9b7f79b3d638e3c3698d/src/cli/cron-cli/shared.ts#L1190)
