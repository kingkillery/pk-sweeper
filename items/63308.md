## Summary

Keep open. Current `main` still exposes `sessionKey`, `label`, and `agentId` as independent optional schema fields, the executor still rejects `sessionKey`+`label` with the exact reported error, and the user-facing `sessions_send` description/docs still do not add the requested mutual-exclusion or precedence guidance. I found no landed main/release change for the relevant files; the provided timeline also points to an open follow-up PR (#63377), which matches the code state.
