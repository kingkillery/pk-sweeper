## Summary

Current `main` does not show a fix for #56197. Embedded auth-error retry exists only for providers that implement runtime-auth refresh, but the bundled `zai` provider does not export that hook, so a Feishu embedded run cannot refresh-and-retry a 401 on the current path. Generic `HTTP 401: Invalid token` errors are also still formatted back to the user rather than being suppressed or rewritten into a retry outcome.

## Evidence
