## Summary

Keep open. Current `main` has related but unreleased CLI-only `models list` speedups, but the gateway path reported here still startup-gates `models.list` and still does a full first-load model-catalog build. There is not strong evidence that issue #62364 is fixed on `main` or shipped in `v2026.4.23`.

## Evidence

- **Issue scope is broader than "many providers only":** The original report measured `models.list` at 33.7s after upgrading to 2026.4.5, and a follow-up comment reported a 49s startup overhead with only Bedrock and 4 models. That weakens any narrow interpretation that this was solved only by reducing multi-provider fanout.
- **Gateway still serves `models.list` by loading the gateway model catalog on demand:** `models.list` still awaits `context.loadGatewayModelCatalog()` before responding, then filters the allowed catalog. There is no gateway-side fast path here.
