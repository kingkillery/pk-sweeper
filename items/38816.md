## Summary

Keep open. Current `main` does not contain the linked fix, and the underlying bug class still exists: bare `agents.defaults.imageModel` refs are still resolved through a generic default-provider path rather than the user’s configured primary provider. The exact bad prefix has changed from `anthropic/` to `openai/` because `DEFAULT_PROVIDER` is now `openai`, but that does not prove the issue is implemented or obsolete.

## Evidence

- **Image fallback still derives its default provider from the bare image-model ref path:** `resolveImageFallbackDefaultProvider()` reads `agents.defaults.imageModel.primary` and, for bare refs, resolves them with `defaultProvider: DEFAULT_PROVIDER`. That means `moondream`/`qwen2.5vl:7b` still inherit the product default provider instead of the user’s primary reply-model provider.
