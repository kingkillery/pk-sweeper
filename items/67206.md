## Summary

Keep open. Current `main` still caches subsystem file loggers without any invalidation when the parent logger is rebuilt, while docs still state that `logging.file` controls the log path. This looks like a real unresolved bug, and the GitHub timeline shows multiple open duplicate PRs rather than an implemented fix on `main`.

## Evidence

- **Subsystem file child is still cached forever on main:** `createSubsystemLogger()` keeps a local `fileLogger` and only initializes it when null. There is no refresh path after the root logger changes, so an existing subsystem logger can keep writing through a stale child logger.
