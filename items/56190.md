## Summary

Current `main` does not implement `${VAR:-default}` env var fallbacks. The config substitution path, env-ref restore path, and config-env staging path still only understand plain `${VAR}` / `$${VAR}` tokens, so this PR remains actionable and should stay open.

## Evidence

- **Substitution parser only handles plain `${VAR}`:** `parseEnvTokenAt` returns only `escaped` and `substitution` tokens, both based on a plain env var name extracted up to the first `}`. There is no `:-default` parsing or `defaultValue` support.
  - file: [src/config/env-substitution.ts:39](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/config/env-substitution.ts#L39)
  - command: `nl -ba src/config/env-substitution.ts | sed -n '39,75p'`
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Missing vars still error or preserve literal placeholder:** When a substitution token is missing or empty, `substituteString` either calls `onMissing` and preserves `${VAR}` or throws `MissingEnvVarError`. There is no fallback branch for `${VAR:-default}`.
  - file: [src/config/env-substitution.ts:113](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/config/env-substitution.ts#L113)
  - command: `nl -ba src/config/env-substitution.ts | sed -n '107,127p'`
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Env-ref detection would still skip fallback-style config chaining:** `applyConfigEnvVars` skips values when `containsEnvVarReference(value)` is true. Since `containsEnvVarReference` has no notion of a resolvable defaulted form, current `main` has not incorporated the PR's fix for config-env staging behavior.
  - file: [src/config/config-env-vars.ts:79](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/config/config-env-vars.ts#L79)
  - command: `nl -ba src/config/config-env-vars.ts | sed -n '79,96p'`
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Write-back restore path still only recognizes `${VAR}`:** `env-preserve.ts` uses `ENV_VAR_PATTERN = /\$\{[A-Z_][A-Z0-9_]*\}/` and `tryResolveString` only resolves `${VAR}` and `$${VAR}`. `${VAR:-default}` would not round-trip on current `main`.
  - file: [src/config/env-preserve.ts:19](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/config/env-preserve.ts#L19)
