## Summary

Current `main` already has a substantial usage dashboard and session-log usage pipeline, but I did not find strong evidence that the issue is fully implemented as filed. The repo now exposes a Usage tab, per-session usage summaries, daily aggregates, per-model/provider breakdowns, and session timeseries/log endpoints; however, the shared usage aggregate shape still has no session-type (`cron`/direct/group/sub-agent) breakdown, and the UI only exposes `chatType` as a filter rather than an aggregate/reporting surface.
