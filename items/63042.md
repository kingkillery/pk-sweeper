## Summary

Keep open. Current `main` partially improved the documentation around token auth, but the reported gaps are still real: `openclaw models auth paste-token` still has no non-interactive `--token`/stdin path, `invalid_type` still does not surface valid credential types, and the isolated-agent runtime guidance still points users to `openclaw agents add <id>` or copying `auth-profiles.json` instead of a concrete custom-provider auth setup path.

## Evidence
