## Summary

Keep open. Current `main` still has both reported gaps: the Discord status log is emitted once before READY and never on the READY event itself, and the lifecycle/reconnect path still has no certificate-expiry-specific escalation or hard process respawn. The issue discussion also adds a second reproduction on Linux/systemd, which points to a real unresolved stability problem rather than a stale macOS-only report.

## Evidence

- **Startup log still snapshots pre-READY state once:** The Discord monitor logs `formatDiscordStartupStatusMessage(...)` once before entering `runDiscordGatewayLifecycle(...)`, using `lifecycleGateway?.isConnected === true` at that instant. If the gateway is not yet connected, the emitted line is the `awaiting gateway readiness` variant, and there is no follow-up log here after READY.
