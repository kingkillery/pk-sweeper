## Summary

Current main already fixes the linked Telegram/API-key auth-routing problem through canonical auth-profile eligibility filtering, and v2026.4.24 includes that behavior and regression coverage. The open PR is obsolete and its own review comments identified fallback/availability gaps that main avoids by filtering at auth profile order resolution.

## Evidence

- **Main rejects mode-mismatched auth profiles before runtime resolution:** resolveAuthProfileEligibility returns mode_mismatch when configured auth.profiles.<id>.mode does not match the stored credential type, except the documented oauth/token compatibility case. This prevents a profile configured as token from resolving an OAuth credential.
