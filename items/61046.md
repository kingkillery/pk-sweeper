## Summary

Keep open. Current main still routes bare configured model keys and bare fallback entries through DEFAULT_PROVIDER in the models-list configured-entry path, so the observable bug this PR targets is not implemented on main. The PR also has a substantive review concern about alias keys remaining tied to the old default-provider alias index, so it should not be conservatively closed as already solved.
