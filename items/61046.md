## Summary

Keep open. Current `main` still shows the original bare-model provider-resolution bug in `resolveConfiguredEntries()`, and the submitted patch has an unresolved alias regression called out in PR review: it would infer `google/...` for bare configured keys while `aliasesByKey` is still indexed under `openai/...`, so alias tags can disappear and alias lookups can stay tied to the wrong provider.

## Evidence

- **Current main still defaults bare fallback/model refs to openai:** `resolveConfiguredEntries()` still calls `resolveModelRefFromString()` with `defaultProvider: DEFAULT_PROVIDER` for fallback/image entries, and still parses bare `agents.defaults.models` keys with `parseModelRef(key, DEFAULT_PROVIDER, ...)`. There is no provider inference here on current `main`, so the reported wrong-provider display remains unfixed.
  - file: [src/commands/models/list.configured.ts:46](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/commands/models/list.configured.ts#L46)

If the PR then rewrites the displayed configured entry to an inferred provider such as `google/...`, `aliasesByKey.get(key)` will miss.
