## Summary

Keep open. Current `main` still classifies cron run status from the isolated-run result, still returns `status: "error"` on non-best-effort announce delivery failures, and still surfaces `lastStatus` in `cron list`. There is no evidence of a fix on `main` after `v2026.4.23`, so this is not safe to close as implemented or obsolete.

## Evidence

- **Run status still drives job error/backoff:** `applyJobResult()` writes `result.status` into both `lastRunStatus` and `lastStatus`, then increments `consecutiveErrors` and applies error backoff whenever that status is `"error"`. That means a delivery-classified error still marks the job itself as errored.
