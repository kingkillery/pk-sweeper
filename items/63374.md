## Summary

Keep open. Current `main` does not implement the requested symmetric eviction delivery for `sessions.messages.subscribe`: lifecycle/session-change broadcasts still target only session-event subscribers, and those broadcasts still use `dropIfSlow: true`. The docs and tests on `main` also describe and assert `sessions.messages.subscribe` as transcript-only behavior. The issue discussion additionally points to a still-open follow-up PR (#69822), which is consistent with the feature not being merged yet.
