## Summary

Current main already solves the user-visible Ollama picker problem through the configured-model catalog path: Ollama setup seeds provider config with suggested/discovered Ollama models, the setup wizard passes Ollama as the preferred provider after auth, and the picker filters to that provider when entries exist. The PR's `augmentModelCatalog` hook is now redundant; the shipped changelog records the same Ollama onboarding/model-picker fix as #55290.

## Best Possible Solution

Close this PR and keep the shipped #55290-style configured-model catalog behavior. If maintainers still want to prevent future provider omissions, track that separately as a generic provider-plugin contract or validation improvement rather than merging an Ollama-only unconditional catalog placeholder.
