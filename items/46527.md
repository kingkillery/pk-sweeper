## Summary

Keep open. Current `main` still computes gateway health channel summaries from config plus optional probe, without threading the live gateway runtime snapshot into `getHealthSnapshot`, and the cached health fast path still ignores runtime freshness. The PR addresses a real gap that is not implemented on `main` yet.

## Evidence

- **Health snapshot still config/probe-only:** `getHealthSnapshot` builds a minimal `snapshot` from `enabled`, `configured`, `probe`, and `lastProbeAt`, then calls `plugin.status.buildChannelSummary(...)`. It does not accept or resolve any gateway runtime snapshot, and it does not use `buildChannelAccountSnapshot`.
