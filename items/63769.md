## Summary

Keep open. Current `main` at `86f8c826e20a01d1bba67c0ba6e73182fbf0455f` still uses overlap-based suffix dedupe in Gateway assistant-text merging, and a read-only reproduction against that exact logic still collapses repeated boundary characters. The cross-tool segmented-merge regression tests are present, but the repeated-character regression coverage described in #63769 is not on `main`.

## Evidence

- **Current merge helper still strips overlap from deltas:** `appendUniqueSuffix()` still returns `base` unchanged when `base.endsWith(suffix)` and otherwise trims the longest suffix/prefix overlap before appending. That is the exact heuristic the issue reports as unsafe for semantically-new `delta` text.
