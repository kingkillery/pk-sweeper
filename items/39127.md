## Summary

Keep open. Current `main` does not implement the requested gateway-owned per-session activity API or WS activity-change event. The gateway method inventory and read scopes expose `sessions.list/get/...` but no `sessions.activity.get` or `sessions.activity.list`; the documented event surface is still `sessions.changed`; and the session row/event payloads only carry coarse fields like `status`, `startedAt`, `endedAt`, `runtimeMs`, and `childSessions`, not the requested runtime-activity/attention model (`busy`, `activeRuns`, `busySince`, `staleRuns`, `attention.state`, `blockedOn`, monotonic `version`, etc.).
