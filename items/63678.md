## Summary

Keep open. Current `main` still uses the strict single-pattern `Vectors:` parser and still treats a parse miss as vector-unavailable, while the test suite only covers the colon-form output. The PR's multi-format parsing fix is not implemented on `main`.

## Evidence

- **Current parser is still strict:** `parseQmdStatusVectorCount` on `main` only matches `Vectors:` with a colon via `/(?:^|\n)\s*Vectors:\s*(\d+)\b/i`, so the PR's claimed `Vectors = 42`, `Vectors:42`, and broader fallback handling is not present.
