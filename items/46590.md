## Summary

Keep open. Current `main` still implements cron as a gateway-owned store at `~/.openclaw/cron/jobs.json`, and the agent config schema/docs do not support an `agents.list[].cron` field, so the requested declarative agent-owned scheduled-task configuration is not implemented on `main`.

## Evidence

- **Cron docs still describe a central gateway store:** The scheduled-tasks docs say cron runs inside the Gateway and that job definitions persist at `~/.openclaw/cron/jobs.json`, with runtime state in `jobs-state.json`. That matches the issue’s current-state description, not the proposed per-agent config.
