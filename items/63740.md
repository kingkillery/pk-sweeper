## Summary

Keep open. Current `main` does not provide strong evidence that issue #63740 is fixed or obsolete: startup still hard-fails if a packaged `dist` chunk is corrupted, and the newer packaged-dist inventory checks are wired into update verification rather than normal CLI startup. The reporter’s proposed `tail -n 200` root cause is not supported by current runtime source, but that only makes the corruption unexplained, not disproven.

A corrupted packaged `dist/run-main-*.js` chunk would still brick the CLI before recovery logic can run.
