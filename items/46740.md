## Summary

Keep open. Current `main` does not implement this PR's behavior: non-ACP runtime error codes still collapse to `ACP_TURN_FAILED`, and the existing regression test still expects plain `acpx exited with code 1` to surface as `ACP_TURN_FAILED` rather than `ACP_BACKEND_UNAVAILABLE`.

## Evidence

- **Manager still downgrades unknown stream error codes to ACP_TURN_FAILED:** When the runtime emits an `error` event, the manager wraps it with `normalizeAcpErrorCode(event.code)`. That helper returns `ACP_TURN_FAILED` for any code that is not already one of the ACP error-code constants.
  - file: [src/acp/control-plane/manager.core.ts:801](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/acp/control-plane/manager.core.ts#L801)
  - command: `nl -ba src/acp/control-plane/manager.core.ts | sed -n '799,806p'; nl -ba src/acp/control-plane/manager.utils.ts | sed -n '85,96p'`
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **Current test still expects plain acpx exit text to map to ACP_TURN_FAILED:** `manager.test.ts` still asserts that a runtime failure thrown as `new Error("acpx exited with code 1")` rejects with `code: "ACP_TURN_FAILED"`. That is the exact behavior this PR was trying to change.
  - file: [src/acp/control-plane/manager.test.ts:1933](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/acp/control-plane/manager.test.ts#L1933)
  - command: `nl -ba src/acp/control-plane/manager.test.ts | sed -n '1933,1950p'`
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **ACPX wrapper on main no longer contains the proposed classification logic:** The bundled ACPX extension now just delegates `ensureSession` and `runTurn` directly into `acpx/runtime`; there is no local silent-exit remapping in `extensions/acpx/src/runtime.ts` on current main.
  - file: [extensions/acpx/src/runtime.ts:274](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/extensions/acpx/src/runtime.ts#L274)
  - command: `nl -ba extensions/acpx/src/runtime.ts | sed -n '274,280p'`
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **Bundled acpx version emits generic runtime output codes, not ACP_BACKEND_UNAVAILABLE:** OpenClaw pins `acpx` 0.5.3. In that package's runtime manager, runtime failures are turned into error events with `code: normalized.code`; `normalizeOutputError` produces generic output codes like `RUNTIME`, so OpenClaw's manager will fall back to `ACP_TURN_FAILED` unless a separate mapping is added.
  - file: [extensions/acpx/package.json:7](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/extensions/acpx/package.json#L7)
