## Summary

Keep open. Current main has outbound Discord mention rewriting, but it is still account-scoped cache-dependent and preserves unknown @Name text unchanged. I found no deterministic config-backed alias/userId mapping or inbound-activity cache seeding path that satisfies the issue’s central request.

## Best Possible Solution

Keep #67587 open and implement the fix in the bundled Discord plugin: add an explicit account-scoped alias-to-userId config surface or directory/teammate source, seed the outbound resolver from that deterministic source, optionally seed from trusted inbound metadata, and add tests proving @AgentName rewrites to <@userId> with an empty runtime cache. Update the Discord docs/config reference for the new field and collision behavior.

- A deterministic alias feature should require explicit ID-backed, account-scoped mappings to avoid mentioning the wrong Discord user when display names collide.
- Seeding from inbound activity would need collision and trust rules, especially for nicknames/global names in shared guilds.
