## Summary

Keep open. Current `main` still defaults Docker users toward container loopback/localhost for LM Studio and Ollama, the bundled `docker-compose.yml` still lacks a `host.docker.internal` mapping, and the only linked fix is open PR #68702 rather than merged code.

## Evidence

- **Bundled Docker compose still lacks host alias mapping:** `docker-compose.yml` defines the `openclaw-gateway` service, published ports, and healthcheck, but there is no `extra_hosts` entry for `host.docker.internal`, so the main-branch Docker stack does not yet include the alias proposed in the linked fix.
  - file: [docker-compose.yml:2](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/docker-compose.yml#L2)
  - command: `nl -ba docker-compose.yml | sed -n '1,220p'`
  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)

  - file: [extensions/ollama/src/setup.ts:406](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/extensions/ollama/src/setup.ts#L406)
