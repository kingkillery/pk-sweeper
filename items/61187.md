## Summary

Current `main` still lacks the `augmentModelCatalog` hooks this PR adds for Kimi and Moonshot, and the model-picker path depends on that hook for provider-owned supplemental rows. The bug described in the PR remains plausible on `main`, so there is not strong evidence to close it as already implemented or obsolete.

## Evidence

- **Kimi plugin on main has no catalog augmentation hook:** `api.registerProvider(...)` defines `catalog`, replay, thinking, and stream hooks, but no `augmentModelCatalog`, so Kimi-only picker rows are not added to the final merged catalog.
  - file: [extensions/kimi-coding/index.ts:32](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/extensions/kimi-coding/index.ts#L32)
  - command: `nl -ba extensions/kimi-coding/index.ts | sed -n '1,180p'`
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Moonshot plugin on main also lacks catalog augmentation:** Moonshot uses `defineSingleProviderPluginEntry(...)` with `catalog.buildProvider/buildStaticProvider`, but it does not register `augmentModelCatalog` on current `main`.
  - file: [extensions/moonshot/index.ts:16](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/extensions/moonshot/index.ts#L16)

- **The provider hook is the intended seam for picker-visible synthetic rows:** The provider contract explicitly documents `augmentModelCatalog` as the final catalog augmentation hook for extra rows that should appear in `models list` and model pickers.
  - file: [src/plugins/types.ts:1461](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/plugins/types.ts#L1461)

- **The picker loads the global model catalog, then filters it:** `promptDefaultModel(...)` calls `loadModelCatalog({ useCache: false })`, then `maybeFilterModelsByProvider(...)`. If provider-owned rows are missing from the final catalog, the picker falls back to whatever global rows are present.
  - file: [src/flows/model-picker.ts:437](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/flows/model-picker.ts#L437)

Without the hook, no supplemental provider rows are appended.
