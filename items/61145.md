## Summary

Keep open. Current main does not implement the requested opt-in WebChat/TUI default peer routing: the TUI default session path still resolves empty input to the global or main session only, and the `gateway.webchat.defaultPeerId` config is absent from current types, schema, and WebChat docs. The PR discussion also shows a concrete review defect was raised and then addressed by the author on the PR branch, so the open PR still appears to contain useful work rather than something safe to close.
