## Summary

Keep open. Current `main` still logs WhatsApp inbound messages with only `from`, `to`, `body`, media fields, and `timestamp`; it does not include participant identity or quoted-reply context in the gateway inbound log, even though the same function already carries those fields into the queued inbound message object.

## Evidence

There is no `senderJid`, `senderE164`, `senderName`, `replyToId`, `replyToBody`, or `replyToSender` in the logged object.
