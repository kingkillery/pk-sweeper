## Summary

Keep open. Current `main` does not implement the requested Feishu behavior broadly enough to justify closing this PR, and the PR as submitted has a concrete review-blocking bug: it introduces duplicate object keys, so the new unconditional `MessageThreadId` assignment would be overridden.
