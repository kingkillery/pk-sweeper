## Summary

Keep open. Current `main` still stores the web chat draft in reactive `@state()` and still mutates that state on every textarea input, so the performance fix proposed in PR #62790 is not already implemented. The follow-up explicit `requestUpdate()` sync points from the PR are also absent in current `main`, which means the PR has not been superseded by an equivalent landing.
