## Summary

Keep open: current `main` still implements `extensions/memory-lancedb` as vector-only LanceDB recall. I found no BM25/FTS index creation, no hybrid query path, and no user-facing hybrid config for this plugin.

## Evidence

- **Vector-only search implementation:** `MemoryDB.search()` calls `this.table!.vectorSearch(vector).limit(limit).toArray()` and converts `_distance` to a similarity score. There is no FTS index creation or hybrid search branch in this code path.
