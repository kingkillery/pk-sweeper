## Summary

Keep open. Current `main` has partial overlap with the PR's runtime-scoping fix, but the PR's broader provider-ownership work is not clearly implemented on `main`, and the discussion contains credible field evidence that the regression was real. I do not have strong enough evidence to say this PR is obsolete or already shipped.

## Evidence

- **PR discussion shows a real regression, not a speculative cleanup:** The PR body ties this to `memory-lancedb-pro` false validation after upgrading to `2026.4.8`, and comments from `davidbordenwi` and `BSTail` confirm real upgrade failures plus a local hotfix path that cleared the blocker. That makes this unsafe to dismiss without a clearly landed equivalent.
- **Current main covers the narrow 'empty owner stays scoped' case:** `resolvePluginProviders` now has regression coverage that unresolved explicit provider refs stay scoped and return no providers instead of broad-loading. The test at `src/plugins/providers.test.ts:1114` expects `onlyPluginIds: []` and `[]` providers for an unresolved explicit runtime owner path.
