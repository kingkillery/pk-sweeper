## Summary

Keep open. Current `main` still appears to have the reported image-model lookup gap for provider-local image IDs: config pairing returns `provider/model`, but the image runtime path still does a single exact registry lookup after normalization and does not apply the provider-prefixed fallback that OpenClaw already uses in shared media lookup code. The issue timeline also points to follow-up PRs (#68906, #68919), which supports that maintainers have not considered this resolved on `main` yet.

## Evidence

- **Config pairing still returns provider-prefixed refs for provider-local image IDs:** `resolveProviderVisionModelFromConfig` returns `${params.provider}/${id}` unless the configured model id already starts with the same provider prefix. For a Bailian config entry like `qwen3.5-plus`, this still produces `bailian/qwen3.5-plus`.
