## Summary

Keep open. Current main still lacks the proposed stable `openclaw/plugin-sdk/embeddings` subpath, package export, and entrypoint registration. The existing public surface remains `plugin-sdk/memory-core-host-engine-embeddings`, and bundled memory/provider plugins still import that memory-core-named path. The PR also addressed the Greptile docs comment in commit `299d6bd...`, and the author’s qa-channel smoke note points to unrelated PR #67492 rather than a defect in this embeddings change.

## Best Possible Solution

Keep the PR open for maintainer review. The concrete path is to land or independently implement a narrow `src/plugin-sdk/embeddings.ts` facade that re-exports `src/memory-host-sdk/engine-embeddings.ts`, add it to `package.json` exports and `scripts/lib/plugin-sdk-entrypoints.json`, update the public plugin SDK docs, and migrate bundled plugin imports if maintainers want the neutral stable contract.
