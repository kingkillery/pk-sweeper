## Summary

Keep open. Current `main` still only supports session reset policies at `session.reset`, `session.resetByType`, and `session.resetByChannel`. The proposed per-group shape like `channels.telegram.groups.<id>.session.reset` is not present in the Telegram group config type/schema and is not consulted by session reset resolution.

## Evidence

- **Telegram per-group config has no `session` field:** `TelegramGroupConfig` lists group overrides such as `requireMention`, `ingest`, `groupPolicy`, `tools`, `skills`, `topics`, `allowFrom`, and `systemPrompt`, but no nested `session` object or reset override.
  - file: [src/config/types.telegram.ts:247](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/config/types.telegram.ts#L247)
  - command: `nl -ba src/config/types.telegram.ts | sed -n '247,272p'`
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Schema would reject the issue's proposed shape:** `TelegramGroupSchema` is a strict object and enumerates allowed keys for `channels.telegram.groups.<id>`. It does not include `session`, so `channels.telegram.groups.<id>.session.reset` is not a supported config surface on current `main`.
  - file: [src/config/zod-schema.providers-core.ts:131](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/config/zod-schema.providers-core.ts#L131)

- **Session config only exposes global, by-type, and by-channel reset knobs:** `SessionConfig` defines `reset`, `resetByType`, and `resetByChannel`; there is no per-group/per-conversation reset map in the session config contract.
  - file: [src/config/types.base.ts:158](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/config/types.base.ts#L158)
