## Summary

Keep open. Current main still starts Feishu accounts concurrently through the gateway path that the issue reports, so the sequential startup-probe logic is not actually used for normal multi-account gateway startup.

## Evidence

- **Gateway starts channel accounts concurrently:** `startChannel` expands all account IDs and launches each `startAccount(...)` via `Promise.all(...)`, so multiple Feishu accounts boot in parallel during gateway startup.
  - file: [src/gateway/server-channels.ts:299](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/gateway/server-channels.ts#L299)
  - command: `nl -ba src/gateway/server-channels.ts | sed -n '286,336p' && nl -ba src/gateway/server-channels.ts | sed -n '404,432p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Feishu gateway startup always uses single-account mode:** Feishu's gateway entry logs `starting feishu[...]` and calls `monitorFeishuProvider` with `accountId`, matching the issue log shape and forcing the single-account code path.
  - file: [extensions/feishu/src/channel.ts:1203](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/extensions/feishu/src/channel.ts#L1203)
