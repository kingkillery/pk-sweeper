## Summary

Keep #67381 open. Current main still returns the old context-overflow messages that always recommend `agents.defaults.compaction.reserveTokensFloor`, so the linked user-facing problem is not implemented on main. The PR is a plausible focused bug-fix PR against the correct code path, but it should not be auto-closed because its latest review discussion still raises a real correctness concern: the proposed mismatch heuristic can diagnose generic fallback/model-switch overflow as heartbeat bleed without heartbeat-specific evidence.

## Best Possible Solution

Keep the PR open and ask for a revision rather than auto-closing it. The best product path is a small, tested helper on the existing overflow reset paths that reports the active model/context window, preserves the existing reserve-token guidance for normal compaction overflow, and only names heartbeat bleed when the runtime can prove a heartbeat model override was involved; otherwise use neutral small-context or model-mismatch guidance. A focused regression test around both overflow reply paths would make the fix maintainable.
