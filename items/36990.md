## Summary

This issue is a valid security-tool idea, but current OpenClaw scope puts optional/community capability in plugins or ClawHub rather than core. Current `main` already has built-in plugin/skill code-safety scanning and deep audit findings, so this is better tracked as an ecosystem integration than a core cleanup item.

## Close Comment

Closing as `clawhub` after a Codex review. The current project scope keeps optional capability in plugins/ClawHub rather than OpenClaw core (`VISION.md:54`, `VISION.md:68`, `VISION.md:101`), and community extensions are expected to publish on ClawHub (`docs/plugins/community.md:9`). Current `main` also already ships built-in code-safety scanning for plugin installs and gateway-backed skill installs (`src/plugins/install-security-scan.runtime.ts:750`, `src/plugins/install-security-scan.runtime.ts:845`) plus deep audit findings for installed plugins and skills (`src/security/audit-extra.async.ts:818`, `docs/gateway/security/audit-checks.md:99`). That makes Clawguard a reasonable ecosystem/plugin/ClawHub direction, but not a core OpenClaw issue to keep open.
