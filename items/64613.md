## Summary

Keep open. Current `main` still sanitizes `chat.history` for envelopes/internal-context delimiters, inline directive tags, tool-call XML, commentary-only assistant rows, and exact control tokens, but it does not filter plain heartbeat prompt text or arbitrary memory-system injection text before returning transcript rows to WebChat/Control UI.

## Evidence

- **Gateway history path has no heartbeat/system-prompt filter:** `chat.history` reads raw transcript messages, then applies `stripEnvelopeFromMessages(...)` and `sanitizeChatHistoryMessages(...)`. There is no call to `filterHeartbeatPairs(...)` or any generic system-injection filter in this path.
  - file: [src/gateway/server-methods/chat.ts:1896](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/gateway/server-methods/chat.ts#L1896)

  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **Gateway sanitizer only strips envelopes, inbound metadata, and internal-context wrappers:** `stripEnvelopeFromMessages` removes inbound metadata and envelope wrappers and runs `stripInternalRuntimeContext(...)`, but it does not match plain `Read HEARTBEAT.md ...` text or arbitrary plugin-provided memory prompt prose.
  - file: [src/gateway/chat-sanitize.ts:52](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/gateway/chat-sanitize.ts#L52)
