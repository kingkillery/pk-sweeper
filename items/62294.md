## Summary

Keep open. Current `main` still matches the reported heartbeat behavior: only interval wakes are gated by `nextDueMs`, non-interval wakes still advance the schedule as a fresh run, the scheduler is still a plain `setTimeout(...).unref()` with no overdue/drift catch-up, and the launchd plist builder still does not emit `ProcessType`. The issue's referenced fix PRs were provided as still open, so there is not strong evidence the item was implemented on `main`.

## Evidence

- **Interval gate still only applies to `reason === "interval"`:** `run()` computes `isInterval = reason === "interval"`; inside the agent loop it skips only when `isInterval && now < agent.nextDueMs`, then advances the schedule for any non-disabled run. That preserves the reported bypass for non-interval wake reasons.
