## Summary

Keep #30529 open. Current main already has automatic Discord typing for inbound replies and heartbeat delivery, but the requested message-tool typing action/explicit start-stop control is still not exposed in the shared action contract, Discord action discovery, or CLI/docs surface. The issue comments specifically called out explicit `action: typing` control as useful, so closing this as implemented would drop meaningful requested scope.
