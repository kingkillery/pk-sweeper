## Summary

Keep open. Current `main` still uses the pre-fix `skills.update` write path, there is no merged regression coverage for the reported disable flow, and a newer linked issue plus an open PR indicate the underlying bug is still being worked rather than already fixed on `main`.

## Evidence

- **`skills.update` still rewrites via `writeConfigFile` on current main:** Current main loads the config, mutates `skills.entries`, and persists with `writeConfigFile(...)` directly in the gateway handler. It does not use the source-snapshot mutation path proposed in the later fix PR, so the suspected write-path bug is still present in checked-out `main`.
