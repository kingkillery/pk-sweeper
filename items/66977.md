## Summary

Keep open. Current `main` still attempts to load `sqlite-vec` through `node:sqlite` extension loading, and a local Node 24.14.1 reproduction still fails with `Cannot enable extension loading because it was disabled at database creation.` OpenClaw now mitigates the impact by persisting embeddings in `chunks.embedding` and falling back to in-process cosine similarity when `sqlite-vec` is unavailable, but that is a degradation path, not a fix for the macOS `OMIT_LOAD_EXTENSION` platform-compat problem described in the issue.

## Evidence

- **Current loader still depends on node:sqlite extension loading:** `loadSqliteVecExtension` still calls `db.enableLoadExtension(true)` and then `loadExtension`/`sqliteVec.load`; there is no alternate macOS-specific backend or non-extension path here.
  - file: [packages/memory-host-sdk/src/host/sqlite-vec.ts:16](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/packages/memory-host-sdk/src/host/sqlite-vec.ts#L16)

- **Local reproduction still matches the reported runtime failure:** On this checkout's runtime (`node v24.14.1`), creating `new DatabaseSync(':memory:', { openWithExtensions: true })` and calling `enableLoadExtension(true)` still prints `Cannot enable extension loading because it was disabled at database creation.`
  - command: `node -e "const {DatabaseSync}=require('node:sqlite'); const db=new DatabaseSync(':memory:',{openWithExtensions:true}); try { db.enableLoadExtension(true); console.log('enable ok'); } catch (e) { console.log(String(e&&e.message||e)); }"`

## Risks / Open Questions

- The underlying macOS `node:sqlite`/`load_extension` incompatibility still appears real on current runtime, so closing would likely hide a legitimate platform gap.
- Current fallback preserves semantic recall but not native `sqlite-vec` acceleration, so the reported expectation of matching Linux extension-backed behavior is still unmet.
- The reporter's `embedding=[]` symptom may involve a separate provider or QMD-path issue; that ambiguity cuts against a high-confidence auto-close.
