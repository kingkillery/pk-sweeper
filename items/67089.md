## Summary

Keep open. Current `main` does not ship the proposed config-HMAC integrity feature, and the PR head has unresolved review blockers including a compile-breaking syntax error and an upgrade-path lockout risk. The scope-enforcement part is partly redundant because `main` already reserves all `config.*` gateway methods to `operator.admin` via shared method policy, but that is not enough to treat the PR as implemented or safely closeable.

## Risks / Open Questions

- The PR's `no-sig && suspicious` rejection path would lock out existing installations that have a normal `openclaw.json` but no preexisting sidecar signature file.
- The review thread identifies a compile-breaking syntax error in `src/config/io.ts` on the PR head, so the branch is not a safe merge/close candidate.
- The PR's HMAC token lookup bypasses injected `homedir` according to review, which weakens test coverage for the new integrity path and makes the branch harder to trust without further fixes.
