## Summary

Current `main` already has Control UI image upload UX, but the issue asks for broader attachment upload and processing (PDF/Word/Excel/docs). That broader request is not implemented, so there is not strong evidence to close this item.

## Evidence

- **Control UI has upload button, drag/drop, and paste handlers:** The chat view renders a file input plus paperclip button, handles `drop` on the chat surface, and intercepts `paste` for clipboard images.
