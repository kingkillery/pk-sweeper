## Summary

Current `main` does not implement this change: `workspaceAccess: "none"` is still read-only in the Docker/browser mount path, FS bridge, and memory-flush gate. The PR addresses a real behavior gap and the author responded to the in-thread hash/memory review comments, but the proposed fix is not yet the best complete fix because current OpenShell remote FS handling and public sandboxing/session-compaction docs still model `"none"` as non-writable.
