## Close Comment

Closing as a `cannot_reproduce` Codex review. The title-casing bug described in the original report does not match current `main`, and the reporter’s own follow-up on 2026-03-16 says that diagnosis was incorrect: the title-cased value was the display `name`, while the API-facing `id` passed through unchanged. On current `main`, OpenClaw still sends `model.id` directly in transport code (`src/agents/openai-transport-stream.ts:1126`) and persists `modelId` directly in session snapshots (`src/agents/pi-embedded-runner/replay-history.ts:547`). The Codex provider normalization also only canonicalizes the legacy `gpt-5.4-codex` alias and otherwise preserves ids as-is (`extensions/openai/openai-codex-provider.ts:151`). Separately, current `main` now suppresses stale `gpt-5.3-codex-spark` rows entirely with an explicit recovery hint (`extensions/openai/openai-codex-provider.ts:481`, `src/agents/pi-embedded-runner/model.test.ts:994`, `docs/concepts/model-providers.md:84`). If there is still a reproducible bug here, it should be reopened as a new issue with the exact failover path and logs for that newer behavior.
