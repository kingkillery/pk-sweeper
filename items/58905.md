## Evidence

- **Issue report has concrete repro and logs:** Issue #58905 describes webhook mode with appId, appSecret, verificationToken, and encryptKey; Feishu URL verification sends an encrypted body without X-Lark-Signature; OpenClaw responds 401 before returning the challenge. The issue includes a gateway log snippet and raw tcpdump request headers/body.
- **Related PR #58938 is still open and treats this as a real bug:** PR #58938 says the bug is that Feishu webhook mode rejects initial url_verification with 401 when encryptKey is configured, proposes detecting/responding to url_verification before ordinary signature auth, and adds unsigned plaintext/encrypted challenge plus unsigned non-challenge guard tests.
