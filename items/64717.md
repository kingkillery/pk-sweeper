## Summary

Keep open. Current `main` still ignores configured Telegram text chunk limits in `sendMessageTelegram` chunk planning, so the PR addresses a live bug rather than obsolete work.

## Evidence

- **Current send path still hardcodes 4000:** `buildChunkedTextPlan` in `extensions/telegram/src/send.ts` still calls `splitTelegramHtmlChunks(rawText, 4000)`, `splitTelegramPlainTextChunks(fallbackText, 4000)`, and `splitTelegramPlainTextFallback(..., 4000)`. That means `channels.telegram.textChunkLimit` is not consulted for this outbound path on current `main`.
  - file: [extensions/telegram/src/send.ts:751](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/telegram/src/send.ts#L751)
  - command: `nl -ba extensions/telegram/src/send.ts | sed -n '735,775p'`
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **Config-aware helper already exists:** OpenClaw already exposes `resolveTextChunkLimit(cfg, provider, accountId, opts)` and returns provider/account overrides before falling back. The PR is reusing an existing contract, not inventing new behavior.
  - file: [src/auto-reply/chunk.ts:56](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/auto-reply/chunk.ts#L56)

  - file: [extensions/telegram/src/bot-core.ts:435](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/telegram/src/bot-core.ts#L435)

  - file: [docs/channels/telegram.md:691](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/docs/channels/telegram.md#L691)

## Risks / Open Questions

- Configured lower per-channel or per-account Telegram chunk limits are still ignored in the `sendMessageTelegram` HTML/plain-text chunk-planning path on `main`.
- Because `send.ts` does not share the config-aware limit resolution already used elsewhere in Telegram, behavior can diverge between send flows and draft/streaming flows.
- Without the missing regression test, this inconsistency can persist or regress unnoticed.
