## Summary

Keep open. Current `main` and the latest release still clear Feishu docs with a single `documentBlock.list` call and no pagination in `clearDocumentContent`, so the reported large-document `write` bug does not appear implemented on `main` or covered by tests.

## Evidence

- **Current main still uses a single-page clear:** `clearDocumentContent` calls `client.docx.documentBlock.list` once, filters only `existing.data?.items`, and deletes `0..childIds.length` with no `page_token` loop.
