## Summary

Keep open. Current `main` already accepts browser-style clients in some pieces of the Gateway protocol, but the specific request in #66983 is not implemented: there is still no explicit `web` node platform in the node-command policy, and the current auth/pairing rules intentionally do not extend trusted-proxy or `dangerouslyDisableDeviceAuth` bypasses to node-role sessions.

## Evidence

- **Protocol accepts arbitrary platform strings:** `connect.params.client.platform` is a generic non-empty string, so the wire protocol does not reject `platform: "web"` outright.
  - file: [src/gateway/protocol/schema/frames.ts:24](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/src/gateway/protocol/schema/frames.ts#L24)

  - file: [src/gateway/node-command-policy.ts:59](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/src/gateway/node-command-policy.ts#L59)
