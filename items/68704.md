## Summary

Current `main` does not implement Gemma-specific stripping of replayed `reasoning_content` on OpenAI-compatible transports. The shared OpenAI-compatible replay policy still preserves prior thinking blocks by default, and the only explicit `reasoning_content` replay/strip handling in-tree is provider-specific DeepSeek logic. With a concrete report, a corroborating comment, and a linked open PR (#68763), this should stay open for maintainer follow-up rather than conservative cleanup.

## Evidence

- **Shared OpenAI-compatible replay policy lacks thinking-strip path:** `buildOpenAICompatibleReplayPolicy(...)` only sets tool-call ID sanitization and turn validation flags for OpenAI-compatible APIs. It does not set `dropThinkingBlocks` or any Gemma/model-specific replay override.
