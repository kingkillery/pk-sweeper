## Summary

Current `main` does not fully implement the requested always-visible token usage indicator for Control UI/WebChat. The codebase has a conditional high-usage warning and per-message context percentages, but not a persistent cumulative `used/limit (%)` counter in the chat header or status bar.

## Evidence

- **Chat notice only appears near the limit:** `getContextNoticeViewModel` suppresses the UI unless token totals are fresh and usage is at least 85% of the context window, so it is not a persistent indicator.
  - file: [ui/src/ui/chat/context-notice.ts:53](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/ui/src/ui/chat/context-notice.ts#L53)
  - command: `nl -ba ui/src/ui/chat/context-notice.ts | sed -n '44,83p'`
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **Notice is rendered inline above the input, not as a header/status-bar counter:** The chat view inserts `renderContextNotice(...)` in the transcript/input area, after fallback/compaction indicators and before the new-messages button.
  - file: [ui/src/ui/views/chat.ts:1192](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/ui/src/ui/views/chat.ts#L1192)
  - command: `nl -ba ui/src/ui/views/chat.ts | sed -n '1189,1193p'`
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **Existing `% ctx` UI is per assistant message, not cumulative session usage:** `grouped-render.ts` computes `contextPercent` from that message group's prompt tokens (`input + cacheRead + cacheWrite`) and renders it as a footer badge like `44% ctx`; it is not the persistent session-level `used/limit (%)` counter requested in the issue.
  - file: [ui/src/ui/chat/grouped-render.ts:427](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/ui/src/ui/chat/grouped-render.ts#L427)
  - command: `nl -ba ui/src/ui/chat/grouped-render.ts | sed -n '427,485p'`
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **Current docs still point users to `/status` and `/usage` for current token usage:** The token-usage docs describe `/status` and `/usage` as the ways to see current token usage in chat surfaces; they do not document an always-on Control UI/WebChat counter.
  - file: [docs/reference/token-use.md:62](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/docs/reference/token-use.md#L62)
  - command: `nl -ba docs/reference/token-use.md | sed -n '62,72p'`
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
