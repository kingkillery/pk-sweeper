## Summary

Keep PR #61416 open. Current main does not implement the requested `img_src` passthrough: the SearXNG result type, normalizer, and result payload mapper still only preserve `url`, `title`, `content`, and derived `siteName`. The provided PR remains a small, relevant plugin-local change, and the existing Greptile review positively supports that it is additive and test-covered rather than obsolete or superseded.
