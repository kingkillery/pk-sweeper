## Summary

Keep open. Current `main` does not implement the PR's Feishu `update` card-action flow or the Signal `signal-core` build fix, and the PR head still appears incomplete because `cardUpdate` is only wired into the single-agent dispatcher path, not the `broadcastAgents` active-dispatch path.

## Risks / Open Questions

- Closing as `implemented_on_main` would be incorrect because `main` still lacks the new Feishu `update` interaction kind, the pending-card-update flow, and the Signal `signal-core` build-fix surface.
- Merging the PR as-is still looks risky for Feishu group chats with `broadcastAgents`: the active broadcast dispatcher appears to bypass `cardUpdate`, which would fall back to normal reply delivery instead of patching the original card in place.
