## Summary

Keep open. Current `main` still implements Telegram ACP configured bindings only for group/forum-topic style conversations, not plain Telegram DMs, and the docs/tests still describe that narrower scope.

## Evidence

- **Telegram configured-binding compiler still rejects non-group chats:** `normalizeTelegramAcpConversationId()` returns `null` unless the parsed Telegram conversation has a `chatId` starting with `-`, so positive Telegram DM chat ids are excluded before ACP binding resolution.
  - file: [extensions/telegram/src/channel.ts:233](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/extensions/telegram/src/channel.ts#L233)
  - command: `nl -ba extensions/telegram/src/channel.ts | sed -n '233,268p'`
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **Inbound matcher still rejects non-group chats:** `matchTelegramAcpConversation()` applies the same `chatId.startsWith("-")` gate to inbound conversations, so even if a binding existed, non-group Telegram conversations would not match here.
  - file: [extensions/telegram/src/channel.ts:244](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/extensions/telegram/src/channel.ts#L244)
  - command: `nl -ba extensions/telegram/src/channel.ts | sed -n '244,268p'`

- **Current tests encode the same limitation:** `src/acp/persistent-bindings.test.ts` has a Telegram-specific test named `skips telegram non-group topic configs`, asserting that a positive Telegram conversation id does not resolve to a configured ACP binding.
  - file: [src/acp/persistent-bindings.test.ts:601](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/acp/persistent-bindings.test.ts#L601)

  - file: [docs/channels/telegram.md:514](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/docs/channels/telegram.md#L514)

  - file: [docs/tools/acp-agents.md:169](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/docs/tools/acp-agents.md#L169)

## Risks / Open Questions

- The report's root-cause section understates the gap: plain DM ids also fail because the Telegram ACP binding parser is topic-shaped, not just because of the negative-chat-id guard.
- Telegram DM topics/thread routing exist elsewhere on main, so future work here needs to distinguish plain DM configured bindings from existing topic/thread handling to avoid regression or scope confusion.
