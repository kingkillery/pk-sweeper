## Summary

Keep open. Current `main` does not implement `agents.list[].plugins.allow/deny` or any equivalent per-agent plugin lifecycle scoping. Plugin activation is still global at `plugins.allow`/`plugins.deny`, hooks run over all registered plugins without an agent-specific filter, and the only existing per-agent control is tool policy, which does not solve hook execution or prompt/context injection.

## Risks / Open Questions

- Closing this would be inaccurate because the requested config surface and runtime behavior are both still missing on `main`.
- Current per-agent `tools.allow/deny` is only a partial workaround; it does not prevent plugin hooks, context injection, or other non-tool plugin work from running for that agent.
