## Summary

Current `main` does not implement per-device or per-platform `typingIndicator` behavior for Feishu. The channel still models `typingIndicator` as a single boolean at the top level or per account, and reply dispatch only checks that merged account-level boolean. The inbound Feishu message context also carries no device/platform field, so there is no existing routing surface for desktop-vs-mobile typing behavior.

## Evidence

- **Schema only allows boolean typingIndicator:** `typingIndicator` is defined as `z.boolean().optional()` in the shared Feishu config shape, with a top-level default of `true`; there is no object or per-device schema.
