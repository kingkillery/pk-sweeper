## Summary

Keep PR #61298 open. Current main at b6af40f1f11e08d3c73d79c678351306bef4acbe does not implement the requested Android assistant launch hardening: App Action prompts are still only trimmed, assistant launch handling is not fingerprinted/restored through activity saved state, and `MainViewModel` routing state is still transient rather than restored via `SavedStateHandle`. The provided PR discussion also supports that the PR addresses real lifecycle and sanitization gaps, with only a maintainability concern noted by Greptile.
