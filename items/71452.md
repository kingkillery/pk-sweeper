## Summary

Keep open. Current main at be1d656 partially handles Teams-side pagination for pinned messages and channel lists, but it still contains the exact generic 25-row display cap reported in the issue. The CLI read, pins, and search text output paths still route through `renderMessageList`, so non-JSON output cannot show more than 25 returned rows even when the underlying action returns more. No related item in the provided context supersedes this request.

## Best Possible Solution

Keep this open and implement a narrow follow-up in the message command formatting/action contract: pass the requested limit or pagination metadata into `renderMessageList`, remove or make configurable the hardcoded 25-row cap, add regression tests for more than 25 read/list-pins/search rows, and ensure provider-specific pagination such as Teams `@odata.nextLink` remains covered.
