## Summary

Current `main` partially fixed the report: the cron tool and gateway schema now accept and pass through `payload.model` and `payload.timeoutSeconds`, and cron runtime/tests show those fields are honored. But `payload.contextTokens` and `maxSpendUsd` are still not part of the current cron payload contract anywhere on `main`, so the issue remains unresolved overall rather than safely closable.

## Evidence

- **Cron tool schema includes model/timeout only:** The agent-facing cron tool payload schema exposes `model` and `timeoutSeconds` alongside other agentTurn fields, but there is no `contextTokens` or `maxSpendUsd` field in the schema.
  - file: [src/agents/tools/cron-tool.ts:99](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/agents/tools/cron-tool.ts#L99)

- **Regression test locks current payload field list:** The schema test explicitly expects `job.payload` and `patch.payload` to contain `model` and `timeoutSeconds` and does not list `contextTokens` or `maxSpendUsd`, which matches the current implementation.
  - file: [src/agents/tools/cron-tool.schema.test.ts:91](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/agents/tools/cron-tool.schema.test.ts#L91)

  - file: [src/gateway/protocol/schema/cron.ts:4](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/gateway/protocol/schema/cron.ts#L4)
