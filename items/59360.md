## Summary

Keep open. The issue is recent, author is not protected, labels are unprotected, and current main still contains the reported Feishu card note behavior: `resolveCardNote` builds a colon-delimited footer (`Agent:`, `Model:`, `Provider:`) and that note is passed into both streaming and structured-card sends. The commenter’s proposed workaround/patch aligns with the problem statement, but it has not landed in main.
