## Summary

Current main still intentionally rejects Control UI sessions without device identity from non-localhost insecure HTTP contexts. The existing docs and tests describe the supported workarounds, but the user-facing report remains observable, #56830 is still an open/unmerged PR, and #71669 only tracks a narrower runtime-warning improvement. There is not a strong conservative close reason yet.
