## Summary

Keep open: current `main` still resolves `openclaw agent --agent <id>` without `--session-id` to the agent's existing main session, and omitted delivery channel still falls back to the prior `last` route. I did not find a warning banner, fresh-session flag, or behavior change that would justify closing this.

## Evidence

- **Agent runs default to the agent main session when `--session-id` is absent:** `resolveSessionKeyForRequest` computes `explicitSessionKey` from `resolveExplicitAgentSessionKey({ agentId })` whenever `sessionId` is missing, then reuses that session entry if it is still fresh. The same file sets `sessionId` to the existing stored id when the session is fresh instead of forcing a new one.
  - file: [src/agents/command/session.ts:147](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/src/agents/command/session.ts#L147)
  - command: `nl -ba src/agents/command/session.ts | sed -n '142,255p'`
  - sha: [ec8dbc459558](https://github.com/openclaw/openclaw/commit/ec8dbc4595583816f71af2568df61dbe7c090e1f)
- **Gateway handler mirrors the same default-main-session behavior:** In the `agent` RPC handler, `requestedSessionKey` is derived from `resolveExplicitAgentSessionKey({ cfg, agentId })` whenever `request.sessionId` is absent, so `--agent` alone targets the stored main session rather than creating a fresh session.
  - file: [src/gateway/server-methods/agent.ts:517](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/src/gateway/server-methods/agent.ts#L517)
  - command: `nl -ba src/gateway/server-methods/agent.ts | sed -n '512,540p'`
  - sha: [ec8dbc459558](https://github.com/openclaw/openclaw/commit/ec8dbc4595583816f71af2568df61dbe7c090e1f)
- **Tests explicitly expect resume of the existing main session:** The current gateway test suite asserts that whitespace/absent `sessionId` with `agentId: "main"` reuses `existing-session-id` and keeps `sessionKey` as `agent:main:main`. That is the behavior reported in the issue, not a fixed regression test.
  - file: [src/gateway/server-methods/agent.test.ts:1046](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/src/gateway/server-methods/agent.test.ts#L1046)
  - command: `nl -ba src/gateway/server-methods/agent.test.ts | sed -n '1046,1072p'`
  - sha: [ec8dbc459558](https://github.com/openclaw/openclaw/commit/ec8dbc4595583816f71af2568df61dbe7c090e1f)
- **Omitted channel still resolves through `last` delivery state:** `resolveAgentDeliveryPlan` normalizes a missing requested channel to `"last"`, and `resolveSessionDeliveryTarget` then maps `"last"` to the stored `lastChannel`/`lastTo` from session state. This matches the issue's claim that delivery defaults to the prior route.
  - file: [src/infra/outbound/agent-delivery.ts:51](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/src/infra/outbound/agent-delivery.ts#L51)

I did not find docs for a `--new-session`/`--fresh` flag or any documented warning on resume.
