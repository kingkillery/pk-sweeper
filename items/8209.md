## Summary

Current `main` has partial cron observability (`cron` event broadcasts, cron run history, and background-task records), but it does not implement the requested native start/completion integration surface. Outbound webhook delivery is only wired for `finished` events, and the core cron event payload still omits requested fields like `jobName` and `agentId`. No built-in `cron.webhooks.onJobStart/onJobComplete` or equivalent telemetry config was found.
