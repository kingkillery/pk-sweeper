## Summary

Keep open. Current `main` still omits `cfg` when memory-search resolution calls `getMemoryEmbeddingProvider`, so the PR addresses a real unfixed bug rather than something already implemented or obsolete.

## Evidence

- **Current main still drops `cfg` in memory-search resolution:** At `86f8c826e20a01d1bba67c0ba6e73182fbf0455f`, `mergeConfig` still calls `getMemoryEmbeddingProvider(provider)` and `getMemoryEmbeddingProvider(fallback)` without the config, and `resolveMemorySearchConfig` still calls `mergeConfig(defaults, overrides, agentId)` plus `getMemoryEmbeddingProvider(resolved.provider)` without forwarding `cfg`. This matches the PR's proposed fix and shows the bug is still present on `main`.
  - file: [src/agents/memory-search.ts:154](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/agents/memory-search.ts#L154)
  - command: `nl -ba src/agents/memory-search.ts | sed -n '145,392p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Provider runtime already supports config-aware resolution:** `getMemoryEmbeddingProvider` accepts `cfg?: OpenClawConfig` and forwards it to `resolvePluginCapabilityProvider`. The caller omission in `memory-search.ts` therefore changes behavior, because the runtime path is explicitly designed to resolve providers with config when available.
  - file: [src/plugins/memory-embedding-provider-runtime.ts:33](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/plugins/memory-embedding-provider-runtime.ts#L33)
  - command: `nl -ba src/plugins/memory-embedding-provider-runtime.ts | sed -n '1,60p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Capability resolution uses `cfg` to build allowlist compat config:** `resolvePluginCapabilityProvider` and `resolvePluginCapabilityProviders` derive bundled-plugin allowlist compat from `params.cfg` before loading the runtime registry. If callers pass `undefined`, this path cannot preserve the source config's `plugins.allow` state.
  - file: [src/plugins/capability-provider-runtime.ts:65](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/plugins/capability-provider-runtime.ts#L65)
  - command: `nl -ba src/plugins/capability-provider-runtime.ts | sed -n '65,142p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Related discussion points to the same unfixed callsite:** The provided PR discussion includes an April 17, 2026 reproduction on `2026.4.15` with a stack trace landing at `mergeConfig -> getMemoryEmbeddingProvider`, and notes at least five open reports of the same false `plugins.allow is empty` warning. That discussion is consistent with the current `main` code path.
