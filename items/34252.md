## Summary

Keep #34252 open. Current main has useful partial work toward provider-specific Ollama baseUrl routing, but I do not see enough end-to-end or documented support to conservatively close a user request for multiple independently configured Ollama providers/hosts. The issue discussion includes multiple users reporting this for GPU/CPU split, multi-host Ollama, and an `ollama-jetson` auth failure, so the remaining surface is meaningful.

## Evidence

- **Issue scope:** The provided issue context asks for multiple Ollama instances, each with a unique baseUrl, and comments report `ollama-jetson` failing auth plus multi-host Ollama needs. That is broader than a single custom base URL.
- **Partial implementation: provider-specific baseUrl lookup:** The Ollama provider's createStreamFn now passes the selected provider id into `resolveConfiguredOllamaProviderConfig`, which can select the matching configured provider baseUrl.
