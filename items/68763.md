## Summary

Keep open. Current `main` still lacks any Gemma-specific "strip all prior reasoning from replay history" policy, and this PR's approach is incomplete because `resolveTranscriptPolicy` prefers provider-owned replay hooks for the common Ollama/OpenAI-compatible routes where the bug was reported.

## Evidence

- **Current replay cleanup only applies Claude-style drop logic:** `sanitizeSessionHistory()` still gates replay cleanup on `policy.dropThinkingBlocks` only. That path preserves the latest assistant thinking block instead of stripping reasoning from all prior assistant turns, which does not satisfy the Gemma 4 contract described in the linked bug.

  - file: [src/agents/pi-embedded-runner/thinking.ts:115](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/src/agents/pi-embedded-runner/thinking.ts#L115)
