## Summary

Keep open. Current `main` does not implement this PR's timestamp backfill, and the motivating bug appears only partially mitigated by a separate macOS-side lossy decode. Legacy cron jobs can still lack required `createdAtMs`/`updatedAtMs`, so the core contract remains inconsistent even if the macOS Cron tab no longer hard-fails.

## Evidence
