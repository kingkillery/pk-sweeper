## Summary

Keep open. Current main still has the reported mismatch: `models status --agent <id>` resolves and uses the target `agentDir` for the non-probe auth overview, but the live probe planning path still calls `ensureAuthProfileStore()` without an agent directory, so omitted-agent fallback resolves the default OpenClaw agent store. The linked fix PR #67263 matches the right direction, but its changes are not present on current main and the provided PR context says it was closed unmerged.

## Best Possible Solution

Keep #67235 open and revive or reimplement the narrow #67263 fix: add an optional agentDir to buildProbeTargets and runAuthProbes, pass the agentDir already resolved by modelsStatusCommand, and add regression tests that assert the target agentDir reaches ensureAuthProfileStore while the no-agent path still uses the default store.
