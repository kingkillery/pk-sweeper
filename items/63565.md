## Summary

Current `main` still sends built-in `web_fetch` and the Firecrawl provider through strict guarded-fetch paths that do not opt into env-proxy mode. The corresponding tests explicitly assert that strict mode ignores env proxies, so the reported proxy-only failure remains plausible and is not implemented on `main`.

## Evidence

- **Built-in web_fetch still omits env-proxy opt-in:** `runWebFetch` calls `fetchWithWebToolsNetworkGuard` without passing `useEnvProxy: true`; only URL, redirects, timeout, lookupFn, optional policy, and headers are forwarded.
  - file: [src/agents/tools/web-fetch.ts:415](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/agents/tools/web-fetch.ts#L415)
