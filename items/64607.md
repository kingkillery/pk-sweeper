## Summary

Current `main` already supports inline image/audio/video attachments in the Control UI, so the issue premise is partly outdated. But there is still no clear implementation of automatic web/link previews or unfurling, and the rich-output path for web content is a separate manual `[embed ...]` mechanism rather than the requested general inline preview behavior. This is partial coverage, not a full match for the issue.
