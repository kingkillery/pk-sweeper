## Summary

Keep open. Current `main` still lacks both proposed fixes, so this is not stale or already implemented, but the PR as submitted has unresolved correctness risks: the `typeof` change can swallow `tool_calls` when a provider emits `content: ""` in the same chunk, and the `logging.file` hot-reload change would clear the paired source snapshot that later config writes rely on to preserve source refs/secrets.
