## Summary

Keep open. Current `main` still has the original 5s/2s/3s polling, so this PR is not already implemented, and the existing review comment identifies a real regression risk: increasing logs polling to 5s would make the live Logs tab skip data on busy gateways because each `logs.tail` fetch still caps at 500 lines / 250 KB and the server resets to the newest `maxBytes` chunk once backlog exceeds that window.

## Evidence

- **Current main still uses the original polling cadence:** `startNodesPolling`, `startLogsPolling`, and `startDebugPolling` still poll at 5000 ms, 2000 ms, and 3000 ms on `main`, so the PR has not been implemented upstream.
  - file: [ui/src/ui/app-polling.ts:15](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/ui/src/ui/app-polling.ts#L15)

  - sha: [ec8dbc459558](https://github.com/openclaw/openclaw/commit/ec8dbc4595583816f71af2568df61dbe7c090e1f)
- **Logs fetches still use a 250 KB window:** The Control UI default state still sets `logsLimit = 500` and `logsMaxBytes = 250_000`, and `loadLogs()` passes those values to `logs.tail` on every poll.
  - file: [ui/src/ui/app.ts:519](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/ui/src/ui/app.ts#L519)

- **Gateway log tail resets when backlog exceeds maxBytes:** `readLogSlice()` treats `size - start > maxBytes` as overflow, marks the response as `reset = true`, and jumps the read start to `size - maxBytes`. That means older log lines are skipped if more than the allowed byte window arrives between polls.
  - file: [src/logging/log-tail.ts:84](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/src/logging/log-tail.ts#L84)

- **Tab switches already force a reset fetch:** When the user enters the Logs tab, `refreshActiveTab()` immediately calls `loadLogs(app, { reset: true })`. That helps on tab entry, but it does not solve data loss during steady-state live tailing if the polling interval is lengthened without widening the fetch window.
  - file: [ui/src/ui/app-settings.ts:397](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/ui/src/ui/app-settings.ts#L397)
