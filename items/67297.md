## Summary

Keep open. Current `origin/main` still computes the preflight and memory-flush gates from `reserveTokensFloor`, so the user-configured `agents.defaults.compaction.reserveTokens` still does not affect those thresholds on main. That matches the PR’s stated bug, and there is no shipped fix on main to justify closure. A duplicate open PR (#69379) exists, but duplication is not an allowed close reason here and does not prove the behavior is already resolved.

## Evidence

There is no read of configured `compaction.reserveTokens` in this path.
