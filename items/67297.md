## Summary

PR #67297 should close as superseded by #69379. Current main still has the underlying reserveTokens/reserveTokensFloor asymmetry, but #69379 is the stronger canonical fix for the same #66830 root cause because it updates the shared gate semantics, adds regression tests, and documents the change path more completely.

## Best Possible Solution

Close #67297 as superseded and focus maintainer review on #69379, or on an equivalent single canonical patch that updates the shared memory-flush/preflight reserve semantics, preserves the documented `reserveTokens` vs `reserveTokensFloor` contract, and lands regression coverage for #66830.
