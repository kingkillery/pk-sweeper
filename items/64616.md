## Summary

Keep open. Current `main` still lacks the ACP zombie-session reconciliation described in PR #64616, and the PR as posted still has unresolved review concerns.

## Evidence

- **Current main still treats any ACP store entry as a live backing session:** `hasBackingSession` reads the ACP session entry and returns `Boolean(acpEntry.entry)`, and `shouldMarkLost` only checks `!hasBackingSession(task)`. There is no current-main logic that inspects ACP `state`, `lastActivityAt`, or missing transcript state for stale ACP tasks.
