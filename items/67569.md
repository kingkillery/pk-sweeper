## Summary

Keep open. Current main still lacks the core behavior this PR is trying to add: gateway model catalog loads do not populate `MODEL_CONTEXT_TOKEN_CACHE`, and `sessions.list` builds Control UI rows synchronously without loading the catalog. The PR addresses a real linked UI bug, but it also has unresolved review concerns around cache key semantics and the proposed 200k fallback, so the best maintainer path is review/update rather than auto-close.

## Best Possible Solution

Keep the PR open for maintainer review. The best fix belongs in core gateway/session context resolution: make `sessions.list` resolve discovery-backed context windows without starting unsafe async work, populate or query the model catalog cache with non-colliding provider-aware semantics, reset cache state cleanly in tests, and add regression coverage for Anthropic/Claude, provider aliases, slash-containing IDs, and non-200k discovery-only models.
