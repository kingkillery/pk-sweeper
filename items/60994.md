## Summary

Keep open. Current `main` still routes model-provider HTTP through the SSRF guard and only permits private/LAN targets when `models.providers.<id>.request.allowPrivateNetwork` is explicitly enabled, so the issue’s primary report is not implemented away. The secondary slug-generator timeout complaint does appear fixed on `main`, but that only resolves part of the issue.
