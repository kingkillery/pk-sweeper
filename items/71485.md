## Summary

Current main fixes the reported Feishu card-mode UX problem for the default `main` agent. `resolveCardHeader` now returns `undefined` when the agent id is `main` and no explicit outbound identity name/emoji supplies a title, and both streaming and static card paths pass that optional header through. A regression test covers `renderMode: "card"` with streaming enabled and disabled. The fix is on main after `v2026.4.24`, so I could not identify a released version that contains it yet.

## Best Possible Solution

Close this issue as implemented on main. Keep the current behavior that suppresses only the meaningless default `main` header while preserving explicit identity/non-main headers; use the narrower related Feishu card-header customization items for any future config surface.
