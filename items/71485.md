## Summary

Keep open. Current `main` still adds a Feishu card header derived from the agent identity or `agentId` (so the default agent shows `main`), the Feishu config surface does not expose any header/title disable option, and the tests/docs still reflect that behavior.

## Evidence

- **Card header falls back to agentId:** `resolveCardHeader()` builds the Feishu card header from `identity?.name?.trim() || agentId`, which explains why the default agent renders `main` as the card title when no custom identity name is set.
  - file: [extensions/feishu/src/reply-dispatcher.ts:69](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/extensions/feishu/src/reply-dispatcher.ts#L69)
  - command: `nl -ba extensions/feishu/src/reply-dispatcher.ts | sed -n '65,95p'`
  - sha: [ec8dbc459558](https://github.com/openclaw/openclaw/commit/ec8dbc4595583816f71af2568df61dbe7c090e1f)
- **Card replies always pass a header on the main reply path:** When Feishu replies use card rendering, `createFeishuReplyDispatcher()` unconditionally computes `cardHeader` and passes it to `sendStructuredCardFeishu()`. That means the structured-card path on current `main` does not produce headerless cards.
  - file: [extensions/feishu/src/reply-dispatcher.ts:470](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/extensions/feishu/src/reply-dispatcher.ts#L470)
  - command: `nl -ba extensions/feishu/src/reply-dispatcher.ts | sed -n '455,495p'`
  - sha: [ec8dbc459558](https://github.com/openclaw/openclaw/commit/ec8dbc4595583816f71af2568df61dbe7c090e1f)
- **Structured cards support omitting headers, but current reply path does not use that option:** `buildStructuredCard()` only adds `card.header` when `options?.header` is present, so the underlying card builder can be headerless. However, the active Feishu reply dispatcher always supplies a header, so the requested behavior is not implemented on `main`.
  - file: [extensions/feishu/src/send.ts:631](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/extensions/feishu/src/send.ts#L631)
  - command: `nl -ba extensions/feishu/src/send.ts | sed -n '616,705p'`
  - sha: [ec8dbc459558](https://github.com/openclaw/openclaw/commit/ec8dbc4595583816f71af2568df61dbe7c090e1f)
- **No config knob exists for disabling the card header/title:** The Feishu config schema exposes `renderMode`, `streaming`, `replyInThread`, `typingIndicator`, and related settings, but there is no `showHeader`, `hideTitle`, or equivalent option in the shared/top-level Feishu config shape.
  - file: [extensions/feishu/src/config-schema.ts:156](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/extensions/feishu/src/config-schema.ts#L156)
  - command: `nl -ba extensions/feishu/src/config-schema.ts | sed -n '38,70p;146,190p;220,236p'`
  - sha: [ec8dbc459558](https://github.com/openclaw/openclaw/commit/ec8dbc4595583816f71af2568df61dbe7c090e1f)
- **User-facing docs list Feishu options but no header toggle:** The Feishu docs document streaming and typing-related options, and the configuration table lists those settings, but there is no documented option to suppress the card header/title while keeping card rendering enabled.
  - file: [docs/channels/feishu.md:238](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/docs/channels/feishu.md#L238)
