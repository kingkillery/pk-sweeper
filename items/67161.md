## Summary

Current `main` still appears vulnerable to this ACP restart path. Gateway restart deferral only tracks queue/reply/embedded/task activity, not active ACP turns, and the startup ACP reconcile path is best-effort metadata repair rather than recovery of an interrupted in-flight ACP turn. I did not find code or tests on `main` that prove #67161 is fixed.

## Evidence

- **Gateway restart deferral does not account for ACP turns:** `requestGatewayRestart()` computes pending work from command queue size, pending replies, embedded runs, and task runs only. There is no ACP session/turn counter in the restart gate, so config-triggered restarts can still proceed without ACP-specific drain protection.
  - file: [src/gateway/server-reload-handlers.ts:212](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/src/gateway/server-reload-handlers.ts#L212)

  - sha: [7875092f4ddc](https://github.com/openclaw/openclaw/commit/7875092f4ddc6f2dbe218bea8506c9ac1da87c39)
- **Process-wide pre-restart check uses the same non-ACP counters:** The SIGUSR1 pre-restart deferral check sums the same four counters and likewise omits ACP activity, reinforcing that ACP turns are not part of the restart drain contract on current `main`.
  - file: [src/gateway/server.impl.ts:315](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/src/gateway/server.impl.ts#L315)

- **ACP startup recovery is only a best-effort identity reconcile:** On startup, the gateway only calls `reconcilePendingSessionIdentities({ cfg })` and logs `checked/resolved/failed`. This is metadata reconciliation after boot, not explicit recovery of an interrupted in-flight ACP turn or protection against `ACP_TURN_FAILED` during restart.

- **Identity reconcile has narrow scope and skips some pending-session states:** `reconcilePendingSessionIdentities()` only inspects ACP sessions whose identity is still pending and already has a stable runtime session id; it skips others entirely. The accompanying test locks that behavior in, so restart recovery is not comprehensive for all interrupted ACP states.
