## Summary

Keep open. Current `main` contains routing fixes for the cross-channel leakage symptom and a DM-scope migration cleanup path, but the remaining issue claim is narrower: that upgrading to `2026.3.2` reset an existing persisted `session.dmScope` to `main`. In the inspected checkout, onboarding defaults explicitly preserve existing `dmScope` values, and I did not find a separate upgrade/config migration writer that proves or disproves the reported reset. Without config snapshots or a reproducible 2026.3.2 upgrade path, the evidence is not strong enough to close conservatively.
