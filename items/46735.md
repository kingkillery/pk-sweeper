## Summary

Current `main` does not show this NVIDIA-specific normalization implemented in the embedded-runner wrapper path, so there is not strong evidence to close the PR. The generic OpenAI transport now downgrades unsupported `minimal` effort to `low`, but that logic is gated on model metadata and lives in `openai-transport-stream.ts`, while `applyExtraParamsToAgent()` still has no NVIDIA/NVIDIA NIM wrapper and the bundled NVIDIA plugin does not provide one.
