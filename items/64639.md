## Summary

Keep open. Current `main` still treats `~/.agents/skills` as an unconditional built-in skill root, and the documented/configured `skills.load` surface does not provide any way to disable that source. Existing mitigations only limit visibility or prompt size after loading; they do not stop the personal source from being scanned.

## Evidence

- **Loader still unconditionally scans `~/.agents/skills`:** `loadSkillEntries()` resolves the OS home directory, builds `~/.agents/skills`, and always calls `loadSkills({ source: "agents-skills-personal" })` before merging results. There is no config gate around that source.

## Risks / Open Questions

- Closing as `implemented_on_main` would be incorrect: the requested source-disable behavior is not present in code, types, docs, or tests.
- Closing as `clawhub` would be a scope mismatch: this is a core loader/config control over built-in skill roots, not a new optional skill/plugin capability.
- Users with large `~/.agents/skills` trees can reduce visible prompt surface with allowlists or prompt caps, but current main still scans that personal source, so the report remains materially valid.
