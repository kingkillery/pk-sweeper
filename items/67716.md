## Summary

Keep #67716 open. Current main still schedules deferred background maintenance only for turn maintenance, while bootstrap maintenance is called through the foreground path and background runs are the only path that opt into consuming deferred compaction debt. The hot-cache policy details appear to live outside this repo in the lossless-claw plugin, but the OpenClaw host scheduling/API surface still has a plausible unresolved gap for debt created outside normal turn maintenance.

## Best Possible Solution

Keep the issue open and fix it with a narrow host-side regression path first: model a context-engine maintain call that records deferred debt during bootstrap/reconcile, then ensure OpenClaw schedules or runs an allowed background maintenance pass to consume it. In parallel, coordinate with lossless-claw on the hot-cache policy so deferred debt is either paid down, explicitly superseded, or surfaced through a maintain result/reason that OpenClaw can schedule around.
