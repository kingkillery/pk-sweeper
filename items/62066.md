## Summary

Keep open. Current `main` partially addresses the Slack-thread case with a Slack-specific `channels.slack.thread.requireExplicitMention` option, but it does not implement the requested cross-channel `requireMention: "strict"`/implicit-mention opt-out. Discord still treats reply-to-bot as an implicit mention with no comparable strict-mode policy hook.

## Evidence

- **Slack has only a channel-specific workaround, not the requested generic strict mode:** Slack config now exposes `thread.requireExplicitMention`, documented as suppressing implicit thread mentions only for Slack threads.
