## Summary

Keep open. Current `main` has only the narrower `chatRunId`-based deferral for `session.message` reloads and still reloads on every `final` chat event. The PR’s remaining behavior on tracked finals, matching follow-up `session.message` suppression, and broader regression coverage is not implemented on `main`, so this is not closeable as `implemented_on_main` or `cannot_reproduce`.
