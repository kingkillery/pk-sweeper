## Summary

Current OpenClaw already includes the shipped #29112 hybrid-recall fix that preserves keyword-backed memory hits when strict hybrid scoring plus default `minScore` would otherwise return an empty result set. The latest release and current `main` still carry that logic, so this issue's reported regression is already implemented on the maintained line.

## Evidence

- **Shipped fix noted in changelog:** `CHANGELOG.md:3748` explicitly records `#29112`: "Memory/Hybrid recall: when strict hybrid scoring yields no hits, preserve keyword-backed matches using a text-weight floor so freshly indexed lexical canaries no longer disappear behind minScore filtering." The surrounding header at `CHANGELOG.md:3622` places that in release `2026.2.27`.
  - file: [CHANGELOG.md:3748](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/CHANGELOG.md#L3748)

- **Current implementation matches the fix:** `extensions/memory-core/src/memory/manager.ts:415-437` keeps keyword-backed matches when strict hybrid scoring returns nothing above `minScore`, using `relaxedMinScore = Math.min(minScore, hybrid.textWeight)` and retrying on keyword-backed rows instead of returning an empty list.
  - file: [extensions/memory-core/src/memory/manager.ts:415](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/extensions/memory-core/src/memory/manager.ts#L415)

## Close Comment

Closing from a Codex review because the reported failure mode is already implemented in the current OpenClaw line. The shipped changelog entry for `#29112` in `CHANGELOG.md:3748` says memory hybrid recall now preserves keyword-backed matches behind the `minScore` threshold, and the current implementation in `extensions/memory-core/src/memory/manager.ts:415-437` still does exactly that via the text-weight floor fallback. The CLI path in `extensions/memory-core/src/cli.runtime.ts:1177-1212` just returns `manager.search()` results and prints `No matches.` when that search is empty, so this specific regression path is already covered by the shipped fix. If someone is still seeing empty results on a current build, please open a fresh issue with exact current version, `openclaw memory status --json --deep`, and current config/runtime details, because that would likely be a different bug than the one tracked here.
