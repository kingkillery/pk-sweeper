## Summary

Keep open. Current `main` still materializes full session rows before `search`, `activeMinutes`, and `limit`, and each row can still trigger transcript fallback work plus child-session scans. The optimization described in PR #68878 is not implemented on `main` and was not shipped in `v2026.4.23`.

## Evidence
