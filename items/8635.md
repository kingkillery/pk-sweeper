## Summary

Current `main` does not expose a structured token-usage API from `session_status`. The tool still takes only `sessionKey`/`model`, renders usage into a text status card, and returns only `ok`, `sessionKey`, `changedModel`, and `statusText` in `details`.

## Evidence

- **`session_status` schema has no usage/output fields:** The tool schema only defines optional `sessionKey` and `model` parameters; there is no request or response contract for structured usage/budget data.
  - file: [src/agents/tools/session-status-tool.ts:55](https://github.com/openclaw/openclaw/blob/c0a7b6a510758645c5267734b984f4885a605b67/src/agents/tools/session-status-tool.ts#L55)
  - command: `nl -ba src/agents/tools/session-status-tool.ts | sed -n '1,80p'`
  - sha: [c0a7b6a51075](https://github.com/openclaw/openclaw/commit/c0a7b6a510758645c5267734b984f4885a605b67)
- **`session_status` returns text plus minimal metadata only:** The execute path builds `statusText` and returns `details` with only `ok`, `sessionKey`, `changedModel`, and `statusText`. No `usage`, `lastTurn`, or budget object is returned.
  - file: [src/agents/tools/session-status-tool.ts:547](https://github.com/openclaw/openclaw/blob/c0a7b6a510758645c5267734b984f4885a605b67/src/agents/tools/session-status-tool.ts#L547)
