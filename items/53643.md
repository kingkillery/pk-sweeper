## Summary

Keep open. I did not find strong evidence that current `main` fixes this report. The current Agents > Skills UI still hides workspace skills inside collapsed `<details>` groups by default, and skill status is still computed directly from requirement metadata for workspace-installed skills, so both the "list seems to disappear" and "shows missing even though usable" complaints remain plausible on `main`. Existing UI tests only cover the tab-count wiring, not workspace-skill visibility persistence or status rendering.

## Evidence

- **Workspace skill groups still default closed in the agent Skills panel:** `renderAgentSkills` groups filtered skills, then `renderAgentSkillGroup` collapses `workspace` and `built-in` groups by default via `<details ?open=${!collapsedByDefault}>`. That means workspace-installed skills can still appear hidden again after the panel rerenders.
  - file: [ui/src/ui/views/agents-panels-tools-skills.ts:837](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/ui/src/ui/views/agents-panels-tools-skills.ts#L837)
  - sha: [ec8dbc459558](https://github.com/openclaw/openclaw/commit/ec8dbc4595583816f71af2568df61dbe7c090e1f)
- **Current skill status still comes from requirement evaluation, including workspace installs:** `buildSkillStatus` resolves the skill source, evaluates requirements, and marks `eligible` from `requirementsSatisfied` without any special handling for workspace-installed ClawHub skills that are functionally usable despite metadata mismatch. That leaves the reported false "missing" status plausible on current `main`.
  - file: [src/agents/skills-status.ts:170](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/src/agents/skills-status.ts#L170)
  - sha: [ec8dbc459558](https://github.com/openclaw/openclaw/commit/ec8dbc4595583816f71af2568df61dbe7c090e1f)
- **Current UI tests do not cover this regression surface:** The nearby agent UI test only checks whether the Skills tab count is shown for the selected agent. It does not test workspace-skill group expansion/persistence, rerender behavior after navigation, or the rendered status for workspace-installed skills.
  - file: [ui/src/ui/views/agents.test.ts:126](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/ui/src/ui/views/agents.test.ts#L126)
  - sha: [ec8dbc459558](https://github.com/openclaw/openclaw/commit/ec8dbc4595583816f71af2568df61dbe7c090e1f)
- **No targeted fix surfaced in local history search:** A local history search over Control UI and skills-related commits in the touched areas only surfaced unrelated recent UI fixes plus the generic `fix: keep control ui bundle browser-safe` change, not a clear fix for agent skill-list visibility or false missing status.
  - command: `git log --oneline --all --grep='Control UI' --grep='skills' -i -- ui/src/ui src/agents src/gateway | head -n 120`
