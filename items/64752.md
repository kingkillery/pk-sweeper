## Summary

Keep open. Current `main` still turns Telegram `message_reaction` updates into queued system/context events only; it does not request a wake/heartbeat or otherwise start an agent turn from the reaction itself. The reporter's exact config key is stale on current `main`, but the underlying bug remains unresolved.
