## Summary

Close as `cannot_reproduce`: on current `main`, `isSafeRelativeModulePath` is only used behind `z.string().refine(...)`, and OpenClaw currently depends on Zod 4.3.6; Zod’s docs say refinements do not run after an `invalid_type` failure and that Zod ensures the value matches the schema type before calling the refinement callback. citeturn1view1turn1view2 The reported non-string `TypeError` path is therefore not reachable from the current schema, and the PR’s added tests also conflict with current exports/path rules.
