## Summary

Current `main` ships Active Memory as a bundled plugin, but there is still no evidence that the scoped install path `@openclaw/active-memory` is supported. The plugin has a manifest and docs for config-based enablement, while the bundled-install fallback code only recognizes npm specs that come from bundled package metadata; `extensions/active-memory` has no `package.json`, so that fallback cannot map `@openclaw/active-memory` to the bundled plugin. This means the report still describes a real mismatch worth keeping open.

## Evidence

- **Active Memory exists on main as a bundled plugin:** The repository contains a bundled Active Memory plugin manifest with id `active-memory`.
  - file: [extensions/active-memory/openclaw.plugin.json:2](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/active-memory/openclaw.plugin.json#L2)
  - command: `nl -ba extensions/active-memory/openclaw.plugin.json | sed -n '1,40p'`
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **Current user-facing docs enable it through config, not npm/ClawHub install:** The Active Memory docs tell users to add `plugins.entries."active-memory"` to `openclaw.json` and restart the gateway; they do not document installing `@openclaw/active-memory`.
  - file: [docs/concepts/active-memory.md:21](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/docs/concepts/active-memory.md#L21)

  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **Bundled plugin install docs only guarantee bare-id installs:** The plugins CLI docs say a bundled plugin is installed directly when the bare install spec matches a bundled plugin id, using `diffs` as the example. They do not say scoped specs like `@openclaw/<id>` resolve to bundled plugins.
  - file: [docs/cli/plugins.md:117](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/docs/cli/plugins.md#L117)

- **CLI fallback for scoped npm specs requires bundled npm metadata:** `resolveBundledInstallPlanForNpmFailure` only falls back when `findBundledSource` can resolve the failed npm spec, and bundled sources only get `npmSpec` from bundled package metadata (`install.npmSpec` or `packageName`). Without that metadata, `@openclaw/active-memory` cannot map back to the bundled plugin.
  - file: [src/cli/plugin-install-plan.ts:68](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/cli/plugin-install-plan.ts#L68)

- **Active Memory has no bundled package metadata to back a scoped npm alias:** `extensions/active-memory` contains `index.ts`, tests, and `openclaw.plugin.json`, but no `package.json`, unlike bundled plugins that expose scoped package names.
