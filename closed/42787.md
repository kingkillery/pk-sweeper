## Summary

Closing PR #42787 as cannot reproduce. Current main still omits `--bind` from the generated gateway service args, but the supervised gateway path already preserves bind mode through config loading: the installed service environment carries profile/state/config context, `gateway run` loads config at startup, and bind defaults to `cfg.gateway?.bind` when no CLI `--bind` is present.

## Evidence

- **Gateway install plan keeps config context for supervised services:** `buildGatewayInstallPlan()` returns service environment from `buildGatewayInstallEnvironment()` after generating args. The install path passes full config into that environment builder, not into gateway args.
  - file: [src/commands/daemon-install-helpers.ts:275](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/commands/daemon-install-helpers.ts#L275)

  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Installed service environment preserves profile/state/config path:** `buildServiceEnvironment()` writes `OPENCLAW_PROFILE`, `OPENCLAW_STATE_DIR`, and `OPENCLAW_CONFIG_PATH` into the managed service environment, along with `HOME` and the gateway port.
  - file: [src/daemon/service-env.ts:273](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/daemon/service-env.ts#L273)

- **Gateway runtime resolves bind from config when CLI flag is absent:** `gateway run` loads config via `readGatewayStartupConfig()` and then computes `bindExplicitRawStr` from `opts.bind ?? cfg.gateway?.bind`, so supervised launches without `--bind` still honor configured `gateway.bind`.
  - file: [src/cli/gateway-cli/run.ts:414](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/cli/gateway-cli/run.ts#L414)

- **Status path also treats bind as config-driven, not arg-driven:** Daemon status parses port from service args, but bind mode comes from `params.daemonCfg.gateway?.bind ?? "loopback"`, reinforcing that bind is expected to come from config rather than embedded service args.
  - file: [src/cli/daemon-cli/status.gather.ts:265](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/cli/daemon-cli/status.gather.ts#L265)
