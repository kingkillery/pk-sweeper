## Summary

Current `main` already supersedes this PR with a broader Discord partial-channel fix: slash-command dispatch now uses safe channel-property access and fetched thread-parent fallback, component interactions use the same safe access path, regression tests cover the DM/thread/component cases raised in this PR discussion, and the changelog already tracks the fix as unreleased work for `2026.4.24`.

## Evidence

This directly addresses the partial-channel getter/proxy failure mode described in the PR.
