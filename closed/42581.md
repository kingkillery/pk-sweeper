## Summary

Current `main` already routes ACP-bound turns through the `reply_dispatch` hook and preprocesses inbound media inside the ACP runtime before ACP prompt construction, which covers the regression this PR targeted.

## Evidence

- **ACP dispatch now happens through the reply-dispatch hook before normal reply generation:** `dispatchReplyFromConfig()` invokes `hookRunner.runReplyDispatch(...)` and returns early when the hook handles the turn, so ACP dispatch is no longer waiting on the older in-function path described in the PR.
