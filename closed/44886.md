## Summary

Keep open. Current main still exposes Telegram `direct.<peer>.tools` in config and applies direct DM prompt config, but the tool-policy enforcement path still resolves only group/channel policy and does not read Telegram `direct` entries. The linked #53530 context is useful partial work, but its own comment says it does not yet adapt Telegram's legacy `direct` surface.
