## Summary

Keep open. Current `main` and the latest shipped release still delegate `streamable-http` to `@modelcontextprotocol/sdk` without any OpenClaw-side Accept-header override or regression test. The bundled SDK version (`1.29.0`) does send `Accept: application/json, text/event-stream` on POST, but its follow-up GET stream still sends only `Accept: text/event-stream`, which matches the later issue comment reporting `GET /mcp ... 406 Not Acceptable`. That is not strong evidence that the issue is fully implemented or obsolete.
