## Summary

Close PR #42567. Current main intentionally preserves an existing named account's disabled state when patching shared account config, and the proposed change would regress non-setup callers by silently re-enabling disabled accounts.

## Evidence

- **Shared helper still preserves existing disabled accounts:** On current main, `patchScopedAccountConfig` keeps `existingAccount.enabled` when it is already a boolean instead of forcing `true`.
