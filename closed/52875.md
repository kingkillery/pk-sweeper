## Summary

Keep open. Current main does not implement the requested recovery path: `sessions.send` still targets an existing session key and returns a session-not-found error when the store entry/sessionId is missing, there is no `sessions.start`/`sessions_start` surface, and the timeout response still lacks the requested accepted/messageId delivery confirmation. PR #52921 is a relevant open unmerged attempt, but the issue should not be closed as implemented or superseded while the regression remains on main.
