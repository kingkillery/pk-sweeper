## Summary

Keep open. Current `main` still implements restart-required config changes as a gateway-wide drain: reload deferral watches total gateway activity, new enqueues are globally rejected while draining, and the default timeout remains 300000 ms. Main has improved the user-facing reply for drain errors, but it does not implement the scoped/non-blocking behavior requested in #66130.
